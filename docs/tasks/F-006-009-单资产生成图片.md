# F-006-009 单资产生成图片

状态：等待用户确认  
所属菜单：M-006 资产中心  
对应功能文档：`docs/features/M-006-资产中心.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：单个角色/场景/道具打开生成弹窗，支持参考图、提示词、模型、分辨率和结果选择。
为什么现在做：批量生成不适合精修单个资产，用户需要挑选满意的历史图。
做完后有什么用：用户能给单个资产生成多张历史图，并选择一张作为主图。
这一步不碰什么：不处理 clip/audio，不做视频生成，不做 ComfyUI 工作流编辑器。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `generateImage.vue`、`/assetsGenerate/generateAssets`、`/assets/getImage`、`/assets/saveAssets`。

只做：
  1. 单资产生成图片弹窗
  2. 上传参考图
  3. 编辑/AI 润色提示词
  4. 选择图片模型和分辨率
  5. 单次生成图片
  6. 查询历史图片
  7. 选择图片作为主图
  8. 删除历史图片
  9. 自定义上传图片作为结果

不做：
  1. 批量生成
  2. 视频生成
  3. 图片工作流编辑
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/assets/components/generateImage.vue` | 左侧参数，右侧历史结果选择 |
| `Toonflow-app/src/routes/assetsGenerate/polishAssetsPrompt.ts` | 单个 AI 润色提示词 |
| `Toonflow-app/src/routes/assetsGenerate/generateAssets.ts` | 插入 o_image，调用图片模型，保存图片 |
| `Toonflow-app/src/routes/assets/getImage.ts` | 查询当前资产历史图片并标 selected |
| `Toonflow-app/src/routes/assets/saveAssets.ts` | 保存当前选中图片或自定义上传 |
| `Toonflow-app/src/routes/assets/delImage.ts` | 删除历史图片，但源码先删库再查路径有风险 |

参考项目关键事实：

```txt
1. 生成前 prompt、resolution、model 必填。
2. 支持参考图 base64。
3. 右侧历史图片可选择，确认选择后更新资产 imageId。
4. 自定义上传图片会插入结果列表。
5. 删除历史图片源码先删 o_image 再查路径，文件删除失效；VT Studio 必须先查路径再删。
6. saveAssets 做了太多事，VT Studio 要拆清“保存提示词”和“选择图片”。
```

## 3. 用户操作

```txt
入口：
  资产中心 role/scene/prop 表格行 -> 生成按钮。

按钮/操作：
  1. 上传参考图。
  2. 编辑提示词。
  3. 点击 AI 润色。
  4. 选择模型和分辨率。
  5. 点击生成。
  6. 选择历史图片。
  7. 删除历史图片。
  8. 自定义上传图片。
  9. 确认选择主图。

弹窗/表单：
  单资产生成图片弹窗。

成功反馈：
  生成成功显示结果；确认后主图更新。

失败反馈：
  参数缺失、模型失败、文件缺失、删除失败。
```

## 4. 要做什么功能

### 1. 打开生成弹窗并读取历史图

怎么做：
- 输入：assetId。
- 输出：资产信息和历史图片列表。
- 写什么数据：不写。
- 状态怎么变：dialogOpen=true，loading -> loaded。
- 异常怎么处理：资产不存在提示并关闭。
- 限制：只支持 role/scene/prop。

### 2. AI 润色提示词

怎么做：
- 输入：asset name、description、type。
- 输出：prompt。
- 写什么数据：写 assets.prompt 和 prompt_status。
- 状态怎么变：润色中 -> 成功/失败。
- 异常怎么处理：视觉手册缺失或模型失败写 prompt_error。
- 限制：clip/audio 不支持。

### 3. 单次生成图片

怎么做：
- 输入：prompt、model、resolution、referenceImage。
- 输出：历史图片记录。
- 写什么数据：写媒体占位、图片文件、状态。
- 状态怎么变：running -> succeeded/failed。
- 异常怎么处理：失败写 error_reason。
- 限制：生成结果先进入历史图，不自动覆盖主图，除非产品确认。

### 4. 选择主图

怎么做：
- 输入：assetId、mediaId。
- 输出：资产 current_media_id 更新。
- 写什么数据：更新 assets.current_media_id。
- 状态怎么变：selected 更新。
- 异常怎么处理：mediaId 不属于该资产时拒绝。
- 限制：再次点击同一历史图不能把主图误清空。

### 5. 删除历史图片

怎么做：
- 输入：mediaId。
- 输出：删除结果。
- 写什么数据：先查路径，再删文件，再删记录；如是主图则清理或切换。
- 状态怎么变：历史图列表刷新。
- 异常怎么处理：文件删除失败提示。
- 限制：不能先删记录再查路径。

## 7. 偏差

```txt

  1. 保存提示词和选择图片拆成不同能力。
  2. 删除图片先查路径再删文件。
  3. 不允许误清当前 imageId。

原因：
  参考项目接口语义太重且有文件删除 bug。

是否写入 04：
  写入。
```
