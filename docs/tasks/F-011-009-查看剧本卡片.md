# F-011-009 查看剧本卡片

状态：等待用户确认

对应功能文档：`docs/features/M-011-剧本Agent.md`

## 0. 快速理解

```txt
一句话：右侧“剧本”tab 用卡片展示分集剧本，支持单卡折叠和全部折叠。
为什么现在做：Agent 生成的剧本不能只藏在聊天里，必须有可管理的列表。
做完后有什么用：用户能快速浏览每集剧本内容。
这一步不碰什么：不编辑、不删除、不生成新剧本。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow script tab 的剧本卡片展示。

只做：
  1. 剧本 tab
  2. 剧本卡片网格
  3. 内容预览
  4. 单卡折叠/展开
  5. 一键全部折叠/展开
  6. 空状态

不做：
  1. 编辑剧本
  2. 删除剧本
  3. 新增剧本
  4. 剧本导出
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/scriptAgent/index.vue` | script tab 中两列卡片网格展示 script 列表 |
| `Toonflow-web/src/stores/scriptAgent.ts` | planData.script 是 `{ id?, name, content }[]` |

参考项目关键事实：

```txt
1. 无剧本时显示 t-empty。
2. 有剧本时两列 card 网格。
3. 卡片显示 #序号、名称、折叠按钮、编辑、删除、正文。
4. 折叠状态按 `id:{id}` 或 `index:{index}` 记录。
5. 右下角悬浮按钮用于全部折叠/展开。
```

## 3. 用户操作

```txt
入口：
  剧本 Agent 右侧 -> 剧本 tab。

按钮/操作：
  1. 点击单张卡片折叠/展开。
  2. 点击全部折叠/展开。

弹窗/表单：
  无。

成功反馈：
  卡片内容折叠状态更新。

失败反馈：
  无剧本显示空状态。
```

## 4. 要做什么功能

### 1. 展示剧本卡片

怎么做：
- 输入：scripts 数组。
- 输出：卡片列表。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：content 为空显示无内容。
- 限制：内容用 pre/Markdown 预览时不能撑破布局。

### 2. 单卡折叠/展开

怎么做：
- 输入：script id 或临时 index。
- 输出：该卡片折叠状态。
- 写什么数据：不写。
- 状态怎么变：collapsedCards 更新。
- 异常怎么处理：无 id 时按 index fallback。
- 限制：优先使用稳定 id。

### 3. 全部折叠/展开

怎么做：
- 输入：当前 scripts。
- 输出：全部卡片折叠状态。
- 写什么数据：不写。
- 状态怎么变：collapsedCards 批量更新。
- 异常怎么处理：无剧本时按钮禁用。
- 限制：不改变剧本内容。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  折叠状态优先使用稳定 script.id，不依赖 index。

原因：
  index 在删除或排序后不稳定。

是否写入 04：
  不需要，属于 UI 状态修正。
```
