# F-008-010 分镜图片生成

状态：等待用户确认  
所属菜单：M-008 生产工作台  
对应功能文档：`docs/features/M-008-生产工作台.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：选中分镜后批量生成分镜图片，支持强制重新生成和状态轮询。
为什么现在做：分镜图片是视频生成、预览和导出的核心素材。
做完后有什么用：分镜卡片能从文字 prompt 生成图片，失败能看到原因。
这一步不碰什么：不生成视频，不做图片工作流编辑，不导出剪映。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/production/storyboard/batchGenerateImage` 和轮询。

只做：
  1. 选中分镜批量生成图片
  2. compulsory 强制生成
  3. 关联资产图片作为参考图
  4. 写分镜图片状态和文件
  5. 轮询分镜图片状态
  6. 失败原因展示

不做：
  1. 视频生成
  2. 图片工作流编辑器
  3. 分镜取消生成
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/production/node/storyboard.vue` | 批量生成图片调用 productionAgentStore().batchGenerateStoryboard |
| `Toonflow-app/src/routes/production/storyboard/batchGenerateImage.ts` | 查分镜、项目模型、关联资产，后台生成图片 |
| `Toonflow-app/src/routes/production/storyboard/pollingImage.ts` | 轮询 state 不为生成中的分镜 |

参考项目关键事实：

```txt
1. compulsory=true 时所有选中分镜都重新生成。
2. compulsory=false 时 shouldGenerateImage=0 的分镜置未生成，shouldGenerateImage=1 的置生成中。
3. 关联资产顺序按 o_assets2Storyboard rowid 保留。
4. 图片生成使用项目 imageModel、imageQuality、artStyle、videoRatio。
5. 保存路径 `/{projectId}/assets/{scriptId}/{uuid}.jpg`。
6. 失败写 reason 和 state=生成失败。
7. 生产页没有找到分镜图片取消按钮。
```

## 3. 用户操作

```txt
入口：
  分镜节点 -> 勾选分镜 -> 批量生成图片。

按钮/操作：
  1. 勾选分镜。
  2. 点击批量生成。
  3. 可选择强制生成。

弹窗/表单：
  可选确认框。

成功反馈：
  分镜变生成中，完成后显示图片。

失败反馈：
  未选中、项目未配置图片模型、关联资产缺图、模型失败。
```

## 4. 要做什么功能

### 1. 启动分镜图片生成

怎么做：
- 输入：storyboardIds、projectId、scriptId、compulsory。
- 输出：启动结果和初始状态。
- 写什么数据：更新 storyboards.state/shouldGenerateImage。
- 状态怎么变：未生成/失败/成功 -> running 或 skipped。
- 异常怎么处理：未选中或分镜不存在提示。
- 限制：必须当前剧本分镜。

### 2. 构建参考图和 prompt

怎么做：
- 输入：分镜 prompt、关联资产 ids。
- 输出：图片模型请求。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：关联资产图片缺失时按规则提示或跳过参考图。
- 限制：关联资产顺序必须保留。

### 3. 保存生成结果

怎么做：
- 输入：图片模型返回结果。
- 输出：分镜图片文件。
- 写什么数据：写文件，更新 storyboards.file_path/state/reason。
- 状态怎么变：running -> succeeded/failed。
- 异常怎么处理：保存文件失败写失败原因。
- 限制：renderer 不直接写文件。

### 4. 轮询分镜图片状态

怎么做：
- 输入：running storyboardIds。
- 输出：完成或失败的分镜。
- 写什么数据：不写。
- 状态怎么变：页面更新。
- 异常怎么处理：轮询失败保留下一轮。
- 限制：轮询不重复生成。
