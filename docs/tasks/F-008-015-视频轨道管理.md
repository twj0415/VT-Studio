# F-008-015 视频轨道管理

状态：等待用户确认  
所属菜单：M-008 生产工作台  
对应功能文档：`docs/features/M-008-生产工作台.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：视频工作台可以新增、删除轨道，更新轨道 prompt、时长和已选候选视频。
为什么现在做：视频轨道是后续剪映导出的基础数据，必须稳定。
做完后有什么用：每条轨道都能管理时长、提示词和最终选用的视频。
这一步不碰什么：不生成视频，不导出剪映，不做 WebAV 编辑器。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow addTrack/deleteTrack/updateVideoPrompt/updateVideoDuration/selectVideo。

只做：
  1. 新增轨道
  2. 删除轨道
  3. 更新轨道 prompt
  4. 更新轨道时长
  5. 选择候选视频
  6. 保证导出可读稳定字段

不做：
  1. 视频生成
  2. 剪映导出
  3. WebAV 时间线保存
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-app/src/routes/production/workbench/addTrack.ts` | Date.now 创建 trackId，插入 o_videoTrack |
| `Toonflow-app/src/routes/production/workbench/deleteTrack.ts` | 删除 o_videoTrack，把 o_storyboard.trackId=null |
| `Toonflow-app/src/routes/production/workbench/updateVideoPrompt.ts` | 更新 track prompt |
| `Toonflow-app/src/routes/production/workbench/updateVideoDuration.ts` | 更新 track duration |
| `Toonflow-app/src/routes/production/workbench/selectVideo.ts` | 写 o_videoTrack.videoId |

参考项目关键事实：

```txt
1. addTrack 中查 vendor 模型列表变量没使用，VT Studio 不照搬。
2. deleteTrack 不删除轨道下 o_video，也不删视频文件，VT Studio 必须定规则。
3. o_videoTrack 同时有 videoId 和 selectVideoId，源码实际用 videoId，VT Studio 只能保留一个正式字段。
4. 更新时长要受视频模型允许 duration 限制。
5. M-010 剪映导出应读取已选择候选视频，不随机取第一个。
```

## 3. 用户操作

```txt
入口：
  视频工作台轨道列表。

按钮/操作：
  1. 新增轨道。
  2. 删除轨道。
  3. 编辑 prompt。
  4. 修改时长。
  5. 选择候选视频。

弹窗/表单：
  删除确认；时长选择或输入。

成功反馈：

失败反馈：
  模型未配置、时长不支持、删除失败、候选视频不属于轨道。
```

## 4. 要做什么功能

### 1. 新增轨道

怎么做：
- 输入：projectId、scriptId、duration。
- 输出：新 videoTrack。
- 写什么数据：写 video_tracks。
- 状态怎么变：列表新增轨道。
- 异常怎么处理：duration 不合法或缺模型配置时失败。
- 限制：duration 从模型允许值中选择。

### 2. 删除轨道

怎么做：
- 输入：trackId。
- 输出：删除结果。
- 写什么数据：删除 video_tracks，处理 storyboards.track_id，删除或保留候选视频按规则。
- 状态怎么变：轨道从列表移除。
- 异常怎么处理：事务失败回滚。
- 限制：如果删除候选视频，必须清文件。

### 3. 更新轨道 prompt 和时长

怎么做：
- 输入：trackId、prompt 或 duration。
- 输出：更新后的轨道。
- 写什么数据：更新 video_tracks。
- 状态怎么变：dirty -> saved。
- 异常怎么处理：保存失败回滚 UI。
- 限制：duration 要 clamp 到模型允许范围。

### 4. 选择候选视频

怎么做：
- 输入：trackId、videoId。
- 输出：selected_video_id。
- 写什么数据：更新 video_tracks.selected_video_id。
- 状态怎么变：该候选被标记为已选。
- 异常怎么处理：video 不属于 track 时拒绝。
- 限制：正式字段只保留 selected_video_id，不混 videoId/selectVideoId。
