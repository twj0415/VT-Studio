# F-002-016 退出登录/清理登录状态

状态：已由 CORE-010 覆盖，不单独执行  
所属菜单：M-002 设置  
对应功能文档：`docs/features/M-002-设置.md`  
原则：先确认本文档，再改代码

说明：

```txt
本任务原本是 M-002 设置下的独立退出登录功能。
实际实现已在 CORE-010 登录和本地用户中完成：退出确认、清理 token/user/userId/currentProject、跳转登录页。
后续不要再单独执行本任务，除非要重做退出登录策略。
```

## 0. 快速理解

```txt
一句话：设置里提供退出登录按钮，确认后清理本地登录状态并回到登录页。
为什么现在做：CORE-010 有底层退出能力，本功能负责设置页退出入口。
做完后有什么用：用户能明确退出当前本地用户，避免半登录状态。
这一步不碰什么：不删除数据库、不删除模型密钥、不删除素材、不做服务端 logout。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow logoutConfig，同时修正 token/userId 清理不一致问题。

只做：
  1. 退出登录入口
  2. 警告说明
  3. 二次确认
  4. 清理 token/user/userId/currentProject
  5. 跳转登录页

不做：
  1. 清空数据库
  2. 清模型配置
  3. 清素材文件
  4. 远程退出接口
```

参考项目关键事实：

```txt
4. VT Studio 必须同时清 token/user/userId/currentProject。
```

## 3. 用户操作

```txt
入口：
  设置弹窗 -> 退出登录/清理登录状态。

按钮/操作：
  1. 点击退出登录。
  2. 确认弹窗里点击确认。
  3. 或取消退出。

弹窗/表单：
  退出确认弹窗。

成功反馈：
  退出成功，跳转登录页。

失败反馈：
  清理失败、跳转失败。
```

## 4. 要做什么功能

### 1. 显示退出警告

怎么做：
- 输入：无。
- 输出：说明退出只清登录状态，不清业务数据。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：无。
- 限制：文案必须说清不会删除项目和素材。

### 2. 二次确认

怎么做：
- 输入：用户点击退出。
- 输出：确认或取消。
- 写什么数据：不写。
- 状态怎么变：弹窗打开/关闭。
- 异常怎么处理：弹窗异常时不执行退出。
- 限制：不能点按钮就直接退出。

### 3. 清理登录状态

怎么做：
- 输入：确认退出。
- 输出：清理成功。
- 写什么数据：清 renderer 本地状态。
- 清理内容：
  - token
  - user
  - userId
  - currentProject
  - 当前项目上下文缓存
- 状态怎么变：已登录 -> 未登录。
- 异常怎么处理：清理失败提示。
- 限制：不清模型密钥、数据库、素材。

### 4. 跳转登录页

怎么做：
- 输入：清理成功。
- 输出：路由到登录页。
- 写什么数据：不写。
- 状态怎么变：业务页面不可访问。
- 异常怎么处理：跳转失败时刷新 auth 状态。
- 限制：依赖 CORE-010 路由守卫。
