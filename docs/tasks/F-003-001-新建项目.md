# F-003-001 新建项目

状态：等待用户确认  
所属菜单：M-003 项目管理  
对应功能文档：`docs/features/M-003-项目管理.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：项目页点击“新建项目”，填写项目资料、模型、画质、比例、手册后创建项目。
为什么现在做：项目是小说、剧本、资产、生产的上层上下文，没有项目后续业务都没归属。
做完后有什么用：用户能创建一个完整可打开的项目，后续导入小说或写剧本都挂到这个项目下。
这一步不碰什么：不做小说导入、不做剧本生成、不创建业务素材、不做手册编辑本身。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的 addProject 流程，实现项目创建。

只做：
  1. 项目页新建按钮
  2. 项目弹窗新增模式
  3. 项目字段填写和校验
  4. 图片模型、视频模型、视频模式、画质、比例选择
  5. 视觉手册和导演手册选择
  6. 创建项目记录
  7. 创建或准备项目业务目录规则
  8. 创建成功后刷新项目列表

不做：
  1. 编辑项目
  2. 删除项目
  3. 打开项目跳转
  4. 新增/编辑视觉手册
  5. 新增/编辑导演手册
```

| `Toonflow-app/src/routes/project/getVisualManual.ts` | 查询视觉手册 |
| `Toonflow-app/src/routes/project/queryDirectorManual.ts` | 查询导演手册 |

参考项目关键事实：

```txt
1. 新建项目主要写数据库，源码未找到完整创建本地项目目录逻辑。
2. projectType 只看到 novel/script。
3. 图片质量是 1K/2K/4K。
4. 视频比例是 16:9 / 9:16。
5. 图片模型只选 enabled 供应商里的 image 模型。
6. 视频模型只选 enabled 供应商里的 video 模型。
7. 视频模式来自选中视频模型的 mode 能力。
8. 视觉手册和导演手册是必选。
```

## 3. 用户操作

```txt
入口：
  左侧全局菜单 -> 项目 -> 新建项目按钮。

按钮/操作：
  1. 点击新建项目。
  2. 填写项目来源类型、名称、题材/类型、简介。
  3. 选择图片模型和图片质量。
  4. 选择视频模型、视频模式、视频比例。
  5. 选择视觉手册和导演手册。
  6. 点击确认。

弹窗/表单：
  项目弹窗。

成功反馈：
  创建成功，关闭弹窗，刷新项目列表。

失败反馈：
  必填缺失、模型不可用、手册缺失、保存失败。
```

## 4. 要做什么功能

### 1. 打开新增项目弹窗

怎么做：
- 输入：点击新建按钮。
- 输出：空项目表单。
- 写什么数据：不写。
- 状态怎么变：dialogShow=true，editProjectData=null。
- 异常怎么处理：无。
- 限制：打开时刷新模型、视觉手册、导演手册列表。

### 2. 填写项目基础信息

怎么做：
- 输入：projectType、name、type、intro。
- 输出：表单状态。
- 写什么数据：确认前不写。
- 状态怎么变：表单 dirty。
- 异常怎么处理：空值显示字段错误。
- 限制：projectType 先只允许 `novel/script`。

### 3. 选择模型和画质

怎么做：
- 输入：imageModel、imageQuality、videoModel、mode、videoRatio。
- 输出：模型配置。
- 写什么数据：确认前不写。
- 状态怎么变：选择视频模型后刷新 mode 选项。
- 异常怎么处理：模型列表为空显示配置模型提示；模型失效不能提交。
- 限制：图片质量只允许 1K/2K/4K；视频比例只允许 16:9/9:16。

### 4. 选择视觉手册和导演手册

怎么做：
- 输入：visualManualId/path、directorManualId/path。
- 输出：项目手册引用。
- 写什么数据：确认前不写。
- 状态怎么变：卡片选中。
- 异常怎么处理：手册列表为空或选中手册失效时不能提交。
- 限制：本任务只选择已有手册，不新增/编辑手册。

### 5. 提交创建项目

怎么做：
- 输入：完整表单。
- 输出：新项目记录。
- 写什么数据：写 projects 表；准备项目业务目录信息。
- 状态怎么变：创建中 -> 成功关闭弹窗 -> 刷新列表。
- 异常怎么处理：数据库失败、路径准备失败、字段非法返回明确 msg。
- 限制：业务文件目录默认放 userData/projects/{projectId}，不能放源码目录。

### 6. 创建后刷新列表

怎么做：
- 输入：创建成功。
- 输出：项目列表。
- 写什么数据：不写。
- 状态怎么变：项目卡片新增。
- 异常怎么处理：刷新失败时提示并保留创建成功结果。
- 限制：不自动打开项目，除非后续单独确认。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 明确项目业务目录在 userData/projects/{projectId}，不放源码目录。
  2. 字段命名不用 artStyle 这种旧名。
  3. 创建时校验模型和手册引用有效。

原因：
  VT Studio 是本地桌面工具，项目目录必须专业、可迁移、可清理。

是否写入 04：
  项目目录规则如和参考项目不同，已有本地文件层偏差覆盖；不新增。
```
