# F-006-010 取消生成

状态：等待用户确认

## 0. 快速理解

```txt
一句话：参考项目后端有取消图片生成接口，但资产中心页面没有直接按钮，第一版先定边界。
为什么现在做：取消生成如果只改状态会变成假取消，必须和任务队列统一。
做完后有什么用：后续需要取消时知道传 imageId/mediaId，不会传资产 id 乱改。
这一步不碰什么：第一版资产中心不强行加取消按钮，不假装能取消供应商任务。
```

## 1. 本次做什么

```txt
目标：
  记录并规范 Toonflow `/assetsGenerate/cancelGenerate` 的取消能力，作为后续统一任务取消的资产侧边界。

只做：
  1. 明确取消目标是图片记录 id
  2. 明确资产中心第一版不加取消按钮也算对齐
  3. 明确如果加取消按钮必须接 task cancel handler
  4. 防止生成完成后覆盖取消状态

不做：
  1. 第一版 UI 取消按钮
  2. 供应商级真实取消协议
  3. 批量取消
  4. 视频取消
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-app/src/routes/assetsGenerate/cancelGenerate.ts` | 接收 o_image.id，把 state 改成生成失败 |
| `Toonflow-app/src/routes/assetsGenerate/generateAssets.ts` | 生成完成前检查 o_image.state，如果已是生成失败，不再写完成 |
| `Toonflow-app/src/routes/assetsGenerate/batchGenerateImageAssets.ts` | 批量生成也会检查状态 |
| `Toonflow-web/src/views/cornerScape/index.vue` | 角景页使用取消生成 |
| `Toonflow-web/src/views/assets/index.vue` | 资产中心页没找到直接取消按钮 |

参考项目关键事实：

```txt
1. 取消接口参数是 o_image.id，不是资产 id。
2. 取消效果是把图片 state 改为生成失败。
3. 生成 route 完成前会检查状态，已失败就不写完成。
4. 资产中心页面没有直接取消按钮，角景页有使用。
5. VT Studio 如果做取消，应接统一任务队列，不只是改状态。
```

## 3. 用户操作

```txt
入口：
  第一版资产中心无直接入口；后续如增加，入口在生成中图片状态上。

按钮/操作：
  后续可能点击取消生成。

弹窗/表单：
  取消确认框。

成功反馈：
  状态变为已取消或失败，生成结果不会覆盖。

失败反馈：
  无生成中记录、任务不支持取消、取消失败。
```

## 4. 要做什么功能

### 1. 明确取消目标

怎么做：
- 输入：mediaId/imageId。
- 输出：取消请求。
- 写什么数据：更新媒体状态或任务状态。
- 状态怎么变：running -> cancelled/failed。
- 异常怎么处理：id 不存在或不属于当前项目时拒绝。
- 限制：不能传 assetId 当取消目标。

### 2. 防止取消后被覆盖

怎么做：
- 输入：生成完成回调。
- 输出：检查当前状态。
- 写什么数据：仅当状态仍为 running 时写 succeeded。
- 状态怎么变：cancelled 不被覆盖。
- 异常怎么处理：状态已取消时丢弃生成结果并清理临时文件。
- 限制：所有图片生成实现都必须检查。

### 3. 接入统一任务取消

怎么做：
- 输入：taskId/mediaId。
- 输出：业务 cancel handler。
- 写什么数据：更新 tasks 和 media 状态。
- 状态怎么变：running -> cancelled。
- 异常怎么处理：供应商不支持真实取消时标记本地取消，并忽略后续结果。
- 限制：没有 cancel handler 的任务不能显示可取消按钮。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  取消状态用 cancelled，不用“生成失败”伪装取消。

原因：
  用户取消和模型失败是两种不同结果。

是否写入 04：
  写入，属于状态语义增强。
```
