所属菜单：M-001 全局导航与项目入口

## 0. 快速理解

```txt
一句话：应用启动时把 Electron 环境、窗口标题栏、运行地址、主题、语言这些基础状态先初始化好。
为什么现在做：全局布局、登录、设置、项目页面都会依赖这些状态。
做完后有什么用：桌面端打开后高度、标题栏、主题、语言、主进程地址都能稳定工作。
这一步不碰什么：不做具体菜单页面，不做设置项编辑，不做项目业务。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的 App 初始化流程，但改成 VT Studio 的 Electron 安全调用方式。

只做：
  1. 检测是否 Electron 桌面环境。
  2. 获取本地主进程服务地址或 IPC 能力状态。
  3. 显示/隐藏自定义标题栏。
  4. 给 body 设置桌面端 class。
  5. 初始化主题。
  6. 初始化语言。
  7. 初始化 Markdown 链接打开规则。
  8. loading 状态结束。

不做：
  1. 不做业务数据初始化。
  2. 不做版本检查轮询。
  3. 不做登录表单。
  4. 不直接使用 Toonflow 的 `toonflow://` 协议名。
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/App.vue` | 启动 loading，fetch `toonflow://getAppUrl` 获取后端地址，设置 `isElectron`，初始化主题和语言 |
| `Toonflow-web/src/components/titleBar.vue` | Electron 下显示自定义标题栏 |
| `Toonflow-web/src/stores/setting.ts` | 保存 `isElectron`、`baseUrl`、`themeSetting`、`language` |

参考项目关键事实：

```txt
1. App 启动先判断 Electron，再决定是否显示 titleBar。
2. Electron 下页面高度要减去 32px 标题栏。
3. baseUrl 由本地协议返回，默认值是 http://localhost:10588/api。
4. 初始化会调用 initTheme 和浏览器语言识别。
5. Markdown 链接点击会走统一打开逻辑，不能让链接在桌面壳里乱跳。
```

## 3. 用户操作

```txt
入口：
  用户打开 VT Studio 桌面程序。

按钮/操作：
  无，启动自动执行。

弹窗/表单：
  无。

成功反馈：
  主窗口显示，loading 结束，标题栏和主题正确。

失败反馈：
  初始化失败时显示可读错误，不能白屏。
```

## 4. 要做什么功能

### 1. 启动 loading

怎么做：
- 输入：应用启动事件。
- 输出：初始化期间显示 loading。
- 写什么数据：不写业务数据。
- 状态怎么变：`appInitializing=true -> false`。
- 异常怎么处理：即使部分初始化失败，也要结束 loading 并显示错误。
- 限制：不能因为主进程地址获取失败一直卡 loading。

### 2. Electron 环境检测

怎么做：
- 输入：`window.vtStudio` 是否存在、主进程返回的 runtime 信息。
- 输出：`settingStore.isElectron`。
- 写什么数据：写前端状态，不写数据库。
- 状态怎么变：Web 默认 false，Electron 成功检测为 true。
- 异常怎么处理：检测失败按 Web 降级，但记录错误。
- 限制：renderer 不直接访问 Node、process、fs。

### 3. 本地能力地址初始化

怎么做：
- 输入：主进程提供的本地 API 地址或 IPC 能力清单。
- 输出：`baseUrl` 或 `runtime.capabilities`。
- 写什么数据：按设置规则持久化可持久化字段。
- 状态怎么变：默认地址被主进程返回值覆盖。
- 异常怎么处理：保留默认值并提示本地服务不可用。
- 限制：不要继续使用 `toonflow://getAppUrl` 名称；VT Studio 要走 `window.vtStudio.runtime.getInfo()` 或同类 API。

### 4. 标题栏和高度修正

怎么做：
- 输入：`isElectron`。
- 输出：标题栏显示状态、内容区高度 class、窗口按钮状态。
- 写什么数据：不写。
- 状态怎么变：Electron 下 body 加 `is-electron`，内容高度减 32px。
- 异常怎么处理：无标题栏信息时使用普通窗口布局。
- 限制：设置弹窗、主内容不能遮住标题栏拖拽区。

标题栏按钮必须做：

```txt
最小化：window.vtStudio.window.minimize()
最大化/还原：window.vtStudio.window.toggleMaximize()
关闭：window.vtStudio.window.close()
最大化状态：window.vtStudio.window.getState() 或事件同步
拖拽区：标题区域可拖拽，按钮区域不可拖拽
不使用 toonflow://windowMinimize/windowMaximize/windowClose
```

### 5. 主题初始化

怎么做：
- 输入：本地保存的主题设置，默认主题。
- 输出：CSS 变量、html/body class。
- 写什么数据：必要时写前端持久化设置。
- 状态怎么变：启动时一次性应用主题。
- 异常怎么处理：无效主题回退默认主题。
- 限制：多主题体系后续归 F-002-001，本任务只保证启动应用。

### 6. 语言初始化

怎么做：
- 输入：本地保存语言、浏览器语言。
- 输出：i18n 当前语言。
- 写什么数据：必要时写前端持久化设置。
- 状态怎么变：启动时确定语言。
- 异常怎么处理：未知语言回退 zh-CN。
- 限制：语言切换页面归 F-002-002。

### 7. Markdown 链接打开规则

怎么做：
- 输入：用户点击 Markdown 内容里的链接。
- 输出：通过受控外链能力打开。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：外链不在白名单或打开失败时提示。
- 限制：不能在 renderer 里直接打开任意 URL。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  不使用 `toonflow://getAppUrl`，改为 VT Studio 的 preload API。

原因：
  VT Studio 不能保留 Toonflow 协议名，页面也不能直接访问非受控协议。

是否写入 04：
  是，属于底层调用方式偏差。
```
