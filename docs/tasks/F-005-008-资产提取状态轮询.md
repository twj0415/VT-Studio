# F-005-008 资产提取状态轮询

状态：等待用户确认  
所属菜单：M-005 剧本  
对应功能文档：`docs/features/M-005-剧本.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：剧本资产提取是异步的，页面要轮询提取状态并刷新卡片。
为什么现在做：HTTP 返回“开始提取”后，用户还需要知道什么时候成功或失败。
做完后有什么用：提取完成后卡片自动更新资产 tag，失败能看到原因。
这一步不碰什么：不重新提取，不取消任务，不自动生成图片。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/script/pollScriptAssets` 的 3 秒轮询。

只做：
  1. 找出正在提取的剧本
  2. 每 3 秒轮询状态
  3. 状态变化后刷新剧本列表
  4. 展示失败原因
  5. 启动时修复残留 running 状态

不做：
  1. 自动重试
  2. 取消提取
  3. 图片生成
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/script/index.vue` | `scripts.filter(s => s.extractState == 0)`，每 3 秒轮询 |
| `Toonflow-app/src/routes/script/pollScriptAssets.ts` | 查询 whereIn ids 且 extractState 不为生成中，返回状态变化的剧本 |
| `Toonflow-app/src/lib/fixDB.ts` | 启动时把残留 extractState=0 改成失败 |

参考项目关键事实：

```txt
1. 只轮询 extractState=0 正在提取，不轮询等待状态 2。
2. 轮询间隔 3 秒。
3. 返回状态变化后前端刷新剧本列表。
4. fixDB 没有处理残留 extractState=2，VT Studio 要定规则。
5. 源码 whereNot 使用了中文“生成中”，但 extractState 实际是数字，VT Studio 必须统一枚举。
```

## 4. 要做什么功能

### 1. 识别提取中剧本

怎么做：
- 输入：当前剧本列表。
- 输出：extract_status=running 的 scriptIds。
- 写什么数据：不写。
- 状态怎么变：有 running 才启动轮询。
- 异常怎么处理：无 id 的剧本忽略。
- 限制：状态用明确枚举，不用数字或中文混查。

### 2. 轮询提取状态

怎么做：
- 输入：scriptIds。
- 输出：已完成或失败的状态。
- 写什么数据：不写。
- 状态怎么变：每 3 秒请求一次。
- 异常怎么处理：请求失败不清空列表，下一轮继续。
- 限制：离开页面停止轮询。

### 3. 刷新剧本列表

怎么做：
- 输入：状态变化结果。
- 输出：最新剧本卡片。
- 写什么数据：不写。
- 状态怎么变：running -> succeeded/failed。
- 异常怎么处理：刷新失败提示。
- 限制：保留当前搜索条件和选中状态合理清理。

### 4. 启动修复

怎么做：
- 输入：应用启动时残留 running/waiting。
- 输出：失败或重新排队状态。
- 写什么数据：更新 scripts.extract_status 和 extract_error。
- 状态怎么变：running -> failed；waiting 按规则处理。
- 异常怎么处理：修复失败写日志。
- 限制：不能让旧任务永久卡在等待。
