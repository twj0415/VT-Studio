# F-004-002 原文列表、搜索、分页

状态：等待用户确认  
所属菜单：M-004 小说/原文  
对应功能文档：`docs/features/M-004-小说原文.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：做原文章节列表，支持按章节名搜索、分页、查看正文和事件摘要状态。
为什么现在做：导入后必须能管理和检查章节，Agent 后续也要读取这些数据。
做完后有什么用：用户能看到每章正文、事件摘要、生成中和失败原因。
这一步不碰什么：不做全文搜索，不做事件重新生成，不做剧本生成。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/novel/getNovel` 的章节列表能力。

只做：
  1. 当前项目章节列表
  2. 按章节名搜索
  3. 分页
  4. 正文预览和查看详情
  5. 事件摘要展示
  6. 事件生成中和失败原因展示

不做：
  1. 正文全文搜索
  2. 卷名搜索
  3. 事件内容搜索
  4. 事件库管理
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/novel/index.vue` | 表格展示章节、正文预览、事件摘要、生成状态和操作 |
| `Toonflow-app/src/routes/novel/getNovel.ts` | 按 projectId 查询章节，支持 chapter 搜索和分页 |
| `Toonflow-app/src/routes/novel/getNovelIndex.ts` | 查询章节索引信息 |

参考项目关键事实：

```txt
1. 列表只查当前项目。
2. 搜索只确认按 chapter 字段搜索。
3. 正文超过 80 字显示查看详情。
4. eventState=0 时事件列显示 loading。
5. eventState=-1 且 event 为空时显示生成失败，可查看 errorReason。
```

## 3. 用户操作

```txt
入口：
  项目内顶部菜单 -> 原文。

按钮/操作：
  1. 输入章节名关键词。
  2. 点击搜索。
  3. 翻页。
  4. 查看正文详情。
  5. 查看事件摘要或失败原因。

弹窗/表单：
  正文/事件详情预览弹窗。

成功反馈：
  表格展示章节。

失败反馈：
  查询失败提示；无数据显示空状态。
```

## 4. 要做什么功能

### 1. 查询当前项目章节

怎么做：
- 输入：projectId、page、pageSize、chapterKeyword。
- 输出：章节列表和 total。
- 写什么数据：不写。
- 状态怎么变：loading=true -> false。
- 异常怎么处理：查询失败显示错误状态。
- 限制：没有当前项目不能查询。

### 2. 按章节名搜索

怎么做：
- 输入：章节名关键词。
- 输出：匹配章节列表。
- 写什么数据：不写。
- 状态怎么变：搜索后 page=1。
- 异常怎么处理：空关键词返回全部章节。
- 限制：先只按章节名搜索，对齐参考项目。

### 3. 展示正文和事件摘要

怎么做：
- 输入：chapterData、event、eventState、errorReason。
- 输出：表格预览和详情弹窗。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：正文或事件为空时显示空文案。
- 限制：长文本不撑破表格。

### 4. 展示事件状态

怎么做：
- 输入：eventState。
- 输出：未生成/生成中/成功/失败显示。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：未知状态显示未知并记录日志。
- 限制：内部状态不使用中文枚举。
