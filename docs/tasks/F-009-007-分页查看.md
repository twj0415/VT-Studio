# F-009-007 分页查看

状态：等待用户确认  
所属菜单：M-009 任务中心  
对应功能文档：`docs/features/M-009-任务中心.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：任务中心列表加分页，支持切页和切每页数量。
为什么现在做：任务数据会越来越多，不分页会卡，也不方便定位。
做完后有什么用：用户能按页查看任务，不会一次加载全部历史任务。
这一步不碰什么：不做无限滚动，不做任务详情，不做导出任务列表。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `page/limit/total` 的任务分页。

只做：
  1. 分页器
  2. page/pageSize 参数
  3. total 展示
  4. 切页和切 pageSize 后刷新列表

不做：
  1. 虚拟滚动
  2. 导出 CSV
  3. 任务详情弹窗
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/task/index.vue` | `pagination.page`、`pagination.limit` 通过分页器 v-model 控制 |
| `Toonflow-app/src/routes/task/getTaskApi.ts` | `offset=(page-1)*limit`，另查 count total |

参考项目关键事实：

```txt
1. 默认 page=1，limit=10。
2. 切筛选会把 page 重置为 1。
3. 刷新不会重置 page。
4. 后端返回 data 和 total。
```

## 4. 要做什么功能

### 1. 页码切换

怎么做：
- 输入：page。
- 输出：对应页任务。
- 写什么数据：不写。
- 状态怎么变：page 更新，loading=true -> false。
- 异常怎么处理：失败时回退到原 page 或保留旧数据。
- 限制：page 必须大于等于 1。

### 2. 每页数量切换

怎么做：
- 输入：pageSize。
- 输出：按新数量查询的任务。
- 写什么数据：不写。
- 状态怎么变：pageSize 更新，通常 page=1。
- 异常怎么处理：非法 pageSize 返回参数错误。
- 限制：pageSize 只能来自允许选项，例如 10/20/50/100。

### 3. total 展示

怎么做：
- 输入：total。
- 输出：分页器总数。
- 写什么数据：不写。
- 状态怎么变：查询成功后更新 total。
- 异常怎么处理：total 缺失时按 0 处理并记录服务异常。
- 限制：total 必须由服务层 count 查询返回。

