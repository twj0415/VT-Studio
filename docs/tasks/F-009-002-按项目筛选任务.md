# F-009-002 按项目筛选任务

状态：等待用户确认  
所属菜单：M-009 任务中心  
对应功能文档：`docs/features/M-009-任务中心.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：任务中心增加项目下拉，能看全部项目，也能只看某个项目的任务。
为什么现在做：任务会跨项目产生，不按项目筛选后期很难定位问题。
做完后有什么用：用户能快速找到某个项目里的生成、导出、Agent 任务。
这一步不碰什么：不改任务写入逻辑，不做项目 CRUD，不做自动刷新。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/task/getProject` 的项目筛选能力，同时修正“全部项目”回退当前项目的语义问题。

只做：
  1. 项目筛选下拉
  2. 查询项目选项
  3. 全部项目筛选
  4. 指定项目筛选
  5. 筛选变化后重置第 1 页

不做：
  1. 新建项目
  2. 编辑项目
  3. 删除项目
  4. 任务分类筛选
  5. 任务状态筛选
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/task/index.vue` | 页面进入后调用 `getProject`，前端补“全部”选项 |
| `Toonflow-app/src/routes/task/getProject.ts` | 查询 `o_project` 的 id/name，过滤 name 为空，按 name groupBy |
| `Toonflow-app/src/routes/task/getTaskApi.ts` | projectId 有值时加 where；空值时参考前端可能回退当前项目 |

参考项目关键事实：

```txt
1. 项目选项 value 是项目 id，label 是项目名。
2. 前端会补“全部”选项。
3. Toonflow 空 projectId 在某些上下文会回退当前项目，导致“全部”不一定是真全部。
4. getProject 按 name groupBy，同名项目 id 可能不稳定；VT Studio 不能照搬。
```

## 3. 用户操作

```txt
入口：
  任务中心筛选栏。

按钮/操作：
  选择“全部项目”或某个项目。

弹窗/表单：
  无。

成功反馈：
  表格刷新为对应项目任务。

失败反馈：
  项目选项加载失败时显示空选项和错误提示；任务查询失败显示错误提示。
```

## 4. 要做什么功能

### 1. 查询项目筛选选项

怎么做：
- 输入：无。
- 输出：`[{ label, value }]`，第一项是全部项目。
- 写什么数据：不写。
- 状态怎么变：项目下拉 loading -> 可选。
- 异常怎么处理：查询失败时保留“全部项目”，并提示项目列表加载失败。
- 限制：同名项目也必须保留不同 id。

### 2. 全部项目筛选

怎么做：
- 输入：projectId 为空。
- 输出：所有项目任务。
- 写什么数据：不写。
- 状态怎么变：筛选变化后 page=1，再刷新任务列表。
- 异常怎么处理：查询失败显示错误。
- 限制：服务层不能把空 projectId 回退当前项目。

### 3. 指定项目筛选

怎么做：
- 输入：projectId。
- 输出：该项目下的任务列表。
- 写什么数据：不写。
- 状态怎么变：page=1，保留其他筛选条件。
- 异常怎么处理：项目不存在时返回空列表或明确提示。
- 限制：只允许查当前本地数据库已有项目。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  “全部项目”真正查全部，不回退当前项目。

原因：
  “全部”语义必须准确，否则任务中心排查问题会误导用户。

是否写入 04：
  写入，属于对 Toonflow 语义缺陷的修正。
```
