说明：

本任务原本是 M-002 设置下的独立用户配置功能。
实际实现已在 CORE-010 登录和本地用户中完成：设置页读取当前用户、修改用户名和密码、同步当前登录用户状态。
后续不要再单独执行本任务，除非要重做用户配置策略。

## 0. 快速理解

```txt
一句话：设置页里查看和修改本地用户的用户名、密码。
为什么现在做：CORE-010 负责底层登录，本功能负责设置页用户配置入口。
做完后有什么用：用户能在设置里改本地账号信息。
这一步不碰什么：不做注册、验证码、云账号、多用户权限。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow loginConfig，但落成本地用户配置页面。

只做：
  1. 读取当前本地用户
  2. 修改用户名
  3. 修改密码
  4. 表单校验
  5. 保存后同步当前登录用户状态

不做：
  1. 云账号
  2. 注册/找回密码
  3. 多账号列表
  4. 权限管理
```

| 参考文件 | 关键逻辑 |

参考项目关键事实：

```txt
1. 用户表只有 name、password、id 这类基础字段。
2. 密码明文保存。
3. 这是设置页用户配置，不是登录页本身。
4. VT Studio 底层登录方案已经在 CORE-010 定义。
```

## 3. 用户操作

```txt
入口：
  设置弹窗 -> 登录/用户配置。

按钮/操作：
  1. 查看当前用户名。
  2. 输入新用户名。
  3. 输入新密码。
  4. 点击保存。

弹窗/表单：
  用户配置表单。

成功反馈：
  保存成功，当前用户信息同步更新。

失败反馈：
  用户名过短/过长、密码过短/过长、用户不存在、保存失败。
```

## 4. 要做什么功能

### 1. 读取本地用户

怎么做：
- 输入：无。
- 输出：`id`、`name`。
- 写什么数据：不写。
- 状态怎么变：表单填入当前用户。
- 异常怎么处理：用户缺失提示重新初始化默认数据。
- 限制：不返回密码明文；如需要改密码，输入框为空代表不展示原密码。

### 2. 修改用户名

怎么做：
- 输入：`id`、`name`。
- 输出：保存成功。
- 写什么数据：写 users.name。
- 状态怎么变：当前登录用户同步更新。
- 异常怎么处理：name 为空、长度不合法、重复时失败。
- 限制：第一版只有本地默认用户，不做多账号唯一性复杂逻辑。

### 3. 修改密码

怎么做：
- 输入：`id`、`password`。
- 输出：保存成功。
- 写什么数据：写 users.password。
- 状态怎么变：后续登录使用新密码。
- 异常怎么处理：密码长度不合法时失败。
- 限制：是否 hash 由 CORE-010 决定；本页不单独改变存储策略。
