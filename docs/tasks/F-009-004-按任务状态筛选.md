# F-009-004 按任务状态筛选

状态：等待用户确认

## 0. 快速理解

```txt
一句话：任务中心增加状态筛选，能只看进行中、成功、失败、已取消任务。
为什么现在做：排查失败任务和查看进行中任务是任务中心最常用操作。
做完后有什么用：用户不用在全部任务里翻，可以直接定位失败或运行中的任务。
这一步不碰什么：不改变任务状态，不做取消按钮，不做重试按钮。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 状态筛选，并把 VT Studio 内部状态统一成英文枚举。

只做：
  1. 状态筛选下拉
  2. 全部/进行中/已完成/生成失败/已取消筛选
  3. 内部状态到中文展示映射
  4. 筛选变化后重置第 1 页

不做：
  1. 修改任务状态
  2. 取消任务
  3. 重试任务
  4. 业务轮询
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/task/index.vue` | 状态下拉固定：全部、进行中、已完成、生成失败 |
| `Toonflow-app/src/routes/task/getTaskApi.ts` | state 有值时加 where 条件 |
| `Toonflow-app/src/utils/taskRecord.ts` | `done(1)` 写已完成，`done(-1, reason)` 写生成失败 |

参考项目关键事实：

```txt
1. Toonflow 状态是中文字符串：进行中、已完成、生成失败。
2. Toonflow 没有取消状态。
3. VT Studio 已在任务队列设计中包含 pending/running/succeeded/failed/cancelled。
```

## 4. 要做什么功能

### 1. 状态下拉展示

怎么做：
- 输入：固定状态映射。
- 输出：状态筛选选项。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：无。
- 限制：UI 显示中文，查询传内部英文状态。

### 2. 按状态查询任务

怎么做：
- 输入：status。
- 输出：匹配状态的任务列表。
- 写什么数据：不写。
- 状态怎么变：page=1，保留项目和分类筛选。
- 异常怎么处理：非法 status 返回参数错误。
- 限制：不能用中文状态直接查库。

### 3. 状态展示映射

怎么做：
- 输入：pending/running/succeeded/failed/cancelled。
- 输出：待执行/进行中/已完成/生成失败/已取消。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：未知状态显示“未知”并记录日志。
- 限制：状态颜色保持一致，失败红色、成功绿色、运行蓝色。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 内部状态不存中文。
  2. 增加 cancelled 状态。

原因：
  中文状态不利于跨模块统一；取消状态是 VT Studio 任务队列已有能力。

是否写入 04：
  cancelled 状态属于增强，写入 04。
```
