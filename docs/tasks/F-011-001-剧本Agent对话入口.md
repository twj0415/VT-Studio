## 0. 快速理解

```txt
一句话：做剧本 Agent 左侧聊天入口，能发送、流式接收、停止生成和显示连接状态。
为什么现在做：剧本 Agent 是从原文到剧本的核心生产入口。
做完后有什么用：用户可以和 Agent 对话，让它生成故事骨架、改编策略和剧本。
这一步不碰什么：不实现全部子 Agent，不写生产工作台，不直接让页面读数据库或 skill 文件。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow scriptAgent 页面左侧聊天入口和 Socket 流式消息语义。

不做：

  4. 生产 Agent
```

参考项目关键事实：

```txt

3. 参考项目用 Socket.IO；VT Studio 已在 CORE-008 确认继续使用 Socket.IO，不能再改成 IPC event。
4. stop 前端发送 messageId，后端实际 abort 当前 controller。

```

## 4. 要做什么功能

### 1. 初始化聊天会话

怎么做：
- 输入：projectId。
- 输出：会话连接和欢迎消息。
- 写什么数据：不写业务数据。
- 状态怎么变：disconnected -> connecting -> connected。
- 异常怎么处理：缺 projectId 或连接失败显示错误状态。
- 限制：隔离 key 必须是 `projectId:scriptAgent`。

### 2. 发送用户消息

怎么做：
- 输入：用户文本。
- 输出：user 消息和 assistant 流式消息。
- 写什么数据：后端可写 Agent 记忆。
- 状态怎么变：idle -> pending -> streaming -> complete/error。
- 异常怎么处理：空文本不发送；失败时 assistant 消息标 error。
- 限制：页面只调用 Agent 会话 API，不直接调用模型。

### 3. 接收流式消息

怎么做：
- 输入：Agent 事件。
- 输出：聊天消息内容。
- 写什么数据：XML 完成写入见后续任务，不在本任务完成。
- 状态怎么变：streaming 中持续追加内容。
- 异常怎么处理：事件格式非法时标记消息 error。
- 限制：必须支持 thinking、markdown、toolcall、error、stop 语义。

必须兼容的事件：

```txt
message：兼容旧的整条消息推送
message:update：更新整条消息状态
content:add：新增 thinking/markdown/toolcall/suggestion 内容块
content:update：更新内容块状态
error：服务端错误
connect/disconnect/connect_error：连接状态
```

### 4. 停止生成

怎么做：
- 输入：当前 messageId。
- 输出：停止事件。
- 写什么数据：可写 stopped 状态和记忆。
- 状态怎么变：streaming -> stop。
- 异常怎么处理：没有运行中消息时按钮禁用。
- 限制：停止当前生成，不清空历史消息。

### 5. 离开页面释放连接

怎么做：
- 输入：页面卸载。
- 输出：关闭会话监听。
- 写什么数据：不写。
- 状态怎么变：connected -> disconnected。
- 异常怎么处理：释放失败写日志。
- 限制：不能残留重复事件监听。
