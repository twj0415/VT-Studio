## 0. 快速理解

```txt
一句话：设置弹窗左侧列出 16 个设置模块，点击切换右侧内容。
为什么现在做：设置入口只是壳，菜单结构要先固定，后续 M-002 每个设置功能才有挂载位置。
做完后有什么用：所有设置模块都有统一入口和稳定 key。
这一步不碰什么：不实现每个设置模块的业务表单。
```

```txt
入口：
  打开设置弹窗后左侧设置菜单。

按钮/操作：
  点击外观、语言、模型供应商、模型映射、Agent、提示词、Skill、记忆、用户、数据库、文件、其他、请求、开发者、关于、退出登录。

弹窗/表单：
  设置弹窗内左侧菜单和右侧内容区。

成功反馈：
  activeMenu 切换，右侧对应模块显示。

失败反馈：
  模块未实现或加载失败时显示明确占位。
```

## 4. 要做什么功能

### 1. 设置菜单配置

怎么做：
- 输入：固定设置菜单列表。
- 输出：左侧菜单。
- 写什么数据：不写。
- 状态怎么变：弹窗打开后显示菜单。
- 异常怎么处理：配置缺失时显示错误占位。
- 限制：设置菜单固定，不从数据库动态生成。

### 2. activeMenu 切换

怎么做：
- 输入：点击菜单 key。
- 输出：当前 activeMenu。
- 写什么数据：写前端状态。
- 状态怎么变：右侧组件切换。
- 异常怎么处理：未知 key 回退 `ui`。
- 限制：关闭弹窗不重置 activeMenu。

### 3. about 更新红点

怎么做：
- 输入：`needUpdate` 和菜单 key。
- 输出：about 菜单显示红点。
- 写什么数据：不写。
- 状态怎么变：needUpdate true 时 about 显示 dot。
- 异常怎么处理：无。
- 限制：只有 about 显示更新红点。

### 4. 旧 key 修正

怎么做：
- 输入：参考项目 key。
- 输出：VT Studio 正式 key。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：无。
- 限制：`agentConfog` 改成 `agentConfig`，不能继承错字。
