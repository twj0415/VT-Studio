# F-008-007 文字节点编辑

状态：等待用户确认  
所属菜单：M-008 生产工作台  
对应功能文档：`docs/features/M-008-生产工作台.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：剧本、导演计划、分镜表三个文字节点可以打开 Markdown 编辑弹窗保存。
为什么现在做：Agent 生成内容需要人工修正，生产流程里的文本也要可编辑。
做完后有什么用：用户能手动调整剧本内容、导演计划和分镜表。
这一步不碰什么：不上传图片，不生成分镜图，不编辑视频轨道。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow script/scriptPlan/storyboardTable 节点编辑能力。

只做：
  1. script 节点编辑
  2. scriptPlan 节点编辑
  3. storyboardTable 节点编辑
  4. Markdown 编辑弹窗
  5. 保存后调用 saveFlowData

不做：
  1. 图片上传
  2. 分镜面板生成
  3. 视频工作台编辑
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/production/node/script.vue` | 剧本节点 Markdown 展示和编辑 |
| `Toonflow-web/src/views/production/node/scriptPlan.vue` | 导演计划节点 Markdown 展示和编辑 |
| `Toonflow-web/src/views/production/node/storyboardTable.vue` | 分镜表 Markdown 展示和编辑 |
| `Toonflow-app/src/routes/production/saveFlowData.ts` | 保存 flowData |

参考项目关键事实：

```txt
1. 编辑弹窗 90vw。
2. 这三个节点都用 Markdown 编辑器。
3. 粘贴/拖拽图片和视频会被阻止。
4. script 节点显示剧本内容，不显示空状态。
5. scriptPlan/storyboardTable 为空显示 t-empty。
```

## 3. 用户操作

```txt
入口：
  生产流程图对应文字节点。

按钮/操作：
  1. 点击编辑。
  2. 修改 Markdown。
  3. 点击保存。

弹窗/表单：
  Markdown 编辑弹窗。

成功反馈：
  节点内容更新并保存。

失败反馈：
  保存失败提示，保留编辑内容。
```

## 4. 要做什么功能

### 1. 编辑剧本文本节点

怎么做：
- 输入：flowData.script。
- 输出：更新后的 script。
- 写什么数据：保存 flowData；是否同步 scripts.content 按规则确认。
- 状态怎么变：编辑中 -> 保存中 -> 保存成功。
- 异常怎么处理：保存失败保留弹窗。
- 限制：不能上传媒体。

### 2. 编辑导演计划

怎么做：
- 输入：flowData.scriptPlan。
- 输出：更新后的 scriptPlan。
- 写什么数据：保存 flowData。
- 状态怎么变：编辑中 -> 保存成功。
- 异常怎么处理：保存失败提示。
- 限制：只改导演计划字段。

### 3. 编辑分镜表

怎么做：
- 输入：flowData.storyboardTable。
- 输出：更新后的 storyboardTable。
- 写什么数据：保存 flowData。
- 状态怎么变：编辑中 -> 保存成功。
- 异常怎么处理：保存失败提示。
- 限制：不自动生成分镜面板。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  script 节点是否同步 scripts.content 需要明确；建议同步真实剧本表或提示会影响剧本。

原因：
  参考项目 getFlowData 总是用最新 o_script.content，若只改 flowData.script 下次刷新会丢。

是否写入 04：
  如果同步 scripts.content，写入偏差；如果只编辑快照，必须提示刷新会覆盖。
```
