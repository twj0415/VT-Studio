状态：等待用户确认

原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：做应用左侧固定菜单，只放“项目”和“任务中心”这种全局入口。
为什么现在做：项目内菜单必须和全局菜单分开，否则后面页面会混乱。
做完后有什么用：无论有没有打开项目，都能进入项目列表和任务中心。
这一步不碰什么：不做小说、剧本、资产、生产这些项目内入口。
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/pages/workbench/index.vue` | 左侧全局菜单固定为项目、任务中心 |
| `Toonflow-web/src/router/index.ts` | 当前路由 path 决定菜单高亮 |

参考项目关键事实：

```txt
1. 左侧全局菜单不放小说、剧本、角景、生产、资产。
2. 项目内功能只有打开项目后才出现在顶部菜单。
3. 全局菜单项是固定配置，不从数据库读取。
4. 无项目状态下仍可进入项目和任务中心。
5. Toonflow 代码里有 `needProject` 判断，但当前左侧菜单没有使用。
```

失败反馈：

## 4. 要做什么功能

### 1. 全局菜单渲染

怎么做：
- 输入：固定菜单配置。
- 输出：左侧菜单。
- 写什么数据：不写。
- 状态怎么变：页面布局显示。
- 异常怎么处理：无。
- 限制：菜单项固定，不从数据库动态生成。

### 2. 项目入口

怎么做：
- 输入：点击项目。
- 输出：跳转 `/project`。
- 写什么数据：不写。
- 状态怎么变：路由切换，activeMenu 变为 `/project`。
- 异常怎么处理：跳转失败显示错误。
- 限制：不要求当前项目。

### 3. 任务中心入口

怎么做：
- 输入：点击任务中心。
- 输出：跳转 `/task`。
- 写什么数据：不写。
- 状态怎么变：路由切换，activeMenu 变为 `/task`。
- 异常怎么处理：跳转失败显示错误。
- 限制：任务中心必须能看全部项目任务，不依赖 currentProject。

### 4. 菜单高亮

怎么做：
- 输入：当前 route.path。
- 输出：active menu key。
- 写什么数据：不写。
- 状态怎么变：对应菜单高亮。
- 异常怎么处理：未知路由不高亮或回退项目。
- 限制：不能用 currentProject 判断全局菜单高亮。
