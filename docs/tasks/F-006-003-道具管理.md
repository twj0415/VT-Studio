# F-006-003 道具管理

状态：等待用户确认  
所属菜单：M-006 资产中心  
对应功能文档：`docs/features/M-006-资产中心.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：资产中心做道具 tab，管理道具资产的列表、新增、编辑、删除和图片状态。
为什么现在做：道具会参与剧本资产提取、分镜关联和图片生成。
做完后有什么用：用户能维护道具名称、描述、提示词、主图和衍生子道具。
这一步不碰什么：不做提示词批量生成，不做图片生成，不改生产页。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `type=tool` 的道具资产管理，同时统一 VT Studio 正式命名。

只做：
  1. 道具 tab
  2. 道具父子资产列表
  3. 新增道具
  4. 编辑道具
  5. 删除/批量删除道具
  6. 展示主图、提示词、描述、备注、创建时间

不做：
  1. 批量生成提示词
  2. 批量生成图片
  3. 单资产生成图片
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/assets/index.vue` | tool tab 展示为“道具” |
| `Toonflow-app/src/routes/assets/getAssetsApi.ts` | 按 type=tool 查询资产 |
| `Toonflow-app/src/routes/assets/addAssets.ts` | 新增 `type=tool` |
| `Toonflow-app/src/routes/assetsGenerate/generateAssets.ts` | tool 图片保存目录是 props |

参考项目关键事实：

```txt
1. 展示叫道具，源码 type 是 tool，图片路径有 props。
2. 视觉手册 key：父道具 art_prop，衍生道具 art_prop_derivative。
3. VT Studio 正式命名要统一，建议业务名 prop，道具兼容 Toonflow tool。
4. 删除风险同角色/场景。
```

## 3. 用户操作

```txt
入口：
  项目内顶部菜单 -> 资产 -> 道具 tab。

按钮/操作：
  新增、编辑、删除、批量删除、展开子道具、勾选。

弹窗/表单：
  新增/编辑道具弹窗；删除确认框。

成功反馈：
  保存或删除成功，刷新列表。

失败反馈：
  必填缺失、生成中禁止操作、删除失败。
```

## 4. 要做什么功能

### 1. 查询道具列表

怎么做：
- 输入：projectId、name、page、pageSize。
- 输出：道具父子资产树。
- 写什么数据：不写。
- 状态怎么变：loading -> loaded。
- 异常怎么处理：查询失败显示错误状态。
- 限制：兼容参考 type=tool，VT Studio 正式类型需统一。

### 2. 新增/编辑道具

怎么做：
- 输入：name、describe、remark、prompt。
- 输出：道具资产。
- 写什么数据：写或更新 assets。
- 状态怎么变：保存中 -> 列表刷新。
- 异常怎么处理：name/describe 缺失不能保存。
- 限制：不能混用 tool/props 做正式字段。

### 3. 删除道具

怎么做：
- 输入：assetIds。
- 输出：删除结果。
- 写什么数据：删除道具、子道具、媒体、关系和文件。
- 状态怎么变：删除中 -> 列表刷新。
- 异常怎么处理：生成中禁止删除；失败回滚。
- 限制：批量删除必须完整级联。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  正式命名建议用 prop，兼容导入/参考时识别 tool。

原因：
  Toonflow 里 tool/props 混用，长期会造成目录和类型混乱。

是否写入 04：
  写入，属于命名统一偏差。
```

## 8. 验收

```txt
1. 道具 tab 能查询当前项目道具。
2. 可新增道具。
3. 可编辑道具。
4. 可删除和批量删除道具。
5. 生成中道具不能删除。
6. 子道具能展开。
7. 删除清理关系和文件。
8. 页面不直接访问 SQLite 或文件系统。
```
