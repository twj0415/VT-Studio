# F-009-006 刷新任务

状态：等待用户确认  
所属菜单：M-009 任务中心  
对应功能文档：`docs/features/M-009-任务中心.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：任务中心加刷新按钮，手动重新拉取当前筛选下的任务。
为什么现在做：参考项目第一版没有自动轮询，手动刷新最稳。
做完后有什么用：用户能随时更新任务状态，并且不会丢失当前筛选和分页。
这一步不碰什么：不做自动轮询，不重复执行任务，不改任务状态。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 右上角刷新按钮。

只做：
  1. 刷新按钮
  2. 保留当前项目/分类/状态筛选
  3. 保留当前分页
  4. 刷新 loading 和错误提示

不做：
  1. 自动轮询
  2. WebSocket 推送
  3. 重试失败任务
  4. 重置筛选
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/task/index.vue` | 右上角刷新按钮直接调用 `getTaskList` |
| `Toonflow-app/src/routes/task/getTaskApi.ts` | 按当前参数重新查任务 |

参考项目关键事实：

```txt
1. 刷新按钮不重置 page。
2. 刷新保留 projectId、taskClass、taskState、page、limit。
3. 参考项目没看到任务中心自动轮询。
```

## 3. 用户操作

```txt
入口：

  点击刷新。

弹窗/表单：
  无。

成功反馈：
  表格数据更新。

失败反馈：
  显示刷新失败提示，保留旧列表。
```

## 4. 要做什么功能

### 1. 手动刷新列表

怎么做：
- 输入：当前筛选和分页状态。
- 输出：最新任务列表。
- 写什么数据：不写。
- 状态怎么变：refreshing=true -> false。
- 异常怎么处理：失败提示，不清空旧数据。
- 限制：刷新不会执行任务。

### 2. 保留筛选和分页

怎么做：
- 输入：projectId、category、status、page、pageSize。
- 输出：相同条件下的新数据。
- 写什么数据：不写。
- 状态怎么变：筛选状态不变。
- 异常怎么处理：如果当前页超出 total，可回退最后一页并重新查。
- 限制：不能像筛选变化一样强制回到第一页。

```txt
  刷新失败时保留旧数据并明确提示。

原因：

```
