# F-011-003 清理 Agent 记忆

状态：等待用户确认  
所属菜单：M-011 剧本 Agent  
对应功能文档：`docs/features/M-011-剧本Agent.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：剧本 Agent 设置菜单里支持清理 message、summary 或全部记忆。
为什么现在做：记忆错了会持续影响 Agent 判断，必须能清。
做完后有什么用：用户能重置当前项目剧本 Agent 的历史上下文。
这一步不碰什么：不删除剧本数据，不删除项目，不清生产 Agent 记忆。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/agents/clearMemory` 清理能力。

只做：
  1. 清 message 记忆
  2. 清 summary 记忆
  3. 清全部记忆
  4. 二次确认
  5. 清理后刷新历史消息

不做：
  1. 清其他 Agent 记忆
  2. 删除工作区计划数据
  3. 删除剧本表
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |

参考项目关键事实：

```txt

2. 清理前有确认框。

4. handleClearMemory 类型里有 reconnect，但实际不是记忆类型。
```


## 4. 要做什么功能

### 1. 清 message 记忆

怎么做：
- 输入：projectId、type=message。
- 输出：删除结果。
- 写什么数据：删除当前 isolationKey 下 message 记忆。
- 状态怎么变：确认 -> 清理中 -> 刷新历史。
- 异常怎么处理：删除失败提示。
- 限制：不删 summary。

### 2. 清 summary 记忆

怎么做：
- 输入：projectId、type=summary。
- 输出：删除结果。
- 写什么数据：删除当前 isolationKey 下 summary 记忆。
- 状态怎么变：确认 -> 清理中 -> 刷新历史。
- 异常怎么处理：删除失败提示。
- 限制：不删 message。

### 3. 清全部记忆

怎么做：
- 输入：projectId、type=all。
- 输出：删除结果。
- 写什么数据：删除当前 isolationKey 下所有记忆。
- 状态怎么变：确认 -> 清理中 -> 只保留欢迎消息。
- 异常怎么处理：删除失败提示。
- 限制：只清 scriptAgent，不清 productionAgent。
