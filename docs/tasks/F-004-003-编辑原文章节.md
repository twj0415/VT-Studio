# F-004-003 编辑原文章节

状态：等待用户确认  
所属菜单：M-004 小说/原文  
对应功能文档：`docs/features/M-004-小说原文.md`

## 0. 快速理解

```txt
一句话：用户可以编辑某一章的章节名、正文和事件摘要。
为什么现在做：导入解析可能不准，原文错误会影响事件分析和剧本 Agent。
做完后有什么用：用户能修正章节内容，并让后续生成基于正确文本。
这一步不碰什么：不做批量编辑，不做自动重写剧本，不做旧 editNodel 命名。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `editNodel.vue` 和 `/novel/updateNovel`，同时修正旧拼写。

只做：
  1. 编辑章节按钮
  2. 编辑弹窗
  3. 回填章节名、正文、事件摘要
  4. 保存章节
  5. 正文变更后的事件状态处理规则

不做：
  1. 批量编辑
  2. 自动重新生成事件
  3. 自动更新已生成剧本
```

## 3. 用户操作

```txt
入口：
  原文列表 -> 某行编辑按钮。

按钮/操作：
  1. 点击编辑。
  2. 修改章节名、正文、事件摘要。
  3. 点击保存。

弹窗/表单：
  编辑章节弹窗。

成功反馈：
  保存成功，关闭弹窗，刷新列表。

失败反馈：
  必填缺失、保存失败、章节正在分析不可编辑。
```

## 4. 要做什么功能

### 1. 打开编辑弹窗

怎么做：
- 输入：章节 id。
- 输出：章节表单。
- 写什么数据：不写。
- 状态怎么变：dialogOpen=true，form 回填。
- 异常怎么处理：章节不存在时提示并刷新列表。
- 限制：事件分析中的章节不能编辑，或必须先取消/等待完成。

### 2. 保存章节修改

怎么做：
- 输入：章节名、正文、事件摘要。
- 输出：更新后的章节。
- 写什么数据：更新章节表。
- 状态怎么变：保存中 -> 成功关闭 -> 刷新列表。
- 异常怎么处理：正文为空不能保存；数据库失败回滚。
- 限制：只允许更新当前项目下章节。

### 3. 正文变更后的事件状态处理

怎么做：
- 输入：原正文、新正文。
- 输出：事件状态。
- 写什么数据：如果正文变更，标记事件摘要需要重新分析。
- 状态怎么变：event_status 变为 stale 或清空事件状态，按最终枚举定。
- 异常怎么处理：状态更新失败时整体保存失败。
- 限制：不能继续把旧 event 当作新正文的有效摘要。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 不保留 editNodel 拼写。
  2. 正文变更后标记事件摘要需要重新分析。

原因：
  旧摘要基于旧正文，继续使用会误导剧本 Agent。

是否写入 04：
  正文变更导致事件摘要失效是合理修正，可写入偏差记录。
```
