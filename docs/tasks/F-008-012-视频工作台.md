## 0. 快速理解

```txt
一句话：生产页工作台节点进入视频工作台，管理分镜对应的视频轨道和候选视频。
为什么现在做：后续视频提示词、视频生成和剪映导出都依赖稳定轨道数据。
做完后有什么用：用户能看到每条轨道、分镜素材、已选视频和生成入口。
这一步不碰什么：不做剪映导出，不做内置视频编辑器完整闭环，不生成提示词。
```

## 1. 本次做什么

```txt
目标：

只做：
  1. 视频工作台入口
  2. 获取视频工作台数据
  3. 展示轨道列表
  4. 展示分镜、资产、绑定音频参考
  5. 展示候选视频和已选视频
  6. 快速预览/视频生成/视频编辑器三个模式入口

不做：
  1. 生成视频提示词
  2. 生成视频候选
  3. 剪映导出
  4. WebAV mp4 导出
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/production/node/workbench.vue` | 工作台节点入口 |
| `Toonflow-web/src/views/production/components/workbench/*` | 快速预览、视频生成、视频编辑器模式 |
| `Toonflow-app/src/routes/production/workbench/getGenerateData.ts` | 获取项目视频模型、分镜、轨道、候选视频、资产音频绑定 |
| `Toonflow-app/src/routes/production/workbench/getAudioBindAssetsList.ts` | 查询绑定音频资产 |

参考项目关键事实：

```txt
1. workbench 有快速预览、视频生成、视频编辑器三个模式。
2. getGenerateData 读取 o_project、o_storyboard、o_videoTrack、o_video、o_assets 等。
3. 没配视频模型时参考项目返回 200 success 文案，VT Studio 应返回明确业务错误。
4. 视频编辑器到剪映导出的完整闭环不在 M-008。
```

```txt

  1. 打开视频工作台。
```
