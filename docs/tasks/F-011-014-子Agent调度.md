## 0. 快速理解

```txt
一句话：剧本 Agent 不是单轮聊天，要有决策层、执行层和监督层子 Agent。
为什么现在做：参考项目的专业性在多 Agent 调度和工具调用，不能简化成普通聊天。
做完后有什么用：Agent 能按任务选择生成骨架、策略、剧本或做监督检查。
这一步不碰什么：不做生产 Agent，不做视频生成，不做资产中心。
```

## 3. 用户操作

```txt
入口：
  用户在剧本 Agent 聊天框发送任务。

按钮/操作：
  1. 发送需求。
  2. 可停止当前生成。
  3. 可查看工具调用和子 Agent 输出。

弹窗/表单：
  无。

成功反馈：
  Agent 输出聊天内容，并把结构化结果写入右侧工作区。

失败反馈：
  模型失败、工具失败、XML 非法时显示 error，并记录失败原因。
```

## 4. 要做什么功能

### 1. 决策层 Agent

怎么做：
- 输入：用户消息、项目资料、记忆、可用工具。
- 输出：决策回复或子 Agent 调用。
- 写什么数据：写 message 记忆。
- 状态怎么变：Agent 会话 streaming。
- 异常怎么处理：模型失败写 error 消息。
- 限制：必须使用 script_agent_decision skill，不做普通单 prompt 聊天。

### 2. 故事骨架子 Agent

怎么做：
- 输入：决策层任务、原文/事件/计划上下文。
- 输出：`<storySkeleton>` XML。
- 写什么数据：XML 完成后写故事骨架。
- 状态怎么变：子消息 streaming -> complete。
- 异常怎么处理：输出不含 storySkeleton 时标记失败。
- 限制：模型 key 来自 Agent 模型配置。

### 3. 改编策略子 Agent

怎么做：
- 输入：故事骨架、原文事件、用户目标。
- 输出：`<adaptationStrategy>` XML。
- 写什么数据：XML 完成后写改编策略。
- 状态怎么变：子消息 streaming -> complete。
- 异常怎么处理：输出格式错误不写入。
- 限制：不能把策略写进故事骨架字段。

### 4. 剧本生成子 Agent

怎么做：
- 输入：故事骨架、改编策略、已有剧本、原文上下文。
- 输出：一个或多个 `<scriptItem>` XML。
- 写什么数据：写 scripts 表和 agent_work_data。
- 状态怎么变：子消息 streaming -> complete。
- 异常怎么处理：scriptItem name/content 缺失时拒绝写入。
- 限制：不按 name 覆盖已有剧本。

### 5. 监督层 Agent

怎么做：
- 输入：生成结果和目标要求。
- 输出：监督意见或修正建议。
- 写什么数据：写监督消息记忆，不直接覆盖业务数据。
- 状态怎么变：supervision streaming -> complete。
- 异常怎么处理：失败不回滚已保存结果，但提示监督失败。
- 限制：监督层不能绕过 XML 写入规则直接改表。

### 6. 任务中心记录

怎么做：
- 输入：一次 Agent 生成请求。
- 输出：任务记录。
- 写什么数据：写 tasks。
- 状态怎么变：running -> succeeded/failed/cancelled。
- 异常怎么处理：停止时写 cancelled 或 stopped 状态映射。
- 限制：这是 VT Studio 增强，需写 04。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. Agent 生成建议写任务中心。
  2. XML 非法时不写入业务数据。
  3. 不按 name 覆盖 scriptItem。

原因：
  长耗时 Agent 需要可追踪；结构化写入必须保护业务数据。

是否写入 04：
  写入。
```

## 8. 验收

```txt
1. 用户发送消息能进入决策层。
2. 决策层能调用上下文工具。
3. 能调用故事骨架子 Agent。
4. 能调用改编策略子 Agent。
5. 能调用剧本生成子 Agent。
6. 能调用监督层。
7. 子 Agent 输出 XML 合法才写入工作区。
8. 停止生成能中断当前请求。
9. 失败有错误消息和任务失败原因。
10. 页面不直接读取 skill 文件或调用模型 SDK。
```
