# F-006-006 素材预览

状态：等待用户确认  
所属菜单：M-006 资产中心  
对应功能文档：`docs/features/M-006-资产中心.md`

## 0. 快速理解

```txt
一句话：资产中心所有图片、视频、音频都必须能安全预览。
为什么现在做：资产看不到就无法判断是否可用，后续生产也会混乱。
做完后有什么用：用户能查看图片大图、播放视频和音频，并看到文件缺失状态。
这一步不碰什么：不生成素材，不编辑视频，不直接暴露本地路径。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 图片/视频/音频预览能力，同时改成主进程受控文件 URL。

只做：
  1. 图片缩略图和大图预览
  2. 视频预览弹窗
  3. 音频预览播放器
  4. 文件缺失状态
  5. 受控文件 URL

不做：
  1. 素材生成
  2. 视频剪辑
  3. 文件管理器打开目录
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/assets/index.vue` | 图片用 image viewer，视频/audio 打开媒体预览 |
| `Toonflow-app/src/routes/assets/getAssetsApi.ts` | role/tool/scene 用 smallImageUrl，clip/audio 用 fileUrl |
| `Toonflow-app/src/routes/assets/getMaterialData.ts` | 按 filePath 转 URL |

参考项目关键事实：

```txt
1. 图片、视频、音频按扩展名判断媒体类型。
2. 图片大图可查看。
3. 视频和音频走媒体预览弹窗。
4. filePath 是内部路径，不是最终展示 URL。
5. VT Studio 必须由 main 服务生成受控 URL。
```

## 3. 用户操作

```txt
入口：
  资产中心各 tab 的预览列、素材卡片、媒体弹窗。

按钮/操作：
  1. 点击图片查看大图。
  2. 点击视频播放。
  3. 点击音频播放。

弹窗/表单：
  图片 viewer；媒体预览弹窗。

成功反馈：
  媒体正常显示或播放。

失败反馈：
  文件不存在、格式不支持、加载失败。
```

## 4. 要做什么功能

### 1. 生成受控预览 URL

怎么做：
- 输入：mediaId 或业务对象 id。
- 输出：短期可用预览 URL。
- 写什么数据：不写。
- 状态怎么变：加载中 -> 可预览。
- 异常怎么处理：文件不存在返回缺失状态。
- 限制：renderer 不接触裸 filePath。

### 2. 图片预览

怎么做：
- 输入：图片 URL。
- 输出：缩略图和大图。
- 写什么数据：不写。
- 状态怎么变：加载成功/失败。
- 异常怎么处理：加载失败显示占位。
- 限制：生成中状态显示 loading，不尝试加载空图。

### 3. 视频/音频预览

怎么做：
- 输入：媒体 URL。
- 输出：video/audio 播放器。
- 写什么数据：不写。
- 状态怎么变：播放/暂停。
- 异常怎么处理：格式不支持或文件缺失显示错误。
- 限制：不编辑媒体内容。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  使用 main 侧受控预览 URL，不直接把 filePath 给页面。

原因：
  防止任意本地路径暴露，统一文件权限边界。

是否写入 04：
  如本地文件层已记录，不重复。
```
