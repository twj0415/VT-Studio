# F-011-006 获取计划数据

状态：等待用户确认  
所属菜单：M-011 剧本 Agent  
对应功能文档：`docs/features/M-011-剧本Agent.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：读取右侧工作区的故事骨架、改编策略和剧本列表。
为什么现在做：Agent 对话产出的内容要落到右侧工作区，不只是聊天文本。
做完后有什么用：用户打开页面能看到已有计划和剧本卡片。
这一步不碰什么：不编辑内容，不写 XML，不删除剧本。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/scriptAgent/getPlanData`，同时修正 schema 不一致问题。

只做：
  1. 读取故事骨架
  2. 读取改编策略
  3. 读取真实剧本列表
  4. 首次进入创建空工作区数据
  5. 统一 planData 类型

不做：
  1. 编辑故事骨架
  2. 编辑改编策略
  3. 编辑剧本卡片
  4. XML 写入
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/scriptAgent/index.vue` | 进入页面调用 `/scriptAgent/getPlanData` |
| `Toonflow-web/src/stores/scriptAgent.ts` | planData 包含 storySkeleton、adaptationStrategy、script 数组 |
| `Toonflow-app/src/routes/scriptAgent/getPlanData.ts` | 查询 `o_agentWorkData key=scriptAgent`，同时读取 `o_script` 拼 script |
| `Toonflow-app/src/lib/initDB.ts` | `o_agentWorkData` 保存工作区 data |

参考项目关键事实：

```txt
1. 首次没有 o_agentWorkData 时创建空 storySkeleton 和 adaptationStrategy。
2. 剧本列表不是只从 agentWorkData 读取，而是从 o_script 查出来拼上。
3. getPlanData 返回 storySkeleton、adaptationStrategy、script。
4. 参考项目 schema 有冲突：setPlanData schema 没声明 script，但代码又使用 script。
```

## 4. 要做什么功能

### 1. 读取工作区数据

怎么做：
- 输入：projectId。
- 输出：storySkeleton、adaptationStrategy。
- 写什么数据：首次没有时创建空工作区记录。
- 状态怎么变：loading=true -> false。
- 异常怎么处理：JSON 损坏时返回错误并提示修复。
- 限制：key 固定为 scriptAgent。

### 2. 读取真实剧本列表

怎么做：
- 输入：projectId。
- 输出：scripts 数组。
- 写什么数据：不写。
- 状态怎么变：planData.scripts 更新。
- 异常怎么处理：读取失败时右侧剧本 tab 显示错误。
- 限制：剧本事实以 scripts 表为准，不只信 agentWorkData 快照。

### 3. 统一 planData 类型

怎么做：
- 输入：工作区 data 和 scripts 表。
- 输出：统一 DTO。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：缺字段补默认值。
- 限制：前端、服务、Agent 工具共用同一类型。
