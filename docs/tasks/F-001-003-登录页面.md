## 0. 快速理解

```txt
一句话：做本地登录页，输入用户名密码，登录成功后写本地 session 并进入项目页。
为什么现在做：路由守卫需要一个明确入口，退出登录后也需要回到这里。
做完后有什么用：本地用户状态闭环，后面项目、任务、设置可以读取 userId。
这一步不碰什么：不做注册、不做云端账号、不做第三方登录。
```

## 3. 用户操作

```txt
入口：
  访问 `/login`，或路由守卫跳转到登录页。

按钮/操作：
  1. 输入用户名。
  2. 输入密码。
  3. 可切换语言。
  4. 可修改请求地址。
  5. 点击登录。

弹窗/表单：
  登录表单。

成功反馈：
  提示登录成功，进入项目页。

失败反馈：
  用户名密码为空、用户不存在、密码错误、本地服务不可用都要提示。
```

## 4. 要做什么功能

### 1. 登录表单

怎么做：
- 输入：用户名、密码。
- 输出：待提交表单数据。
- 写什么数据：提交前不写。
- 状态怎么变：输入态、提交中、提交完成。
- 异常怎么处理：空字段直接提示。
- 限制：密码输入框必须隐藏明文。

### 2. 请求地址配置入口

怎么做：
- 输入：用户修改 baseUrl。
- 输出：更新前端请求地址。
- 写什么数据：写前端设置持久化。
- 状态怎么变：baseUrl 更新。
- 异常怎么处理：地址格式明显错误时提示。
- 限制：这是登录页快捷入口，完整请求地址设置归 F-002-013。

### 3. 语言切换

怎么做：
- 输入：用户选择语言。
- 输出：i18n 语言切换。
- 写什么数据：写前端语言设置。
- 状态怎么变：登录页文案立即切换。
- 异常怎么处理：未知语言回退默认中文。
- 限制：完整语言设置归 F-002-002。

### 4. 登录提交

怎么做：
- 输入：用户名、密码。
- 输出：登录结果 `{ code, data, msg }`。
- 写什么数据：成功后写 token、userId、userName。
- 状态怎么变：提交中 -> 成功/失败。
- 异常怎么处理：失败显示 `msg`，网络或 IPC 异常显示可读错误。
- 限制：返回格式必须用 VT Studio 统一 `{ code, data, msg }`。

### 5. 登录成功跳转

怎么做：
- 输入：登录成功数据。
- 输出：跳转 `/project`。
- 写什么数据：写本地 session。
- 状态怎么变：登录页离开，进入项目列表。
- 异常怎么处理：跳转失败时保留 session 并提示。
- 限制：不能跳到旧 currentProject 页面。
