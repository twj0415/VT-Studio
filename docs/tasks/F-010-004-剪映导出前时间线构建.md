# F-010-004 剪映导出前时间线构建

状态：等待用户确认  
所属菜单：M-010 导出  
对应功能文档：`docs/features/M-010-导出.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：剪映导出前先从稳定数据构建 timeline，不能临时猜。
为什么现在做：剪映草稿导出是 VT Studio 增强能力，必须有确定数据源。
做完后有什么用：导出服务能知道每个 clip 来自哪条轨道、哪个视频、时长是多少。
这一步不碰什么：不写剪映草稿文件，不校验素材，不复制素材。
```

## 1. 本次做什么

```txt
目标：
  为剪映草稿导出构建统一 timeline，第一版建议读取 M-008 视频工作台已选视频轨道。

只做：
  1. 读取当前项目和剧本
  2. 读取视频工作台轨道
  3. 读取每条轨道 selected_video_id
  4. 构建统一 timeline clips
  5. 计算开始时间和时长
  6. 记录来源对象

不做：
  1. 素材校验
  2. 写剪映草稿
  3. WebAV 时间线导出
  4. 复制素材
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-app/src/routes/production/workbench/getGenerateData.ts` | 读取视频轨道、分镜、候选视频、资产 |
| `Toonflow-web/src/views/production/components/workbench/editVideo/*` | 另有前端临时时间线，但没有后端保存闭环 |
| `docs/features/M-010-导出.md` | 明确剪映导出是 VT Studio 增强，建议第一版读取 M-008 视频工作台已选轨道 |

参考项目关键事实：

```txt
1. Toonflow 没有剪映草稿导出实现。
2. 内置视频编辑器时间线没有找到后端保存接口。
3. 视频工作台轨道和候选视频存在数据库里，更适合作为第一版导出依据。
4. M-010 必须读取已选择候选视频，不随机取第一个候选。
```

## 3. 用户操作

```txt
入口：
  视频工作台 -> 剪映草稿导出。

按钮/操作：
  点击导出前，系统自动构建 timeline。

弹窗/表单：
  导出配置弹窗可在 F-010-006 做。

成功反馈：
  构建出可校验的 timeline。

失败反馈：
  没有轨道、轨道没有已选视频、时长非法。
```

## 4. 要做什么功能

### 1. 读取导出数据源

怎么做：
- 输入：projectId、scriptId。
- 输出：videoTracks、selectedVideos、storyboards。
- 写什么数据：不写。
- 状态怎么变：building -> loaded。
- 异常怎么处理：没有轨道或没有已选视频返回明确错误。
- 限制：第一版只用视频工作台数据源。

### 2. 构建 timeline clips

怎么做：
- 输入：轨道和 selected video。
- 输出：标准 timeline clips。
- 写什么数据：不写。
- 状态怎么变：timeline ready。
- 异常怎么处理：duration 缺失时用轨道 duration 或视频元数据兜底。
- 限制：每个 clip 必须记录来源类型和来源 id。

### 3. 计算时间轴

怎么做：
- 输入：clips 顺序和 duration。
- 输出：start、duration、end。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：duration <= 0 直接失败。
- 限制：统一时间单位，内部建议毫秒或微秒明确记录。

### 4. 输出导出 DTO

怎么做：
- 输入：timeline。
- 输出：JianyingExportTimeline。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：字段缺失返回结构化错误。
- 限制：不包含临时 Vue 状态。
