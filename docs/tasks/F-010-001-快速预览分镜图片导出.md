# F-010-001 快速预览分镜图片导出

状态：等待用户确认  
所属菜单：M-010 导出  
对应功能文档：`docs/features/M-010-导出.md`

## 0. 快速理解

```txt
一句话：快速预览里勾选分镜图片，按当前顺序打包导出 zip。
为什么现在做：这是参考项目已有的轻量导出能力，和视频/剪映导出不是一回事。
做完后有什么用：用户能把选中的分镜图快速导出给别人看或备份。
这一步不碰什么：不导出剧本 txt，不合成 mp4，不生成剪映草稿。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 快速预览里的分镜图片 zip 导出。

只做：
  1. 快速预览入口
  2. 读取当前剧本分镜图片
  3. 分镜播放预览
  4. 勾选/全选分镜
  5. 拖拽调整导出顺序
  6. 还原排序
  7. 导出选中图片 zip
  8. 导出失败列表提示

不做：
  1. WebAV mp4 导出
  2. 剪映草稿导出
  3. 剧本 zip 导出
  4. 保存拖拽排序到数据库
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/production/components/workbench/preview.vue` | 快速预览、播放控制、分镜勾选、拖拽排序、导出图片 zip |
| `Toonflow-app/src/routes/production/getStoryboardData.ts` | 按 scriptId/projectId 读取分镜、关联资产和图片 |

参考项目关键事实：

```txt
1. 数据来自 `/production/getStoryboardData`。
2. 分镜按 o_storyboard.index asc 排序。
3. duration 为空时前端按 3 秒播放。
4. 导出只导出 selected=true 的分镜图片。
5. 前端用 JSZip fetch 图片 URL 打包，文件名是 分镜{id}.{ext}。
6. 拖拽排序只影响当前预览和导出顺序，不写数据库。
7. 导出失败只 console.error，VT Studio 必须把失败分镜列出来。
```

## 3. 用户操作

```txt
入口：
  生产工作台 -> 视频工作台 -> 快速预览。

按钮/操作：
  1. 上一张、播放/暂停、下一张。
  2. 点击进度条跳转。
  3. 勾选分镜。
  4. 全选。
  5. 拖拽调整顺序。
  6. 还原排序。
  7. 点击导出图片。

弹窗/表单：
  未选择提示框；还原排序确认框；导出失败详情。

成功反馈：
  zip 导出成功，显示保存路径或下载完成。

失败反馈：
  未选择、图片缺失、文件读取失败、zip 写入失败。
```

## 4. 要做什么功能

### 1. 读取分镜预览数据

怎么做：
- 输入：projectId、scriptId。
- 输出：shotList。
- 写什么数据：不写。
- 状态怎么变：loading -> loaded。
- 异常怎么处理：读取失败显示错误空态。
- 限制：只读当前剧本分镜。

### 2. 预览播放

怎么做：
- 输入：shotList、duration。
- 输出：当前分镜和播放进度。
- 写什么数据：不写。
- 状态怎么变：playing/paused。
- 异常怎么处理：没有图片时显示缺失状态但仍可浏览。
- 限制：duration 为空默认 3 秒。

### 3. 勾选和排序

怎么做：
- 输入：用户选择和拖拽顺序。
- 输出：导出列表顺序。
- 写什么数据：不写数据库。
- 状态怎么变：selected/order 更新。
- 异常怎么处理：还原排序失败回到接口初始顺序。
- 限制：拖拽排序只影响本次导出。

### 4. 导出图片 zip

怎么做：
- 输入：选中分镜图片列表。
- 输出：zip 文件。
- 写什么数据：写导出文件或返回 blob。
- 状态怎么变：exporting -> succeeded/failed。
- 异常怎么处理：逐张记录失败，最后展示失败列表。
- 限制：图片路径必须通过受控 URL 或 main 文件服务读取。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  导出失败要列出失败分镜和原因。

原因：
  只 console.error 用户无法知道哪些图没导出。

是否写入 04：
  不需要，属于错误提示增强。
```
