# F-011-005 原文事件完成检查

状态：等待用户确认  
所属菜单：M-011 剧本 Agent  
对应功能文档：`docs/features/M-011-剧本Agent.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：进入剧本 Agent 时检查原文事件是否还有生成中的章节。
为什么现在做：剧本 Agent 依赖事件摘要，事件没完成时生成质量会不稳定。
做完后有什么用：用户能知道当前原文事件还没准备好，可以选择继续或回去处理。
这一步不碰什么：不自动触发事件分析，不阻塞所有操作，不做旧“强制生成”假按钮。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 原文事件未完成提示，同时修正“蒙层按钮只关闭不生成”的语义问题。

只做：
  1. 进入页面读取原文列表
  2. 检查 event_status=running 的章节
  3. 显示事件未完成提示
  4. 允许用户关闭提示继续
  5. 提供去原文页处理的入口建议

不做：
  1. 自动重新生成事件
  2. 强制生成按钮
  3. 剧本生成本身
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/scriptAgent/index.vue` | 调 `/novel/getNovelData`，存在 `eventState === 0` 时显示 forceGenerateMask |
| `Toonflow-app/src/routes/novel/getNovelData.ts` | 返回当前项目全部章节 |

参考项目关键事实：

```txt
1. 只要存在 eventState=0，就显示提示蒙层。
2. 蒙层按钮只关闭蒙层，没有触发事件分析。
3. 名称叫 forceGenerateMask，但实际没有强制生成逻辑。
4. VT Studio 不能做成“点击强制生成”但背后什么都不做。
```

## 3. 用户操作

```txt
入口：
  打开剧本 Agent 页面自动检查。

按钮/操作：
  1. 关闭提示继续。
  2. 可跳转到原文页查看事件状态。

弹窗/表单：
  事件未完成提示蒙层或提示条。

成功反馈：
  没有生成中事件时不提示。

失败反馈：
  读取原文失败时显示轻量错误，不阻止聊天入口加载。
```

## 4. 要做什么功能

### 1. 读取原文事件状态

怎么做：
- 输入：projectId。
- 输出：章节事件状态统计。
- 写什么数据：不写。
- 状态怎么变：checking=true -> false。
- 异常怎么处理：读取失败提示，允许继续进入页面。
- 限制：只检查当前项目。

### 2. 显示未完成提示

怎么做：
- 输入：runningCount。
- 输出：提示用户还有事件分析未完成。
- 写什么数据：不写。
- 状态怎么变：showEventWarning=true。
- 异常怎么处理：无。
- 限制：不能说“强制生成”却不执行生成。

### 3. 关闭或跳转

怎么做：
- 输入：用户点击继续或去原文页。
- 输出：关闭提示或路由跳转。
- 写什么数据：不写。
- 状态怎么变：showEventWarning=false。
- 异常怎么处理：跳转失败提示。
- 限制：继续生成要明确风险。
