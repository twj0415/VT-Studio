# F-008-005 保存流程数据

状态：等待用户确认  
所属菜单：M-008 生产工作台  
对应功能文档：`docs/features/M-008-生产工作台.md`

## 0. 快速理解

```txt
一句话：把生产工作区快照保存到数据库，同时同步分镜排序。
为什么现在做：Agent 和用户编辑后的生产数据需要保存，刷新后不能丢。
做完后有什么用：导演计划、分镜表、流程快照和分镜顺序可以持久化。
这一步不碰什么：不生成图片，不生成视频，不导出剪映。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/production/saveFlowData`，并区分快照和业务事实。

只做：
  1. 保存 flowData 快照
  2. 创建或更新 production agentWorkData
  3. 同步 storyboard.index
  4. 防止前端临时字段污染业务表
  5. 保存失败提示

不做：
  1. 保存视频编辑器临时时间线
  2. 生成分镜
  3. 生成视频
  4. 剪映导出
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-app/src/routes/production/saveFlowData.ts` | 按 projectId + episodesId + key 保存 `o_agentWorkData.data` |
| `Toonflow-web/src/stores/productionAgent.ts` | setFlowData 保存工作区 |
| `Toonflow-web/src/views/production/index.vue` | XML 完成或用户编辑后触发保存 |

参考项目关键事实：

```txt
1. key=productionAgent。
2. 如果 data.storyboard 全都有真实 id，会按数组顺序更新 o_storyboard.index。
3. 没有 o_agentWorkData 时插入，有则更新。
4. flowData 是工作区快照，真实资产/分镜/视频仍以业务表为准。
5. Agent schema 不包含 workbench，但前端 flowData 有 workbench。
```

## 3. 用户操作

```txt
入口：
  文字节点保存、Agent XML 完成、流程保存触发。

按钮/操作：
  1. 编辑节点后保存。
  2. Agent 生成完成自动保存。

弹窗/表单：
  由具体编辑任务提供。

成功反馈：
  保存成功或静默保存状态。

失败反馈：
  保存失败提示，不能显示为已保存。
```

## 4. 要做什么功能

### 1. 保存 flowData 快照

怎么做：
- 输入：projectId、scriptId、flowData。
- 输出：保存结果。
- 写什么数据：agent_work_data key=productionAgent。
- 状态怎么变：saving -> saved。
- 异常怎么处理：JSON 序列化失败或数据库失败提示。
- 限制：只能保存允许字段。

### 2. 同步分镜排序

怎么做：
- 输入：flowData.storyboard。
- 输出：storyboards.index 更新。
- 写什么数据：按当前数组顺序更新真实分镜 index。
- 状态怎么变：事务内完成。
- 异常怎么处理：存在无 id 分镜时不做排序同步或先持久化。
- 限制：只更新 index，不覆盖分镜图片、prompt、videoDesc。

### 3. 过滤临时字段

怎么做：
- 输入：前端 flowData。
- 输出：可保存 DTO。
- 写什么数据：过滤后写入。
- 状态怎么变：无。
- 异常怎么处理：非法字段记录日志。
- 限制：workbench 前端字段和 Agent schema 分开建模。
