# F-002-008 记忆配置

状态：已完成  
所属菜单：M-002 设置  
对应功能文档：`docs/features/M-002-设置.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：在开发者模式里设置 Agent 记忆和 embedding 参数，比如 ONNX 模型路径、摘要阈值、RAG 数量，并提供受控清空记忆入口。
为什么现在做：CORE-011 记忆层需要这些参数，Agent 后续检索历史和摘要都依赖它。
做完后有什么用：高级用户能调整记忆长度、摘要策略、RAG 返回数量，也能安全清理指定范围记忆。
这一步不碰什么：不做 Agent 对话，不做 embedding 模型下载，不做全库危险清空默认入口。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow memoryConfig，但把清空范围和模型路径校验做清楚。

只做：
  1. 查询记忆配置
  2. 保存记忆配置
  3. 恢复默认配置
  4. 清空记忆
  5. 参数范围校验
  6. ONNX 模型路径存在性校验
  7. 记忆统计
  8. 清空全部记忆的强确认短语

不做：
  1. ONNX 模型下载器
  2. Agent 页面接入
  3. Skill embedding 重建进度
  4. 云端向量库
  5. 自动下载/安装 ONNX 模型
  6. 普通用户主入口
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/components/setting/components/memoryConfig.vue` | 记忆配置表单、保存、清空记忆、恢复默认 |
| `Toonflow-app/src/routes/setting/memoryConfig/getMemory.ts` | 读取 modelOnnxFile/modelDtype/messagesPerSummary 等配置 |
| `Toonflow-app/src/routes/setting/memoryConfig/sureMemory.ts` | 保存记忆配置到 o_setting |
| `Toonflow-app/src/routes/setting/memoryConfig/delAllMemory.ts` | 清空 memories |

参考项目关键事实：

```txt
1. modelOnnxFile 默认 ["all-MiniLM-L6-v2","onnx","model_fp16.onnx"]。
2. modelDtype 默认 fp16。
3. messagesPerSummary、shortTermLimit、summaryMaxLength、summaryLimit、ragLimit、deepRetrieveSummaryLimit 都是可配置数字。
4. delAllMemory 是直接清空 memories，VT Studio 必须增加范围和确认。
```

## 3. 用户操作

```txt
入口：
  设置 -> 开发者模式 -> 记忆配置。

定位：
  高级能力，不放普通用户主入口。
  原因：参数会影响 Agent 召回质量，清空记忆会删除用户创作上下文。

按钮/操作：
  1. 修改 ONNX 模型文件路径。
  2. 修改 dtype。
  3. 修改摘要和检索数量。
  4. 点击保存配置。
  5. 点击恢复默认。
  6. 点击清空记忆。

弹窗/表单：
  1. 记忆配置表单。
  2. 清空记忆确认弹窗。
  3. 全部清空必须输入确认短语。

成功反馈：
  保存成功、恢复默认成功、清空成功。

失败反馈：
  模型路径不存在、路径越界、数字越界、确认短语错误、清空失败。
```

## 4. 要做什么功能

### 1. 查询配置

怎么做：
- 输入：无。
- 输出：记忆配置对象、模型状态、记忆统计。
- 写什么数据：不写。
- 状态怎么变：表单填入当前配置。
- 异常怎么处理：配置缺失时用默认值。
- 限制：不直接读写 memories 内容。
  - modelOnnxFile 只显示相对模型路径数组，不显示绝对路径。

### 2. 保存配置

怎么做：
- 输入：模型路径、dtype、各数量参数。
- 输出：保存成功。
- 写什么数据：写 settings。
- 状态怎么变：后续 memory service 使用新配置。
- 异常怎么处理：参数越界、模型路径非法、保存失败时提示。
- 限制：修改配置不会自动重建旧 embedding。
  - ONNX 文件不存在允许保存，但状态标红并提示 embedding 会降级。
  - 路径必须在 userData/models 下，不能是绝对路径。

### 3. 恢复默认

怎么做：
- 输入：无。
- 输出：默认配置。
- 写什么数据：用户确认后写 settings。
- 状态怎么变：表单恢复默认值。
- 异常怎么处理：保存失败提示。
- 限制：恢复默认不清空记忆。

### 4. 记忆统计

怎么做：
- 输入：无。
- 输出：总数、message 数、summary 数、按 isolationKey 聚合数量。
- 写什么数据：不写。
- 状态怎么变：页面展示统计。
- 异常怎么处理：查询失败显示错误。
- 限制：不展示记忆正文。

### 5. 清空记忆

怎么做：
- 输入：范围和确认。
- 输出：清空成功。
- 写什么数据：删除 memories 或按范围删除。
- 状态怎么变：记忆数量减少。
- 异常怎么处理：事务失败回滚。
- 限制：
  - 必须二次确认。
  - 第一版支持指定隔离范围和全部清空。
  - 指定隔离范围输入 projectId、agentType、episodesId。
  - 全部清空必须输入确认短语：清空全部记忆。
  - 页面不提供无确认的一键全删。

### 6. ONNX 模型路径校验

怎么做：
- 输入：modelOnnxFile。
- 输出：存在/不存在。
- 写什么数据：不写。
- 状态怎么变：表单显示校验结果。
- 异常怎么处理：不存在时允许保存但标记不可用。
- 限制：只允许 userData/models 或规定模型目录，不允许任意路径。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 清空记忆增加范围和二次确认。
  2. ONNX 路径由 main 校验。
  3. 修改配置不自动重建旧 embedding。
  4. 记忆配置放开发者模式，不放普通入口。
  5. 全部清空必须输入确认短语。

原因：
  记忆是用户创作上下文，不能一个按钮无差别清空。

是否写入 04：
  如不保留全库一键清空默认入口，不算偏差，属于安全修正。
```
