## 0. 快速理解

```txt
一句话：先把事件管理边界定清楚：第一版以章节事件摘要为准，不照搬无闭环的旧事件列表。
为什么现在做：参考项目 `event.vue` 存在但没有在主页面挂载，直接做会偏题。
做完后有什么用：后续 Agent 读取上下文不会混用两套事件数据。
这一步不碰什么：不做旧 `eventAnalysis.vue`，不调用不存在的接口。
```

## 1. 本次做什么

目标：
  整理并落地“章节级事件摘要”的管理能力，明确旧事件库不是第一版正式能力。

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/novel/index.vue` | 主页面展示章节级 `o_novel.event` |
| `Toonflow-web/src/views/novel/components/event.vue` | 有事件列表、删除、批量删除 UI，但主页面未找到挂载 |
| `Toonflow-web/src/views/novel/components/eventAnalysis.vue` | 调用 `/novel/event/eventAnalysis` |
| `Toonflow-app/src/routes/novel/event/getEvent.ts` | 读取 `o_event/o_eventChapter/o_novel` |
| `Toonflow-app/src/routes/novel/event/generateEvents.ts` | 生成章节级 `o_novel.event`，要求 novelIds |

参考项目关键事实：

```txt
1. 主页面用的是章节级 event 字段。
2. event.vue 读取 o_event/o_eventChapter，但没有找到和 o_novel.event 的完整同步闭环。
3. event.vue 调 generateEvents 时没有传 novelIds，而后端要求 novelIds。
4. eventAnalysis.vue 调的后端 route 没找到。
5. 剧本 Agent 工具读取的是章节事件，不依赖独立事件库。
```

## 4. 要做什么功能

### 1. 统一事件来源

怎么做：
- 输入：当前项目章节。
- 输出：章节级事件摘要。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：事件为空时显示未分析或空状态。
- 限制：第一版以 source_chapters.event_summary 为准。

### 2. 事件摘要查看和编辑

怎么做：
- 输入：章节 event_summary。
- 输出：详情查看；编辑时走 F-004-003。
- 写什么数据：编辑时更新章节事件摘要。
- 状态怎么变：编辑保存后刷新列表。
- 异常怎么处理：保存失败提示。
- 限制：不建立第二套事件表作为主来源。

### 3. 旧事件组件处理

怎么做：
- 输入：参考项目 event.vue、eventAnalysis.vue。
- 输出：记录为旧痕迹，不作为第一版实现依据。
- 写什么数据：必要时写 04 偏差。
- 状态怎么变：无。
- 异常怎么处理：无。
- 限制：不能调用不存在的 `/novel/event/eventAnalysis`。
