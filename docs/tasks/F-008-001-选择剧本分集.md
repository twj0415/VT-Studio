# F-008-001 选择剧本/分集

状态：等待用户确认  
所属菜单：M-008 生产工作台  
对应功能文档：`docs/features/M-008-生产工作台.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：生产页先选择当前剧本/分集，再加载这个分集的流程数据。
为什么现在做：生产工作台所有分镜、资产、视频轨道都挂在具体剧本下。
做完后有什么用：用户能切换不同剧本分集，并看到对应生产流程。
这一步不碰什么：不生成分镜，不生成视频，不做剪映导出。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 生产页顶部剧本选择逻辑。

只做：
  1. 读取当前项目剧本列表
  2. 默认选择第一个剧本
  3. 切换剧本/分集
  4. 切换时加载 flowData
  5. Agent 生成中切换确认

不做：
  1. 新增剧本
  2. 编辑剧本
  3. 删除剧本
  4. 生成流程内容
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/production/index.vue` | 顶部 select 选择剧本，默认第一个，切换后加载 flowData |
| `Toonflow-app/src/routes/script/getScrptApi.ts` | 获取当前项目剧本列表 |
| `Toonflow-app/src/routes/production/getFlowData.ts` | 根据 projectId + episodesId 获取流程数据 |

参考项目关键事实：

```txt
1. 生产页必须先有剧本，没有剧本时不能空跑流程图。
2. 默认选择第一个剧本。
3. 切换剧本时重新加载 flowData、资产、分镜、视频轨道。
4. Production Agent pending/streaming 时切换剧本会弹确认。
5. episodesId 实际对应 scriptId。
```

## 3. 用户操作

```txt
入口：
  项目内顶部菜单 -> 生产。

按钮/操作：
  1. 打开生产页。
  2. 从顶部下拉选择剧本/分集。
  3. Agent 生成中切换时确认。

弹窗/表单：
  生成中切换确认框。

成功反馈：
  流程图加载当前剧本数据。

失败反馈：
  没有剧本、剧本被删除、流程数据加载失败。
```

## 4. 要做什么功能

### 1. 加载剧本列表

怎么做：
- 输入：projectId。
- 输出：当前项目 scripts。
- 写什么数据：不写。
- 状态怎么变：loading -> loaded。
- 异常怎么处理：加载失败显示错误；无剧本显示空状态和去剧本页入口。
- 限制：只读当前项目。

### 2. 默认选择剧本

怎么做：
- 输入：scripts。
- 输出：currentScriptId。
- 写什么数据：不写。
- 状态怎么变：未选中 -> 第一个剧本。
- 异常怎么处理：scripts 为空不选择。
- 限制：不能使用已删除或无效 scriptId。

### 3. 切换剧本

怎么做：
- 输入：scriptId。
- 输出：新的生产上下文。
- 写什么数据：不写。
- 状态怎么变：currentScriptId 更新，flowData 重新加载。
- 异常怎么处理：Agent 生成中先确认；加载失败回退旧剧本或显示错误。
- 限制：切换时清理旧轮询和临时选择状态。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  episodesId 正式命名为 scriptId 或 episodeId，不保留含糊命名。

原因：
  参考项目 episodesId 实际指剧本 id，容易误解。

是否写入 04：
  不需要，属于命名规范；如迁移旧数据再记录。
```
