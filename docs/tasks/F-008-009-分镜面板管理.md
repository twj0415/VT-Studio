## 0. 快速理解

```txt
一句话：生产页分镜节点管理分镜卡片，支持选择、编辑、删除、批量删除和插入新分镜。
为什么现在做：分镜是图片生成、视频轨道和最终导出的核心结构。
做完后有什么用：用户能维护每个镜头的画面描述、图片提示词、时长和关联资产。
这一步不碰什么：不批量生成分镜图片，不生成视频，不导出剪映。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow storyboard 节点的分镜面板管理。

只做：
  1. 分镜卡片网格
  2. 单选/全选/清空
  3. 编辑分镜 prompt 和 videoDesc
  4. 单条删除
  5. 批量删除
  6. 左右插入新分镜
  7. 分镜网格预览和下载入口

不做：
  1. 分镜图片批量生成
  2. 视频生成
  3. 剪映导出
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/production/node/storyboard.vue` | 分镜网格、选择、删除、编辑、插入、预览 |
| `Toonflow-app/src/routes/production/storyboard/addStoryboard.ts` | 新增单条分镜，同时新建 videoTrack |
| `Toonflow-app/src/routes/production/storyboard/batchAddStoryboardInfo.ts` | 批量新增分镜、写资产关系和轨道 |
| `Toonflow-app/src/routes/production/storyboard/editStoryboardInfo.ts` | 更新 prompt/videoDesc |
| `Toonflow-app/src/routes/production/storyboard/removeFrame.ts` | 删除单条分镜 |
| `Toonflow-app/src/routes/production/storyboard/batchDelete.ts` | 批量删除 |

参考项目关键事实：

```txt
1. 每张分镜有 id、index、duration、prompt、videoDesc、associateAssetsIds、src、state、reason、trackId、flowId。
2. 单条删除最后一个 track 分镜会删 videoTrack。
3. batchDelete 没有同等轨道清理逻辑，VT Studio 必须修正。
4. addStoryboard 接收 shouldGenerateImage，但源码写库时用 src ? 1 : 0，入参被忽略。
5. 新增分镜前端传 flowId，但后端 schema 没接收，VT Studio 必须修正。
```

## 3. 用户操作

```txt
入口：
  生产流程图 -> 分镜节点。

按钮/操作：
  1. 点击分镜卡片选择。
  2. 全选、清空选择。
  3. 编辑分镜信息。
  4. 删除单条或批量删除。
  5. 左右插入新分镜。
  6. 打开网格预览。

弹窗/表单：
  编辑分镜信息弹窗；删除确认；预览弹窗。

成功反馈：
  分镜列表更新。

失败反馈：
  删除失败、编辑失败、插入失败。
```

## 4. 要做什么功能

### 1. 展示分镜卡片

怎么做：
- 输入：storyboard 列表。
- 输出：分镜网格。
- 写什么数据：不写。
- 状态怎么变：根据 state 显示未生成/生成中/成功/失败。
- 异常怎么处理：图片缺失显示缺失状态。
- 限制：分镜排序按 index。

### 2. 编辑分镜信息

怎么做：
- 输入：storyboardId、prompt、videoDesc。
- 输出：更新结果。
- 写什么数据：更新 storyboards.prompt/video_desc。
- 状态怎么变：保存中 -> 刷新。
- 异常怎么处理：保存失败提示。
- 限制：不改图片和 track。

### 3. 插入新分镜

怎么做：
- 输入：插入位置、参考图、prompt、duration、videoDesc、关联资产。
- 输出：新分镜。
- 写什么数据：写 storyboards、video_tracks、storyboard_asset_links。
- 状态怎么变：列表插入并重新排序。
- 异常怎么处理：事务失败回滚。
- 限制：flowId 和 shouldGenerateImage 必须正确保存。

### 4. 删除分镜

怎么做：
- 输入：storyboardIds。
- 输出：删除结果。
- 写什么数据：删除分镜、资产关系、图片工作流、必要时清轨道和文件。
- 状态怎么变：列表刷新。
- 异常怎么处理：失败回滚。
- 限制：批量删除也要处理轨道清理。
