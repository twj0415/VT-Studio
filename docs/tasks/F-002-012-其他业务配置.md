## 0. 快速理解

```txt
一句话：集中管理会影响业务结果的杂项配置，比如章节正则、请求超时、画布滚轮、生成并发、剧本长度。
为什么现在做：这些值会影响小说导入、生产工作台、批量生成和剧本编辑，不能散在页面里。
做完后有什么用：用户能调关键业务参数，后续功能都从统一配置读取。
这一步不碰什么：不做具体小说导入、不做生产工作台、不做生成任务。
```

## 4. 要做什么功能

### 1. 章节拆分正则

怎么做：
- 输入：正则字符串，支持 `/pattern/flags`。
- 输出：保存成功。
- 写什么数据：写设置。
- 状态怎么变：后续小说导入使用新规则。
- 异常怎么处理：编译失败不保存，并提示错误位置。
- 限制：不能让非法正则进入配置。

### 2. 恢复默认正则

怎么做：
- 输入：无。
- 输出：默认正则。
- 写什么数据：写设置。
- 状态怎么变：章节拆分恢复默认。
- 异常怎么处理：保存失败提示。
- 限制：只恢复正则，不改其他配置。

### 3. 请求超时

怎么做：
- 输入：秒。
- 输出：毫秒配置。
- 写什么数据：写设置。
- 状态怎么变：后续请求使用新 timeout。
- 异常怎么处理：小于最小值或非数字失败。
- 限制：页面显示秒，内部存毫秒。

### 4. 画布滚轮方式

怎么做：
- 输入：`zoom/scroll`。
- 输出：保存成功。
- 写什么数据：写设置。
- 状态怎么变：生产工作台 pan-on-scroll/zoom-on-scroll 改变。
- 异常怎么处理：非法值回退 zoom。
- 限制：必须持久化，不照搬源码可能不持久化的问题。

### 5. 交互态显示

怎么做：
- 输入：boolean。
- 输出：保存成功。
- 写什么数据：写设置。
- 状态怎么变：生产工作台交互 class 控制。
- 异常怎么处理：非法值回退 true。
- 限制：只影响 UI 状态，不影响数据。

### 6. 资产生成并发数

怎么做：
- 输入：数字。
- 输出：保存成功。
- 写什么数据：写设置 `assetsBatchGenerateSize`。
- 状态怎么变：后续批量任务按该并发分组。
- 异常怎么处理：小于 1、过大、非数字失败。
- 限制：
  - 建议默认值 `5`。
  - 建议最大值 `8`，避免本地机器或中转模型服务被并发打爆。

### 7. 剧本单集长度限制

怎么做：
- 输入：数字。
- 输出：保存成功。
- 写什么数据：写设置。
- 状态怎么变：后续新增/编辑剧本按该长度校验。
- 异常怎么处理：小于 100 或非数字失败。
- 限制：只限制单集文本长度，不自动截断用户内容。

## 6. VT Studio 怎么落

```txt
能力名：
  settings.business

调用链：
  renderer 设置页 -> settings store/service -> 后续业务模块读取

需要新增：
  设置页其他业务配置组件
  business settings 类型和默认值

需要修改：
  小说导入、请求封装、生产工作台、批量生成、剧本编辑后续统一读取本配置

当前代码现实：
  1. 当前 renderer 侧还没有统一的 business settings store。
  2. 后续很多能力会落在 main/services，不适合只靠 renderer localStorage 持有。
  3. 所以会影响业务产物或 main 行为的字段，更适合落 `app_settings` / settings 表统一读取。
```
