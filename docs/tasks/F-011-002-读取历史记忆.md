# F-011-002 读取历史记忆

状态：等待用户确认  
所属菜单：M-011 剧本 Agent  
对应功能文档：`docs/features/M-011-剧本Agent.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：进入剧本 Agent 时读取这个项目的历史对话记忆。
为什么现在做：Agent 不能每次打开都忘记之前聊过什么。
做完后有什么用：用户能继续上次的剧本规划上下文。
这一步不碰什么：不清记忆，不做向量检索实现，不生成剧本。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/agents/getMemory`，按 `projectId:scriptAgent` 隔离读取 message 类型记忆。

只做：
  1. 进入页面后读取历史 message 记忆
  2. 转成聊天消息格式
  3. 和欢迎消息合并展示
  4. 读取失败兜底

不做：
  1. 清理记忆
  2. 记忆摘要生成
  3. RAG 检索
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/scriptAgent/index.vue` | 如果只有欢迎消息，调用 `/agents/getMemory` 读取历史 |
| `Toonflow-app/src/routes/agents/getMemory.ts` | 按 isolationKey/type 读取 memories |
| `Toonflow-app/src/agents/scriptAgent/index.ts` | Agent 运行时写入 memory |

参考项目关键事实：

```txt
1. isolationKey 是 `${projectId}:scriptAgent`。
2. 读取 message 类型历史消息。
3. Memory 里还有 summary 类型，但聊天列表主要展示 message。
4. 读取失败没有完整页面兜底，VT Studio 要补。
```

## 3. 用户操作

```txt
入口：
  打开剧本 Agent 页面自动读取。

按钮/操作：
  无。

弹窗/表单：
  无。

成功反馈：
  历史消息显示在聊天列表。

失败反馈：
  读取失败显示轻量提示，不阻止新对话。
```

## 4. 要做什么功能

### 1. 读取历史消息

怎么做：
- 输入：projectId、type=message。
- 输出：历史消息列表。
- 写什么数据：不写。
- 状态怎么变：memoryLoading=true -> false。
- 异常怎么处理：读取失败提示，但保留欢迎消息。
- 限制：只读当前项目的 scriptAgent 记忆。

### 2. 转换消息格式

怎么做：
- 输入：memory role/name/content。
- 输出：聊天 message。
- 写什么数据：不写。
- 状态怎么变：messages 追加历史记录。
- 异常怎么处理：空 content 的记忆跳过。
- 限制：不能把 summary 直接当普通聊天逐条显示。

### 3. 避免重复加载

怎么做：
- 输入：当前 messages。
- 输出：只在需要时读取。
- 写什么数据：不写。
- 状态怎么变：避免重复 append。
- 异常怎么处理：重复消息按 id 去重。
- 限制：刷新页面可以重新读取，页面内重连不重复塞消息。

## 7. 偏差

```txt

  读取失败不阻塞新对话，且消息按 id 去重。

原因：
  记忆损坏不应该导致 Agent 页面不可用。

是否写入 04：
  不需要，属于容错增强。
```

## 8. 验收

```txt
1. 打开页面能读取当前项目历史消息。
2. 只读取 scriptAgent 隔离 key。
3. 不读取其他项目记忆。

5. 读取失败仍可发送新消息。
```
