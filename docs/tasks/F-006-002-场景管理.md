## 0. 快速理解

```txt
一句话：资产中心做场景 tab，管理场景资产的列表、新增、编辑、删除和图片状态。
为什么现在做：场景资产会参与分镜、角景、图片生成和视频生产。
做完后有什么用：用户能维护场景名称、描述、提示词、主图和衍生子场景。
这一步不碰什么：不做分镜生成，不做视频生成，不做生产页衍生资产调度。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `type=scene` 的资产管理能力。

只做：
  1. 场景 tab
  2. 场景父子资产列表
  3. 新增场景
  4. 编辑场景
  5. 删除/批量删除场景
  6. 展示主图、提示词、描述、备注、创建时间

不做：
  1. 批量生成提示词
  2. 批量生成图片
  3. 生产工作台衍生资产
```

参考项目关键事实：

```txt
1. 场景类型是 scene。
2. 场景视觉手册用于后续提示词润色：父场景 key=art_scene，衍生场景 key=art_scene_derivative。
3. 场景和角色/道具的列表逻辑一致。
4. 删除源码同样存在批量删除不完整风险。
```

## 3. 用户操作

```txt
入口：
  项目内顶部菜单 -> 资产 -> 场景 tab。

按钮/操作：
  新增、编辑、删除、批量删除、展开子场景、勾选。

弹窗/表单：
  新增/编辑场景弹窗；删除确认框。

成功反馈：
  保存或删除成功，刷新列表。

失败反馈：
  必填缺失、生成中禁止操作、删除失败。
```

## 4. 要做什么功能

### 1. 查询场景列表

怎么做：
- 输入：projectId、name、page、pageSize。
- 输出：场景父子资产树。
- 写什么数据：不写。
- 状态怎么变：loading -> loaded。
- 异常怎么处理：查询失败显示错误状态。
- 限制：只查 type=scene。

### 2. 新增/编辑场景

怎么做：
- 输入：name、describe、remark、prompt。
- 输出：场景资产。
- 写什么数据：写或更新 assets。
- 状态怎么变：保存中 -> 列表刷新。
- 异常怎么处理：name/describe 缺失不能保存。
- 限制：不能修改资产类型。

### 3. 删除场景

怎么做：
- 输入：scene assetIds。
- 输出：删除结果。
- 写什么数据：删除场景、子场景、媒体、关系和文件。
- 状态怎么变：删除中 -> 列表刷新。
- 异常怎么处理：生成中禁止删除；失败回滚。
- 限制：批量删除必须完整级联。
