# F-004-005 事件分析生成

状态：等待用户确认  
所属菜单：M-004 小说/原文  
对应功能文档：`docs/features/M-004-小说原文.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：用户选中章节后，用模型生成每章事件摘要。
为什么现在做：剧本 Agent 依赖章节事件，不先分析事件，后面生成会缺上下文。
做完后有什么用：每章会有事件摘要，失败也能看到原因。
这一步不碰什么：不写剧本，不做独立事件库，不让页面直接调模型 SDK。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/novel/event/generateEvents` 和 `cleanNovel.ts` 的事件摘要生成。

只做：
  1. 选择章节触发事件分析
  2. 二次确认
  3. 设置章节事件状态为生成中
  4. 调用文本模型生成事件摘要
  5. 成功写事件摘要
  6. 失败写失败原因

不做：
  1. 剧本 Agent 对话
  2. 独立事件库管理
  3. 事件批量编辑
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/novel/index.vue` | 选中章节后点击事件分析，弹确认，调用生成接口 |
| `Toonflow-app/src/routes/novel/event/generateEvents.ts` | 把选中章节 eventState=0、event=null，然后异步 cleanNovel |
| `Toonflow-app/src/utils/cleanNovel.ts` | 用 `u.Ai.Text("universalAi").invoke` 生成事件摘要 |
| `Toonflow-app/src/routes/novel/getNovelEventState.ts` | 轮询事件生成结果 |

参考项目关键事实：

```txt
1. 只分析用户选中的章节。
2. concurrentCount 来自 otherSetting.assetsBatchGenereateSize。
3. 提示词优先取 o_prompt type=eventExtraction 的 useData，再取 data，再取默认 event prompt。
4. 成功写 event 和 eventState=1。
5. 失败写 event=null、eventState=-1、errorReason。
6. Toonflow 没有把事件分析写入 o_tasks。
```

## 3. 用户操作

```txt
入口：
  原文列表 -> 勾选章节 -> 事件分析按钮。

按钮/操作：
  1. 勾选章节。
  2. 点击事件分析。
  3. 二次确认。

弹窗/表单：
  确认框。

成功反馈：
  章节事件列变成生成中，后续轮询更新。

失败反馈：
  未选章节、模型配置缺失、提示词缺失、模型调用失败。
```

## 4. 要做什么功能

### 1. 触发事件分析

怎么做：
- 输入：chapterIds、projectId。
- 输出：生成任务启动结果。
- 写什么数据：更新章节 event_status=running、event_summary=null。
- 状态怎么变：selected 清空，列表刷新显示生成中。
- 异常怎么处理：未选章节不能提交；章节不存在返回错误。
- 限制：只处理当前项目章节。

### 2. 调用模型生成事件摘要

怎么做：
- 输入：章节正文、事件提取提示词、universalAi 模型配置。
- 输出：事件摘要文本。
- 写什么数据：成功写 event_summary，失败写 event_error。
- 状态怎么变：running -> succeeded/failed。
- 异常怎么处理：模型异常、空输出、提示词缺失分别写失败原因。
- 限制：renderer 不直接调用模型 SDK。

### 3. 并发控制

怎么做：
- 输入：并发数设置。
- 输出：按并发执行的章节事件分析。
- 写什么数据：不额外写。
- 状态怎么变：每章独立完成。
- 异常怎么处理：单章失败不影响其他章节继续。
- 限制：设置字段正式命名不能保留 `assetsBatchGenereateSize` 拼写错误。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  建议事件分析写入任务中心。

原因：
  事件分析是模型异步任务，失败原因应在任务中心可查。

  如果接入任务中心，需要写 04。
```
