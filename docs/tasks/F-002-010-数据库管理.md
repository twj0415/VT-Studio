# F-002-010 数据库管理

状态：已完成  
所属菜单：M-002 设置  
对应功能文档：`docs/features/M-002-设置.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：在开发者模式里提供数据库查看、备份、从受控备份恢复、清表、清空全部数据的管理能力。
为什么现在做：后续项目、剧本、资产、任务都会写数据库，必须先有备份和高风险操作规则。
做完后有什么用：高级用户能备份数据、从受控备份恢复、排查数据库状态，也能在明确确认后清理数据。
这一步不碰什么：不做业务表设计，不做云同步，不做自动迁移策略以外的复杂数据修复。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的 dbConfig 能力，但把高风险操作做得更安全。

只做：
  1. 查看数据库信息
  2. 导出数据
  3. 导入数据
  4. 查看表列表
  5. 清空指定表
  6. 清空全部数据
  7. 导入/清空前 running 任务检查
  8. 危险操作二次确认
  9. 备份列表

不做：
  1. 数据库结构重设计
  2. 业务数据编辑器
  3. 云端备份
  4. 自动合并冲突数据
  5. 跳过 migration 的直接覆盖
  6. 让 renderer 传任意文件路径覆盖数据库
  7. 删除项目素材目录
```

| `Toonflow-app/src/routes/setting/dbConfig/clearTable.ts` | 清空指定表 |
| `Toonflow-app/src/routes/setting/dbConfig/clearData.ts` | 清空全部数据，并重新初始化 |

参考项目关键事实：

```txt
1. 导入数据格式要求包含 tables。
2. clearData 会清空并重新初始化数据库。
3. 清空表有确认，清空全部有二次确认。
4. 参考项目没有明确运行中任务检查；VT Studio 必须补。
```

## 3. 用户操作

```txt
入口：
  设置 -> 开发者模式 -> 数据库管理。

定位：
  高级能力，不放普通用户主入口。
  原因：导入、清表、清空全部都是不可逆高风险操作。

按钮/操作：
  1. 查看数据库信息。
  2. 点击导出数据。
  3. 从受控备份列表选择备份并导入。
  4. 选择一张表。
  5. 点击清空表。
  6. 点击清空全部数据。

弹窗/表单：
  1. 备份选择。
  2. 清空表确认弹窗。
  3. 清空全部数据二次确认，要求输入确认关键字。

成功反馈：
  备份成功、导入成功、清空成功、重新初始化成功。

失败反馈：
  备份文件不存在、版本不兼容、running 任务未停止、清空失败、迁移失败。
```

## 4. 要做什么功能

### 1. 查看数据库信息

怎么做：
- 输入：无。
- 输出：数据库路径、大小、表数量、迁移版本、最近备份时间。
- 写什么数据：不写。
- 状态怎么变：页面 loading -> 信息卡片。
- 异常怎么处理：数据库不存在或连接失败时显示错误。
- 限制：绝对路径可以在设置页显示，但不能进普通错误 detail。

### 2. 备份列表

怎么做：
- 输入：无。
- 输出：`userData/exports/database` 下的 `.sqlite` 备份文件列表。
- 写什么数据：不写。
- 状态怎么变：页面显示可恢复备份。
- 异常怎么处理：目录不存在时返回空列表。
- 限制：
  - 只列受控备份目录。
  - 不允许页面输入任意路径。

### 3. 导出数据

怎么做：
- 输入：无。
- 输出：备份文件路径。
- 写什么数据：写入 `userData/exports/database`。
- 状态怎么变：导出中 -> 成功显示路径和打开目录按钮。
- 异常怎么处理：路径无权限、写入失败、数据库锁定时失败。
- 限制：格式必须固定，不允许导出半成品结构。
  - 第一版固定 SQLite 备份，不做 JSON。
  - 返回相对/文件名和受控目录信息，不让 renderer 指定输出路径。

### 4. 导入数据

怎么做：
- 输入：备份文件名。
- 输出：导入结果。
- 写什么数据：导入前先备份当前库；导入后执行 migration。
- 状态怎么变：导入前校验 -> 自动备份 -> 导入 -> migration -> 成功。
- 异常怎么处理：备份不存在、不是 sqlite、版本不兼容、migration 失败时提示自动备份路径。
- 限制：
  - 有 running 任务时阻止导入，要求先停止任务。
  - 只能导入 `userData/exports/database` 下的备份文件。
  - 导入过程由 main 关闭数据库连接、替换文件、重开连接并运行 migration。
  - 导入前强制自动备份当前库。

### 5. 表列表和选择表

怎么做：
- 输入：无。
- 输出：表名、行数、说明、影响模块。
- 写什么数据：不写。
- 状态怎么变：选择表后显示表风险说明。
- 异常怎么处理：读取表列表失败显示错误。
- 限制：系统表和 migration 表默认不允许清空。

### 6. 清空指定表

怎么做：
- 输入：表名。
- 输出：清空成功。
- 写什么数据：删除指定表数据。
- 状态怎么变：二次确认后执行，完成后刷新表行数。
- 异常怎么处理：表不存在、受保护表、running 任务存在、事务失败时失败。
- 限制：必须列出影响模块；高风险表需要输入表名确认。
  - schema_migrations、sqlite_sequence 等系统表禁止清空。
  - 清空前强制自动备份。

### 7. 清空全部数据

怎么做：
- 输入：确认关键字。
- 输出：清空业务数据并重新初始化成功。
- 写什么数据：清空业务数据，重新执行默认数据初始化。
- 状态怎么变：确认 -> 自动备份 -> 清空 -> 初始化 -> 重启提示或刷新状态。
- 异常怎么处理：确认关键字不匹配不执行；初始化失败时提示备份路径。
- 限制：不能删除数据库文件本身；不能删除用户项目素材目录。
  - 清空前强制自动备份。
  - 保留 schema_migrations，清空业务表后运行 runSeed。

### 8. running 任务检查

怎么做：
- 输入：无。
- 输出：是否存在 running 任务。
- 写什么数据：不写。
- 状态怎么变：危险操作前阻断或提示。
- 异常怎么处理：检查失败时默认阻止危险操作。
- 限制：导入、清表、清空全部都必须检查。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 导入前自动备份当前库。
  2. 危险操作前检查 running 任务。
  3. 受保护表不能直接清空。
  4. 只允许从受控备份目录导入，不接收任意路径。
  5. 固定 SQLite 备份格式，不做 JSON 半成品。
  6. 放开发者模式，不放普通入口。

原因：
  桌面项目数据不可恢复，必须比参考项目更稳。

是否写入 04：
  不需要，属于安全增强。
```

```txt
8. 清空表前显示表名和影响模块。

11. 清库不删除素材文件和项目目录。
12. 页面不直接访问 SQLite。
13. 所有 IPC 返回 `{ code, data, msg }`。
14. 备份列表只来自 `userData/exports/database`。
15. 受保护表不能清空。
```
