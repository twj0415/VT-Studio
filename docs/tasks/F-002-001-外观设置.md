## 0. 快速理解

```txt
一句话：补齐 VT Studio 的外观主题底座，让设置页能真正控制全局模式、主题系列和字号。
为什么现在做：当前只有 token 层的亮/暗变量，还没有完整的 theme store、初始化和系统监听链路。
做完后有什么用：用户能切自动/浅色/深色、切工作风/暖色/Studio 主题系列、调字号，而且刷新、重启、系统主题变化都能真的走通。
这一步不碰什么：不做主题市场，不做每个页面单独配色，不做云同步。
```

## 4. 要做什么功能

### 1. 颜色模式

怎么做：
- 输入：`auto/light/dark`。
- 输出：当前生效模式。
- 写什么数据：写本地外观设置。
- 状态怎么变：documentElement 更新 `theme-mode` 和 `dark` class。
- 异常怎么处理：非法值回退 `auto` 或 `light`。
- 限制：auto 模式跟随系统，不手动锁死。
  - 当前实际要求：不能只改 class，必须补启动初始化和系统监听，否则刷新后会丢状态或跟随失效。

### 2. 多主题系列预设

怎么做：
- 输入：`studio / warm / work`。
- 输出：CSS 变量。
- 写什么数据：写本地外观设置。
- 状态怎么变：整套界面 token 和 TDesign 品牌色一起切换。
- 异常怎么处理：未知 preset 回退 `studio`。
- 限制：
  - 第一版主题系列固定维护，不开放普通模式任意 HEX。
  - 除了 `--vt-brand` 系列，还要同步 TDesign 品牌色变量，不然组件颜色不会一起变。

### 3. 字体大小

怎么做：
- 输入：`12/13/14/16/18/20/22`。
- 输出：根节点 font-size。
- 写什么数据：写本地外观设置。
- 状态怎么变：整体 rem 基准更新。
- 异常怎么处理：非法值回退 14。
- 限制：不能用 viewport 宽度动态缩放字体。
  - 当前实际要求：只能改根节点 `font-size`，不能去逐页散改字号。

### 4. 多主题系列扩展口

怎么做：
- 输入：`themePresetId`，第一版内置 `studio / warm / work`。
- 输出：一组 CSS variables。
- 写什么数据：写本地外观设置。
- 状态怎么变：主题系列切换时整套颜色语言一起变化。
- 异常怎么处理：未知 preset 回退 `studio`。
- 限制：这是 VT Studio 增强，不代表参考项目已有多主题系列。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 第一版直接支持多主题系列，不只是预留 themePresetId。
  2. 普通模式取消任意 HEX 主色，改为受控主题系列。
  3. 强制页面使用统一 CSS variables。

原因：
  你希望后续支持不止浅色/深色的多种配色系列，但不能让页面拼拼凑凑。

是否写入 04：
  如果正式增加多主题系列，需要写入 04；当前只是预留规则，不写。
```
