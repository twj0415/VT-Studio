## 0. 快速理解

```txt
一句话：进入页面前检查本地登录状态，没登录就跳登录页，登录页本身放行。
为什么现在做：项目和设置都需要稳定 user 语义，不能出现旧 userId 残留。
做完后有什么用：页面访问顺序清楚，退出登录后不会继续用旧状态。
这一步不碰什么：不做完整云账号，不做远程鉴权。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的 token 路由守卫语义，同时适配 VT Studio 本地优先策略。

只做：
  1. 定义公开路由。
  2. 定义需要登录的路由。
  3. 进入非登录页前检查本地 token/user 状态。
  4. 未登录跳 `/login`。
  5. 已登录访问 `/login` 时跳 `/project`。
  6. 清理无效 userId/currentProject。

不做：
  1. 不做后端远程 token 校验。
  2. 不做注册、找回密码。
  3. 不做云端账号体系。
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/router/index.ts` | `/login` 放行，其它路由检查 `localStorage.token`，没有 token 跳登录 |
| `Toonflow-web/src/pages/login/index.vue` | 登录成功写 `token` 和 `userId` 后跳 `/project` |
| `Toonflow-web/src/pages/workbench/index.vue` | 主布局依赖当前路由和项目上下文 |

参考项目关键事实：

```txt
1. Toonflow 路由守卫只看 localStorage.token。
2. token 不存在时直接跳 `/login`。
3. token 存在时不校验过期，不请求后端验证。
4. VT Studio 本地优先，可以弱化登录，但不能丢掉“未登录/已登录”状态边界。
```

## 3. 用户操作

```txt
入口：
  用户打开任意路由，或退出登录后再次进入业务页面。

按钮/操作：
  无，路由切换自动触发。

弹窗/表单：
  无。

成功反馈：
  已登录进入目标页面；未登录跳登录页。

失败反馈：
  路由状态异常时清理旧状态并提示重新登录。
```

## 4. 要做什么功能

### 1. 公开路由放行

怎么做：
- 输入：目标路由。
- 输出：`/login` 放行。
- 写什么数据：不写。
- 状态怎么变：路由继续。
- 异常怎么处理：无。
- 限制：公开路由只包含登录和必要初始化页。

### 2. 登录状态检查

怎么做：
- 输入：本地 token、userId、本地默认用户状态。
- 输出：是否允许进入目标路由。
- 写什么数据：不写数据库；可清理无效前端缓存。
- 状态怎么变：未登录跳 `/login`。
- 异常怎么处理：token 有但 userId 缺失时按无效登录处理。
- 限制：不能只看 currentProject，当用户不存在时不能继续业务页面。

### 3. 已登录访问登录页处理

怎么做：
- 输入：用户访问 `/login`。
- 输出：已登录跳 `/project`。
- 写什么数据：不写。
- 状态怎么变：路由替换。
- 异常怎么处理：跳转失败提示。
- 限制：避免登录页和项目页之间循环跳转。

### 4. 无效状态清理

怎么做：
- 输入：token 缺失、userId 缺失、用户记录不存在。
- 输出：清理 token、userId、currentProject。
- 写什么数据：清前端持久化状态。
- 状态怎么变：进入登录页。
- 异常怎么处理：清理失败时仍阻止进入业务页面。
- 限制：不清模型密钥和项目数据，退出登录清理范围归 F-002-016。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  Toonflow 只看 token；VT Studio 还要保证本地 userId/session 不冲突。

原因：
  本地桌面应用弱化云登录后，更要避免旧 userId 污染数据。

是否写入 04：
  是，属于登录弱化偏差。
```
