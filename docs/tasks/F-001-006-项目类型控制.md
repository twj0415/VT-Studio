# F-001-006 项目类型控制

状态：等待用户确认  
所属菜单：M-001 全局导航与项目入口  
对应功能文档：`docs/features/M-001-全局导航与项目入口.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：根据项目类型控制顶部菜单，小说项目显示“小说”和“剧本 Agent”，剧本项目不显示。
为什么现在做：不同项目来源能走的流程不同，菜单不能让用户点进无意义页面。
做完后有什么用：项目菜单和项目类型一致，不会出现剧本项目还显示原文处理入口。
这一步不碰什么：不做新建项目表单，不修改项目类型定义。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的 novel 项目菜单显示逻辑，并修正旧字段命名问题。

只做：
  1. 读取 currentProject.projectType。
  2. 小说项目显示小说和剧本 Agent。
  3. 非小说项目隐藏小说和剧本 Agent。
  4. 剧本、角景、生产、资产有项目就显示。
  5. 修正 `nodelOnly` 旧拼写为正式语义 `novelOnly`。

不做：
  1. 不做项目类型创建。
  2. 不做项目打开校验。
  3. 不做菜单页面业务。
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/pages/workbench/index.vue` | 顶部菜单用 `project.projectType === "novel" || !menu.nodelOnly` 判断显示 |
| `Toonflow-web/src/views/project/index.vue` | 项目打开后写入当前项目上下文 |

参考项目关键事实：

```txt
1. 小说和剧本 Agent 菜单标了 `nodelOnly: true`。
2. 实际语义是 novelOnly，`nodelOnly` 是旧拼写问题。
3. 条件是 projectType 为 novel 时显示 novelOnly 菜单。
4. 剧本、角景、生产、资产不受 novelOnly 限制。
5. 没有项目时整个顶部菜单不显示。
```

## 3. 用户操作

```txt
入口：
  用户打开某个项目后看到顶部菜单。

按钮/操作：
  1. 打开小说项目。
  2. 打开剧本项目。
  3. 切换不同项目。

弹窗/表单：
  无。

成功反馈：
  顶部菜单根据项目类型自动变化。

失败反馈：
  项目类型未知时隐藏类型限定菜单，并提示项目配置异常。
```

## 4. 要做什么功能

### 1. 小说项目菜单显示

怎么做：
- 输入：`currentProject.projectType = "novel"`。
- 输出：显示小说、剧本 Agent、剧本、角景、生产、资产。
- 写什么数据：不写。
- 状态怎么变：顶部菜单项完整显示。
- 异常怎么处理：无。
- 限制：只在 currentProject 有效时显示。

### 2. 剧本项目菜单显示

怎么做：
- 输入：`currentProject.projectType = "script"` 或非 novel。
- 输出：隐藏小说、剧本 Agent；显示剧本、角景、生产、资产。
- 写什么数据：不写。
- 状态怎么变：菜单项过滤。
- 异常怎么处理：当前 route 如果在 `/novel` 或 `/scriptAgent`，自动回 `/script`。
- 限制：不能让用户停留在当前项目不支持的页面。

### 3. 项目类型变化处理

怎么做：
- 输入：currentProject 切换或项目类型变化。
- 输出：重新计算可见菜单。
- 写什么数据：不写。
- 状态怎么变：菜单列表和高亮更新。
- 异常怎么处理：当前路由不合法时跳默认项目内页面。
- 限制：不能继续展示上一个项目的菜单。

### 4. 旧字段命名修正

怎么做：
- 输入：菜单配置。
- 输出：使用 `novelOnly`。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：无。
- 限制：VT Studio 正式代码不保留 `nodelOnly`。
