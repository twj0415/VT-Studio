# F-010-005 导出素材校验

状态：等待用户确认  
所属菜单：M-010 导出  
对应功能文档：`docs/features/M-010-导出.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：剪映导出前必须检查所有图片、视频、音频文件是否存在且可用。
为什么现在做：不能边写草稿边失败，缺素材必须提前告诉用户。
做完后有什么用：导出失败能精确定位到轨道、clip、来源对象和路径。
这一步不碰什么：不写草稿文件，不修复素材，不生成缺失素材。
```

## 1. 本次做什么

```txt
目标：
  对剪映导出 timeline 做素材校验，阻止半成品草稿输出。

只做：
  1. 校验 timeline clips
  2. 校验文件路径非空
  3. 校验文件存在
  4. 校验格式支持
  5. 校验读取权限
  6. 输出结构化失败列表

不做：
  1. 写剪映草稿
  2. 复制素材
  3. 自动补生成素材
  4. 修复数据库路径
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `docs/features/M-010-导出.md` | 明确剪映导出素材校验必须列出轨道、clip、素材类型、来源、路径、失败原因 |
| `Toonflow-app/src/routes/production/workbench/getFileUrl.ts` | 参考项目有补真实 URL 的逻辑，但不是剪映校验 |
| `Toonflow-web/src/views/production/components/workbench/editVideo/utils/mediaLoader.ts` | 前端加载缩略图/波形，失败有默认值 |

参考项目关键事实：

```txt
1. Toonflow 没有剪映草稿导出校验实现。
2. WebAV 和快速预览更多是在前端加载失败时处理，不适合剪映草稿。
3. VT Studio 必须在 main 服务里校验本地文件路径。
4. 校验失败要阻止导出，不写半成品草稿。
```

## 3. 用户操作

```txt
入口：
  点击剪映草稿导出后自动执行校验。

按钮/操作：
  1. 点击导出。
  2. 如果校验失败，查看失败列表。

弹窗/表单：
  校验失败详情弹窗。

成功反馈：
  校验通过，进入草稿导出步骤。

失败反馈：
  显示缺失素材列表，并阻止导出。
```

## 4. 要做什么功能

### 1. 校验路径和文件存在

怎么做：
- 输入：timeline clips。
- 输出：valid 或 failure list。
- 写什么数据：不写。
- 状态怎么变：validating -> passed/failed。
- 异常怎么处理：路径为空、文件不存在、权限不足都生成校验错误。
- 限制：校验在 main 服务做。

### 2. 校验素材格式

怎么做：
- 输入：filePath、mediaType。
- 输出：格式是否支持。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：格式不支持写 failure。
- 限制：第一版只校验剪映草稿支持的基础视频/音频/图片格式。

### 3. 输出失败列表

怎么做：
- 输入：校验错误。
- 输出：结构化 failures。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：无。
- 限制：每条必须包含 trackId、clipId、mediaType、sourceType、sourceId、path、reason。

### 4. 阻止导出

怎么做：
- 输入：failures。
- 输出：阻止进入 F-010-006。
- 写什么数据：不写草稿。
- 状态怎么变：export blocked。
- 异常怎么处理：用户修复后可重新校验。
- 限制：不能写半成品草稿目录。
