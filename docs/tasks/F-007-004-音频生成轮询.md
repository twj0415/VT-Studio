状态：等待用户确认

## 0. 快速理解

```txt
一句话：这里的“音频轮询”其实是轮询音频绑定状态，不是轮询 TTS 生成。
为什么现在做：批量绑定音频是异步模型任务，页面要自动更新结果。
做完后有什么用：绑定完成后卡片自动显示音色，失败后显示原因。
这一步不碰什么：不生成音频文件，不做 TTS，不轮询视频。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/cornerScape/pollingAudio`，但把命名解释清楚。

只做：
  1. 找出 audioBindStatus=running 的资产
  2. 每 3 秒轮询绑定状态
  3. 成功后刷新 boundAudios
  4. 失败后展示 audioBindError
  5. 离开页面停止轮询

不做：
  1. TTS 文件生成状态
  2. 音频文件上传状态
  3. 视频生成轮询
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/cornerScape/index.vue` | computed audioBindData 找 audioBindState=生成中，每 3 秒轮询 |
| `Toonflow-app/src/routes/cornerScape/pollingAudio.ts` | 查询 id in ids 且 audioBindState != 生成中 的资产 |

参考项目关键事实：

```txt
1. audioBindState=生成中 才轮询。
2. 每 3 秒请求一次。
3. 返回完成项后刷新 getAllAssets，以更新 relepedAudio。
4. 源码没有失败原因字段，VT Studio 要补。
5. 这不是音频文件生成。
```

## 4. 要做什么功能

### 1. 识别绑定中资产

怎么做：
- 输入：dataList。
- 输出：audioBindStatus=running 的 assetIds。
- 写什么数据：不写。
- 状态怎么变：有 running 才启动轮询。
- 异常怎么处理：无 id 的项忽略。
- 限制：使用明确状态枚举。

### 2. 轮询绑定状态

怎么做：
- 输入：assetIds。
- 输出：已完成或失败的资产状态。
- 写什么数据：不写。
- 状态怎么变：每 3 秒请求一次。
- 异常怎么处理：请求失败不清空列表，下一轮继续。
- 限制：页面卸载停止 timer。

### 3. 刷新绑定结果

怎么做：
- 输入：状态变化结果。
- 输出：最新 boundAudios。
- 写什么数据：不写。
- 状态怎么变：running -> succeeded/failed。
- 异常怎么处理：刷新失败提示。
- 限制：保留当前选择状态合理清理。
