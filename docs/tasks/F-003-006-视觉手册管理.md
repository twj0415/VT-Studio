# F-003-006 视觉手册管理

状态：等待用户确认  
所属菜单：M-003 项目管理  
对应功能文档：`docs/features/M-003-项目管理.md`

## 0. 快速理解

```txt
一句话：管理项目可选的视觉手册，包括新增、编辑、删除、预览和选择。
为什么现在做：项目创建必须选视觉手册，后续角色、场景、分镜提示词都会依赖视觉规范。
做完后有什么用：用户能维护自己的视觉风格手册，并在项目里选择使用。
这一步不碰什么：不做图片生成，不做导演手册，不做 Skill 市场。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的 VisualManual 能力，把视觉手册作为本地 Markdown 文件集合管理。

只做：
  1. 查询视觉手册列表
  2. 新增视觉手册
  3. 编辑视觉手册
  4. 删除视觉手册
  5. 上传/预览封面图
  6. 12 个 Markdown tab 编辑
  7. 项目选择视觉手册

不做：
  1. 导演手册
  2. 图片生成
  3. Skill 市场下载
  4. 自动优化手册内容
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/project/components/projectDialog.vue` | 项目弹窗中展示视觉手册卡片，支持选择、新增、编辑、删除、预览 |
| `Toonflow-app/src/routes/project/getVisualManual.ts` | 读取视觉手册列表 |
| `Toonflow-app/src/routes/project/addVisualManual.ts` | 新增视觉手册 |
| `Toonflow-app/src/routes/project/editVisualManual.ts` | 编辑视觉手册 |
| `Toonflow-app/src/routes/project/deleteVisualManual.ts` | 删除视觉手册 |

参考项目关键事实：

```txt
1. 视觉手册来自 skills/art_skills。
2. 字段包含 name、stylePath、images、data。
3. 编辑时 stylePath 禁用修改。
4. 封面上传用 image/*，前端转 base64，后端保存 jpg。
5. 视觉手册有 12 个 Markdown tab。
```

## 3. 用户操作

```txt
入口：
  新建/编辑项目弹窗 -> 视觉手册区域。

按钮/操作：
  1. 点击视觉手册卡片选择。
  2. 点击新增视觉手册。
  3. 点击编辑视觉手册。
  4. 点击删除视觉手册。
  5. 点击预览封面。
  6. 上传封面图。
  7. 编辑 Markdown tab。

弹窗/表单：
  视觉手册编辑弹窗。

成功反馈：
  新增成功、编辑成功、删除成功、选择成功。

失败反馈：
  名称为空、目录名非法、封面缺失、tab 内容缺失、路径越界、被项目引用不能删除。
```

## 4. 要做什么功能

### 1. 查询视觉手册列表

怎么做：
- 输入：无。
- 输出：视觉手册列表。
- 写什么数据：不写。
- 状态怎么变：项目弹窗显示卡片。
- 异常怎么处理：目录不存在时显示空状态或初始化默认手册。
- 限制：只读取 art skills 白名单目录。

### 2. 新增视觉手册

怎么做：
- 输入：name、stylePath、images、12 个 tab 内容。
- 输出：新增手册。
- 写什么数据：创建手册目录，写 Markdown 文件和封面图。
- 状态怎么变：新增后列表刷新并可选择。
- 异常怎么处理：目录名非法、目录已存在、封面为空、tab 为空时失败。
- 限制：stylePath 必须安全，不能路径穿越。

### 3. 编辑视觉手册

怎么做：
- 输入：手册 id/path、name、images、tab 内容。
- 输出：保存成功。
- 写什么数据：更新 Markdown 文件和封面。
- 状态怎么变：列表和当前选择刷新。
- 异常怎么处理：文件缺失、写入失败、路径越界时失败。
- 限制：编辑时 stylePath 禁止修改。

### 4. 删除视觉手册

怎么做：
- 输入：手册 id/path。
- 输出：删除成功。
- 写什么数据：删除手册目录或标记删除。
- 状态怎么变：列表移除。
- 异常怎么处理：被项目引用时阻止或要求先解除引用。
- 限制：递归删除前必须校验目录在 art skills 根目录内。

### 5. 预览封面

怎么做：
- 输入：封面路径。
- 输出：media URL。
- 写什么数据：不写。
- 状态怎么变：打开图片预览。
- 异常怎么处理：图片不存在时显示占位。
- 限制：走 CORE-012 媒体服务，不暴露绝对路径。

  2. 图片预览走受控 media URL。
  3. 路径安全更严格。

原因：

是否写入 04：
