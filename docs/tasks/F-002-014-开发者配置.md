# F-002-014 开发者配置

状态：等待用户确认  
所属菜单：M-002 设置  
对应功能文档：`docs/features/M-002-设置.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：给开发和排错用的开关放到设置里，比如 DevTools、AI DevTools、本地存储查看。
为什么现在做：后续功能复杂，开发期需要排查工具，但不能让普通用户误操作。
做完后有什么用：开发环境能快速打开调试工具，查看和清理本地存储。
这一步不碰什么：不做业务设置，不给生产环境默认暴露危险工具。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow devConfig，同时明确 VT Studio 生产环境的隐藏和保护规则。

只做：
  1. 打开 DevTools
  2. AI DevTools 开关
  3. localStorage 列表
  4. 搜索、新增、编辑、删除、复制、清空 localStorage
  5. JSON 格式化
  6. 开发环境显示控制

不做：
  1. 任意执行脚本
  2. 直接改数据库
  3. 直接改文件系统
  4. 生产环境默认显示危险按钮
```

  3. 搜索 localStorage。

## 4. 要做什么功能

### 1. 打开 DevTools

怎么做：
- 输入：无。
- 输出：DevTools 打开。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：非 Electron 或生产受限时提示不可用。
- 限制：生产环境默认隐藏或需要开发者模式。

### 2. AI DevTools 开关

怎么做：
- 输入：enabled。
- 输出：保存成功。
- 写什么数据：写设置项 `switchAiDevTool`。
- 状态怎么变：开关更新。
- 异常怎么处理：保存失败回滚。
- 限制：值统一为 boolean，兼容参考项目 `"1"/"0"`。

### 3. localStorage 列表和搜索

怎么做：
- 输入：搜索关键词。
- 输出：key/value 列表。
- 写什么数据：不写。
- 状态怎么变：过滤列表。
- 异常怎么处理：读取失败提示。
- 限制：敏感 key 默认脱敏显示。

### 4. 新增/编辑 localStorage

怎么做：
- 输入：key、value。
- 输出：保存成功。
- 写什么数据：写 localStorage。
- 状态怎么变：列表刷新。
- 异常怎么处理：key 为空失败；JSON 格式化失败只提示，不强制要求 value 是 JSON。
- 限制：不能编辑受保护 key，除非开发者模式二次确认。

### 5. 删除/清空 localStorage

怎么做：
- 输入：key 或清空确认。
- 输出：删除成功。
- 写什么数据：删除 localStorage 项。
- 状态怎么变：列表刷新，可能触发登录状态变化。
- 异常怎么处理：删除失败提示。
- 限制：清空前必须提示会影响登录、语言、外观、当前项目等状态。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 生产环境默认不暴露危险工具。
  2. 敏感 localStorage key 脱敏。
  3. 清空 localStorage 要明确影响范围。

原因：
  开发者工具是排错能力，不是普通业务能力。

是否写入 04：
  不需要。
```
