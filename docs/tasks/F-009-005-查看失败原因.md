# F-009-005 查看失败原因

状态：等待用户确认  
所属菜单：M-009 任务中心  
对应功能文档：`docs/features/M-009-任务中心.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：失败任务不能只显示“失败”，必须能看到失败原因。
为什么现在做：模型、文件、导出失败如果没原因，后续排查会很痛苦。
做完后有什么用：用户能直接知道是 API Key、模型参数、文件路径还是素材缺失导致失败。
这一步不碰什么：不做重试，不展开完整日志，不暴露敏感信息。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 失败 tooltip，并加强敏感信息保护。

只做：
  1. 失败原因列
  2. 失败状态 tooltip
  3. 无失败原因兜底文案
  4. 敏感信息脱敏规则

不做：
  1. 任务日志详情
  2. 重试任务
  3. 打开本地日志文件
  4. 修改失败原因
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/task/index.vue` | 失败状态用红色，hover tooltip 展示 `row.reason` |
| `Toonflow-app/src/utils/taskRecord.ts` | 失败时 `done(-1, reason)` 写入 `o_tasks.reason` |
| `Toonflow-app/src/utils/ai.ts` | 模型失败时把 `u.error(e).message` 写入任务失败原因 |

参考项目关键事实：

```txt
1. 表格本身有 reason 列。
2. 状态为生成失败时，状态文字外包 tooltip。
3. reason 为空时显示 noFailReason。
4. Toonflow 没看到敏感信息脱敏逻辑；VT Studio 必须补。
```

## 3. 用户操作

```txt
入口：
  任务中心任务表格。

按钮/操作：
  1. 查看失败原因列。
  2. 鼠标悬停失败状态。

弹窗/表单：
  无。

成功反馈：
  显示失败原因。

失败反馈：
  失败原因为空时显示“无失败原因”。
```

## 4. 要做什么功能

### 1. 失败原因列

怎么做：
- 输入：failReason。
- 输出：表格列文本，过长省略。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：空值显示“无失败原因”。
- 限制：不能直接显示密钥、token、完整 stack。

### 2. 失败 tooltip

怎么做：
- 输入：status=failed、failReason。
- 输出：状态标签 hover 展示原因。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：无原因显示兜底文案。
- 限制：只有失败状态需要 tooltip。

### 3. 失败原因脱敏

怎么做：
- 输入：服务层错误文本。
- 输出：脱敏后的 failReason。
- 写什么数据：不写；任务写入时由任务服务写脱敏结果。
- 状态怎么变：无。
- 异常怎么处理：脱敏异常时返回通用错误文案。
- 限制：apiKey、Authorization、Bearer、完整本地用户目录路径要处理。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  失败原因做脱敏，不把原始 detail 全量返回页面。

原因：
  本地路径、API Key、供应商响应可能包含敏感内容。

是否写入 04：
  如果 04 已记录错误 detail 只进日志，不重复；否则补充。
```

## 8. 验收

```txt
1. 失败任务行能看到失败原因。
2. 失败状态 hover 有 tooltip。
3. reason 为空时显示无失败原因。
4. 成功和进行中任务不展示失败 tooltip。
5. API Key、token 不会显示在页面。
6. 页面不直接读取日志文件。
```
