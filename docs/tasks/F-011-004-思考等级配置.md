# F-011-004 思考等级配置

状态：等待用户确认  
所属菜单：M-011 剧本 Agent  
对应功能文档：`docs/features/M-011-剧本Agent.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：如果剧本 Agent 模型支持 think，就显示思考等级切换。
为什么现在做：不同模型对推理深度支持不同，不能强行给所有模型显示。
做完后有什么用：用户可以按需要选择关闭、轻度、深度、极限思考。
这一步不碰什么：不改模型供应商配置，不训练模型，不改 Agent prompt。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 根据模型详情显示思考等级按钮，并把配置发给 Agent 会话。

只做：
  1. 查询 scriptAgent 模型详情
  2. 判断是否支持 think
  3. 显示思考等级按钮
  4. 切换 thinkLevel
  5. 通知 Agent 会话更新配置

不做：
  1. 模型供应商配置
  2. Agent 模型映射配置
  3. 模型调用协议改造
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/scriptAgent/index.vue` | 调 `/project/getModelDetails key=scriptAgent`，判断是否显示思考按钮 |

参考项目关键事实：

```txt
1. 思考等级选项：关闭、轻度、深度、极限，对应 0/1/2/3。
2. 只有模型支持 think 时显示。
3. 源码里 thinlLevel 有拼写错误，VT Studio 不能照搬。
4. 切换配置没有明确失败提示，VT Studio 要补。
```

## 3. 用户操作

```txt
入口：
  剧本 Agent 聊天区设置或思考按钮。

按钮/操作：
  选择关闭、轻度、深度、极限。

弹窗/表单：
  弹出菜单。

成功反馈：
  当前思考等级更新。

失败反馈：
  更新失败提示并回滚。
```

## 4. 要做什么功能

### 1. 判断模型是否支持思考

怎么做：
- 输入：projectId、agentKey=scriptAgent。
- 输出：模型详情和 think 支持状态。
- 写什么数据：不写。
- 状态怎么变：supportsThinking=true/false。
- 异常怎么处理：查询失败时隐藏思考按钮并提示模型配置异常。
- 限制：不能默认所有模型都支持。

### 2. 切换思考等级

怎么做：
- 输入：thinkLevel。
- 输出：Agent 会话配置更新。
- 写什么数据：可写会话内存状态，不写业务表。
- 状态怎么变：当前等级更新。
- 异常怎么处理：更新失败回滚到旧等级。
- 限制：thinkLevel 只能是 0/1/2/3。

### 3. 发送给 Agent 服务

怎么做：
- 输入：thinkLevel。
- 输出：后续模型调用使用新配置。
- 写什么数据：不写。
- 状态怎么变：agent session thinkConfig 更新。
- 异常怎么处理：会话未连接时提示无法更新。
- 限制：配置影响后续生成，不改已经生成中的消息。
