# F-008-003 刷新流程数据

状态：等待用户确认  
所属菜单：M-008 生产工作台  
对应功能文档：`docs/features/M-008-生产工作台.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：生产页加刷新按钮，从数据库重新构建当前剧本的流程数据。
为什么现在做：资产、分镜、视频可能被其他功能更新，生产页要能重新同步真实数据。
做完后有什么用：用户能手动刷新，避免页面显示旧数据。
这一步不碰什么：不重新生成内容，不保存修改，不重置剧本选择。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `refFlowData` 刷新流程。

只做：
  1. 顶部刷新按钮
  2. 重新调用 getFlowData
  3. 用数据库最新数据覆盖展示
  4. 刷新后自动布局或保持布局规则
  5. 保留当前剧本选择

不做：
  1. 保存流程
  2. 生成分镜
  3. 生成视频
  4. 清空工作区
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/production/index.vue` | 顶部刷新按钮调用 `refFlowData`，重新获取 flowData 并布局 |
| `Toonflow-app/src/routes/production/getFlowData.ts` | 从 o_agentWorkData、o_script、o_assets、o_storyboard 重建 flowData |

参考项目关键事实：

```txt
1. script 始终使用最新 o_script.content。
2. assets 会用数据库最新资产覆盖 flowData.assets。
3. storyboard 会从 o_storyboard 重建。
4. agentWorkData 是工作区快照，不是唯一业务事实。
5. 刷新后会自动布局。
```

```txt
  点击刷新。

弹窗/表单：

成功反馈：

失败反馈：
  刷新失败提示，保留旧数据。
```

## 4. 要做什么功能

### 1. 手动刷新 flowData

怎么做：
- 输入：projectId、scriptId。
- 输出：最新 flowData。
- 写什么数据：不写。
- 状态怎么变：refreshing=true -> false。
- 异常怎么处理：失败提示并保留旧 flowData。
- 限制：不重置当前剧本选择。

### 2. 数据事实覆盖

怎么做：
- 输入：agentWorkData 快照和业务表。
- 输出：合并后的 flowData。
- 写什么数据：不写。
- 状态怎么变：节点数据更新。
- 异常怎么处理：业务表缺失时显示缺失状态。
- 限制：业务事实以 scripts/assets/storyboards/videos 为准。

### 3. 刷新后布局

怎么做：
- 输入：新 nodes。
- 输出：布局后的节点位置。
- 写什么数据：不写。
- 状态怎么变：画布重新 fit 或保持定位策略。
- 异常怎么处理：布局失败保留旧位置。
- 限制：不覆盖用户已保存的位置，除非点击自动布局。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  刷新失败保留旧数据并提示。

原因：
  避免用户一次刷新失败导致流程图清空。

是否写入 04：
  不需要，属于体验增强。
```

## 8. 验收

```txt
1. 点击刷新会重新获取 flowData。
2. 刷新保留当前剧本选择。
3. 刷新后脚本内容来自最新 scripts。
4. 刷新后资产来自最新 assets。
5. 刷新后分镜来自最新 storyboards。
6. 刷新失败保留旧数据。
7. 页面不直接访问 SQLite。
```
