# F-006-007 批量生成提示词

状态：等待用户确认  
所属菜单：M-006 资产中心  
对应功能文档：`docs/features/M-006-资产中心.md`

## 0. 快速理解

```txt
一句话：选中角色/场景/道具后，用模型批量生成或润色图片提示词。
为什么现在做：图片生成前需要高质量提示词，不能让用户每个资产手写。
做完后有什么用：资产会自动填充 prompt，失败能看到原因。
这一步不碰什么：不生成图片，不处理 clip/audio，不做 ComfyUI workflow。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/assetsGenerate/batchPolishAssetsPrompt`。

只做：
  1. 角色/场景/道具批量选择
  2. 批量生成提示词入口
  3. 并发控制
  4. 根据类型读取视觉手册
  5. 调用文本模型生成 prompt
  6. 写 prompt_status 和 prompt_error
  7. 轮询提示词状态

不做：
  1. 批量生成图片
  2. 单资产生成图片
  3. clip/audio 提示词
  4. ComfyUI 工作流
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/assets/index.vue` | role/tool/scene 顶部批量生成 -> 生成提示词 |
| `Toonflow-app/src/routes/assetsGenerate/batchPolishAssetsPrompt.ts` | 按 concurrentCount 并发生成 prompt，写 promptState/promptErrorReason |
| `Toonflow-app/src/routes/assets/pollingPromptAssets.ts` | 轮询 promptState 不为生成中的资产 |
| `Toonflow-app/src/routes/assetsGenerate/polishAssetsPrompt.ts` | 单个提示词润色逻辑，读取视觉手册 |

参考项目关键事实：

```txt
1. 只支持 role/tool/scene。
2. 视觉手册 key：role=art_character，scene=art_scene，tool=art_prop。
3. 子/衍生资产使用 derivative key。
4. 模型使用 universalAi。
5. 批量接口后端要求 otherTextPrompt 必填，但资产中心调用没传，这是源码冲突；VT Studio 必须修正为可选或传空字符串。
6. promptState 状态混用“生成失败”和“失败”，VT Studio 必须统一。
```

## 3. 用户操作

```txt
入口：
  资产中心 role/scene/prop tab -> 勾选资产 -> 批量生成 -> 生成提示词。

按钮/操作：
  1. 勾选资产。
  2. 点击生成提示词。

弹窗/表单：
  可选确认弹窗或附加指令输入。

成功反馈：
  资产 prompt 状态变生成中，完成后展示 prompt。

失败反馈：
  未选中、视觉手册缺失、模型失败、输出为空。
```

## 4. 要做什么功能

### 1. 启动批量提示词生成

怎么做：
- 输入：assetIds、otherTextPrompt 可选。
- 输出：启动结果。
- 写什么数据：assets.prompt_status=running。
- 状态怎么变：idle/failed -> running。
- 异常怎么处理：未选中或包含生成中资产时提示。
- 限制：只允许 role/scene/prop。

### 2. 根据类型读取视觉手册

怎么做：
- 输入：asset type、是否衍生资产、project visualManual。
- 输出：对应视觉手册文本。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：手册缺失写 prompt_status=failed 和原因。
- 限制：道具正式类型 prop 需要兼容 Toonflow tool。

### 3. 调用文本模型生成 prompt

怎么做：
- 输入：资产名称、描述、视觉手册、附加指令。
- 输出：prompt 文本。
- 写什么数据：成功写 assets.prompt，失败写 prompt_error。
- 状态怎么变：running -> succeeded/failed。
- 异常怎么处理：模型失败、空输出、超时分别写原因。
- 限制：renderer 不直接调用模型。

### 4. 轮询提示词结果

怎么做：
- 输入：running assetIds。
- 输出：非 running 的资产状态。
- 写什么数据：不写。
- 状态怎么变：页面更新。
- 异常怎么处理：轮询失败保留下一轮。
- 限制：轮询只读状态，不重新执行生成。
