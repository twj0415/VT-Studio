# F-002-004 模型专用提示词模板

状态：已完成  
所属菜单：M-002 设置  
对应功能文档：`docs/features/M-002-设置.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：给图片/视频模型配置专用提示词模板，让生成链路能按模型拿到更合适的 prompt。
为什么现在做：后续图片、视频生成不能每个页面自己写 fallback，否则模型效果和排错都会失控。
做完后有什么用：某个图片/视频模型可以绑定专用模板；没有绑定时走统一 fallback。
这一步不碰什么：不做文本模型，不做正式生成，不改供应商接入，不改内置核心提示词内容。
```

## 2. 当前专业设计

```txt
入口定位：
  高级能力，放在设置 -> 开发者模式/高级模型能力下。
  普通用户主要使用“模型服务”和“提示词管理”，不暴露复杂映射概念。

数据边界：
  F-002-006 的 prompts 表只保存内置核心提示词：事件提取、资产提取、视频提示词生成、音色绑定。
  F-002-004 新建 model_prompt_templates 和 model_prompt_mappings。
  不复用 prompts 做模型模板库，避免污染内置 prompt。

模型范围：
  第一版只支持 image/video。
  text 模型不做，因为 Agent 和文本生成已经由模型服务默认文本模型 + Agent 高级覆盖管理。

绑定粒度：
  connection_id + model_name + model_type + model_mode。
  model_mode 可为空，表示该模型默认模板。
  model_mode 预留给视频模型，避免后续视频模式接入时返工。
```

## 5. 要做什么功能

### 1. 查询模型映射列表

怎么做：
- 输入：无。
- 输出：按连接分组的 image/video 模型，每个模型带绑定状态和 fallback 状态。
- 写什么数据：不写。
- 状态怎么变：loading -> 分组列表。
- 异常怎么处理：模型服务配置损坏时显示错误。
- 限制：不展示 text 模型。

### 2. 查询模板列表

怎么做：
- 输入：可选 `modelType=image/video`。
- 输出：模板列表。
- 写什么数据：不写。
- 状态怎么变：弹窗打开或刷新时加载。
- 异常怎么处理：空列表显示空状态。
- 限制：image 只能绑定 imagePrompt，video 只能绑定 videoPrompt。

### 3. 新增模板

怎么做：
- 输入：`name`、`type`、`content`。
- 输出：新模板。
- 写什么数据：写 model_prompt_templates。
- 状态怎么变：新增后出现在模板列表。
- 异常怎么处理：name 为空、content 为空、type 不支持、同类型重名失败。
- 限制：第一版模板均为用户模板，不能创建 builtin。

### 4. 编辑模板

怎么做：
- 输入：`id`、`name`、`type`、`content`。
- 输出：更新后的模板。
- 写什么数据：更新 model_prompt_templates。
- 状态怎么变：已绑定该模板的模型后续生成使用新内容。
- 异常怎么处理：模板不存在、内容为空、同类型重名失败。
- 限制：编辑类型时必须保证现有引用仍兼容；第一版建议不允许把被引用模板改成另一类型。

### 5. 删除模板

怎么做：
- 输入：`id`。
- 输出：删除成功。
- 写什么数据：删除 model_prompt_templates。
- 状态怎么变：模板从列表移除。
- 异常怎么处理：模板不存在、被引用时直接阻止并返回引用模型。
- 限制：第一版不做强制删除，不自动清绑定。

### 6. 绑定模板

怎么做：
- 输入：`connectionId`、`modelName`、`modelType`、`modelMode?`、`templateId`。
- 输出：绑定后的映射。
- 写什么数据：upsert model_prompt_mappings。
- 状态怎么变：模型绑定状态立即更新。
- 异常怎么处理：模型不存在、模板不存在、类型不匹配、连接不存在。
- 限制：同一个 connectionId + modelName + modelType + modelMode 只能有一条有效绑定。

### 7. 清除绑定

怎么做：
- 输入：`connectionId`、`modelName`、`modelType`、`modelMode?`。
- 输出：清除成功。
- 写什么数据：删除对应 model_prompt_mappings。
- 状态怎么变：模型回到 fallback 状态。
- 异常怎么处理：不存在时视为成功。
- 限制：不删除模板。

### 8. 失效状态

怎么做：
- 输入：模型列表、模板列表、映射列表。
- 输出：正常、未绑定、模板失效、模型失效。
- 写什么数据：不写。
- 状态怎么变：页面显示提示。
- 异常怎么处理：生成前必须阻断失效映射。
- 限制：不能静默忽略。

## 7. fallback 规则

```txt
image 模型：
  有绑定模板 -> 使用绑定模板。
  无绑定模板 -> 使用内置通用图片模板；第一版如果没有内置通用图片模板，则返回“未绑定，生成任务需显式提示词”。

video 模型：
  有 mode 专用绑定 -> 使用 mode 专用模板。
  无 mode 绑定但有模型默认绑定 -> 使用模型默认模板。
  无模型默认绑定 -> 使用 F-002-006 的 videoPromptGeneration。

注意：
  fallback 只在 service 中统一处理。
  生成页面不能各自写 fallback 文案。
```

## 9. 偏差

```txt
和 Toonflow 不同的地方：
  1. 不使用 fileName/path 作为正式字段。
  2. 不复用 prompts 表做模型模板库。
  3. 不放普通用户主入口。
  4. 删除被引用模板直接阻止，不做强制删除。
  5. 预留 model_mode。

原因：
  VT Studio 已经把模型服务和内置提示词管理拆开，继续照搬 Toonflow 会破坏当前架构。

是否写入 04：
  不需要，属于架构收口和安全增强。
```
