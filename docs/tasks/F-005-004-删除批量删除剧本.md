# F-005-004 删除/批量删除剧本

状态：等待用户确认  
所属菜单：M-005 剧本  
对应功能文档：`docs/features/M-005-剧本.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：删除剧本不是只删卡片，要同步清理资产关系、Agent 工作区、分镜和视频。
为什么现在做：剧本是生产链路的根，删除不完整会留下大量脏数据。
做完后有什么用：用户能安全删除无用剧本，不影响其他剧本数据。
这一步不碰什么：不删除项目，不删除原文，不清理无关资产库。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/script/delScript`，并补齐提取中保护和文件清理规则。

只做：
  1. 单条删除剧本
  2. 批量删除剧本
  3. 删除确认
  4. 提取中保护
  5. 级联清理剧本相关数据
  6. 删除后刷新列表和缓存

不做：
  1. 项目删除
  2. 原文删除
  3. 清空资产中心全部资产
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/script/index.vue` | 单删和批删都有确认；批量删除提取中的剧本会提示 |
| `Toonflow-app/src/routes/script/delScript.ts` | 删除 o_script、o_scriptAssets、o_agentWorkData、o_storyboard、o_assets2Storyboard、o_video |
| `Toonflow-web/src/stores/imageListCache.ts` | 删除后清理缓存 |

参考项目关键事实：

```txt
1. 删除剧本影响 Agent 工作数据、分镜、视频和剧本资产关系。
2. 批量删除时提取中的剧本不允许删。
3. 单条删除源码没有同等提取中保护，VT Studio 必须补。
4. 删除会尝试删除 storyboard.filePath 对应文件，但清理范围需要明确。
```

## 3. 用户操作

```txt
入口：
  剧本卡片删除图标；顶部批量删除按钮。

按钮/操作：
  1. 勾选剧本。
  2. 点击批量删除。
  3. 点击单卡删除。
  4. 二次确认。

弹窗/表单：
  删除确认框，说明影响范围。

成功反馈：
  删除成功，刷新列表。

失败反馈：
  未选择、剧本提取中、删除失败、文件清理失败。
```

## 4. 要做什么功能

### 1. 单条删除剧本

怎么做：
- 输入：scriptId。
- 输出：删除结果。
- 写什么数据：删除 scripts 和关联数据。
- 状态怎么变：删除中 -> 刷新列表。
- 异常怎么处理：提取中禁止删除；事务失败回滚。
- 限制：必须二次确认并说明影响范围。

### 2. 批量删除剧本

怎么做：
- 输入：scriptIds。
- 输出：删除结果。
- 写什么数据：批量删除 scripts 和关联数据。
- 状态怎么变：删除中 -> 清空选择 -> 刷新列表。
- 异常怎么处理：包含提取中剧本时阻止。
- 限制：无选中时禁用或提示。

### 3. 级联清理

怎么做：
- 输入：scriptIds。
- 输出：级联清理结果。
- 写什么数据：删除剧本资产关系、Agent 工作区、分镜、分镜资产关系、视频轨道/视频候选。
- 状态怎么变：事务内完成。
- 异常怎么处理：数据库失败回滚；文件删除失败按策略提示。
- 限制：只清理这些剧本相关数据，不删无关资产库父资产。

### 4. 清理文件和缓存

怎么做：
- 输入：分镜文件路径、视频文件路径。
- 输出：文件删除结果。
- 写什么数据：文件系统删除由 main 服务执行。
- 状态怎么变：缓存刷新。
- 异常怎么处理：文件不存在可忽略，权限失败要提示。
- 限制：不能让 renderer 直接删文件。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  单条删除也保护提取中的剧本。

原因：
  批量和单条删除行为必须一致，避免后台任务写入已删除剧本。

是否写入 04：
  写入，属于修正源码保护不完整。
```
