# F-011-007 编辑故事骨架

状态：等待用户确认  
所属菜单：M-011 剧本 Agent  
对应功能文档：`docs/features/M-011-剧本Agent.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：右侧“故事骨架”tab 支持查看 Markdown 和手动编辑保存。
为什么现在做：Agent 生成的骨架需要用户可检查、可修正。
做完后有什么用：用户不用重新对话，也能直接调整故事结构。
这一步不碰什么：不生成新骨架，不编辑改编策略，不改剧本列表。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 故事骨架 tab 的 MdPreview + 编辑弹窗。

只做：
  1. 故事骨架 Markdown 展示
  2. 空状态
  3. 编辑按钮
  4. Markdown 编辑弹窗
  5. 保存到 agent_work_data

不做：
  1. Agent 自动生成
  2. 改编策略编辑
  3. 剧本卡片编辑
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/scriptAgent/index.vue` | storySkeleton tab 有内容用 MdPreview，无内容 t-empty，编辑打开 editMdPreivew |
| `Toonflow-app/src/routes/scriptAgent/updateData.ts` | 更新 `o_agentWorkData.data.storySkeleton` |

参考项目关键事实：

```txt
1. storySkeleton 是 planData 顶层字段。
2. 保存调用 updateData。
3. Markdown 编辑器阻止图片拖拽上传。
4. 源码组件 editMdPreivew 有拼写问题，VT Studio 不能照搬。
```

## 3. 用户操作

```txt
入口：
  剧本 Agent 右侧 -> 故事骨架 tab。

按钮/操作：
  1. 点击编辑。
  2. 修改 Markdown。
  3. 点击保存。

弹窗/表单：
  Markdown 编辑弹窗。

成功反馈：
  保存成功，预览更新。

失败反馈：
  保存失败提示。
```

## 4. 要做什么功能

### 1. 展示故事骨架

怎么做：
- 输入：planData.storySkeleton。
- 输出：Markdown 预览或空状态。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：Markdown 渲染失败显示纯文本兜底。
- 限制：不允许 Markdown 图片拖拽上传。

### 2. 编辑并保存故事骨架

怎么做：
- 输入：Markdown 文本。
- 输出：更新后的 storySkeleton。
- 写什么数据：更新 agent_work_data.data.storySkeleton。
- 状态怎么变：保存中 -> 成功关闭弹窗。
- 异常怎么处理：保存失败保留弹窗内容。
- 限制：只更新 storySkeleton，不覆盖 scripts。
