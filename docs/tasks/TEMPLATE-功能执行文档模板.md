# 功能ID 功能名称

状态：等待用户确认  
所属菜单：M-xxx  
对应功能文档：`docs/features/M-xxx-xxx.md`  
原则：先确认本文档，再改代码

文件命名规则：

```txt
正式功能 task 必须用真实功能 ID 命名，并且一个 features 文件只对应一个前缀：
  docs/features/00-公共底层.md        -> CORE-xxx
  docs/features/M-001-全局导航与项目入口.md -> F-001-xxx
  docs/features/M-002-设置.md         -> F-002-xxx
  docs/features/M-003-项目管理.md     -> F-003-xxx
  docs/features/M-004-小说原文.md     -> F-004-xxx
  docs/features/M-005-剧本.md         -> F-005-xxx
  docs/features/M-006-资产中心.md     -> F-006-xxx
  docs/features/M-007-角景音频绑定.md -> F-007-xxx
  docs/features/M-008-生产工作台.md   -> F-008-xxx
  docs/features/M-009-任务中心.md     -> F-009-xxx
  docs/features/M-010-导出.md         -> F-010-xxx
  docs/features/M-011-剧本Agent.md    -> F-011-xxx

示例：
  CORE-012-本地媒体访问服务.md
  F-002-003-模型供应商配置.md

模板、扫描、搭框架这类非真实功能不能占用数字编号：
  TEMPLATE-功能执行文档模板.md
  DOC-001-扫描参考项目菜单与功能.md
  FRAME-001-搭建可运行项目框架.md

不允许再用 013、014、015 这种流水号命名正式功能。
不允许同一个 features 文件下混多个前缀。
不允许把模板、扫描、框架准备文档算进正式功能前缀。
```
