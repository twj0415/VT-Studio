# F-001-011 版本检查

状态：等待用户确认  
所属菜单：M-001 全局导航与项目入口

## 0. 快速理解

```txt
一句话：检查当前应用版本是否有更新，有更新就显示提醒和设置红点。
为什么现在做：版本检查会影响全局红点和关于页入口，必须先把规则定清楚。
做完后有什么用：用户能知道是否有新版本，但开发期不自动乱联网。
这一步不碰什么：不做下载安装细节，不写更新器。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 版本检查语义，但清理 Toonflow 更新源，并把开发期默认设为手动检查。

只做：
  1. 手动版本检查入口。
  2. 检查中状态。
  3. 有更新时设置 needUpdate。
  4. 有更新时提示用户。
  5. 打开设置 about 的入口。
  6. 检查失败提示。
  7. 自动检查规则先关闭或可配置。

不做：
  1. 不使用 Toonflow OSS update.json。
  2. 不做下载安装。
  3. 不做增量更新。
  4. 不在开发期默认自动联网。
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/pages/workbench/index.vue` | `startVersionCheck()` 立即检查并每 2 分钟检查；needUpdate 后停止轮询 |
| `Toonflow-app/src/routes/setting/about/checkUpdate.ts` | 根据 source/url 获取 update.json，比对 Major/Minor/Patch |
| `Toonflow-app/src/routes/other/getVersion.ts` | 获取当前应用版本 |
| `Toonflow-web/src/components/setting/components/about.vue` | 关于页展示版本并触发检查更新 |

参考项目关键事实：

```txt
1. 启动后立即 checkVersion。
2. 每 2 分钟检查一次。
3. `needUpdate=true` 后停止轮询。
4. Major/Minor 高于当前版本时 `reinstall=true`。
5. Patch 高于当前版本时 `reinstall=false`。
6. 默认更新源是 Toonflow OSS：`https://toonflow.oss-cn-beijing.aliyuncs.com/update.json`。
7. 源码没有看到版本检查失败时的完整 UI 提示。
```

## 3. 用户操作

```txt
入口：
  主界面版本检查入口，或设置 about 页面。

按钮/操作：
  1. 点击检查版本。
  2. 有更新提示时点击打开设置。
  3. 关闭更新提示。

弹窗/表单：
  更新提示通知或小弹窗。

成功反馈：
  显示当前已是最新，或提示发现新版本。

失败反馈：
  网络失败、更新源未配置、版本信息格式错误都要提示。
```

## 4. 要做什么功能

### 1. 手动检查版本

怎么做：
- 输入：用户点击检查版本。
- 输出：检查结果。
- 写什么数据：写前端状态 `checkingUpdate`、`needUpdate`。
- 状态怎么变：idle -> checking -> success/failed。
- 异常怎么处理：失败显示 msg，不影响主流程。
- 限制：开发期默认不自动发外网请求。

### 2. 更新源读取

怎么做：
- 输入：设置中的 update source/url。
- 输出：更新信息 JSON。
- 写什么数据：不写业务数据。
- 状态怎么变：无。
- 异常怎么处理：源未配置或格式错误返回失败。
- 限制：不能使用 Toonflow 默认 OSS 地址。

### 3. 版本号比对

怎么做：
- 输入：当前版本、远端版本。
- 输出：是否有更新、是否需要重装。
- 写什么数据：不写。
- 状态怎么变：更新 `needUpdate`。
- 异常怎么处理：版本号格式不合法时返回失败。
- 限制：按语义版本 Major/Minor/Patch 比较，不用字符串直接比较。

### 4. 更新提示

怎么做：
- 输入：检查结果。
- 输出：提示用户。
- 写什么数据：写 `needUpdate`。
- 状态怎么变：有更新时显示红点。
- 异常怎么处理：提示显示失败不影响检查结果。
- 限制：更新提示里的打开设置要切到 `about`。

### 5. 自动检查控制

怎么做：
- 输入：设置里的自动检查开关。
- 输出：是否启动定时检查。
- 写什么数据：读写设置。
- 状态怎么变：开启则定时检查，发现更新后停止。
- 异常怎么处理：检查失败只提示或记录，不循环弹窗。
- 限制：默认关闭自动检查；如果开启，轮询间隔不能低于参考项目 2 分钟。
