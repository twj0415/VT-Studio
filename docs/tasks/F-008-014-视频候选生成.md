# F-008-014 视频候选生成

状态：等待用户确认

## 0. 快速理解

```txt
一句话：根据轨道 prompt 和参考素材生成视频候选，用户再选择其中一个作为该轨道最终视频。
为什么现在做：剪映导出和预览都需要稳定的已选视频，而不是临时生成结果。
做完后有什么用：每条轨道可以有多个候选视频，并明确哪个被选中。
这一步不碰什么：不导出剪映，不做完整视频编辑器，不生成 TTS。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/production/workbench/generateVideo`、`batchGenerateVideo`、`checkVideoStateList`、`selectVideo`、`delVideo`。

只做：
  1. 单轨生成视频候选
  2. 批量生成视频候选
  3. 根据 mode 组织参考素材
  4. 轮询视频状态
  5. 选择候选视频
  6. 删除候选视频
  7. 下载候选视频

不做：
  1. 剪映导出
  2. WebAV 合成
  3. 视频轨道新增删除
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-app/src/routes/production/workbench/generateVideo.ts` | 插入 o_video state=生成中，调用视频模型，保存视频 |
| `Toonflow-app/src/routes/production/workbench/batchGenerateVideo.ts` | 批量生成视频 |
| `Toonflow-app/src/routes/production/workbench/checkVideoStateList.ts` | 轮询生成成功/生成失败的视频 |
| `Toonflow-app/src/routes/production/workbench/selectVideo.ts` | 写 o_videoTrack.videoId |
| `Toonflow-app/src/routes/production/workbench/delVideo.ts` | 删除 o_video 并清空引用 |

参考项目关键事实：

```txt
1. generateVideo 立即插入 o_video，返回 videoId。
2. 视频模型参数包括 prompt/referenceList/mode/duration/aspectRatio/resolution/audio。
3. 成功状态源码写“生成成功”，前端有时用“已完成”，VT Studio 必须统一。
4. generateVideo assets 查询为空时可能访问 filePath.type 报错，VT Studio 必须防空。
5. delVideo 只删数据库，不删视频文件，VT Studio 必须补。
```

## 3. 用户操作

```txt
入口：
  视频工作台 -> 视频生成模式。

按钮/操作：
  1. 单轨生成视频。
  2. 批量生成视频。
  3. 选择候选视频。
  4. 删除候选视频。
  5. 下载单个或批量视频。

弹窗/表单：
  视频预览弹窗；删除确认框。

成功反馈：
  视频生成中，完成后可播放和选择。

失败反馈：
  缺 prompt、缺参考素材、模型失败、文件保存失败。
```

## 4. 要做什么功能

### 1. 生成视频候选

怎么做：
- 输入：trackId、prompt、model、mode、duration、resolution、参考素材。
- 输出：videoId。
- 写什么数据：插入 videos 状态 running。
- 状态怎么变：创建候选 -> running。
- 异常怎么处理：参数缺失时不创建候选。
- 限制：只生成当前剧本轨道候选。

### 2. 调用视频模型并保存

怎么做：
- 输入：VideoGenerationRequest。
- 输出：视频文件。
- 写什么数据：写视频文件，更新 videos.file_path/status/error。
- 状态怎么变：running -> succeeded/failed。
- 异常怎么处理：供应商失败、素材缺失、保存失败写 error。
- 限制：不同协议由 Model Adapter 处理，业务页不关心 SDK。

### 3. 轮询视频状态

怎么做：
- 输入：running videoIds。
- 输出：完成或失败视频。
- 写什么数据：不写。
- 状态怎么变：页面更新。
- 异常怎么处理：轮询失败继续下一轮。
- 限制：不重复生成。

### 4. 选择和删除候选

怎么做：
- 输入：trackId、videoId。
- 输出：已选视频或删除结果。
- 写什么数据：选择时写 video_tracks.selected_video_id；删除时删 videos 和文件并清引用。
- 状态怎么变：selected 更新或候选移除。
- 异常怎么处理：video 不属于 track 时拒绝。
- 限制：删除候选必须删除文件。
