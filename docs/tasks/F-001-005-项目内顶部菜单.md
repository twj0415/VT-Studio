# F-001-005 项目内顶部菜单

状态：等待用户确认  
所属菜单：M-001 全局导航与项目入口  
对应功能文档：`docs/features/M-001-全局导航与项目入口.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：打开项目后，在顶部显示小说、剧本 Agent、剧本、角景、生产、资产这些项目内入口。
为什么现在做：项目功能必须由当前项目驱动，不能在没项目时乱进页面。
做完后有什么用：用户进入项目后能按项目上下文切换功能模块。
这一步不碰什么：不做项目类型过滤细节，过滤归 F-001-006。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的项目内顶部菜单结构。

只做：
  1. 有 currentProject 时显示顶部菜单。
  2. 显示项目名称。
  3. 展示项目内菜单入口。
  4. 点击菜单跳对应路由。
  5. 当前项目缺失时隐藏顶部菜单。

不做：
  1. 不做项目打开逻辑。
  2. 不做项目类型过滤。
  3. 不做菜单对应页面业务。
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/pages/workbench/index.vue` | `project?.id` 存在时显示顶部项目菜单 |
| `Toonflow-web/src/router/index.ts` | 顶部菜单路由包含 `/novel`、`/scriptAgent`、`/script`、`/cornerScape`、`/production`、`/assets` |

参考项目关键事实：

```txt
1. 没有当前项目时，不显示顶部项目菜单。
2. 左侧标题有项目名，空时显示“选择项目”。
3. 顶部菜单里包含小说、剧本 Agent、剧本、角景、生产、资产。
4. 资产入口和前面菜单之间有分隔线。
5. 测试页 `/test` 不应该在正式导航暴露。
```

## 3. 用户操作

```txt
入口：
  打开项目后出现的顶部菜单。

按钮/操作：
  1. 点击小说。
  2. 点击剧本 Agent。
  3. 点击剧本。
  4. 点击角景。
  5. 点击生产。
  6. 点击资产。

弹窗/表单：
  无。

成功反馈：
  路由切换，顶部菜单高亮更新。

失败反馈：
  currentProject 丢失时提示并回项目页。
```

## 4. 要做什么功能

### 1. 顶部菜单显示控制

怎么做：
- 输入：currentProject。
- 输出：是否显示顶部菜单。
- 写什么数据：不写。
- 状态怎么变：有项目显示，无项目隐藏。
- 异常怎么处理：项目 id 缺失时隐藏并回项目页。
- 限制：不能用 localStorage 里残留项目直接显示，必须经过项目上下文确认。

### 2. 项目名称展示

怎么做：
- 输入：currentProject.name。
- 输出：顶部或侧边区域显示项目名。
- 写什么数据：不写。
- 状态怎么变：打开项目后更新。
- 异常怎么处理：名称为空显示“选择项目”或默认占位。
- 限制：名称过长要截断或 tooltip，不能挤压菜单。

### 3. 项目内菜单跳转

怎么做：
- 输入：点击菜单项。
- 输出：跳对应路由。
- 写什么数据：不写。
- 状态怎么变：route.path 更新，菜单高亮更新。
- 异常怎么处理：跳转失败提示。
- 限制：进入项目内页面前必须有 currentProject。

### 4. 项目内菜单高亮

怎么做：
- 输入：当前 route.path。
- 输出：activeProjectMenu。
- 写什么数据：不写。
- 状态怎么变：当前模块高亮。
- 异常怎么处理：未知项目内路由不高亮。
- 限制：不要和左侧全局菜单共用 active 状态。
