# F-002-013 请求地址配置

状态：等待用户确认  
所属菜单：M-002 设置  
对应功能文档：`docs/features/M-002-设置.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：管理请求 baseUrl，但 VT Studio 桌面版要弱化普通用户手动改地址，主要给开发者排错。
为什么现在做：参考项目是 Web + 本地 HTTP，VT Studio 是 Electron + preload/main，不能照搬成普通用户必填项。
做完后有什么用：开发环境能切换本地服务地址，生产环境不容易被用户改坏。
这一步不碰什么：不改 IPC 主链路，不把页面重新改回直接 axios 访问后端。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow requestConfig 的能力，但明确 VT Studio 本地桌面版的边界。

只做：
  1. 显示当前本地服务地址
  2. 保存 baseUrl
  3. 重置 baseUrl
  4. 刷新 Electron 本地服务地址
  5. 地址格式校验
  6. 开发者/高级模式显示控制

不做：
  1. 普通业务页面直接用 axios 调后端
  2. 生产环境鼓励用户改地址
  3. 远程云服务切换
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/components/setting/components/requestConfig.vue` | warning alert、API 地址 input、保存、重置、Electron 刷新 |
| `Toonflow-web/src/stores/setting.ts` | baseUrl 本地持久化 |
| `Toonflow-web/src/utils/axios.ts` | axios 请求使用 baseUrl |
| `Toonflow-web/src/App.vue` | Electron 环境通过 `toonflow://getAppUrl` 获取本地服务地址 |
| `Toonflow-web/src/pages/login/index.vue` | 登录页也能设置 baseUrl |

参考项目关键事实：

```txt
1. store 默认值是 http://localhost:10588/api。
2. requestConfig 重置值是 http://localhost:10588，不带 /api。
3. 登录页 placeholder 也不带 /api。
4. VT Studio 必须统一 baseUrl 语义。
5. VT Studio 主链路应走 preload/main，不应该让普通页面绕开 IPC。
```

## 3. 用户操作

```txt
入口：
  设置弹窗 -> 请求地址配置。
  开发者/高级模式下显示。

按钮/操作：
  1. 查看当前 baseUrl。
  2. 输入新 baseUrl。
  3. 点击保存。
  4. 点击重置。
  5. 点击刷新本地服务地址。

弹窗/表单：

成功反馈：

失败反馈：

```

## 4. 要做什么功能

### 1. 显示当前 baseUrl

怎么做：
- 输入：无。
- 输出：当前 baseUrl。
- 写什么数据：不写。
- 状态怎么变：表单显示。
- 异常怎么处理：缺失时显示默认本地服务地址。
- 限制：明确 baseUrl 是否包含 `/api`，不能两套语义混用。

### 2. 保存 baseUrl

怎么做：
- 输入：http(s) URL。
- 输出：保存成功。
- 写什么数据：写本地设置。
- 状态怎么变：后续 axios 开发请求使用新地址。
- 异常怎么处理：格式不匹配不保存。
- 限制：生产桌面版普通功能不依赖它，主链路仍走 IPC。

### 3. 重置 baseUrl

怎么做：
- 输入：无。
- 输出：默认 baseUrl。
- 写什么数据：写本地设置。
- 状态怎么变：恢复默认。
- 异常怎么处理：保存失败提示。
- 限制：默认值必须统一，建议固定 `http://127.0.0.1:{port}/api` 或由 main 返回。

### 4. 刷新本地服务地址

怎么做：
- 输入：无。
- 输出：当前本地服务 URL。
- 写什么数据：写本地设置。
- 状态怎么变：表单更新。
- 异常怎么处理：本地服务未启动时提示。
- 限制：不能使用 Toonflow 的 `toonflow://getAppUrl`，VT Studio 用 preload/main 能力。

### 5. 显示控制

怎么做：
- 输入：是否开发者模式。
- 输出：显示/隐藏。
- 写什么数据：不写。
- 状态怎么变：普通用户默认不显示或只读。
- 异常怎么处理：无。
- 限制：避免用户改坏后整个应用请求失败。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 不使用 toonflow://getAppUrl。
  2. 普通用户弱化该配置。
  3. 统一 baseUrl 是否带 /api。

原因：
  VT Studio 是 Electron 本地桌面应用，页面不应该依赖用户手填后端地址。

是否写入 04：
  如删除普通用户入口，需要写入 04；如果仅放开发者/高级设置，不需要。
```
