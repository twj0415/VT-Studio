# F-008-013 视频提示词生成

状态：等待用户确认  
所属菜单：M-008 生产工作台  
对应功能文档：`docs/features/M-008-生产工作台.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：为视频轨道生成视频模型需要的 prompt，支持单条和批量。
为什么现在做：视频生成不能直接拿分镜描述，必须按模型和模式生成合适提示词。
做完后有什么用：每条视频轨道都有可编辑的视频 prompt，后续才能生成视频候选。
这一步不碰什么：不生成视频文件，不选择候选视频，不做剪映导出。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/production/workbench/generateVideoPrompt` 和 `/batchGeneratePrompt`。

只做：
  1. 单轨视频提示词生成
  2. 批量视频提示词生成
  3. 按视频模型和 mode 读取提示词模板
  4. 写 videoTrack.prompt/state/reason
  5. 轮询提示词生成状态

不做：
  1. 视频候选生成
  2. 轨道删除
  3. 剪映导出
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/production/components/workbench/*` | 单轨/批量生成视频提示词 |
| `Toonflow-app/src/routes/production/workbench/generateVideoPrompt.ts` | 根据模型/mode 读取提示词，调用文本模型 |
| `Toonflow-app/src/routes/production/workbench/batchGeneratePrompt.ts` | 批量生成提示词 |
| `Toonflow-app/src/routes/production/workbench/checkVideoPrompt.ts` | 轮询轨道提示词状态 |

参考项目关键事实：

```txt
1. 视频提示词主要读取 storyboard.videoDesc。
2. 提示词模板按 video model/mode 从模型提示词映射读取。
3. 写入 o_videoTrack.prompt。
4. 状态有 未生成/生成中/已完成/生成失败。
5. 状态要统一，不能和视频候选成功状态混用。
```

## 3. 用户操作

```txt
入口：
  视频工作台 -> 视频生成模式 -> 轨道提示词区域。

按钮/操作：
  1. 点击单轨生成 prompt。
  2. 点击批量生成 prompt。
  3. 手动编辑 prompt。

弹窗/表单：
  可选确认框。

成功反馈：
  轨道 prompt 填充完成。

失败反馈：
  缺视频模型、缺 mode、缺模板、模型失败。
```

## 4. 要做什么功能

### 1. 单轨生成提示词

怎么做：
- 输入：trackId、storyboardId、model、mode。
- 输出：video prompt。
- 写什么数据：更新 video_tracks.prompt/status。
- 状态怎么变：idle -> running -> succeeded/failed。
- 异常怎么处理：模板缺失或模型失败写 reason。
- 限制：只作用于当前剧本轨道。

### 2. 批量生成提示词

怎么做：
- 输入：trackIds、并发数。
- 输出：每条轨道 prompt。
- 写什么数据：批量更新 video_tracks。
- 状态怎么变：每条独立 running -> succeeded/failed。
- 异常怎么处理：单条失败不影响其他轨道继续。
- 限制：并发数来自设置。

### 3. 手动更新提示词

怎么做：
- 输入：trackId、prompt。
- 输出：更新结果。
- 写什么数据：更新 video_tracks.prompt。
- 状态怎么变：dirty -> saved。
- 异常怎么处理：保存失败回滚 UI。
- 限制：不改变候选视频。

### 4. 轮询提示词状态

怎么做：
- 输入：running trackIds。
- 输出：完成或失败状态。
- 写什么数据：不写。
- 状态怎么变：页面更新。
- 异常怎么处理：轮询失败保留下一轮。
- 限制：轮询不重新执行模型。


## 12. 本轮补充

```txt
视频提示词编辑不是普通 textarea，要参考 promptEditor。
必须支持：
1. @图 / @图片 / @视频 / @音频 / @文本 引用。
2. 按模型 mode 限制引用类型和数量。
3. 支持键盘上下选择、Enter/Tab 确认、Esc 关闭。
4. 保存时同时保留 prompt 文本和引用对象。
5. imageReference:N / videoReference:N / audioReference:N 超数量时不能提交。
```
