## 0. 快速理解

```txt
一句话：做原文导入弹窗，支持 txt/docx 或粘贴文本，先解析章节，再勾选保存。
为什么现在做：原文是事件分析、剧本 Agent、剧本生成的输入源。
做完后有什么用：用户能把小说或原文拆成章节保存到当前项目。
这一步不碰什么：不做剧本生成，不做独立事件库，不保存到源码目录。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的 importNovel 流程：读取文本 -> 解析章节 -> 用户勾选 -> 保存章节 -> 自动开始事件摘要。

只做：
  1. 导入原文按钮和弹窗
  2. txt/docx 文件读取
  3. 粘贴文本
  4. 章节解析和预览
  5. 章节勾选
  6. 保存选中章节
  7. 保存后自动触发事件摘要生成状态

不做：
  1. doc 文件导入
  2. 事件列表库
  3. 剧本生成
  4. 原文全文搜索
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/novel/components/importNovel.vue` | 两步导入：上传/粘贴 -> 解析预览 -> 勾选保存 |
| `Toonflow-web/src/utils/parseNovel.ts` | 按卷和章节正则拆分文本；支持自定义章节正则 |
| `Toonflow-app/src/routes/novel/addNovel.ts` | 按项目最大 chapterIndex 继续写入章节，新增后触发事件摘要 |
| `Toonflow-app/src/utils/cleanNovel.ts` | 调 `universalAi` 生成章节事件摘要 |

参考项目关键事实：

```txt
1. 支持 txt 和 docx，不支持 doc。
2. 单文件最大 10MB。
3. 上传文件只读取文本，源码没有保存原始文件副本。
4. 没有章节匹配但文本非空时，整篇作为第 1 章。
5. 默认卷名是“正文卷”。
6. 保存时后端按项目已有最大 chapterIndex 继续累加，不直接信任前端 index。
7. 新增章节 eventState 初始为 0，随后异步生成事件摘要。
```

## 3. 用户操作

```txt
入口：
  项目内顶部菜单 -> 原文 -> 导入原文按钮。

按钮/操作：
  1. 上传 txt/docx 或粘贴文本。
  2. 查看解析出的章节数量。
  3. 点击下一步。
  4. 勾选要保存的章节。
  5. 点击保存。

弹窗/表单：
  两步导入弹窗。

成功反馈：
  保存成功，关闭弹窗，刷新原文列表，事件列显示生成中。

失败反馈：
  文件类型不支持、文件过大、正则错误、未选章节、保存失败。
```

## 4. 要做什么功能

### 1. 读取文件或粘贴内容

怎么做：
- 输入：txt/docx 文件或文本框内容。
- 输出：原始文本 content。
- 写什么数据：确认保存前不写。
- 状态怎么变：content 更新，章节解析结果同步更新。
- 异常怎么处理：doc 和未知 MIME 直接提示不支持。
- 限制：单文件最大 10MB。

### 2. 解析章节

怎么做：
- 输入：content、设置里的 chapterReg。
- 输出：章节数组，含临时 id、卷、章节名、正文。
- 写什么数据：不写。
- 状态怎么变：解析结果为空时不能下一步。
- 异常怎么处理：自定义正则非法时提示，并回退默认正则或阻止继续。
- 限制：row-key 必须用稳定临时 id，不能用章节序号。

### 3. 勾选保存章节

怎么做：
- 输入：用户勾选的章节临时 id。
- 输出：待保存章节列表。
- 写什么数据：写 source_chapters 或等价章节表。
- 状态怎么变：保存中 -> 成功关闭弹窗 -> 列表刷新。
- 异常怎么处理：未选章节不能保存；数据库失败回滚。
- 限制：只保存当前项目下章节。

### 4. 保存后触发事件摘要状态

怎么做：
- 输入：新增章节 id。
- 输出：eventState=running。
- 写什么数据：章节事件状态字段。
- 状态怎么变：新增章节进入事件摘要生成中。
- 异常怎么处理：如果事件生成启动失败，章节保留但 eventState=failed 并写失败原因。
- 限制：事件摘要生成细节放 F-004-005/F-004-006。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. row-key 使用稳定临时 id，避免不同卷同章节序号冲突。
  2. 如果保存原始文件副本，必须放项目业务目录并写入偏差。

原因：
  参考项目用 index 做 row-key 有选择冲突风险；源码没有保存原文件副本。

是否写入 04：
  如果决定保存原始文件副本，需要写 04；只做文本导入则不写。
```
