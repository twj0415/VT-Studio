## 0. 快速理解

```txt
一句话：做章节单删和批量删除，并清理章节关联事件数据。
为什么现在做：导入错了或章节重复时必须能安全删除。
做完后有什么用：用户能清理无效原文，不留下脏关联。
这一步不碰什么：不删除项目，不删除剧本，不做旧 event.vue 的无闭环删除。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 删除章节能力，同时修正事件关系删除风险。

只做：
  1. 单条删除章节
  2. 批量删除章节
  3. 删除确认
  4. 删除章节事件关系
  5. 删除后刷新分页

不做：
  1. 删除项目
  2. 删除剧本
  3. 删除独立事件库完整链路
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/novel/index.vue` | 单条删除和批量删除都有确认，删除后刷新列表 |
| `Toonflow-app/src/routes/novel/delNovel.ts` | 删除章节及事件关系 |
| `Toonflow-app/src/routes/novel/batchDeleteNovel.ts` | 批量删除章节及事件关系 |
| `Toonflow-app/src/routes/novel/event/deletEvent.ts` | 删除事件 |

参考项目关键事实：

```txt
1. 单条删除最后一条时会回退页码。
2. 删除章节会处理 o_eventChapter/o_event。
3. 源码里存在用关系表 id 去删 o_event.id 的风险，VT Studio 不能照搬。
4. eventState=0 生成中的章节，前端编辑/删除按钮禁用。
```

## 3. 用户操作

```txt
入口：
  原文列表操作列删除按钮；顶部批量删除按钮。

按钮/操作：
  1. 勾选章节。
  2. 点击批量删除。
  3. 点击单行删除。
  4. 二次确认。

弹窗/表单：
  删除确认框。

成功反馈：
  删除成功，刷新列表。

失败反馈：
  未选择章节、章节正在分析、删除失败。
```

## 4. 要做什么功能

### 1. 单条删除章节

怎么做：
- 输入：chapterId。
- 输出：删除结果。
- 写什么数据：删除章节和章节事件关系。
- 状态怎么变：删除中 -> 刷新列表；当前页为空时回退上一页。
- 异常怎么处理：章节不存在、正在分析、事务失败都返回明确 msg。
- 限制：必须二次确认。

### 2. 批量删除章节

怎么做：
- 输入：chapterIds。
- 输出：删除结果。
- 写什么数据：事务删除章节和关联关系。
- 状态怎么变：删除中 -> 清空选中 -> 刷新列表。
- 异常怎么处理：任一删除失败整体回滚。
- 限制：不能删除事件分析中的章节。

### 3. 清理事件关系

怎么做：
- 输入：chapterIds。
- 输出：关系清理结果。
- 写什么数据：删除章节事件关系；是否删除孤立事件按规则处理。
- 状态怎么变：无。
- 异常怎么处理：关系删除失败回滚章节删除。
- 限制：必须按 eventId 清理，不按关系表 id 误删事件。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  修正事件关系删除逻辑，不用关系表 id 误删事件。

原因：
  参考源码有误删风险，必须按真实 eventId 清理。

是否写入 04：
  写入，属于修正源码风险。
```

## 8. 验收
