# F-011-008 编辑改编策略

状态：等待用户确认  
所属菜单：M-011 剧本 Agent  
对应功能文档：`docs/features/M-011-剧本Agent.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：右侧“改编策略”tab 支持查看 Markdown 和手动编辑保存。
为什么现在做：改编策略会影响后续剧本生成方向，必须能人工修正。
做完后有什么用：用户可以调整改编原则、节奏和取舍。
这一步不碰什么：不生成新策略，不编辑故事骨架，不改剧本内容。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 改编策略 tab 的 MdPreview + 编辑弹窗。

只做：
  1. 改编策略 Markdown 展示
  2. 空状态
  3. 编辑按钮
  4. Markdown 编辑弹窗
  5. 保存到 agent_work_data

不做：
  1. Agent 自动生成
  2. 故事骨架编辑
  3. 剧本卡片编辑
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |

| `Toonflow-app/src/routes/scriptAgent/updateData.ts` | 更新 `o_agentWorkData.data.adaptationStrategy` |

参考项目关键事实：

```txt
1. adaptationStrategy 是 planData 顶层字段。
2. 保存调用 updateData。
3. Markdown 编辑器阻止图片拖拽上传。
```

## 3. 用户操作

```txt
入口：
  剧本 Agent 右侧 -> 改编策略 tab。

按钮/操作：
  1. 点击编辑。
  2. 修改 Markdown。
  3. 点击保存。

弹窗/表单：
  Markdown 编辑弹窗。

成功反馈：
  保存成功，预览更新。

失败反馈：
  保存失败提示。
```

## 4. 要做什么功能

### 1. 展示改编策略

怎么做：
- 输入：planData.adaptationStrategy。
- 输出：Markdown 预览或空状态。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：Markdown 渲染失败显示纯文本兜底。
- 限制：不允许 Markdown 图片拖拽上传。

### 2. 编辑并保存改编策略

怎么做：
- 输入：Markdown 文本。
- 输出：更新后的 adaptationStrategy。
- 写什么数据：更新 agent_work_data.data.adaptationStrategy。
- 状态怎么变：保存中 -> 成功关闭弹窗。
- 异常怎么处理：保存失败保留弹窗内容。
- 限制：只更新 adaptationStrategy，不覆盖 scripts。
