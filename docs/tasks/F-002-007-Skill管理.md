# F-002-007 Skill 管理

状态：已完成  
所属菜单：M-002 设置  
对应功能文档：`docs/features/M-002-设置.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：设置的开发者模式里管理本地 Skill Markdown，能搜索、查看、编辑和保存。
为什么现在做：Agent 后续会读取 Skill，Skill 内容必须有安全的查看和编辑入口。
做完后有什么用：高级用户能调整本地 Skill 内容；系统能保证不越权写文件，并标记 embedding 需要重建。
这一步不碰什么：不做 Skill 市场，不下载 Skill，不删除 Skill，不做 Agent 调用执行，不自动重建 embedding。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的 skillManagement，同时加强路径安全。

只做：
  1. 查询 Skill 列表
  2. 搜索 Skill
  3. 查看 Skill 内容
  4. 编辑 Markdown
  5. 保存 Skill 内容
  6. 路径白名单校验
  7. 保存前轻量结构校验
  8. 保存后更新 skill_list 的 md5、description、updated_at、state
  9. 显示文件缺失和 embedding 过期状态

不做：
  1. Skill 安装市场
  2. Skill 删除
  3. Skill 自动下载
  4. Agent 执行 Skill
  5. embedding 重建进度 UI
  6. 扫描任意用户目录
  7. 新增 Skill 文件
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/components/setting/components/skillManagement.vue` | 搜索框、Skill 列表、右侧详情、编辑按钮、Markdown 编辑弹窗 |
| `Toonflow-app/src/routes/setting/skillManagement/getSkillList.ts` | 查询 o_skillList |
| `Toonflow-app/src/routes/setting/skillManagement/getSkillContent.ts` | 参数 path，读取 Skill 文件内容 |
| `Toonflow-app/src/routes/setting/skillManagement/saveSkillContent.ts` | 参数 path/content，保存 Skill 文件 |

参考项目关键事实：

```txt
1. Skill 是本地 Markdown 文件，不是纯数据库配置。
2. 数据库保存 Skill 列表和归属关系。
3. 读取和保存都依赖 path。
4. VT Studio 必须限制 path 只能在 skills 根目录内。
```

## 3. 用户操作

```txt
入口：
  设置 -> 开发者模式 -> Skill 管理。

定位：
  高级能力，不放普通用户主入口。
  原因：这是直接编辑 Agent 行为的 Markdown 源文件，普通用户容易误改导致 Agent 失效。

按钮/操作：
  1. 搜索 Skill。
  2. 点击 Skill 查看详情。
  3. 点击编辑。
  4. 修改 Markdown。
  5. 点击保存。
  6. 查看文件缺失、embedding 过期、frontmatter 风险提示。

弹窗/表单：
  Markdown 编辑弹窗。

成功反馈：
  内容保存成功。

失败反馈：
  文件不存在、路径非法、内容为空、frontmatter 缺失、保存失败、权限不足。
```

## 4. 要做什么功能

### 1. 查询 Skill 列表

怎么做：
- 输入：关键词。
- 输出：Skill 列表，包含 id、名称、描述、类型、归属、相对路径、文件状态、embedding 状态、更新时间。
- 写什么数据：不写。
- 状态怎么变：列表刷新。
- 异常怎么处理：读取失败显示错误。
- 限制：
  - 不把绝对路径暴露为可编辑输入。
  - 搜索只匹配 name、description、path、attribution。
  - 列表可展示相对路径，但保存时不能信任页面传回的 path。

### 2. 查看 Skill 内容

怎么做：
- 输入：skillId。
- 输出：Markdown 内容。
- 写什么数据：不写。
- 状态怎么变：右侧详情显示内容。
- 异常怎么处理：文件不存在、路径越界返回失败。
- 限制：
  - 只能读 skills 根目录下文件。
  - 不允许 renderer 直接传 path 读取任意文件。
  - service 必须先用 skillId 查询 skill_list.path，再走 CORE-011 的安全路径规则。

### 3. 编辑 Skill 内容

怎么做：
- 输入：Markdown 内容。
- 输出：编辑态。
- 写什么数据：打开编辑不写。
- 状态怎么变：弹窗打开。
- 异常怎么处理：内容加载失败不能进入编辑。
- 限制：编辑格式是 Markdown 原文，不做富文本转换。

### 4. 保存 Skill 内容

怎么做：
- 输入：skillId、content。
- 输出：保存成功。
- 写什么数据：
  - 写对应 Skill Markdown 文件。
  - 更新 skill_list.md5。
  - 如果 frontmatter 可解析，更新 skill_list.name / description。
  - 更新 skill_list.updated_at。
  - 将 skill_list.state 标记为 -1，表示 embedding 需要重建。
- 状态怎么变：保存后刷新详情。
- 异常怎么处理：路径非法、文件不存在、无权限、写入失败分别提示。
- 限制：
  - 不能写任意本地文件；不能修改 skills 根目录外文件。
  - 不能由页面传 path 决定写入目标。
  - 保存不自动重建 embedding，避免 UI 卡住或 ONNX 缺失时失败。

### 5. 内容结构校验

怎么做：
- 输入：Markdown 内容。
- 输出：允许保存、阻止保存或风险提示。
- 写什么数据：校验阶段不写。
- 状态怎么变：保存按钮前置校验。
- 异常怎么处理：
  - 空内容：直接阻止。
  - 缺少 frontmatter：给风险提示，用户二次确认后允许保存。
  - frontmatter 缺少 name/description：给风险提示，用户二次确认后允许保存。
  - references 类型 description 为空：给风险提示。
- 限制：
  - 第一版做轻量校验，不做完整 Markdown AST 校验。
  - 不强制用户一定保留原描述，但必须明确风险。

### 6. 路径安全

怎么做：
- 输入：skillId 查出的 path。
- 输出：安全的绝对路径或失败。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：`..`、绝对路径越界、软链越界都拒绝。
- 限制：所有读写都必须走 main 文件层。

### 7. 默认文件缺失处理

怎么做：
- 输入：skill_list 中的 path。
- 输出：文件存在/缺失状态。
- 写什么数据：默认不自动补文件。
- 状态怎么变：列表中显示“文件缺失”，查看和保存阻止。
- 异常怎么处理：文件缺失返回明确提示。
- 限制：
  - 不静默创建空 Skill，避免 Agent 读到空文件。
  - 如果后续要恢复默认 Skill 文件，单独做“恢复默认 Skill”增强任务。

```txt
  读 skill_attributions

任务状态：
  不写 tasks 表

轮询/Socket：
  无

  无

  不删除 Skill

```
