# F-008-011 图片工作流编辑

状态：等待用户确认

## 0. 快速理解

```txt
一句话：生产页提供图片工作流编辑器，用上传节点和生成节点编辑分镜图或衍生资产图。
为什么现在做：参考项目这里不是普通上传图，而是一个小型图片生成流程。
做完后有什么用：用户能用参考图、prompt、模型等节点重新生成并保存图片。
这一步不碰什么：不接 ComfyUI workflow，不生成视频，不做剪映导出。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `components/editImage` 图片工作流编辑器。

只做：
  1. 图片工作流全屏弹窗
  2. upload 节点
  3. generated 节点
  4. 节点连线
  5. 参考图和结果图
  6. 保存 imageFlow
  7. 更新分镜或衍生资产图片

不做：
  1. ComfyUI 队列/节点映射
  2. 视频生成
  3. 剪映导出
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/production/components/editImage/*` | 图片工作流编辑器 |
| `Toonflow-web/src/views/production/utils/editImageType.ts` | 区分分镜和衍生资产图类型 |
| `Toonflow-app/src/routes/production/editImage/uploadImage.ts` | 上传工作流图片 |
| `Toonflow-app/src/routes/production/editImage/generateFlowImage.ts` | 调用模型生成工作流图片 |
| `Toonflow-app/src/routes/production/editImage/saveImageFlow.ts` | 保存 flowData |
| `Toonflow-app/src/routes/production/editImage/updateImageFlow.ts` | 更新 flowData |
| `Toonflow-app/src/routes/production/editImage/getImageFlow.ts` | 读取 flowData |

参考项目关键事实：

```txt
1. 节点类型有 upload 和 generated。
2. 顶部有上传、生成、保存等操作。
3. 生成节点包含 prompt、模型、比例、质量等参数。
4. imageFlow 存在 o_imageFlow.flowData。
5. 工作流图片生成保存路径有路径规范问题，VT Studio 必须统一。
6. 这不是 ComfyUI 源码，没有 queue/prompt/history/workflow 节点映射。
```

## 3. 用户操作

```txt
入口：
  分镜卡片或衍生资产卡片 -> 打开图片工作流。

按钮/操作：
  1. 添加上传节点。
  2. 上传参考图。
  3. 添加生成节点。
  4. 编辑 prompt、模型、比例、质量。
  5. 生成图片。
  6. 保存工作流和结果。

弹窗/表单：
  图片工作流全屏弹窗。

成功反馈：
  图片生成成功，保存后对应分镜/衍生资产更新。

失败反馈：
  上传失败、模型失败、保存失败、工作流数据损坏。
```

## 4. 要做什么功能

### 1. 打开和读取工作流

怎么做：
- 输入：targetType、targetId、flowId。
- 输出：工作流节点和已有结果。
- 写什么数据：不写。
- 状态怎么变：弹窗打开，加载工作流。
- 异常怎么处理：flowId 不存在时创建默认工作流。
- 限制：targetType 只允许 storyboard/derivedAsset。

### 2. 上传参考图节点

怎么做：
- 输入：图片文件。
- 输出：参考图节点。
- 写什么数据：写项目临时或业务图片文件。
- 状态怎么变：上传中 -> 成功。
- 异常怎么处理：类型不支持、写文件失败提示。
- 限制：renderer 不直接写文件。

### 3. 生成图片节点

怎么做：
- 输入：prompt、model、ratio、quality、参考图。
- 输出：生成图。
- 写什么数据：写图片文件。
- 状态怎么变：running -> succeeded/failed。
- 异常怎么处理：模型失败写错误。
- 限制：走统一图片 Model Adapter。

### 4. 保存工作流和结果

怎么做：
- 输入：flowData、选中结果图。
- 输出：flowId 和目标图片更新。
- 写什么数据：写 image_flows；更新 storyboards 或 derived asset 图片。
- 状态怎么变：保存中 -> 成功。
- 异常怎么处理：事务失败回滚。
- 限制：工作流保存和目标图片更新要一致。
