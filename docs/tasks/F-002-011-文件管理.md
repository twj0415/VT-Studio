# F-002-011 文件管理

状态：已完成  
所属菜单：M-002 设置  
对应功能文档：`docs/features/M-002-设置.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：在设置里列出安全的本地目录，用户点击后用系统文件管理器打开。
为什么现在做：项目素材、导出、日志、缓存都会落在 userData，用户需要能快速定位，但不能暴露源码目录。
做完后有什么用：用户能一键打开项目目录、导出目录、日志目录、缓存目录，排查和取文件更方便。
这一步不碰什么：不做文件上传、不做文件删除、不做任意路径浏览器，不暴露数据库文件目录。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的 fileManagement/openFolder 能力，但目录范围必须符合 VT Studio runtime 规则。

只做：
  1. 展示可打开目录列表
  2. 打开固定白名单目录
  3. userData 白名单目录不存在时自动创建
  4. 打开失败时给出原因
  5. 按常用、诊断、高级三类展示目录

不做：
  1. 任意路径输入
  2. 文件删除
  3. 文件上传
  4. 素材管理
  5. 打开项目源码 .runtime
  6. 打开数据库文件所在目录
  7. 打开供应商 adapter 代码目录
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/components/setting/components/fileManagement.vue` | 展示可打开目录，点击打开 |
| `Toonflow-app/src/routes/setting/fileManagement/openFolder.ts` | 参数 path，服务端调用系统能力打开文件夹 |

参考项目关键事实：

```txt
1. 参考项目只做打开目录，不做复杂文件管理。
2. 前端传 path 给后端打开。
3. VT Studio 不能允许用户传任意 path，必须由 main 提供白名单目录。
```

## 3. 用户操作

```txt
入口：
  设置 -> 文件管理。

定位：
  普通设置入口，不放开发者模式。
  原因：它只打开白名单目录，不做删除/导入/覆盖；用户找导出文件和日志是高频需求。

按钮/操作：
  1. 查看目录列表。
  2. 点击打开目录。

弹窗/表单：
  无。

成功反馈：
  系统文件管理器打开对应目录。

失败反馈：
  目录不存在、无权限、系统打开失败。
```

## 4. 要做什么功能

### 1. 查询可打开目录

怎么做：
- 输入：无。
- 输出：目录列表，包含 key/name/description/path/exists。
- 写什么数据：不写。
- 状态怎么变：页面加载后显示目录卡片。
- 异常怎么处理：userData 未初始化时显示错误。
- 限制：目录由 main 侧生成，renderer 不拼绝对路径。

### 2. 打开目录

怎么做：
- 输入：目录 key。
- 输出：打开成功。
- 写什么数据：不写。
- 状态怎么变：按钮 loading -> 系统打开。
- 异常怎么处理：目录不存在、权限不足、系统拒绝时返回明确 msg。
- 限制：只允许打开白名单 key，不接受任意 path。

### 3. 目录不存在处理

怎么做：
- 输入：目录 key。
- 输出：创建后打开或提示不存在。
- 写什么数据：必要时创建目录。
- 状态怎么变：exists=false -> 创建 -> exists=true。
- 异常怎么处理：创建失败提示权限问题。
- 限制：只允许创建 userData 下规定目录。

### 4. 白名单目录

怎么做：
- 输入：无。
- 输出：固定目录清单。
- 第一版目录：
  - 常用：项目目录 `projects`
  - 常用：导出目录 `exports`
  - 诊断：日志目录 `logs`
  - 诊断：缓存目录 `cache`
  - 诊断：临时目录 `temp`
  - 高级：本地模型目录 `models`
  - 高级：Skill 目录 `skills`
- 不展示目录：
  - 数据库目录 `database`
  - 供应商 adapter 代码目录 `vendors`
  - 项目源码目录
  - 开发期 `.runtime`
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：无。
- 限制：
  - 禁止默认暴露 `D:\project\vt-studio\.runtime`、源码目录、数据库文件所在目录的直接危险操作入口。
  - `vendors` 不从文件管理打开，避免绕过供应商配置页的引用检查和保存校验。
  - 不做“打开上级目录”，避免用户顺手进入非白名单位置。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 不让前端传任意 path。
  2. 不默认打开项目源码 .runtime。
  3. 目录按 userData/runtime 规范生成。

原因：
  防止路径越权和项目源码污染。

是否写入 04：
  不需要，属于安全增强。
```
