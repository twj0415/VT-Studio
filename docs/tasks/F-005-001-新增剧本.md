# F-005-001 新增剧本

状态：等待用户确认  
所属菜单：M-005 剧本  
对应功能文档：`docs/features/M-005-剧本.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：剧本页新增一集剧本，填写名称和内容，还可以关联已有角色/场景/道具。
为什么现在做：剧本是资产提取、生产工作台、视频生成的基础。
做完后有什么用：用户能手动创建剧本分集，不完全依赖 Agent。
这一步不碰什么：不做批量导入，不提取资产，不生成分镜。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `addScript.vue` 和 `/script/addScript`。

只做：
  1. 新增剧本按钮
  2. 新增弹窗
  3. 剧本名称和内容
  4. txt/docx 读取到内容
  5. 字数上限校验
  6. 关联已有资产
  7. 保存剧本和资产关系

不做：
  1. 批量新增
  2. 编辑已有剧本
  3. 资产提取
  4. 剧本导出
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/script/components/addScript.vue` | 新增弹窗，名称、上传/粘贴内容、关联资产，字数限制 |
| `Toonflow-app/src/routes/script/addScript.ts` | 写 `o_script`，如果 assets 非空写 `o_scriptAssets` |
| `Toonflow-web/src/utils/assetsCheck.ts` | 打开资产选择器，类型限制 role/tool/scene |
| `Toonflow-web/src/stores/setting.ts` | `otherSetting.scriptEpisodeLength` 控制单集字数上限 |

参考项目关键事实：

```txt
1. 支持 txt/docx，不支持 doc。
2. 单文件最大 10MB。
3. 上传只读取文本，不保存原始文件。
4. 内容超过 scriptEpisodeLength 时确认按钮禁用。
5. 关联资产类型限制 role/tool/scene。
6. 新增后写剧本和剧本资产关系。
```

## 3. 用户操作

```txt
入口：
  项目内顶部菜单 -> 剧本 -> 新增剧本。

按钮/操作：
  1. 填写剧本名称。
  2. 上传 txt/docx 或粘贴剧本内容。
  3. 选择关联资产。
  4. 点击确认。

弹窗/表单：
  新增剧本弹窗。

成功反馈：
  新增成功，关闭弹窗，刷新剧本卡片列表。

失败反馈：
  名称为空、内容为空、字数超限、文件不支持、保存失败。
```

## 4. 要做什么功能

### 1. 打开新增弹窗

怎么做：
- 输入：点击新增按钮。
- 输出：空表单。
- 写什么数据：不写。
- 状态怎么变：dialogOpen=true。
- 异常怎么处理：无。
- 限制：打开时读取字数上限和资产选择能力。

### 2. 填写或导入剧本内容

怎么做：
- 输入：文本、txt/docx 文件。
- 输出：scriptContent。
- 写什么数据：确认前不写。
- 状态怎么变：字数统计实时更新。
- 异常怎么处理：doc 和未知 MIME 提示不支持。
- 限制：单文件最大 10MB。

### 3. 选择关联资产

怎么做：
- 输入：资产选择器返回 role/scene/tool。
- 输出：selectedAssets。
- 写什么数据：确认前不写。
- 状态怎么变：资产 tag 展示，可移除。
- 异常怎么处理：资产不存在时提示并移除无效项。
- 限制：不能选择 clip/audio。

### 4. 保存剧本

怎么做：
- 输入：name、content、selectedAssets、projectId。
- 输出：新剧本记录。
- 写什么数据：写 scripts；写 script_asset_links。
- 状态怎么变：保存中 -> 成功关闭 -> 刷新列表。
- 异常怎么处理：事务失败回滚。
- 限制：assets 为空也要明确保存为无关联。
