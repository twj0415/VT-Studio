## 0. 快速理解

```txt
一句话：做一个统一的模型供应商管理页，能配置 API Key、启用供应商、管理模型、测试文本/图片/视频模型。
为什么现在做：后面的 Agent、资产生成、视频生成都依赖供应商和模型，没它就无法稳定调用模型。
做完后有什么用：用户可以自己配置 OpenAI、Claude、Gemini、DeepSeek、本地/自定义供应商，也能测试配置是否可用。
这一步不碰什么：不做正式业务生成，不做项目页，不做资产页，不在 renderer 执行供应商代码。
```

参考项目关键事实：

```txt
1. 供应商不是固定 SDK，而是动态 vendor 脚本 + 数据库配置。
2. vendor 脚本必须导出 vendor、textRequest、imageRequest、videoRequest；VT Studio 还要兼容 ttsRequest。
3. 供应商输入项是动态 schema，不同供应商字段不同。
4. models 是供应商内的数组，不同模型有 type、mode、think、audio、durationResolutionMap 等能力字段。
5. 模型测试是即时操作，不写任务中心。
6. Toonflow 里 ComfyUI 只看到 provider 名称/图标匹配，没有完整 workflow、queue、history 实现。
```

## 4. 要做什么功能

### 1. 查询供应商列表

怎么做：
- 输入：无。
- 输出：供应商数组，包含基础信息、启用状态、输入项、输入值、模型列表、是否内置。
- 写什么数据：不写。
- 状态怎么变：页面从 loading -> 列表可用；默认选中第一个供应商。
- 异常怎么处理：数据库读取失败显示错误状态；供应商代码损坏时该供应商标记为不可用，但不影响其他供应商。
- 限制：不返回密钥明文给日志；页面显示密钥输入框时用 password 类型。

### 2. 启用/禁用供应商

怎么做：
- 输入：`vendorId`、`enabled`。
- 输出：更新后的供应商状态。
- 写什么数据：写供应商配置表的启用状态。
- 状态怎么变：开关立即进入保存中；成功后更新；失败回滚开关。
- 异常怎么处理：供应商不存在、配置损坏、数据库失败都返回 `{ code: 400, data: {}, msg }`。
- 限制：禁用供应商后，后续模型选择不能再选它；已配置的 Agent/项目模型要显示“供应商已禁用”。

### 3. 保存动态输入项

怎么做：
- 输入：`vendorId`、`inputValues`。
- 输出：保存成功。
- 写什么数据：写供应商配置表的 inputValues。
- 状态怎么变：输入框 blur 或保存按钮触发保存。
- 异常怎么处理：必填项为空时不保存；类型不匹配时返回字段错误。
- 限制：输入项 key 必须来自 vendor schema，不能让页面写任意字段。

### 4. 添加模型

怎么做：
- 输入：模型显示名、模型 ID、类型、能力配置。
- 输出：新增后的模型。
- 写什么数据：写供应商 models 数组。
- 状态怎么变：弹窗确认后刷新模型列表。
- 异常怎么处理：模型 ID 为空、同供应商内重复、类型不支持时失败。
- 限制：模型类型统一为 `text/image/video/tts`；Toonflow 文档里旧的 `type=text/image/video` 要扩展兼容 tts。

### 5. 编辑模型

怎么做：
- 输入：`vendorId`、原模型 ID 或模型索引、更新字段。
- 输出：更新后的模型。
- 写什么数据：更新 models 数组中的一项。
- 状态怎么变：编辑弹窗保存后刷新当前供应商。
- 异常怎么处理：找不到模型、改成重复 ID、能力字段非法时失败。
- 限制：如果模型被 Agent、项目、提示词映射引用，允许编辑显示名和能力，不建议直接改 modelName；如改 modelName 必须提示影响。

### 6. 删除模型

怎么做：
- 输入：`vendorId`、`modelName`、`type`。
- 输出：删除成功。
- 写什么数据：从 models 数组移除。
- 状态怎么变：二次确认后删除；列表刷新。
- 异常怎么处理：模型不存在返回失败。
- 限制：如果模型被模型映射、Agent 配置、项目配置引用，必须阻止删除或显示影响范围后确认；不能静默造成后续生成失败。

### 7. 添加供应商

怎么做：
- 输入：供应商代码或从链接读取到的代码。
- 输出：新增供应商配置。
- 写什么数据：写供应商配置表；写 `userData/vendors/{vendorId}.ts` 或等价运行目录。
- 状态怎么变：校验通过后新增到左侧列表并选中。
- 异常怎么处理：id 为空、id 包含英文冒号、id 重复、导出缺失、schema 不合法时失败。
- 限制：供应商代码只允许 main 侧保存和执行；renderer 不能 `eval`。

### 8. 编辑供应商代码

怎么做：
- 输入：`vendorId`、代码文本。
- 输出：保存成功和重新校验后的供应商信息。
- 写什么数据：写供应商代码文件；必要时同步供应商元数据。
- 状态怎么变：代码弹窗保存后重新读取供应商。
- 异常怎么处理：代码语法错误、缺少导出、vendor id 和原 id 不一致时失败。
- 限制：保存前必须做沙盒校验；不能让代码访问任意 Node API。

### 9. 从链接读取供应商代码

怎么做：
- 输入：URL。
- 输出：代码文本。
- 写什么数据：只读取不保存，用户确认添加后才写。
- 状态怎么变：链接加载中 -> 代码预览。
- 异常怎么处理：URL 非 http(s)、请求失败、返回不是文本、内容过大都失败。
- 限制：这是网络能力，正式实现时要考虑网络权限和超时；不能自动信任远程代码。

### 10. 删除供应商

怎么做：
- 输入：`vendorId`。
- 输出：删除成功。
- 写什么数据：删除供应商配置；删除对应供应商代码文件或标记废弃。
- 状态怎么变：二次确认后从列表移除。
- 异常怎么处理：供应商不存在、仍被引用、文件删除失败要给出原因。
- 限制：内置供应商不建议硬删除，可只禁用；被模型映射、Agent、项目引用时必须先解除引用。

### 10.1 高风险代码来源处理

怎么做：
- 输入：文件导入、链接读取、代码编辑器内容。
- 输出：代码预览、vendor 信息、校验结果。
- 写什么数据：确认前不写；确认后写 userData/vendors。
- 状态怎么变：选择/读取 -> 校验 -> 预览 -> 二次确认 -> 保存。
- 异常怎么处理：文件过大、URL 非 http(s)、返回非文本、缺少导出、语法错误时失败；失败不能覆盖旧可用代码。
- 限制：不能从项目源码目录直接运行文件，必须复制到受控 vendors 目录；链接读取只能预览，不能自动保存。

### 11. 文本模型测试

怎么做：
- 输入：`vendorId`、`modelName`、messages/prompt、可选 temperature/maxOutputTokens。
- 输出：模型返回文本、耗时、错误原因。
- 写什么数据：不写数据库。
- 状态怎么变：测试按钮 loading -> 成功/失败。
- 异常怎么处理：供应商未启用、缺 API Key、模型不存在、请求失败、响应格式错误分开提示。
- 限制：测试不写任务中心，不污染 Agent 记忆。

### 12. 图片模型测试

怎么做：
- 输入：`vendorId`、`modelName`、prompt、可选参考图 base64、mode。
- 输出：图片 URL/base64/本地文件结果。
- 写什么数据：测试产生的临时文件可写 cache/temp，不写业务资产表。
- 状态怎么变：测试 loading -> 图片预览。
- 异常怎么处理：模式不支持、图片过大、协议失败、结果为空分开提示。
- 限制：测试图不进入资产中心。

### 13. 视频模型测试

怎么做：
- 输入：`vendorId`、`modelName`、mode、prompt、images/videos/audios、duration/resolution。
- 输出：视频 URL/本地临时文件、耗时、失败原因。
- 写什么数据：临时文件只进 cache/temp。
- 状态怎么变：测试 loading -> 视频预览；如果供应商协议是异步轮询，要显示轮询状态。
- 异常怎么处理：轮询超时、任务失败、音频不支持、分辨率不支持要区分。
- 限制：测试视频不进入生产工作台，不写任务中心。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 供应商动态代码不在 renderer 执行，只在 main 侧沙盒执行。
  2. 删除供应商/模型前增加引用检查，不照搬直接删除。
  3. ComfyUI 暂不混进核心供应商页面的普通字段，只作为增强 adapter 能力规划。
  4. 数据表和字段不用 o_vendorConfig 旧名。

原因：
  桌面应用要保护本地文件、密钥和业务配置；模型引用关系如果被删坏，后续生成链路会直接失败。

是否写入 04：
  动态代码 main 侧执行和 ComfyUI adapter 方式已有基础偏差记录；如后续正式支持 ComfyUI workflow，需要新增偏差。
```
