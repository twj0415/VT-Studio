## 0. 快速理解

```txt
一句话：生产流程图提供自动布局，让节点按参考项目的 LR/TB 规则排好。
为什么现在做：节点越来越多后，用户需要一键整理画布。
做完后有什么用：流程图不会乱堆，主链和资产分支能清楚展示。
这一步不碰什么：不修改业务数据，不生成内容，不保存 flowData。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow LR 手动布局和 TB dagre 布局。

只做：
  1. 自动布局按钮
  2. LR 主链布局
  3. assets 分支位置
  4. TB dagre 布局能力
  5. 拖拽后位置同步到前端状态

不做：
  1. 保存布局到数据库
  2. 编辑节点内容
  3. 自定义布局模板
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/production/utils/dagre.ts` | TB 使用 dagre 自动布局 |
| `Toonflow-web/src/views/production/index.vue` | 自动布局按钮调用 `layoutGraph("LR")` |
| `Toonflow-web/src/views/production/utils/flowBuilder.ts` | 构建默认节点位置 |

参考项目关键事实：

```txt
1. LR：主链手动从左到右，assets 放到 script 下方。
2. TB：使用 dagre。
3. 节点拖拽结束后同步 nodePositions，避免数据更新导致回跳。
4. 按住 Space + 左键拖拽画布。
```

## 3. 用户操作

```txt
入口：
  生产页顶部自动布局按钮。

按钮/操作：
  1. 点击自动布局。
  2. 拖拽节点。
  3. 按住 Space 拖动画布。

弹窗/表单：
  无。

成功反馈：
  节点重新排列。

失败反馈：
  布局失败提示，并保留原位置。
```

## 4. 要做什么功能

### 1. LR 自动布局

怎么做：
- 输入：nodes、edges。
- 输出：从左到右的节点位置。
- 写什么数据：只写前端布局状态。
- 状态怎么变：nodes positions 更新。
- 异常怎么处理：布局失败保留旧位置。
- 限制：不写业务数据库。

### 2. TB dagre 布局

怎么做：
- 输入：nodes、edges。
- 输出：从上到下布局。
- 写什么数据：只写前端布局状态。
- 状态怎么变：nodes positions 更新。
- 异常怎么处理：dagre 失败回退 LR。
- 限制：需要稳定节点尺寸。

### 3. 拖拽位置同步

怎么做：
- 输入：用户拖拽后的 node position。
- 输出：nodePositions 状态。
- 写什么数据：本任务不持久化。
- 状态怎么变：nodePositions 更新。
- 异常怎么处理：无。
- 限制：后续保存布局需单独任务确认。
