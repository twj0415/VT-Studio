# F-001-007 设置弹窗入口

状态：等待用户确认  
所属菜单：M-001 全局导航与项目入口  
对应功能文档：`docs/features/M-001-全局导航与项目入口.md`

## 0. 快速理解

```txt
一句话：左下角提供设置按钮，点击打开统一设置弹窗，有更新时显示红点。
为什么现在做：所有设置模块都要从一个统一入口打开，不能后面散成多个设置页。
做完后有什么用：用户随时能打开设置，后续 M-002 的 16 个设置模块都挂这里。
这一步不碰什么：不做设置弹窗里的具体模块内容。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的统一设置弹窗入口。

只做：
  1. 左下设置按钮。
  2. 点击后设置 `showSetting=true`。
  3. 设置弹窗显示和关闭。
  4. 更新红点显示。
  5. 打开设置时保留 activeMenu。

不做：
  1. 不做 16 个设置模块具体表单。
  2. 不做版本下载。
  3. 不做登录退出。
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/pages/workbench/index.vue` | 左下设置按钮点击 `showSetting = true`，有 `needUpdate` 时显示 dot |
| `Toonflow-web/src/components/setting/index.vue` | 统一设置弹窗，左侧 menu，右侧当前组件 |
| `Toonflow-web/src/stores/setting.ts` | 保存 `showSetting`、`activeMenu`、`needUpdate` |

参考项目关键事实：

```txt
1. 设置是 t-dialog 弹窗，不是独立路由页面。
2. 左下设置按钮有红点，来源是 `needUpdate`。
3. 版本更新通知可以打开设置并切到 about。
4. 关闭设置弹窗不重置 activeMenu。
5. 弹窗宽度约 1200px，高度约 70vh。
```

## 3. 用户操作

```txt
入口：
  主布局左下角设置按钮。

按钮/操作：
  1. 点击设置按钮。
  2. 关闭设置弹窗。
  3. 有更新红点时点击设置。

弹窗/表单：
  统一设置弹窗外壳。

成功反馈：
  设置弹窗打开，当前设置 tab 显示。

失败反馈：
  设置组件加载失败时显示错误空状态。
```

## 4. 要做什么功能

### 1. 设置按钮

怎么做：
- 输入：用户点击设置按钮。
- 输出：打开设置弹窗。
- 写什么数据：写前端状态 `showSetting=true`。
- 状态怎么变：弹窗从关闭变打开。
- 异常怎么处理：状态更新失败时提示。
- 限制：设置入口是全局入口，不依赖当前项目。

### 2. 设置红点

怎么做：
- 输入：`needUpdate`。
- 输出：设置按钮显示红点。
- 写什么数据：不写。
- 状态怎么变：needUpdate true 显示，false 隐藏。
- 异常怎么处理：无。
- 限制：红点只表示版本更新，不表示设置未保存。

### 3. 设置弹窗打开关闭

怎么做：
- 输入：showSetting。
- 输出：弹窗显示状态。
- 写什么数据：写前端状态。
- 状态怎么变：打开/关闭。
- 异常怎么处理：弹窗内部错误显示错误区域，不影响主布局。
- 限制：关闭时不清 activeMenu。

### 4. 设置弹窗层级和标题栏避让

怎么做：
- 输入：Electron 标题栏高度、窗口高度。
- 输出：弹窗在可视区内显示。
- 写什么数据：不写。
- 状态怎么变：样式适配。
- 异常怎么处理：窗口过小时内容滚动。
- 限制：不能遮住标题栏拖拽区。
