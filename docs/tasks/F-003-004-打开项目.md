状态：等待用户确认

## 0. 快速理解

```txt
一句话：点击项目卡片前先校验项目和模型可用，成功后写当前项目上下文并跳到小说或剧本页面。
为什么现在做：后续所有项目内页面都依赖 currentProject，不能点击就盲跳。
做完后有什么用：项目打开前能发现模型失效、项目不存在等问题，并引导用户修复。
这一步不碰什么：不编辑项目、不加载小说正文、不生成内容。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 打开项目逻辑，实现项目校验、上下文写入和路由跳转。

只做：
  1. 点击项目卡片
  2. 查找项目
  3. 校验图片模型
  4. 校验视频模型
  5. 写当前项目上下文
  6. 按 sourceType 跳转
  7. 校验失败打开编辑项目弹窗

不做：
  1. 项目编辑保存
  2. 原文列表加载
  3. 剧本列表加载
  4. Agent 初始化
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/project/index.vue` | 点击项目卡片，按 id 找项目；校验模型；写 project store；按 projectType 跳转 |
| `Toonflow-app/src/routes/modelSelect/getModelDetail.ts` | 校验模型详情是否可用 |
| `Toonflow-app/src/routes/general/getSingleProject.ts` | 其他模块读取当前项目详情 |
| `Toonflow-web/src/stores/project.ts` | 保存当前项目上下文 |

参考项目关键事实：

```txt
1. 找不到项目提示项目不存在。
2. imageModel 或 videoModel 为空时提示模型供应商不可用并打开编辑弹窗。
3. 模型详情校验失败时也打开编辑弹窗。
4. projectType=novel 跳 /novel。
5. projectType=script 跳 /script。
```

## 3. 用户操作

```txt
入口：
  项目页 -> 点击项目卡片。

按钮/操作：
  点击卡片打开项目。

弹窗/表单：
  校验失败时打开项目编辑弹窗。

成功反馈：
  进入项目内小说页或剧本页。

失败反馈：
  项目不存在、图片模型不可用、视频模型不可用、来源类型不支持。
```

## 4. 要做什么功能

### 1. 查找项目

怎么做：
- 输入：项目 id。
- 输出：项目记录。
- 写什么数据：不写。
- 状态怎么变：打开中。
- 异常怎么处理：找不到提示项目不存在并刷新列表。
- 限制：不能只信任前端卡片缓存，main/service 要能二次确认。

### 2. 校验图片模型

怎么做：
- 输入：项目 imageModel/vendorId。
- 输出：可用/不可用。
- 写什么数据：不写。
- 状态怎么变：校验通过继续。
- 异常怎么处理：供应商禁用、模型删除、类型不是 image 时失败并打开编辑弹窗。
- 限制：不调用模型，只查配置。

### 3. 校验视频模型

怎么做：
- 输入：项目 videoModel/vendorId。
- 输出：可用/不可用。
- 写什么数据：不写。
- 状态怎么变：校验通过继续。
- 异常怎么处理：供应商禁用、模型删除、类型不是 video 时失败并打开编辑弹窗。
- 限制：同时校验项目保存的 mode 是否仍在模型能力内。

### 4. 写当前项目上下文

怎么做：
- 输入：项目记录。
- 输出：currentProject。
- 写什么数据：renderer 本地状态和必要上下文。
- 状态怎么变：未打开项目 -> 当前项目。
- 异常怎么处理：写入失败提示刷新。
- 限制：只保存必要字段，不保存整堆业务数据。

### 5. 按来源类型跳转

怎么做：
- 输入：sourceType。
- 输出：目标路由。
- 写什么数据：不写。
- 状态怎么变：路由进入项目内页面。
- 异常怎么处理：未知 sourceType 提示项目类型不支持。
- 限制：`novel -> /novel`，`script -> /script`。
