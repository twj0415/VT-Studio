# F-005-002 批量新增剧本

状态：等待用户确认  
所属菜单：M-005 剧本  
对应功能文档：`docs/features/M-005-剧本.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：把一大段多集剧本按正则拆成多集，预览勾选后批量保存。
为什么现在做：很多剧本是一次性导入多集，手动一集一集新增效率太低。
做完后有什么用：用户能用默认正则、自定义正则或 AI 正则快速拆集入库。
这一步不碰什么：不提取资产，不生成分镜，不自动创建项目。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `batchAddScript.vue`、`parseScript.ts` 和 `/script/batchAddScript`。

只做：
  1. 批量新增弹窗
  2. txt/docx 或粘贴文本
  3. 默认集正则解析
  4. 自定义正则解析
  5. AI 生成正则
  6. 解析结果预览
  7. 勾选集数保存
  8. 字数上限校验

不做：
  1. 资产提取
  2. 剧本编辑
  3. 关联资产手动选择
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/script/components/batchAddScript.vue` | 两步 tabs：输入/解析 -> 勾选/保存 |
| `Toonflow-web/src/utils/parseScript.ts` | 默认按“第 N 集”拆分，支持自定义正则 |
| `Toonflow-app/src/routes/script/batchAddScript.ts` | 批量写 `o_script` |
| `Toonflow-app/src/routes/script/getAiRegex.ts` | 用 `universalAi` 根据文本前 2000 字生成正则 |

参考项目关键事实：

```txt
1. 默认正则是 `/第\s*([0-9０-９零一二三四五六七八九十百千万]+)\s*集\s*([^\n\r]*)/g`。
2. 自定义正则支持 /表达式/flags，没有 g 会自动补 g。
3. 没匹配但文本非空时，全文作为第 1 集。
4. AI 正则要求返回 JavaScript 正则字符串，必须有两个捕获组。
5. 保存前必须勾选要保存的剧本。
6. Toonflow row-key 用 index，有重复风险，VT Studio 必须用稳定临时 id。
```

## 3. 用户操作

```txt
入口：
  剧本页 -> 批量新增剧本。

按钮/操作：
  1. 上传 txt/docx 或粘贴剧本。
  2. 可填写自定义正则。
  3. 可点击 AI 获取正则。
  4. 点击下一步。
  5. 勾选要保存的集数。
  6. 点击保存。

弹窗/表单：
  两步批量导入弹窗。

成功反馈：
  保存成功，关闭弹窗，刷新剧本列表。

失败反馈：
  正则非法、没有解析结果、未勾选、字数超限、保存失败。
```

## 4. 要做什么功能

### 1. 输入和解析剧本文本

怎么做：
- 输入：txt/docx 文件或粘贴文本。
- 输出：原始 content 和解析结果。
- 写什么数据：确认保存前不写。
- 状态怎么变：content 更新后实时解析。
- 异常怎么处理：文件不支持或过大提示。
- 限制：单文件最大 10MB。

### 2. 自定义正则

怎么做：
- 输入：customRegStr。
- 输出：解析结果。
- 写什么数据：不写。
- 状态怎么变：正则非法时输入框 error。
- 异常怎么处理：非法正则不能进入下一步。
- 限制：必须至少能捕获集数和标题。

### 3. AI 生成正则

怎么做：
- 输入：content 前 2000 字。
- 输出：正则字符串。
- 写什么数据：不写。
- 状态怎么变：AI 按钮 loading -> 回填 customRegStr。
- 异常怎么处理：模型失败或返回空字符串时提示。
- 限制：只调用文本模型，不写任务中心。

### 4. 预览勾选保存

怎么做：
- 输入：解析结果 selectedRowKeys。
- 输出：批量剧本记录。
- 写什么数据：写 scripts。
- 状态怎么变：保存中 -> 成功关闭 -> 列表刷新。
- 异常怎么处理：任一写入失败回滚。
- 限制：row-key 使用稳定临时 id，不能用 index。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  解析预览 row-key 不用 index，使用稳定临时 id。

原因：
  不同集或重复编号可能导致勾选错乱。

是否写入 04：
  不需要，属于前端稳定性修正。
```
