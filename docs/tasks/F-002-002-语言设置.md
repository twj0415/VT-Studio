# F-002-002 语言设置

状态：已完成  
所属菜单：M-002 设置  
对应功能文档：`docs/features/M-002-设置.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：补齐 VT Studio 的国际化底座，让设置页和登录页能真正切语言。
为什么现在做：当前 renderer 里还没有 i18n/locale 基础设施，后续页面再继续硬编码中文会越改越乱。
做完后有什么用：用户能切换语言，页面、表单、组件库文案保持一致，而且用户选择不会被启动逻辑覆盖。
这一步不碰什么：不做机器翻译，不做在线语言包下载，不要求一次补完所有业务翻译。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的语言设置，修正启动时覆盖用户选择的问题，明确 VT Studio 多语言落地规则。

只做：
  1. i18n 基础设施
  2. 语言列表
  3. 当前语言保存
  4. 设置页语言切换
  5. 登录页语言切换
  6. TDesign/组件库语言同步
  7. 首次启动读取系统语言
  8. 已选择语言不被系统语言覆盖

不做：
  1. 在线翻译
  2. 远程语言包
  3. 所有页面文案一次性补齐
  4. main 返回 msg 的完整多语言化
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/components/setting/components/languageConfig.vue` | 语言卡片，点击后写 locale/cachedLocale |
| `Toonflow-web/src/locales/index.ts` | 定义 languageList 和语言包 |
| `Toonflow-web/src/App.vue` | 根据 cachedLocale 设置 TDesign globalConfig |
| `Toonflow-web/src/pages/login/index.vue` | 登录页语言下拉 |

参考项目关键事实：

```txt
1. 语言列表：zh-CN、zh-TW、en、th-TH、vi-VN、ja-JP、ru-RU。
2. 业务文案用 i18n JSON。
3. TDesign 只明确配置 zh-CN 和 en，其他语言回退 zhConfig。
4. App 启动时会按 navigator.language 覆盖 cachedLocale，这会覆盖用户已选语言，VT Studio 不能照搬。
```

## 4. 要做什么功能

### 1. 语言列表

怎么做：
- 输入：无。
- 输出：语言选项数组。
- 写什么数据：不写。
- 状态怎么变：页面显示语言卡片。
- 异常怎么处理：语言包缺失时该语言标记不可用。
- 限制：语言 value 必须和 i18n locale 一致。
  - 当前实际要求：如果第一版没有真实翻译内容，不应该把七种语言全部开放给普通用户直接选。

### 0. i18n 基础设施

怎么做：
- 输入：无。
- 输出：`i18n instance + locale store + language list`。
- 写什么数据：首次启动时写本地 locale 初始化状态。
- 状态怎么变：App 启动时就决定当前 locale。
- 异常怎么处理：语言包缺失时回退 `zh-CN`。
- 限制：不能等到打开设置页后才初始化语言。

### 2. 切换语言

怎么做：
- 输入：locale value。
- 输出：当前语言。
- 写什么数据：写本地 locale。
- 状态怎么变：i18n locale 更新，页面文案立即刷新。
- 异常怎么处理：未知 locale 拒绝保存。
- 限制：不能只改业务文案，组件库语言也要同步。

### 3. 登录页语言切换

怎么做：
- 输入：locale value。
- 输出：当前语言。
- 写什么数据：写同一份本地 locale。
- 状态怎么变：登录页文案立即刷新。
- 异常怎么处理：未知 locale 拒绝保存。
- 限制：登录页和设置页必须共用同一 locale 来源。

### 4. 组件库语言同步

怎么做：
- 输入：当前 locale。
- 输出：TDesign globalConfig。
- 写什么数据：不写。
- 状态怎么变：组件内置文案切换。
- 异常怎么处理：组件库不支持的语言回退中文或英文，并记录缺口。
- 限制：不能出现业务文案是英文、组件按钮还是中文的混乱状态。

### 5. 首次启动系统语言

怎么做：
- 输入：系统语言。
- 输出：初始 locale。
- 写什么数据：仅首次启动时写本地 locale。
- 状态怎么变：没有用户选择时才按系统语言初始化。
- 异常怎么处理：系统语言不支持时回退 `zh-CN`。
- 限制：用户已经选择过语言后，不允许启动时再次覆盖。

## 6. VT Studio 怎么落

```txt
能力名：
  language

调用链：
  renderer 设置页/登录页 -> i18n store -> vue-i18n + TDesign globalConfig

需要新增：
  设置页语言组件
  renderer i18n store
  语言列表常量
  locale messages

需要修改：
  App 启动初始化 locale
  登录页语言选择共用 locale store

当前代码现实：
  1. 当前 `src/renderer/src` 下还没有 `i18n`、`locale`、`messages`、`TDesign locale provider` 相关文件。
  2. 也没有 `$t()` / `useI18n()` 接线，当前页面大多还是直接写中文。
  3. 所以这一步不是只做一个语言切换面板，而是先补 renderer 国际化底座，再把设置页和登录页接上。
```

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 不在每次启动时用系统语言覆盖用户已选语言。
  2. 组件库语言必须同步，不只切业务文案。
  3. 第一版如果没有真实翻译内容，不应把七种语言全部开放给普通用户。

原因：
  参考项目启动覆盖用户选择不合理；多语言半套会造成体验混乱。

是否写入 04：
  如果第一版不完整保留七种语言，需要写入 04。
```
