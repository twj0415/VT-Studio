# F-007-003 批量绑定音频

状态：等待用户确认  
所属菜单：M-007 角景音频绑定  
对应功能文档：`docs/features/M-007-角景音频绑定.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：选中多个资产后，用文本模型从已有音色里自动匹配一个音色。
为什么现在做：手动给大量角色/场景/道具绑定音频很慢。
做完后有什么用：系统能按名称和描述自动选择最合适的已有音色。
这一步不碰什么：不生成新音频，不做 TTS，不允许一个资产多个音色。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `/cornerScape/batchBindAudio`。

只做：
  1. 批量选择资产
  2. 查询候选音频父资产
  3. 读取 audioBindPrompt
  4. 调用文本模型匹配 audioId
  5. 写绑定关系
  6. 写 audioBindState 和失败原因

不做：
  1. TTS 生成
  2. 多音色绑定
  3. 上传音频
  4. 语音合成文件
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/cornerScape/index.vue` | 左侧“一键绑定音频”，选中后本地 audioBindState=生成中 |
| `Toonflow-app/src/routes/cornerScape/batchBindAudio.ts` | 查询待绑定资产和候选 audio，用 universalAi 匹配 audioId |
| `Toonflow-app/src/routes/cornerScape/pollingAudio.ts` | 轮询 audioBindState 不为生成中的资产 |

参考项目关键事实：

```txt
1. 候选音频是 assets type=audio 且 assetsId=null 的父资产。
2. 没有候选音频时返回“暂无设置音频，请先前往资产中心上传音频”。
3. 提示词来源 o_prompt type=audioBindPrompt，优先 useData。
4. resultTool schema 描述说列表/空数组，但实际字段是单个 audioId；VT Studio 按单个 audioId 做。
5. 失败只写 audioBindState=生成失败，源码没有独立失败原因字段，VT Studio 要补。
```

## 3. 用户操作

```txt
入口：
  角景页 -> 勾选资产 -> 一键绑定音频。

按钮/操作：
  1. 勾选资产。
  2. 点击一键绑定音频。

弹窗/表单：
  可选确认框。

成功反馈：
  资产显示音频绑定中，完成后显示绑定音色。

失败反馈：
  未选中、没有音频素材、模型失败、无匹配结果。
```

## 4. 要做什么功能

### 1. 启动批量绑定

怎么做：
- 输入：assetIds、projectId。
- 输出：启动结果。
- 写什么数据：assets.audio_bind_status=running。
- 状态怎么变：idle/failed -> running。
- 异常怎么处理：未选中或没有候选音频时提示。
- 限制：只处理 role/scene/prop。

### 2. 模型匹配音色

怎么做：
- 输入：待绑定资产列表、候选音频列表、audioBindPrompt。
- 输出：每个资产的 audioId。
- 写什么数据：不直接在 renderer 写。
- 状态怎么变：每个资产独立完成。
- 异常怎么处理：模型失败、返回非法 audioId、无匹配都写失败原因。
- 限制：输出是单个 audioId，不是数组。

### 3. 写绑定关系

怎么做：
- 输入：assetId、audioId。
- 输出：绑定结果。
- 写什么数据：删除旧关系，插入新关系，更新 audio_bind_status。
- 状态怎么变：running -> succeeded。
- 异常怎么处理：写库失败回滚该资产绑定状态为 failed。
- 限制：一个资产一个音色。

### 4. 写任务中心

怎么做：
- 输入：批量绑定请求。
- 输出：任务记录。
- 写什么数据：tasks。
- 状态怎么变：running -> succeeded/failed。
- 异常怎么处理：失败原因脱敏。
- 限制：这是 VT Studio 增强，写 04。
