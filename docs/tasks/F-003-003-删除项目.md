# F-003-003 删除项目

状态：等待用户确认  
所属菜单：M-003 项目管理  
对应功能文档：`docs/features/M-003-项目管理.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：项目卡片点删除，确认后删除项目数据库关系，并按用户选择处理本地项目文件。
为什么现在做：项目是顶层数据，删除必须一次定清楚，否则后续容易残留原文、剧本、资产、任务和文件。
做完后有什么用：用户能安全删除项目，知道会影响哪些数据。
这一步不碰什么：不做批量删除，不做回收站，不做跨项目数据迁移。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow delProject，并补足本地桌面端文件删除边界。

只做：
  1. 删除按钮
  2. 删除确认弹窗
  3. 列出影响范围
  4. 删除项目相关数据库数据
  5. 可选处理项目本地文件目录
  6. 删除当前项目后清 currentProject
  7. 删除后刷新列表和清缓存

不做：
  1. 批量删除
  2. 回收站
  3. 数据恢复
  4. 手动编辑级联关系
```

```txt
4. VT Studio 必须把数据库删除和文件删除分开记录结果。
```

## 3. 用户操作

```txt
入口：
  项目页 -> 项目卡片 -> 删除按钮。

按钮/操作：
  1. 点击删除。
  2. 查看影响范围。
  3. 选择是否同时删除本地文件。
  4. 输入确认或点击确认。

弹窗/表单：
  删除确认弹窗。

成功反馈：
  删除成功，项目列表刷新。

失败反馈：
  数据库删除失败、文件删除失败、路径非法、项目不存在。
```

## 4. 要做什么功能

### 1. 打开删除确认

怎么做：
- 输入：项目 id。
- 输出：确认弹窗。
- 写什么数据：不写。
- 状态怎么变：弹窗打开。
- 异常怎么处理：项目不存在时刷新列表。
- 限制：点击删除按钮必须阻止打开项目事件。

### 2. 展示影响范围

怎么做：
- 输入：项目 id。
- 输出：原文、剧本、资产、分镜、视频、任务、Agent 记忆、本地目录影响说明。
- 写什么数据：不写。
- 状态怎么变：弹窗显示风险。
- 异常怎么处理：统计失败时仍可显示基础风险，但建议阻止删除。
- 限制：不能只显示一句“确定删除吗”。

### 3. 删除数据库关系

怎么做：
- 输入：项目 id。
- 输出：删除结果。
- 写什么数据：事务删除项目相关表数据。
- 状态怎么变：项目从列表移除。
- 异常怎么处理：任一数据库步骤失败则事务回滚。
- 限制：必须按 projectId 删除，不能误删其他项目。

### 4. 处理本地项目文件

怎么做：
- 输入：项目 workspace_path、是否删除文件。
- 输出：文件删除结果。
- 写什么数据：按用户选择删除项目目录。
- 状态怎么变：目录删除或保留。
- 异常怎么处理：路径不在项目目录内直接拒绝；删除失败记录原因但不伪装成功。
- 限制：文件删除必须独立于数据库结果记录。

### 5. 删除当前项目后的状态

怎么做：
- 输入：删除项目 id。
- 输出：currentProject 清空。
- 写什么数据：清本地项目上下文。
- 状态怎么变：如果删除的是当前打开项目，回到项目列表。
- 异常怎么处理：清理失败提示刷新应用。
- 限制：不保留坏 currentProject。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 文件删除失败不能只打日志，必须返回结果。
  2. 删除前列影响范围。
  3. 删除前检查运行中任务。

原因：
  本地文件是用户资产，删除必须可解释、可确认。

是否写入 04：
  不需要，属于安全增强。
```

## 8. 验收

```txt
1. 删除按钮不会触发打开项目。
2. 删除前有确认弹窗。
3. 确认弹窗列出影响范围。
4. 有运行中任务时阻止删除或要求先停止。
5. 数据库删除使用事务。
6. 文件目录删除有路径安全校验。
7. 文件删除失败会显示失败原因。
8. 删除当前项目后清 currentProject。
9. 删除成功后刷新列表和缓存。
10. 不误删其他项目数据。
```
