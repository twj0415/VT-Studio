## 0. 快速理解

```txt
一句话：资产中心做音频 tab，管理“音色父资产”和它下面的音频文件子项。
为什么现在做：角景页会用这些音色给角色/场景/道具做音频参考绑定。
做完后有什么用：用户能维护音色、性别/描述，以及多个参考音频文件和文本。
这一步不碰什么：不做 TTS 生成，不做音频绑定，不做视频配音。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow `type=audio` 的音频素材管理，同时修正 sex|describe 拼字段问题。

只做：
  1. 音频 tab
  2. 新增音色父资产
  3. 编辑音色父资产
  4. 新增/编辑/删除音频子项
  5. 上传音频文件
  6. 预览音频

不做：
  1. TTS 生成
  2. 角景音频绑定
  3. 批量 AI 绑定音频
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/assets/components/addAudioAssets.vue` | 音频父资产和子音频项表单 |
| `Toonflow-app/src/routes/assets/addAudioAssets.ts` | 写父 o_assets，子 o_assets，音频文件 o_image |
| `Toonflow-app/src/routes/assets/updateAudioAssets.ts` | 更新父资产、更新/新增/删除子音频 |
| `Toonflow-app/src/routes/assets/getAssetsApi.ts` | 音频父资产把 describe 拆成 sex 和 describe |

参考项目关键事实：

```txt
1. 音频资产是两层：父资产=音色，子资产=具体音频文件 + 音频文本。
2. Toonflow 把 sex 和 describe 拼到同一个 describe：sex|describe。
3. 子音频文件写入 /{projectId}/assets/audio/{uuid}.{ext}。
4. 角景页只绑定音频父资产，不在资产中心做绑定。
5. 源码未找到真正 TTS 生成逻辑。
```

## 3. 用户操作

```txt
入口：
  资产中心 -> 音频 tab。

按钮/操作：
  1. 新增音色。
  2. 填写音色名、性别、描述。
  3. 上传音频文件子项。
  4. 填写音频文本和描述。
  5. 编辑音色和子项。
  6. 删除音色或子项。

弹窗/表单：
  新增/编辑音频素材弹窗。

成功反馈：
  保存成功，刷新音频列表。

失败反馈：
  音色名缺失、音频文件不支持、写文件失败、删除失败。
```

## 4. 要做什么功能

### 1. 新增音频父资产

怎么做：
- 输入：name、voiceGender、description、audioItems。
- 输出：父音色资产和子音频资产。
- 写什么数据：写 assets 父项、assets 子项、media_files。
- 状态怎么变：保存中 -> 列表刷新。
- 异常怎么处理：必填缺失或文件写入失败时回滚。
- 限制：不把 gender 和 description 拼到一个字段。

### 2. 编辑音频资产

怎么做：
- 输入：父资产字段和子项列表。
- 输出：更新后的音频资产树。
- 写什么数据：更新父资产；更新/新增/删除子资产和文件记录。
- 状态怎么变：保存中 -> 列表刷新。
- 异常怎么处理：删除子项文件失败要提示。
- 限制：当前子项列表就是最终结果，移除的子项要删除。

### 3. 预览音频子项

怎么做：
- 输入：音频 filePath。
- 输出：受控 audio URL。
- 写什么数据：不写。
- 状态怎么变：播放状态由页面控制。
- 异常怎么处理：文件不存在显示缺失状态。
- 限制：renderer 不直接访问本地路径。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  sex 和 description 拆成正式字段。

原因：
  拼字段会导致搜索、编辑、多语言和后续维护困难。

是否写入 04：
  写入，属于数据结构修正。
```
