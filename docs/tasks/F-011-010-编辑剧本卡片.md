# F-011-010 编辑剧本卡片

状态：等待用户确认  
所属菜单：M-011 剧本 Agent  
对应功能文档：`docs/features/M-011-剧本Agent.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：右侧剧本卡片可以打开编辑弹窗，修改剧本名称和正文。
为什么现在做：Agent 生成的剧本需要人工修正，且要同步到真实剧本表。
做完后有什么用：用户能在 Agent 工作区直接改分集剧本。
这一步不碰什么：不新增资产提取，不删除剧本，不导出剧本。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 剧本卡片编辑能力，同时保证 agentWorkData 和 scripts 表一致。

只做：
  1. 剧本卡片编辑按钮
  2. 编辑弹窗
  3. 修改名称和正文
  4. 保存到 scripts 表
  5. 刷新 planData

不做：
  1. 删除剧本
  2. 新增剧本
  3. 资产提取
  4. 剧本导出
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/views/scriptAgent/index.vue` | 点击编辑打开 80% 宽 t-dialog，MdEditor 编辑 content |
| `Toonflow-app/src/routes/scriptAgent/setPlanData.ts` | 同步 planData 和 `o_script` |
| `Toonflow-app/src/routes/script/updateScript.ts` | 更新剧本内容 |

参考项目关键事实：

```txt
1. 编辑保存会更新 planData.script[index]。
2. 保存后调用 store.setPlanData，再刷新 getPlanData。
3. 参考项目里 updateData schema 要求 script[].id 必填，但新生成剧本可能没有 id。
4. VT Studio 必须统一按稳定 id 更新，不能只按数组 index。
```

## 3. 用户操作

```txt
入口：
  剧本 Agent 右侧 -> 剧本卡片 -> 编辑按钮。

按钮/操作：
  1. 点击编辑。
  2. 修改名称和内容。
  3. 保存。

弹窗/表单：
  剧本编辑弹窗。

成功反馈：
  保存成功，卡片更新。

失败反馈：
  名称为空、内容为空、保存失败。
```

## 4. 要做什么功能

### 1. 打开剧本编辑弹窗

怎么做：
- 输入：scriptId。
- 输出：剧本表单。
- 写什么数据：不写。
- 状态怎么变：dialogOpen=true。
- 异常怎么处理：剧本不存在时提示并刷新列表。
- 限制：优先用 scriptId，不用 name 查找。

### 2. 保存剧本修改

怎么做：
- 输入：scriptId、name、content。
- 输出：更新后的剧本。
- 写什么数据：更新 scripts 表，并同步 agent_work_data 必要快照。
- 状态怎么变：保存中 -> 成功关闭 -> 刷新 planData。
- 异常怎么处理：保存失败保留弹窗内容。
- 限制：事务同步，不能只改右侧卡片。

### 3. 处理新生成但无 id 的剧本

怎么做：
- 输入：没有 id 的剧本卡片。
- 输出：保存时创建真实 scripts 记录。
- 写什么数据：插入 scripts，再回写 id 到工作区数据。
- 状态怎么变：临时剧本 -> 持久化剧本。
- 异常怎么处理：创建失败提示。
- 限制：不能按 name 覆盖已有同名剧本。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  不按 name 覆盖剧本，使用稳定 id。

原因：
  同名剧本会导致错误覆盖。

是否写入 04：
  写入，属于修正源码风险。
```
