# F-002-006 提示词管理

状态：已完成  
所属菜单：M-002 设置  
对应功能文档：`docs/features/M-002-设置.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：管理内置核心提示词，用户能查看和编辑事件分析、资产提取、视频提示词、音频绑定等 prompt。
为什么现在做：这些提示词直接影响后续生成逻辑，必须有统一入口，而不是散落在代码里。
做完后有什么用：后续任务调用提示词时都从统一配置读取，用户也能针对自己的项目调优。
这一步不碰什么：不做模型映射里的新增提示词，不做 Agent 对话，不做提示词版本系统。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow 的 promptManage 能力，实现内置提示词列表、查看、编辑、保存。

只做：
  1. 查询提示词列表
  2. 展示提示词卡片
  3. 打开 Markdown 编辑弹窗
  4. 保存提示词内容
  5. 保存前基础校验
  6. 提示词为空/结构风险提示
  7. 恢复默认

不做：
  1. 模型与提示词映射
  2. 新增任意类型提示词
  3. 删除内置提示词
  4. 版本历史和回滚
  5. prompt 自动优化
```

## 2. 参考项目怎么做

| 参考文件 | 关键逻辑 |
|---|---|
| `Toonflow-web/src/components/setting/components/promptManage.vue` | 提示词卡片列表；点击卡片打开 Markdown 编辑器；确认后保存 |
| `Toonflow-app/src/routes/setting/promptManage/getPrompt.ts` | 返回 o_prompt 列表 |
| `Toonflow-app/src/routes/setting/promptManage/updatePrompt.ts` | 参数 id,data，更新 o_prompt.data |
| `Toonflow-app/src/lib/initDB.ts` | 初始化 eventExtraction、scriptAssetExtraction、videoPromptGeneration、audioBindPrompt |

参考项目关键事实：

```txt
1. 提示词管理只管理内置 prompt，不是模型映射里的 prompt 文件库。
2. 内置类型至少有 eventExtraction、scriptAssetExtraction、videoPromptGeneration、audioBindPrompt。
3. Toonflow 保存时直接更新 data；VT Studio 当前表结构有 data/use_data，必须更专业地保留默认值。
4. VT Studio 中 data 是内置默认提示词，use_data 是用户自定义提示词。
5. 业务读取有效提示词时优先 use_data，空则回退 data。
6. 提示词内容可能包含 XML/resultTool 等结构，保存时不能随便破坏。
```

## 3. 用户操作

```txt
入口：
  设置弹窗 -> 提示词管理。

按钮/操作：
  1. 查看提示词卡片。
  2. 点击某个提示词。
  3. 在 Markdown 编辑器里编辑。
  4. 点击确认保存。
  5. 点击恢复默认，确认后清空自定义内容并回到默认提示词。
  6. 关闭弹窗取消修改。

弹窗/表单：
  Markdown 编辑弹窗。

成功反馈：
  保存成功，新任务使用新提示词。

失败反馈：
  提示词不存在、内容为空、结构明显损坏、保存失败。
```

## 4. 要做什么功能

### 1. 查询提示词列表

怎么做：
- 输入：无。
- 输出：提示词列表，包含 id/name/type/data/useData/effectiveData/isCustomized/updatedAt。
- 写什么数据：不写。
- 状态怎么变：页面 loading -> 卡片列表。
- 异常怎么处理：读取失败显示错误；空列表显示初始化缺失。
- 限制：不返回无关系统配置。

### 2. 展示提示词卡片

怎么做：
- 输入：提示词列表。
- 输出：卡片 UI，显示名称、类型、默认/已自定义状态和摘要。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：内容过长只显示摘要。
- 限制：卡片不要展示全部 prompt，避免页面过重。

### 3. 编辑提示词

怎么做：
- 输入：当前提示词。
- 输出：可编辑 Markdown 文本，默认展示 effectiveData。
- 写什么数据：打开弹窗不写，保存才写。
- 状态怎么变：弹窗打开 -> 编辑中。
- 异常怎么处理：提示词不存在时弹窗不打开。
- 限制：编辑器不允许直接把本地图片拖入 renderer 并转存。

### 4. 保存提示词

怎么做：
- 输入：`id`、`useData`。
- 输出：保存成功。
- 写什么数据：只更新 prompts.use_data，不覆盖 prompts.data。
- 状态怎么变：保存中 -> 成功后关闭弹窗并刷新列表。
- 异常怎么处理：id 不存在、useData 为空、数据库失败返回明确错误。
- 限制：修改后只影响新任务，旧任务不回滚。

### 5. 结构风险校验

怎么做：
- 输入：提示词 type + useData。
- 输出：通过或风险提示。
- 写什么数据：不写。
- 状态怎么变：保存前提示。
- 异常怎么处理：空内容直接阻止；结构明显缺失时返回风险项，由 renderer 二次确认后才允许保存。
- 限制：不要过度解析模型提示词，只检查关键标记，比如 XML 标签、resultTool 名称、必需占位符。

### 6. 恢复默认入口

怎么做：
- 输入：提示词 id。
- 输出：默认内容生效。
- 写什么数据：用户确认后清空 prompts.use_data，不改 prompts.data。
- 状态怎么变：当前提示词回到 data 默认内容，状态变为“默认”。
- 异常怎么处理：默认内容不存在或提示词不存在时失败。
- 限制：参考项目未明确做恢复默认；VT Studio 可作为安全增强，但不能替代保存逻辑。

## 7. 偏差

```txt
和 Toonflow 不同的地方：
  1. 保存前增加空内容和结构风险校验。
  2. 建议增加恢复默认入口。
  3. 不允许删除内置提示词。
  4. 不覆盖 data，用户编辑只写 use_data。

原因：
  提示词是生成链路核心配置，随便删改会导致后续功能整体偏题。
  data 是默认值，use_data 是用户覆盖值，恢复默认只需要清空 use_data。

是否写入 04：
  不需要，属于安全增强。
```

## 12. 补充收口点

这一节把能确定的细节写死，避免后续只做一个“能编辑文本”的空页面。

```txt

CORE-003：prompts 表已由 SQLite 基础层和迁移管理。
CORE-005：错误 detail 只进日志，页面只显示 msg。
CORE-009：默认提示词由默认数据初始化写入。
F-002-017/F-002-005：模型服务和 Agent 高级配置已完成，本任务不再处理模型选择。
```

### 12.2 被谁依赖

```txt
F-004-005 事件分析生成：读取 eventExtraction。
F-005-007 从剧本提取资产：读取 scriptAssetExtraction。
F-008-013 视频提示词生成：读取 videoPromptGeneration，或先经过 F-002-004 的模型提示词映射。
F-007-002/F-007-003 音频绑定：读取 audioBindPrompt。
F-011/F-008 Agent 工具链：需要通过 prompt service 读取结构化提示词，不能在 Agent 代码里硬编码同一份 prompt。
```

### 12.3 提示词枚举和来源

```txt
内置提示词类型必须至少包含：
  eventExtraction
  scriptAssetExtraction
  videoPromptGeneration
  audioBindPrompt

默认内容来源：
  以 CORE-009 seed 写入的默认数据为准。
  restoreDefault 只能恢复 seed 内置默认副本，不能随便生成一段新 prompt。

用户修改后：
  prompts.use_data 保存用户版本。
  新任务读取 effectiveData = use_data || data。
  已创建任务、已生成结果不回滚。
```

### 12.4 保存校验

```txt
useData 不能为空。
useData 不能只包含空白字符。
id/type 不存在时不能保存。
内置类型不能改 type。
保存失败不能覆盖旧内容。
```

轻量结构校验只做“防明显改坏”，不做完整 prompt 语法解析：

```txt
eventExtraction：必须保留事件提取语义，不能保存完全空的 result/XML 指令。
scriptAssetExtraction：必须保留 resultTool、assetsList、角色/场景/道具等输出要求。
videoPromptGeneration：必须保留 storyboardItem、videoDesc、Seedance 2.0、Wan 2.6 等关键结构。
audioBindPrompt：必须保留音色、候选音频、audioId 等匹配说明。
```

如果校验无法精确判断：

```txt
明显错误：阻止保存。
疑似风险：弹二次确认。
正常文本：允许保存。
```

### 12.5 服务契约

```txt
settings.prompt.list
  输入：无
  输出：{ prompts: PromptSummary[] }
  失败：数据库读取失败、默认数据缺失

settings.prompt.update
  输入：id, useData, force?
  输出：{ saved, prompt, warnings }
  失败：提示词不存在、内容为空、数据库失败
  说明：结构风险时 saved=false 且返回 warnings；用户二次确认后带 force=true 保存。

settings.prompt.restoreDefault
  输入：id
  输出：{ prompt }
  失败：默认内容不存在、提示词不存在、数据库失败
```

### 12.6 数据契约

```txt
prompts.type 唯一标识内置语义，不允许页面随便改。
prompts.name 用于展示，可编辑与否由产品决定；第一版建议不编辑内置名称。
prompts.data 存内置默认内容，不被用户编辑覆盖。
prompts.use_data 存用户自定义内容，空字符串表示使用默认内容。
effectiveData = use_data || data，是后续业务实际读取的有效提示词。
prompts.updated_at 每次保存更新。
```

### 12.7 严格不允许遗漏

```txt
不能把提示词写死在某个生成页面里。
不能让页面直接读 SQLite。
不能提供删除内置提示词按钮。
不能让空内容覆盖已有可用提示词。
不能把 F-002-004 的“模型提示词文件库”和本任务的“内置核心提示词”混成一张页面逻辑。
不能覆盖 prompts.data。
restoreDefault 只能清空 prompts.use_data，不能清空默认内容。
```
