# F-011-013 Agent 工具读取上下文

状态：等待用户确认  
所属菜单：M-011 剧本 Agent  
对应功能文档：`docs/features/M-011-剧本Agent.md`  
原则：先确认本文档，再改代码

## 0. 快速理解

```txt
一句话：给剧本 Agent 提供工具，让它能读取原文、事件、计划数据和已有剧本。
为什么现在做：Agent 决策不能靠猜，必须能查真实项目上下文。
做完后有什么用：Agent 可以基于章节事件、原文和已有剧本继续工作。
这一步不碰什么：不写入剧本，不生成图片，不调生产 Agent。
```

## 1. 本次做什么

```txt
目标：
  对齐 Toonflow scriptAgent tools：get_novel_events、get_planData、get_novel_text、get_script_content。

只做：
  1. 读取章节事件
  2. 读取章节原文
  3. 读取当前计划数据
  4. 读取已有剧本内容
  5. 工具参数校验和返回格式

不做：
  1. 写剧本
  2. 删除剧本
  3. 资产生成
  4. 生产工作台工具
```

参考项目关键事实：

```txt
1. get_novel_events 参数是 chapterIndexs: number[]。
2. get_novel_text 参数是 chapterIndex: string。
3. get_planData 参数 key 可为 storySkeleton/adaptationStrategy/script。
4. get_script_content 参数 ids: string[]。
5. tools.ts 里 planData.script 定义成 string，但前端真实是数组，VT Studio 必须修正。
```

## 3. 用户操作

```txt
入口：
  用户和 Agent 对话时，Agent 自动调用工具。

按钮/操作：
  无直接按钮。

弹窗/表单：
  无。

成功反馈：
  Agent 回复更准确，工具调用可在聊天中展示。

失败反馈：
  工具参数错误或数据不存在时，Agent 收到明确错误。
```

## 4. 要做什么功能

### 1. 读取章节事件

怎么做：
- 输入：chapterIndexes。
- 输出：章节编号、标题、事件摘要、状态。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：章节不存在返回空结果和说明。
- 限制：只读当前项目章节。

### 2. 读取章节原文

怎么做：
- 输入：chapterIndex。
- 输出：章节正文。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：章节不存在返回工具错误。
- 限制：长文本要考虑截断或分块策略。

### 3. 读取计划数据

怎么做：
- 输入：key。
- 输出：storySkeleton/adaptationStrategy/scripts 中指定内容。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：key 非法返回错误。
- 限制：scripts 返回数组结构，不返回含糊字符串。

### 4. 读取已有剧本内容

怎么做：
- 输入：scriptIds。
- 输出：剧本名称和内容。
- 写什么数据：不写。
- 状态怎么变：无。
- 异常怎么处理：id 不存在时返回缺失列表。
- 限制：按 id 查询，不按 name 查询。
