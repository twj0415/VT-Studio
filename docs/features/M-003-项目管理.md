## 4. 页面操作细节

项目列表：

```txt
进入项目页时，将当前项目上下文置空
页面挂载后调用项目列表接口
新建项目按钮：editProjectData=null，dialogShow=true
点击项目卡片：执行打开项目逻辑
点击编辑按钮：stop 事件，打开项目弹窗并回填数据
点击删除按钮：stop 事件，打开确认框
```

打开项目：

```txt
1. 从 allProject 中按 id 找项目
2. 找不到时提示项目不存在
3. imageModel 或 videoModel 为空时，提示模型供应商不可用，并打开编辑弹窗
4. imageModel 存在时调用 /modelSelect/getModelDetail 校验
5. videoModel 存在时调用 /modelSelect/getModelDetail 校验
6. 任一模型校验失败，提示模型供应商不可用，并打开编辑弹窗
7. 校验通过后写入 project store
8. projectType=novel 跳转 /novel
9. projectType=script 跳转 /script
```

删除项目：

```txt
点击删除按钮后弹出确认框
确认后调用 /project/delProject
后端按 projectId 级联删除项目相关数据库数据
后端尝试删除 OSS 中 `${projectId}/` 目录，失败只打印日志，不阻塞接口成功
成功后 clearProjectCache(projectId)
成功后刷新项目列表
失败后显示接口错误信息
```

项目弹窗：

```txt
新增时使用默认空表单
编辑时 watch projectData 并回填表单
编辑时如果存在 videoModel，会主动调用 /modelSelect/getModelDetail 拉取 mode 列表用于回显
打开弹窗时刷新视觉手册列表和导演手册列表
确认时逐项校验必填字段
新增时 emit add
编辑时 emit edit
关闭时重置表单
```

视频模式：

```txt
modelSelect 通过 /modelSelect/getModelList 只查询 enable=1 的供应商
图片模型按 type=image 过滤
视频模型按 type=video 过滤
选择视频模型后，modelSelect 返回模型能力数据
从 data.mode 生成 mode 下拉选项
单模式直接保存字符串，例如 text、singleImage、startEndRequired
数组型多参考模式会转成 JSON 字符串保存
页面展示翻译后的模式名称
```

## 8. 不能漏的逻辑

1. 打开项目前必须校验图片模型和视频模型是否可用。
2. 模型不可用时不是直接失败，而是提示并打开编辑弹窗。
3. 新建/编辑项目必须逐项校验必填字段。
4. 视觉手册和导演手册的每个 Markdown tab 都不能为空。
5. 删除项目要二次确认，且后端会级联删除大量业务数据。
6. `artStyle` 是 Toonflow 旧字段名，VT Studio 文档和代码不要把它当正式命名。
7. `editDirectorlManual` 是 Toonflow 拼写错误，VT Studio 必须修正为 `editDirectorManual` 语义。
8. `driector_skills` 是 Toonflow 目录拼写错误，VT Studio 不能继续作为新目录名。
9. 手册新增/编辑源码校验的是 `name`，但真实路径字段是 `stylePath` / `directorManual`；VT Studio 必须校验真实路径字段，防止路径穿越。
10. 删除视觉手册/导演手册会递归删除整个目录，必须二次确认，并处理项目已引用该手册时的影响。

## 9.1 参考项目废弃/未接线接口（不要复刻）

参考项目存在一组 `general/*` 项目接口，与 `project/*` 重复或失效，VT Studio 不照搬，统一并入 `project.*` 能力。详见 `features/00-公共底层.md` 8.8。

| 源码接口 | 状态 | VT Studio 处理 |
|---|---|---|
| `general/getSingleProject` | 前端调 `/project/getSingleProject`，后端挂 `/api/general/getSingleProject`，路径不匹配，调用失效（404） | 并入 `project.get`，按 id 查单个项目 |
| `general/updateProject` | 前端零调用，与 `project/editProject` 重复 | 并入 `project.update`，不另立接口 |
| `general/generalStatistics` | 前端未接线；返回项目下角色/剧本/视频/分镜数量 | **候选增强**：功能有价值，VT Studio 可在项目卡片或项目首页显示概览数据；正式开发 M-003 时再定是否做，做则记入 04 |

## 11. 缺少的逻辑和实现细节

### F-003-001 新建项目

参考项目新建项目主要写数据库，不负责创建完整本地项目目录。VT Studio 是本地桌面端，必须把“项目记录”和“项目文件目录”分清。

必须明确：

```txt
提交前校验：项目名、简介、来源类型、图片模型、图片质量、视频模型、视频模式、画面比例、视觉手册、导演手册
来源类型：先只允许 novel/script
图片质量：参考项目使用 1K/2K/4K
画面比例：用于视频/分镜链路，不能和图片生成分辨率混用
项目业务目录：源码未找到 Toonflow 创建本地目录；VT Studio 建议在首次导入/生成文件时创建
目录位置：默认 userData/projects/<projectId> 或用户选择的工作区，不能放源码目录
```

### F-003-002 编辑项目

编辑项目不只是改名称。参考项目打开项目时会校验模型，模型失效时会打开编辑弹窗让用户修。

必须明确：

```txt
回填字段必须包含模型、画质、比例、手册
模型被删除或禁用：编辑弹窗要能重新选择
手册被删除或路径失效：编辑弹窗要显示失效状态，不能静默空白
保存后：刷新项目卡片，并更新当前项目上下文
```

### F-003-003 删除项目

参考项目删除数据库关系很多，但本地文件目录删除没有完整源码事实。

必须明确：

```txt
删除前必须列出影响：原文、剧本、资产、分镜、视频、任务、Agent 记忆
数据库删除必须事务化
文件删除必须独立记录失败原因
如果选择只删数据库：保留业务文件并提示残留路径
如果选择数据库+文件：先校验路径必须在项目业务目录内
删除当前打开项目：删除成功后清空 currentProject 并回到项目列表
```

### F-003-004 打开项目

打开项目不是路由跳转这么简单，必须先校验模型和项目上下文。

必须明确：

```txt
先查项目是否存在
再校验图片模型和视频模型是否仍存在且供应商启用
校验失败：提示具体哪个模型不可用，并打开编辑项目弹窗
校验成功：写当前项目上下文，再按 sourceType 跳转
sourceType=novel：进入原文页
sourceType=script：进入剧本页
```

### F-003-005 项目模型和画质配置

参考项目模型能力来自供应商 models 配置，不是页面写死。

必须明确：

```txt
图片模型：只显示启用供应商里的 image 模型
视频模型：只显示启用供应商里的 video 模型
视频模式：来自选中 video model 的 mode
视频时长/分辨率：后续 M-008 从 durationResolutionMap 读取
图片质量：1K/2K/4K，不等于视频分辨率
模型不存在：不能保存项目
```

### F-003-006 / F-003-007 视觉手册和导演手册

手册本质是本地 skills 目录里的 Markdown 文件集合。VT Studio 不能只保存一个名字。

必须明确：

```txt
新增手册：必须校验目录名安全、文件完整、封面存在或可为空
编辑手册：每个 Markdown tab 保存到对应文件
删除手册：递归删除目录前二次确认
项目引用中：删除要阻止或清空引用，不能让项目保留坏路径
Toonflow 的 driector_skills 拼写只做兼容读取，新写入目录必须用 director_skills
```
