## 2. Toonflow 底层形态

```txt
前端：Toonflow-web，Vue 页面通过 axios 调本地服务
本地服务：Toonflow-app，Express routes + Socket.IO
数据库：SQLite，Knex + better-sqlite3
数据库初始化：initDB.ts
模型调用：utils/ai.ts + utils/vendor.ts + utils/vm.ts
供应商代码：vendor 目录下的 TypeScript 脚本
任务记录：utils/taskRecord.ts 写 o_tasks
Agent 对话：Socket.IO namespace /socket/scriptAgent 和 /socket/productionAgent
```

VT Studio 技术栈不同，但业务语义要映射到：

```txt
renderer -> window.vtStudio -> main/ipc -> main/services
```

## 8.1.1 本地媒体访问服务（CORE-012）

参考项目用 `utils/oss.ts`：文件存本地 `userData/data/oss/`，但通过一个**本地 HTTP 服务**（`http://localhost:端口/oss/xxx.jpg`）把访问 URL 返回给前端，前端 `<img>` 当网图加载。缩略图（`getSmallImageUrl` + sharp）在源码里**整段被注释掉了，当前是假的**，只返回原图加 `?size=20`。

VT Studio 决策（已写入 04，D-BASE-012）：

```txt
统一用本地 HTTP 媒体服务显示所有本地图片/视频/音频，不区分图片走协议、视频走服务。
原因：最终要导剪映，视频是核心；本地 HTTP 服务天然支持视频拖进度条（range 分段加载），
      自定义协议要自己手写 range；一套方案心智成本最低，且参考项目已跑通。
```

必须做到（不能照搬参考项目的裸奔实现）：

```txt
只绑 127.0.0.1，不对外网开放
加访问 token 校验：只有 VT Studio 前端带 token 能取文件，防止本机其它程序/恶意网页读取创作文件
服务随应用启动/退出，根目录锁死在受控目录内，保留路径逃逸校验
补上参考项目没做完的缩略图：列表用缩略图、点开看原图（复用 utils/image.ts 的 sharp 能力）
```

VT Studio 落点：

```txt
能力归属：CORE-012，是 CORE-004 文件层的延伸
能力名建议：media.getUrl / media.getThumbnailUrl
资产中心、生产工作台、导出在开工前必须先完成 CORE-012
```

## 8.2 Result 和错误规范

Toonflow 多数 route 直接返回数据或抛异常。VT Studio 要统一，避免页面各写各的错误处理。

固定响应结构：

```txt
成功：{ code: 200, data: {}, msg: "成功" }
失败：{ code: 400, data: {}, msg: "错误提示" }
```

说明：

```txt
内部可以用 statusCode/errorKey/detail 细分错误。
返回给页面时 code 只用 200/400。
msg 走统一文案，后续接语言设置。
detail 只进日志，不返回页面，避免泄露本地路径、密钥和 stack。
```

错误必须覆盖：

```txt
参数缺失
数据不存在
数据库失败
文件路径非法
文件不存在
模型未配置
供应商未启用
模型调用失败
任务已取消
权限或登录状态无效
```

页面只负责展示错误，不负责拼业务失败原因。

## 8.4 模型和供应商边界

Toonflow 支持在设置里编辑 vendor 代码并由后端加载执行。VT Studio 可以保留这个能力，但必须在 main 侧受控执行。

必须保留的语义：

```txt
供应商启用/禁用
供应商动态输入项
文本/图片/视频/TTS 模型能力。注意：供应商模型类型用 `tts`，生成结果是 audio base64；资产类型里的 `audio` 不能和供应商模型类型混用。
模型测试
Agent 模型映射
提示词映射
调用失败原因
```

不能做：

```txt
renderer 执行供应商代码
renderer 保存 apiKey 后直接调用 SDK
模型失败只给“失败”两个字
页面绕过任务服务直接更新生成状态
```

## 8.5 Agent 事件契约

Toonflow 用 Socket.IO。VT Studio 已确认继续使用 Socket.IO，与参考项目事件流对齐；CORE-008 已落地本地 HTTP server + Socket.IO namespace。

必须保留的事件语义：

```txt
打开会话
发送用户消息
接收 thinking / reasoning / markdown / toolcall / suggestion
停止生成
更新上下文
读取/保存 plan data
读取/清理记忆
写入业务数据
```

消息状态：

```txt
pending
streaming
complete
stop
error
```

## 8.9 日志与诊断层（CORE-013）

参考项目没有找到统一日志层，VT Studio 不能继续把 `console.info/error` 散在各个服务里。后面模型调用、任务队列、导出、文件服务、Socket 都会打日志，必须先定规则。

终端日志只给人看：

```txt
──────────────────────────── VT Studio 启动 ────────────────────────────

10:30:18 INFO  │ 启动         │ VT Studio 正在启动...

10:30:18 INFO  │ 运行目录     │ 已就绪：开发 userData 路径

10:30:18 INFO  │ 数据库       │ 已连接：表数量、迁移数量

10:30:18 INFO  │ 任务中心     │ 已恢复 N 个运行中任务

10:30:18 INFO  │ 本地服务     │ 已启动：http://127.0.0.1:端口

10:30:18 INFO  │ Agent 连接   │ 已就绪

10:30:18 INFO  │ 桌面窗口     │ 已打开

10:30:18 INFO  │ 启动         │ VT Studio 启动完成

────────────────────────────── 启动完成 ──────────────────────────────
```

详细日志写文件：

```txt
开发：%TEMP%\VT Studio Dev\user-data\logs\main.log
生产：%LOCALAPPDATA%\VT Studio\user-data\logs\main.log
```

日志规则：

```txt
main 侧不允许业务模块直接散写 console，必须走 logger
终端只输出简短中文状态，不输出大对象、stack、密钥、模型请求全文
终端输出固定为：时间 + 级别 + 模块 + 文案
模块名固定列宽，避免左边长短不齐
级别使用颜色：info 绿色、warn 黄色、error/fatal 红色、debug 紫色
启动和完成使用分隔线，方便从 Vite 输出里快速定位 VT Studio 日志
终端每条 VT Studio 日志后空一行，提升扫读性
detail 写 main.log，方便排查，但必须脱敏 token/apiKey/password/secret/authorization
debug/detail 默认只写文件，不刷终端
warn 表示功能跳过但不影响启动
error/fatal 表示功能失败或无法继续
任务日志必须带 taskId
模型日志必须带 provider/model/requestId，密钥和原文不进终端
```

当前已落地：

```txt
src/main/services/logger.ts
src/main/index.ts 启动链路日志
src/main/app/server.ts 本地服务日志
src/main/app/window.ts 窗口加载失败日志
src/main/services/socket/index.ts Agent Socket 日志
src/main/services/database/seed.ts 默认数据和 ONNX 跳过提示
src/main/services/result.ts 服务错误日志
src/main/services/model/vendor-runner.ts 供应商脚本日志转入文件 detail
```

## 10. 本轮复扫补充

### 请求和响应

Toonflow 前端有 axios 封装：baseUrl、timeout、Authorization、401 跳登录、`Network Error` 单独提示。VT Studio 不照搬 axios 调本地服务，但这些语义要落在 `window.vtStudio` 调用层：

```txt
请求超时：由设置或服务默认值控制
登录失效：统一返回 code=400、msg=登录已失效，路由层处理跳转
Network Error：本地服务不可用时显示“本地服务不可用/请重启应用”，不要显示原始英文
所有业务返回：{ code, data, msg }
```

### 图片性能插件

参考项目 `utils/imageOptimizer.ts` 在 `main.ts` 全局安装，做三件事：

```txt
给新增 img 补 loading="lazy"
给新增 img 补 decoding="async"
用 content-visibility:auto 降低图片列表渲染成本
MutationObserver 只处理新增节点
```

VT Studio 不需要一开始做成复杂插件，但资产中心、角景、生产工作台图片列表开工前要有同等规则。若做插件，归公共 UI/性能任务；若分散写在组件里，必须保证不漏。

### 标题栏和本地协议边界

参考项目 `titleBar.vue` 调 `toonflow://windowMinimize`、`toonflow://windowMaximize`、`toonflow://windowClose`，`App.vue` 调 `toonflow://getAppUrl/openurlwithbrowser`。VT Studio 不能保留协议名，统一走：

```txt
window.vtStudio.window.minimize()
window.vtStudio.window.toggleMaximize()
window.vtStudio.window.close()
window.vtStudio.runtime.getInfo()
window.vtStudio.shell.openExternalByKey()
```
