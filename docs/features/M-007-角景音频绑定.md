## 2. 功能边界

这个页面在 Toonflow 里叫角景，但实际不是单纯“音频绑定”。它同时做：

```txt
角色/场景/道具资产看板
批量提示词润色
批量图片生成
单个资产图片重新生成
历史图片切换
单个资产音频参考选择
批量 AI 匹配音频参考
图片生成取消
提示词/图片/音频绑定三类轮询
```

资产来源：

```txt
只取父资产
只取 role / scene / tool
排除 clip
排除 audio
排除 assetsId 不为空的子资产
```

音频绑定不是生成音频。Toonflow 的“批量绑定音频”是：

```txt
读取资产中心已有的 audio 父资产
用 universalAi 文本模型从候选音色中匹配一个
写 o_assetsRole2Audio
```

VT Studio 落地时不要把它做成 TTS 或音频生成。

## 5. 查询角景资产

调用：

```txt
POST /cornerScape/getAllAssets
```

参数：

```txt
projectId
type?: string[]
```

前端触发：

```txt
页面 mounted
素材类型筛选变化
取消生成后刷新
生成/绑定轮询完成后刷新
打开抽屉时刷新当前项
```

查询逻辑：

```txt
1. o_assets left join o_image on o_assets.imageId = o_image.id
2. where o_assets.projectId = projectId
3. where o_assets.type != clip
4. where o_assets.type != audio
5. where o_assets.assetsId = null
6. 如果 type 有值，whereIn o_assets.type
7. 排序：role -> scene -> tool -> 其他
8. 查询 o_assetsRole2Audio，拿每个资产绑定的音频名称
9. 查询每个资产的历史成功图片 o_image where assetsId=资产 id and state=已完成
10. filePath 和历史图片 filePath 转成 small image url
```

返回附加字段：

```txt
historyImages: [{ id, filePath }]
relepedAudio: [{ id, name }]
```

卡片空状态：

```txt
没有 state 且 promptState 不是生成中：显示等待生成
state=生成中 或 promptState=生成中 或 audioBindState=生成中：显示 loading
state=生成失败：显示失败，popup 展示 errorReason
filePath 有值：显示图片
没有数据：显示“请先操作剧本”
```

## 15. 批量绑定音频

按钮：左侧「一键绑定音频」。

前端校验：

```txt
selectedIds.length > 0
```

前端逻辑：

```txt
1. 取选中资产
2. 本地把 item.audioBindState = "生成中"
3. 清空 selectedIds
4. 调 /cornerScape/batchBindAudio
5. watch audioBindData 启动 3 秒轮询
```

调用：

```txt
POST /cornerScape/batchBindAudio
```

参数：

```txt
projectId
assetsIds
concurrentCount = otherSetting.assetsBatchGenereateSize
```

后端逻辑：

```txt
1. 查询待绑定资产：
   o_assets where id in assetsIds and projectId = projectId
   select id/name/describe/type

2. 查询候选音频：
   o_assets where type="audio" and assetsId is null and projectId = projectId
   select id/name/describe

3. 如果没有候选音频：
   返回 400：暂无设置音频，请先前往资产中心上传音频

4. 批量更新待绑定资产：
   audioBindState = "生成中"

5. runWithConcurrency 后台执行，不等待完成

6. 每个资产调用 u.Ai.Text("universalAi").invoke

7. 模型必须调用 resultTool 返回 audioId

8. resultTool 执行：
   删除旧 o_assetsRole2Audio
   如果 result.audioId 有值，插入新关系
   更新 o_assets.audioBindState = "已完成"

9. catch：
   更新 o_assets.audioBindState = "生成失败"
```

候选音频文本：

```txt
- ID:${id} | 名称:${name} | 描述:${describe ?? "无"}
```

待匹配资产文本：

```txt
- ID:${asset.id} | 名称:${asset.name} | 描述:${asset.describe ?? "无"} | 类型：${asset.type}
```

提示词来源：

```txt
o_prompt where type = "audioBindPrompt"
优先 useData
否则 data
```

默认提示词含义：

```txt
你是一个音色匹配助手。
根据角色资产名称和描述，从候选音频列表中选最合适的音色。
优先根据性别、年龄、性格等特征匹配。
同一角色仅可匹配一个音色。
没有合适音色则无需返回 audioId。
```

源码风险：

```txt
resultTool 的 schema 描述写“audioId 列表/空数组”，但实际字段是单个 audioId。
VT Studio 必须按单个 audioId 设计，描述也要修正。
```

## 16. 三类轮询

### 16.1 提示词轮询

条件：

```txt
notCompultedData = dataList.filter(promptState == "生成中")
```

每 3 秒调用：

```txt
POST /assets/pollingPromptAssets
```

参数：

```txt
ids = notCompultedData.map(id)
```

返回非生成中资产后更新：

```txt
promptState
prompt
```

如果有完成项：

```txt
重新调用 /cornerScape/getAllAssets
刷新 historyImages
同步 currentItem.historyImages
```

### 16.2 图片轮询

条件：

```txt
generatingData = dataList.filter(state == "生成中")
```

每 3 秒调用：

```txt
POST /assets/pollingImageAssets
```

返回后更新：

```txt
state
filePath
```

如果有完成项：

```txt
重新调用 /cornerScape/getAllAssets
刷新 historyImages
同步 currentItem.historyImages
```

### 16.3 音频绑定轮询

条件：

```txt
audioBindData = dataList.filter(audioBindState == "生成中")
```

每 3 秒调用：

```txt
POST /cornerScape/pollingAudio
```

参数：

```txt
ids
```

后端：

```txt
o_assets where id in ids and audioBindState != "生成中"
```

返回后更新：

```txt
audioBindState
filePath 如果有
```

如果有完成项：

```txt
重新调用 /cornerScape/getAllAssets
刷新 relepedAudio
同步 currentItem.relepedAudio
```

## 17. 状态枚举

源码状态：

| 字段 | 状态 | 说明 |
|---|---|---|
| `state` | 空/null | 未生成图片 |
| `state` | `生成中` | 图片生成中 |
| `state` | `已完成` | 图片生成成功 |
| `state` | `生成失败` | 图片生成失败或取消 |
| `promptState` | `生成中` | 提示词润色中 |
| `promptState` | `已完成` | 提示词成功 |
| `promptState` | `生成失败` / `失败` | 提示词失败 |
| `audioBindState` | `生成中` | 正在 AI 匹配音色 |
| `audioBindState` | `已完成` | 音频绑定完成 |
| `audioBindState` | `生成失败` | 音频绑定失败 |

VT Studio 要求：

```txt
内部不要直接存中文状态。
统一使用 running/succeeded/failed/cancelled 等枚举。
```

## 19. 源码冲突和风险

1. 页面名是角景，但功能包含图片生成、提示词润色和音频绑定，VT Studio 文档和 UI 要讲清楚用途，避免用户误解。
2. `relepedAudio` 拼写错误，VT Studio 正式字段不能照搬。
3. `o_assetsRole2Audio.assetsRoleId` 命名只写角色，但实际绑定 role/scene/tool，VT Studio 应改成更通用的 `assetId`。
4. `audioBindState` 在 Toonflow 表结构里是 integer，但源码写中文字符串，VT Studio 必须修正。
5. 批量绑定音频的 `resultTool` 描述说列表/空数组，实际只接收单个 `audioId`，VT Studio 必须按单个音色实现。
6. 历史图片取消选择时可能向 `/assets/saveAssets` 传 `imageId=undefined`，有误清当前图风险。
7. 提示词失焦保存复用 `/assets/saveAssets`，语义太重；VT Studio 应拆成 `asset.updatePrompt`。
8. `onUnmounted` 会把本地 `state=生成中` 重置为空，但这只是前端内存，不会取消后端任务；VT Studio 不要把它当取消逻辑。
9. 批量图片生成前端本地设置 `state=生成中`，但如果接口还没成功就失败，状态可能需要回滚。
10. 音频绑定失败只写 `audioBindState=生成失败`，没有失败原因字段，VT Studio 应补 `audioBindErrorReason` 或统一任务失败原因。

## 22. VT Studio 落地要求

1. 角景页复用资产中心的资产生成服务，不能另写一套图片生成逻辑。
2. 单项提示词保存、图片选择、图片生成要拆成清晰服务，避免一个 `saveAssets` 做太多事。
3. 音频绑定只绑定资产中心已有音色，不生成新音频。
4. 一键绑定音频必须读 `audioBindPrompt`，允许用户在提示词管理里修改。
5. 批量功能必须使用统一任务队列和状态，不只靠前端轮询字段。
6. 取消生成必须传图片任务/图片记录 id，不能传资产 id。
7. 所有文件预览 URL 必须由主进程受控生成。
8. Toonflow 的中文状态、错拼字段、泛化不足的关系表名只做参考，不作为 VT Studio 正式命名。

## 23. 缺少的逻辑和实现细节

### F-007-001 角色/场景/音频资产列表

角景页不是单纯列表页，它把 role/scene/tool 和 audio 放在一起，用于生成图片和绑定音色。

必须明确：

```txt
资产来源：当前项目 assets
显示类型：角色、场景、道具、音频
历史图片：来自 o_image
已绑定音频：来自 o_assetsRole2Audio
音频素材：只使用已有 audio 资产，不上传新音频

```

### F-007-002 资产音频绑定

参考项目一次只绑定一个音频，不支持多个音色。

必须明确：

```txt
单选音频：写 assetId -> audioAssetId 关系
移除绑定：删除对应关系
一个资产多个音频：源码未找到，第一版不做
绑定对象：role/scene/tool 都可绑定，关系表正式命名不能只叫 role
```

### F-007-003 / F-007-004 批量绑定音频和轮询

一键绑定音频是模型选择已有音色，不是生成音频文件。

必须明确：

```txt
输入：待绑定资产列表 + 可用音频列表 + audioBindPrompt

成功：写绑定关系，audioBindState=succeeded
失败：写 failed，VT Studio 要补失败原因字段

任务中心：建议写任务记录，但这是 VT Studio 增强
```

### F-007-005 角景页复用资产提示词和图片生成

角景页图片生成不能另起一套逻辑，要复用 M-006。
