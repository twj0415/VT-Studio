### F-002-001 外观设置

前端：`uiConfig.vue`、`utils/theme.ts`、`stores/setting.ts`  
后端：无  
数据来源：Pinia `settingStore.themeSetting`，通过持久化插件保存到本地存储

页面结构：

```txt
颜色模式 radio group
主题色预设色块
自定义主题色 color picker
字体大小 radio group
```

字段和行为：

```txt
themeSetting.mode：auto / light / dark
themeSetting.primaryColor：HEX 主题色
themeSetting.fontSize：12 / 13 / 14 / 16 / 18 / 20 / 22
```

按钮和操作：

| 操作 | 源码行为 | 影响范围 |
|---|---|---|
| 切换自动/浅色/深色 | `watch(themeSetting.mode)` 后调用 `toggleThemeWithTransition`、`applyThemeMode`、`applyThemeColor` | `document.documentElement` 的 `theme-mode` 属性和 `dark` 类 |
| 点击预设色 | 直接写 `themeSetting.primaryColor` | 重新生成 TDesign 品牌色 CSS 变量 |
| 选择自定义颜色 | `t-color-picker` 写 `themeSetting.primaryColor` | 只允许 HEX，不启用 alpha |
| 切换字体大小 | `watch(themeSetting.fontSize)` 后写 `document.documentElement.style.fontSize` | 全局 rem 基准字号 |

主题色处理：

```txt
预设色：#000000 / #0052D9 / #2BA471 / #ED7B2F / #E34D59 / #7B61FF / #111111
normalizeColor：空值或非法 HEX 回退 #0052D9
applyThemeColor：HEX -> HSL -> 10 阶品牌色，写入 --td-brand-color-* 和文本链接色变量
initTheme：App 启动时应用本地保存的 mode / primaryColor / fontSize
auto 模式：监听 prefers-color-scheme 变化，系统变更时自动切换 light/dark
```

VT Studio 落地：

```txt
该功能可以保留为 renderer 本地状态，不需要走 SQLite
如果后续要跨设备同步，再增加 setting service；当前本地桌面版不需要
不能把 TDesign 主题变量散落在页面里，统一走 theme 工具
```

| C-002-006 | 其他配置是否继续使用本地持久化 | 建议保留本地持久化，但把会影响业务产物的字段同步写入 settings 表，便于排查 |

## 11. 缺少的逻辑和实现细节

### F-002-001 外观设置

参考项目不是多套主题系列，只是 `auto/light/dark`、主色、字号。VT Studio 如果要做多种主题配色系列，属于增强，不是 Toonflow 已有逻辑。

必须明确：

```txt
参考项目模式：auto/light/dark
参考项目主色：预设 HEX + 自定义 HEX
参考项目字号：12/13/14/16/18/20/22，写 documentElement font-size
参考项目未找到：主题系列、主题市场、每个模块独立配色
VT Studio 建议：增加 themePresetId，但仍输出统一 CSS variables，不让页面各写一套颜色
```

### F-002-003 模型供应商配置

参考项目供应商不是固定 SDK，而是动态 vendor 脚本。文本模型可通过 AI SDK 工厂函数，图片/视频/TTS 可直接 HTTP、轮询或自定义适配。

必须明确：

```txt
供应商字段：id/version/name/author/description/icon/inputs/inputValues/models
输入项字段：key/label/type/required/placeholder/disabled
模型类型：text/image/video/tts
图片模型能力：mode=text/singleImage/multiReference
视频模型能力：mode、audio、durationResolutionMap
调用出口：textRequest/imageRequest/videoRequest/ttsRequest
执行位置：只能在 main/services 的 Model Adapter 中执行
失败原因：缺 API Key、供应商未启用、模型不存在、协议失败、轮询超时都要分开
```

ComfyUI 细节：

```txt
Toonflow-web 只找到 comfyui provider 图标和名称匹配。
Toonflow-app 未找到 ComfyUI vendor 脚本、workflow 配置、queue/prompt/history 轮询实现。
如果 VT Studio 支持 ComfyUI，必须作为增强能力写入 04，字段至少包括 endpoint、workflow、输入节点映射、输出节点映射、queueId、pollInterval、timeout、cancel。
```

补充：供应商代码相关操作是高风险能力，不能只做普通保存。

```txt
文件导入供应商代码：必须显示来源、供应商 id、导出函数、权限风险，用户二次确认后才保存
链接读取供应商代码：只读取预览，不自动保存；校验 URL、超时、内容类型、大小、导出结构
编辑供应商代码：保存前语法校验和导出校验；保存失败不能覆盖旧可用代码
模型模式：图片 mode 至少覆盖 text/singleImage/multiReference；视频 mode 覆盖 singleImage/startEndRequired/endFrameOptional/startFrameOptional/text/audioReference/videoReference 和 imageReference:N/videoReference:N/audioReference:N
视频参数：durationResolutionMap、audio 必须可配置并参与测试
```

### F-002-004 模型与提示词映射

参考项目按模型绑定提示词文件，不是按页面临时选择提示词。后续图片、视频、事件、资产提取等生成链路都要从这里拿模板。

必须明确：

```txt
绑定对象：供应商模型 + 提示词文件
提示词文件：来自 data/modelPrompt 或 prompt 管理
视频模式：不同 model/mode 可对应不同提示词模板
删除模型：必须处理已绑定提示词是否失效
删除提示词：必须处理已绑定模型是否回退默认
```

### F-002-005 Agent 参数与高级覆盖配置

参考项目有简易/高级模式。VT Studio 保留高级能力，但普通模式不再重复配置模型，默认继承模型服务里的默认文本模型。

必须明确：

```txt
Agent key 必须固定枚举，不能页面随便写字符串
universalAi 是公共文本能力，默认继承默认文本模型
剧本 Agent 和生产 Agent 子能力只有高级模式才能独立覆盖模型
thinkLevel 只在模型支持 think 时显示
maxOutputTokens=0 表示自动，不是 0 token
模型解析顺序必须稳定：Agent 覆盖 -> 默认文本模型 -> 未配置错误
高级模式批量设置：必须支持全选和部分选择，统一更新覆盖模型、temperature、maxOutputTokens
复用 agent_model_configs，不新增 AgentOverride 表
ttsDubbing 不放在本任务做
```

### F-002-006 提示词管理

提示词不是普通文本配置，它会影响事件分析、剧本资产提取、音频绑定、视频提示词等链路。

必须明确：

```txt
提示词类型必须统一枚举
保存前不能破坏 XML/resultTool 所需结构
修改提示词后新任务生效，旧任务不回滚
空提示词不能覆盖已有可用提示词
Markdown 编辑器不能上传本地图片到 renderer
```

### F-002-007 Skill 管理

参考项目 Skill 是本地 Markdown 文件，不是数据库配置。VT Studio 必须保护文件路径。

必须明确：

```txt
读取范围：只允许 skills 根目录下的白名单文件
保存范围：不能路径穿越，不能写任意本地文件
编辑格式：Markdown 原文
保存失败：要区分权限、文件不存在、路径非法
手册管理 M-003 和 Skill 管理共用 skills 文件，但入口语义不同
```

### F-002-008 记忆配置

记忆配置影响 Agent 检索和摘要，不能只做表单保存。

必须明确：

```txt
embedding 模型路径必须由 main 校验存在
messagesPerSummary/shortTermLimit/ragLimit 等要限制范围
清空记忆必须二次确认
清空范围：全部记忆、当前项目记忆、指定 Agent 记忆要分清
修改配置不会自动重建旧 embedding，除非单独提供重建能力
```

### F-002-009 / F-002-016 登录和退出

参考项目登录写 `token/userId`，退出删 `token/user`，存在不一致。VT Studio 必须修正。

必须明确：

```txt
退出时清理 token/user/userId/currentProject
本地优先版本不应保留云账号假象
如果弱化登录，路由守卫要同步调整，不能出现半登录状态
```

### F-002-010 数据库管理

数据库导入导出是高风险能力，不能做成普通按钮。

必须明确：

```txt
导出：输出 SQLite 备份或 JSON 备份的具体格式必须固定
导入：导入前自动备份当前库
清表：必须列出表名、影响模块、不可恢复提示
运行中任务：导入/清表前必须阻止或要求先停止任务
版本迁移：导入旧库后必须执行 migration
```

### F-002-011 文件管理

参考项目只打开固定目录。VT Studio 要符合 runtime 规则，不能打开项目源码里的 `.runtime`。

必须明确：

```txt
可打开目录：userData、项目业务目录、导出目录、日志目录、缓存目录
禁止默认目录：D:\project\vt-studio\.runtime、项目源码 data/temp
打开失败：路径不存在、权限不足、系统拒绝要分别提示
```

### F-002-012 / F-002-013 / F-002-014 其他、请求、开发者配置

这些配置会影响业务结果，不能散在 localStorage 后无人知道。

必须明确：

```txt
章节正则：保存前必须试编译，失败不保存
请求超时：页面显示秒，内部存毫秒
资产生成并发：必须限制最小 1，建议设置最大值，避免压垮模型服务
字典值：资产类型、媒体类型、音频性别、任务状态、模型类型要集中到 constants，不在页面硬编码
请求地址：生产桌面版不建议给普通用户修改，只放开发者模式
AI DevTools：只能开发环境显示，不能在生产误开启
```

### F-002-007 Skill 管理补充

参考项目有 `scanSkills` 工具，但设置页扫描接口未找到后端注册，不能写成已完成链路。

必须明确：

```txt
Skill 列表读取：已有 getSkillList/getSkillContent/saveSkillContent
扫描 Skills：后端 route 未找到，第一版不默认做按钮
如果新增扫描：只扫描受控 skills 根目录，不扫描用户任意目录
扫描结果：写 skill_list 和 skill_attributions，embedding 生成归 CORE-011
```

### F-002-013 请求错误处理补充

参考项目 axios 对 `Network Error` 做了特殊通知。VT Studio 走 preload/main，但仍要有同等用户提示。

```txt
本地服务不可用：提示重启应用或查看开发日志
登录失效：提示登录状态失效并跳登录/初始化页
超时：提示请求超时，并显示当前超时配置
错误格式：页面只读 { code, data, msg }，不读 message/detail
```
