## 2. 功能边界

资产中心负责管理项目里的资产库：

```txt
角色 role
道具 tool
场景 scene
片段素材 clip
音频素材 audio
```

资产中心负责：

```txt
资产列表
新增资产
编辑资产
删除资产
批量删除资产
上传片段素材
新增/编辑音频素材
素材预览
提示词润色
图片生成
生成状态轮询
资产图片选择和保存
```

资产中心不负责：

```txt
把角色/场景/道具绑定到音频
批量自动绑定音频
音频绑定轮询
生产流程里的衍生资产调度
```

这些在 Toonflow 里属于角景页：

```txt
cornerScape/updateAssetsAudio
cornerScape/batchBindAudio
cornerScape/pollingAudio
cornerScape/getAllAssets
```

VT Studio 落地时要保持这个边界：资产中心维护素材，角景页处理绑定和项目内使用。

## 3. 页面结构和入口

入口：项目内顶部菜单「资产」。

页面由 `assets/index.vue` 承载，主要区域：

```txt
资产类型 tab
顶部操作栏
搜索框
资产表格
新增/编辑普通资产弹窗
新增/编辑音频弹窗
批量生成确认弹窗
单资产生成图片弹窗
媒体预览弹窗
图片结果选择区
```

tab 来自 `allThemeData`：

| tab | Toonflow type | 中文显示 | 说明 |
|---|---|---|---|
| 角色 | `role` | 角色 | 可新增、编辑、删除、批量生成提示词、批量生成图片、单资产生成图片 |
| 道具 | `tool` | 道具 | 同角色 |
| 场景 | `scene` | 场景 | 同角色 |
| 片段 | `clip` | 片段 | 上传图片/音频/视频素材，不走提示词和图片生成 |
| 音频 | `audio` | 音频 | 管理音色和音频文件，不走图片生成 |

选择器模式：

```txt
selectorMode：资产选择弹窗模式
allowedTypes：限制可选资产类型
clipMediaTypes：限制 clip 的媒体类型 image/video/audio
multiple：是否多选
```

`assetsCheck.ts` 会把资产中心作为选择器弹出，确认时返回选中的父资产和子资产。

## 4. 顶部按钮和用户动作

每个 tab 顶部都有：

| 按钮/控件 | 显示条件 | 动作 |
|---|---|---|
| 新增 + 当前类型名 | 全部 tab | `role/tool/scene` 打开 `addAssets.vue`；`clip` 打开文件选择；`audio` 打开 `addAudioAssets.vue` |
| 批量生成 | 仅 `role/tool/scene` | 弹出菜单：生成提示词、生成图片 |
| 生成提示词 | 批量生成菜单内 | 选中资产后调用 `/assetsGenerate/batchPolishAssetsPrompt` |
| 生成图片 | 批量生成菜单内 | 选中资产后选择图片模型和分辨率，再调用 `/assetsGenerate/batchGenerateImageAssets` |
| 批量删除 | 全部 tab | 调用 `/assets/batchDelete` |
| 搜索框 | 全部 tab | 输入资产名称 |
| 搜索按钮 | 全部 tab | 重置到第 1 页并调用 `/assets/getAssetsApi` |

生成中限制：

```txt
promptState === "生成中" 的资产不能选择/删除/生成
state === "生成中" 的资产不能选择/删除/生成
```

前端判断函数是 `isGenerating(id)`，它会在父资产和子资产里查同一个 id。

### 5.3 音频素材表格

音频父资产表格列：

```txt

性别 sex
描述 describe

```

音频子资产展开列：

```txt
勾选
预览
音频文本 prompt
操作：删除
```

Toonflow 把音频父资产的 `describe` 拼成：

```txt
sex + "|" + describe
```

```txt
sex = describe.split("|")[0]
describe = describe.split("|")[1]
```

VT Studio 不能照搬这种拼接字段，建议拆成正式字段，例如 `voiceGender` 和 `description`。



## 9. 音频素材新增和编辑

弹窗：`addAudioAssets.vue`

音频资产是两层结构：

| 业务含义 | Toonflow 字段 | 控件 | 必填 | 说明 |

| 业务含义 | Toonflow 字段 | 控件 | 必填 | 说明 |
|---|---|---|---|---|
| 音频文件 | `base64` / `src` | 点击或拖拽上传 | 新增子项时需要 | `accept=audio/*` |
| 音频文本 | `prompt` | 输入框 | 否 | 表格显示为音频文本 |
| 音频名称 | `name` | 自动或输入 | 否 | 默认文件名 |
| 音频描述 | `describe` | 输入框 | 否 | 子音频描述 |

```txt
POST /assets/addAudioAssets
```

参数：

```txt

projectId

```

`assetsItem[]` 新增格式：

```txt
base64
prompt
describe
name
```

后端新增逻辑：

```txt
1. base64 音频写入 /${projectId}/assets/audio/${uuid()}.${ext}
2. 插入父 o_assets：
   name
   describe
   type = "audio"
   projectId
   startTime
3. 每个音频子项插入子 o_assets：
   prompt
   assetsId = 父资产 id
   type = "audio"
   describe
   name
   projectId
   startTime
4. 每个子项插入 o_image：
```

```txt
POST /assets/updateAudioAssets
```

```txt

   - 如果传 id，更新 prompt/describe/name
   - 如果 src 是外部 URL，先用 replaceUrl 转回内部路径
   - 更新 o_image.filePath
3. 对新增子项：
   - base64 写文件
   - 插入子 o_assets
   - 插入 o_image
   - 回写子 o_assets.imageId
4. 对被移除的子项：
   - 查询旧子项
   - o_assets.imageId 置空
   - 删除 o_image
   - 删除 o_assets
```

## 11. 删除逻辑

### 11.1 单条删除

调用：

```txt
POST /assets/delAssets
```

参数：

```txt
id
```

后端逻辑：

```txt
1. 查询 o_image where assetsId = id
2. 尝试删除这些 o_image.filePath 文件
3. 查询 imageIds
4. 将引用这些 imageIds 的 o_assets.imageId 置空
5. 删除 o_image where assetsId = id
6. 删除 o_assets where id = id
7. 删除 o_assets where assetsId = id
```

源码风险：

```txt
删除父资产时，只按 o_image.assetsId = 父 id 查图片。
如果子资产也有图片或音频文件，源码会删子 o_assets，但没有按子资产 id 查询并删除子 o_image 和文件。
VT Studio 必须级联处理父子资产、图片表和本地文件。
```

### 11.2 批量删除

调用：

```txt
POST /assets/batchDelete
```

参数：

```txt
id: number[]
```

后端逻辑：

```txt
delete from o_assets where id in ids
```

源码风险：

```txt
batchDelete 只删 o_assets，不删 o_image，不删文件，不删子资产，不处理 o_assets2Storyboard，不处理 o_scriptAssets。
VT Studio 不能照抄。批量删除必须复用单条删除的完整级联能力。
```

### 11.3 图片删除

调用：

```txt
POST /assets/delImage
```

参数：

```txt
id = o_image.id
```

后端逻辑：

```txt
1. o_assets where imageId=id 更新 imageId=null
2. o_image where id=id delete
3. 再查询 o_image where id=id
4. 尝试删除 filePath
```

源码风险：

```txt
delImage 先删除 o_image，再查询同一个 id，所以文件路径已经查不到，文件删除逻辑基本失效。
VT Studio 必须先查文件路径，再删库，再删文件。
```

## 12. 单资产生成图片

生成前校验：

```txt

resolution 必须有值
model 必须有值
```

### 12.1 智能生成提示词

调用：

```txt
POST /assetsGenerate/polishAssetsPrompt
```

参数：

```txt
projectId
assetsId
type
name
describe
```

后端逻辑：

```txt
1. 查询 o_project.artStyle/type/intro
2. 更新 o_assets.promptState = "生成中"
3. 查询资产是否有 assetsId，用来判断是否衍生资产
4. 根据 type 选择视觉手册 key
5. 调用 u.getArtPrompt(project.artStyle, "art_skills", visualManual)
6. 调用 u.Ai.Text("universalAi").invoke
7. 成功：更新 o_assets.prompt 和 promptState="已完成"
8. 失败：更新 promptState="失败" 和 promptErrorReason
```

视觉手册 key：

## 21. 源码冲突和风险

1. `batchPolishAssetsPrompt.ts` 要求 `otherTextPrompt` 必填，但 `assets/index.vue` 调用时没有传，VT Studio 必须修正。
2. `batchGeneration.vue` 内部调用 `/assets/polishAssetsPrompt` 和 `/assets/generateAssets`，但实际后端路由是 `/assetsGenerate/polishAssetsPrompt` 和 `/assetsGenerate/generateAssets`。该组件像旧版本遗留，VT Studio 不能按它的错误路径实现。
3. `batchGeneration.vue` 保存时调用 `/assets/saveAssets` 传 `filePath`，但后端 `saveAssets.ts` schema 没有 `filePath`，并且要求 `type=role/scene/tool`。该组件保存逻辑和后端不匹配。
4. `batchDelete.ts` 只删 `o_assets`，不删文件、图片、子资产和关系表，VT Studio 必须补完整级联删除。
5. `delImage.ts` 先删 `o_image` 再查文件路径，导致文件删除逻辑失效，VT Studio 必须先查路径。
6. `delAssets.ts` 删除父资产时会删子 `o_assets`，但没有完整删除子资产的 `o_image` 和文件。
7. 音频父资产把 `sex` 和 `describe` 拼到同一个字段，VT Studio 不应照搬。
8. `o_assets.audioBindState` 在 `initDB.ts` 定义成 integer，但源码实际写入中文字符串 `生成中/已完成/生成失败`，VT Studio 必须修正字段类型和枚举。
9. 提示词状态混用 `生成失败` 和 `失败`，VT Studio 必须统一状态枚举。
10. `tool` 图片保存目录是 `props`，展示名称是道具；VT Studio 需要统一内部命名，不要出现 `tool/props` 混乱。
11. 资产中心没有直接取消生成按钮，但后端有取消接口，角景页在用；VT Studio 如果加按钮，要写进偏差记录。
12. 片段上传只在前端 accept 限制 `.png,.jpg,.jpeg,.mp3,.mp4`，后端还支持 `webm/wav` 等 MIME，VT Studio 要统一允许格式。
13. `getMaterialData.ts` 会固定追加 `Toonflow片尾` 和 `/ending.mp4`，VT Studio 不能继续保留 Toonflow 品牌素材。

## 24. VT Studio 落地要求

1. 页面不能直接访问 SQLite、文件系统或模型 SDK，必须走 `window.vtStudio -> main/ipc -> main/services`。
2. 普通资产、图片、音频、片段文件都必须按项目隔离存储。
3. 图片/音频/片段预览 URL 必须由主进程受控生成，不能暴露任意本地路径。
4. 删除必须做事务：资产、图片、关系、文件路径一起处理；文件删除失败要返回可读错误。
5. 生成类功能必须统一任务状态，不直接把中文状态写进数据库枚举。
6. 角色/场景/道具的提示词生成必须依赖项目视觉手册；手册缺失要明确提示。
7. 批量生成必须可限制并发，默认值来自设置里的批量生成数量。
8. `tool` / `props` 这种旧命名差异只作为参考源码记录，VT Studio 正式命名要统一。
9. `audioBindState` 属于角景绑定状态，资产中心只展示必要状态，不在这里做绑定业务。
10. 所有与 Toonflow 源码不同的增强或修正必须写入 `docs/04-对齐验收与偏差记录.md`。

## 25. 缺少的逻辑和实现细节

### F-006-001 / F-006-002 / F-006-003 角色、场景、道具管理

参考项目资产是 `role/scene/tool` 三类，字段主要是名称、描述、提示词、图片。源码未找到三视图、方位图、角色设定多图位的专门结构。

必须明确：

```txt
资产图形式：主图 imageId + 历史图片列表 o_image
源码未找到：三视图、正侧背、表情差分、动作方位图字段
如果 VT Studio 做方位图：必须作为增强，设计 asset_images.purpose/view 字段
角色/场景/道具新增：不需要父资产选择
父子资产来源：剧本提取、生产衍生、音频素材子项等链路
```

字典值必须统一：

```txt
资产类型：role/scene/prop/tool/clip/audio 要统一正式命名
Toonflow 展示叫道具，源码有 tool/props 混用；VT Studio 建议统一 prop
音频性别：参考项目是自由字段 sex，VT Studio 建议字典化，但不改变业务语义
媒体类型：image/video/audio 由文件 MIME 和扩展名共同判断
```

### F-006-004 片段素材管理

片段素材可以是图片、视频、音频，不只是视频片段。

必须明确：

```txt
允许格式：前端 accept 和后端 MIME 要统一
上传位置：项目业务目录，不进源码目录
预览：图片用 image viewer，视频/audio 用受控 file URL
删除：必须删除资产记录、媒体记录、文件和被工作台引用关系
固定片尾：Toonflow 会加入 Toonflow 片尾素材，VT Studio 不能保留旧品牌
```

### F-006-005 音频素材管理

参考项目音频资产是“音色父资产 + 音频文件子资产”，不是 TTS 生成中心。

必须明确：

```txt
父资产：音色名称、性别/描述
子资产：音频文件、文本、文件路径
绑定音频：在 M-007 角景页做，不在资产中心做
生成音频：角景页源码未找到真正 TTS 生成，不能误做成 TTS 功能
```

### F-006-006 素材预览

预览不能直接暴露本地路径。

必须明确：

```txt
filePath 是内部文件路径，不是 renderer 可直接访问 URL
所有预览 URL 由 main 服务生成短期可用地址
图片大图、音频、视频都要处理文件不存在
文件不存在时显示缺失状态，不让页面崩溃
```

### F-006-007 / F-006-008 批量生成提示词和图片

批量生成依赖视觉手册、模型配置和并发数。

必须明确：

```txt
适用类型：只支持 role/scene/tool
提示词来源：资产名称、描述、视觉手册、otherTextPrompt
图片分辨率：参考项目选择 1K/2K/4K
图片比例：参考源码资产图生成里固定 aspectRatio=16:9，未找到可配置比例
并发数：来自设置 assetsBatchGenerateSize
失败：必须写 promptErrorReason 或 image errorReason
任务中心：模型适配层写任务记录，页面不要自己假造任务
```

### F-006-009 单资产生成图片

单资产生成支持参考图、提示词、模型、分辨率、结果选择。

必须明确：

```txt
参考图：转 base64 后传 referenceList
生成结果：先写历史图片，不自动覆盖主图
保存选中：用户选择某张图后才更新资产 imageId
取消选择：不能把 imageId 写成 undefined
删除历史图：先查路径再删文件，再删记录
```

### F-006-010 取消生成

资产中心页面源码未找到直接取消按钮，后端取消接口被角景页使用。

必须明确：

```txt
取消目标：图片记录 id，不是资产 id
取消效果：状态变失败/取消，并阻止后续轮询继续覆盖
第一版资产中心不加取消按钮也算对齐
如果加取消按钮：写入 04 偏差记录
```

### ComfyUI 和多协议模型

参考项目没有 ComfyUI 真实工作流实现，只有 provider 图标识别。VT Studio 如果支持 ComfyUI，不能混在普通 HTTP 图片模型里糊弄。

必须明确：

```txt
普通图片模型：prompt/referenceList/size/aspectRatio -> image URL/base64
ComfyUI：workflow JSON + 节点输入映射 + queue/poll/history + 输出文件解析
RunningHub/其他工作流平台：也要作为独立 adapter protocol
业务页面只认统一 ImageGenerationRequest，不知道底层协议
```
