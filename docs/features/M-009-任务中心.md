## 7. 后端接口细节

`POST /task/getTaskApi`

请求参数：

```txt
page: number
limit: number
projectId?: number | null
taskClass?: string | null
state?: string | null
```

查询逻辑：

```txt
1. offset = (page - 1) * limit
2. o_tasks leftJoin o_project
3. taskClass 有值时 andWhere o_tasks.taskClass
4. state 有值时 andWhere o_tasks.state
5. projectId 有值时 andWhere o_tasks.projectId
6. select o_tasks.* 和 o_project.*
7. orderBy o_tasks.id desc
8. offset/limit 分页
9. 另查 count(*) total
```

返回结构：

```txt
{
  data: TaskItem[],
  total: number
}
```

`POST /task/getProject`

```txt
查询 o_project select id,name groupBy name
过滤 name 为空的数据
前端补 “全部” 选项
```

`POST /task/getTaskCategories`

```txt
查询 o_tasks select taskClass groupBy taskClass
过滤 taskClass 为空的数据
前端补 “全部” 选项
```

`POST /task/taskDetails`

```txt
参数：taskId
查询：o_tasks where id=taskId first
当前 views/task/index.vue 未调用
```

## 8. 任务记录写入

任务记录工具：

```txt
utils/taskRecord.ts
```

创建任务：

```txt
u.task(projectId, taskClass, modelName, { describe, content })
```

写入逻辑：

```txt
1. content 是 string：直接写 relatedObjects。
2. content 是 object：JSON.stringify 后写 relatedObjects。
3. content 是 function：抛“不支持的类型”。
4. 插入 o_tasks：projectId、taskClass、relatedObjects、model、describe、state=进行中、startTime=Date.now()。
5. 返回 done 函数。
6. done(1)：更新 state=已完成，reason=null。
7. done(-1, reason)：更新 state=生成失败，写 reason。
```

`utils/ai.ts` 自动记录：

```txt
withTaskRecord 会在 u.Ai.Image / u.Ai.Video run 时创建任务。
成功时 taskRecord(1)。
失败时 taskRecord(-1, u.error(e).message)。
```

## 11. 源码冲突和风险

1. `getTaskApi.ts` 里 `select("o_tasks.*", "o_project.* ")` 带空格，且项目字段可能覆盖同名字段；VT Studio 查询要明确字段别名。
2. 当前正式表格没有项目名称列，但接口 leftJoin 了项目表；如果要显示项目名，需要明确字段别名。
3. `projectId` 前端空值时会回退到当前项目 `project.value?.id`，所以“全部项目”在已有当前项目时可能不是真正全部。
4. `getProject.ts` 按 name groupBy，只选 id,name；如果有同名项目，id 取值不稳定。
5. `taskDetails.ts` 存在但当前正式页面没使用。
6. `taskList/index.vue` 是旧视图，未在 router 注册，还调用不存在的 `/task/getMyTaskApi`。
7. 任务没有取消状态，VT Studio 如果新增 cancelled 是增强和偏差。
8. 任务没有重试入口，VT Studio 如果新增重试是增强和偏差。
