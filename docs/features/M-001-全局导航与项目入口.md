## 9. 登录入口和鉴权

登录页面：

```txt
pages/login/index.vue
```

登录表单：

| 字段 | 控件 | 必填 | 说明 |
|---|---|---|---|
| `username` | t-input | 是 | 用户名 |
| `password` | t-input password | 是 | 密码 |
| `baseUrl` | 设置弹窗 t-input | 否 | 请求地址，默认来自 setting store |

登录行为：

```txt
1. 用户名或密码为空：warning enterUsernameAndPassword。
2. POST /login/login。
3. 成功：localStorage.token=data.token，localStorage.userId=data.id。
4. 跳转 /project。
5. 成功 toast。
6. 失败：toast e.message。
```

后端 `/login/login`：

```txt
参数：username, password
查 o_user where name=username
用户不存在：400 登录失败
密码和用户名匹配：读取 o_setting key=tokenKey
生成 JWT，expiresIn=180Days
返回 token = "Bearer " + token, name, id
密码错误：400 用户名或密码错误
```

VT Studio 注意：

```txt
项目目标是本地优先，登录可以弱化，但必须记录偏差。
如果保留登录，也不能把本地桌面核心功能强依赖远程账号。
```

## 13. 源码冲突和风险

1. `nodelOnly` 拼写错误，实际语义是 novelOnly；VT Studio 正式命名要修正。
2. `agentConfog` 拼写错误，VT Studio 正式命名要改为 agentConfig。
3. 外链全部指向 Toonflow：GitHub、反馈腾讯文档、更新 JSON。

8. `handleClick` 有 `needProject` 逻辑，但当前菜单数据没有使用 needProject。

## 17. 缺少的逻辑和实现细节

### F-001-001 应用初始化和标题栏

参考项目 Electron 下显示 `titleBar.vue`，窗口按钮通过 `toonflow://windowMinimize/windowMaximize/windowClose` 调主进程。VT Studio 不保留 Toonflow 协议名，必须改成 preload 白名单。

必须明确：

```txt
标题栏只在 Electron 桌面端显示
窗口按钮：最小化、最大化/还原、关闭
最大化状态：按钮图标或 tooltip 要能同步当前窗口状态
拖拽区：标题文字区域可拖拽，按钮区域不可拖拽
内容高度：Electron 下减去 32px 标题栏高度
能力入口：window.vtStudio.window.minimize/toggleMaximize/close/getState
```

### F-001-004 左侧全局菜单

参考项目只放两个全局入口：项目和任务中心。VT Studio 实现时不要把小说、剧本、资产、生产放到左侧全局菜单；这些必须等打开项目后出现在顶部项目菜单。

必须明确：

```txt
全局菜单项来源：固定配置，不从数据库读取
高亮依据：当前路由 path
无项目状态：左侧仍可进入项目和任务中心
任务中心项目筛选：不能依赖当前项目，必须能看全部项目任务
```

### F-001-005 项目内顶部菜单

参考项目顶部菜单依赖当前项目，并且小说、剧本 Agent 只对 `novel` 项目显示。VT Studio 后续做项目类型时，必须保证菜单由项目上下文驱动，不要用页面自己判断。

必须明确：

```txt
project.sourceType=novel：显示原文、剧本 Agent、剧本、角景、生产、资产
project.sourceType=script：不显示原文和剧本 Agent
没有当前项目：顶部项目菜单不显示
项目被删除或打开失败：清空当前项目上下文并回到项目页
```

### F-001-007 设置入口

设置是弹窗入口，不是独立设置页。后续所有设置模块都要挂在 M-002 的统一弹窗里，不能在菜单里拆成多个页面入口。

必须明确：

```txt
打开设置：只改 settingStore.showSetting
关闭设置：保留当前设置 tab
设置红点：只用于更新提醒，不能混成设置未保存状态
Electron 标题栏：设置弹窗不能遮住窗口拖拽区和标题栏
```

### F-001-002 / F-001-003 登录路由和鉴权跳转

参考项目有 token 路由守卫。VT Studio 如果本地优先弱化登录，也必须保留“未初始化/未登录/已登录”的状态语义，不能直接删掉鉴权入口。

必须明确：

```txt
首次启动：进入项目页还是初始化/登录页
退出登录：是否清当前项目、模型密钥、记忆缓存
本地用户：是否只是本机资料，不做云端账号
路由守卫：不允许出现无 token 但页面仍使用旧 userId 的状态
```

参考项目外链、版本源和欢迎页都带 Toonflow 旧品牌。VT Studio 不能保留这些旧链接。

