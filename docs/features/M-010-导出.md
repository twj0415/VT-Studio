## 4. 功能拆分

| 功能 ID | 功能 | 要做到什么 |
|---|---|---|
| F-010-001 | 快速预览分镜图片导出 | 读取分镜、播放预览、勾选分镜、全选、拖拽排序、还原排序、导出选中图片 zip |
| F-010-002 | 内置视频编辑器时间线构建 | 素材库拖拽到轨道，自动找可用轨道，生成 clip，加载缩略图/波形，支持切割、删除、撤销、重做、转场 |

| F-010-004 | 剪映导出前时间线构建 | 从 M-008 视频轨道或内置编辑器轨道生成确定的 timeline，不临时猜数据 |
| F-010-005 | 导出素材校验 | 校验图片、视频、音频路径和 clip 引用，缺失时列出对象、路径、来源并阻止导出 |
| F-010-006 | Windows 剪映专业版草稿导出 | 主进程写入剪映草稿目录，生成草稿结构，记录导出结果和失败原因 |

## 5. 快速预览和图片 zip 导出

数据读取：

```txt
POST /production/getStoryboardData
参数：scriptId, projectId
后端：routes/production/getStoryboardData.ts
读取：o_storyboard、o_assets2Storyboard、o_assets、o_image
排序：o_storyboard.index asc
```

返回给前端的主要字段：

| 字段 | 来源 | 说明 |
|---|---|---|
| `id` | `o_storyboard.id` | 前端转成 string |
| `duration` | `o_storyboard.duration` | 播放和进度条使用；空值时前端按 3 秒 |
| `filePath` | `o_storyboard.filePath` | 后端转 smallImageUrl；导出图片从这里 fetch |
| `prompt` | `o_storyboard.prompt` | 右侧图片 prompt |
| `index` | `o_storyboard.index` | 初始排序 |
| `characters[]` | `o_assets2Storyboard -> o_assets -> o_image` | 关联资产名称、类型、头像 |

页面按钮和操作：

| 操作 | 源码行为 | 失败或空状态 |
|---|---|---|
| 上一张 | `prevShot`，切到前一个分镜，停止播放 | 第一张禁用 |
| 播放/暂停 | `togglePlay`，每 50ms 更新时间，到当前分镜 duration 后自动下一张 | 最后一张结束后停止 |
| 下一张 | `nextShot`，切到下一个分镜，停止播放 | 最后一张禁用 |
| 点击进度条分段 | `jumpToShot(index)` | 无 |
| 拖拽进度条 | 按总时长计算目标分镜和当前 elapsed | 无 |
| 点击底部分镜 | 选中并滚动到当前分镜 | 无 |
| Space | 切换当前分镜 selected | 无 |
| 全选 | 把所有 `shot.selected` 设为当前 checkbox 值 | 无 |
| 拖拽排序 | 只改变当前预览列表顺序 | 不写数据库 |
| 还原排序 | 用打开页面时的 `initialOrder` 还原 | 会弹确认框 |
| 导出图片 | 只导出 selected=true 的分镜图片 | 没选时 DialogPlugin.alert |

图片 zip 导出逻辑：

```txt
1. 从 shotList 过滤 selected=true。
2. filePath 会去掉 ? 后面的参数。
3. 没有选中任何分镜时弹 alert，不继续。
4. 用 fetch(filePath) 读取图片 Blob。
5. 用 JSZip 写入 zip，文件名是 分镜{id}.{ext}。
6. zip 文件名是 分镜压缩包.zip。
7. 创建 a 标签下载，5 秒后 revokeObjectURL。
```

源码问题：

```txt
导出图片失败只 console.error，不在界面列出失败图片。
拖拽排序只影响当前预览和导出顺序，不保存到数据库。
initialOrder 在 shotList 异步加载前创建，初始为空；还原排序在源码里可能无法按接口初始顺序恢复。
```

VT Studio 落地要求：

```txt
保留“只导出选中分镜”的语义。
导出失败要把失败分镜 id 和路径提示出来。
桌面端可由 main 服务打包 zip，renderer 只触发 window.vtStudio.export.storyboardImages。
如果继续用浏览器 JSZip，也不能直接访问本地裸路径，必须使用受控文件 URL。
```

## 6. 内置视频编辑器时间线

组件结构：

```txt
editVideo/index.vue
mediaLibrary.vue
videoPreview.vue
propertyPanel.vue
vue-clip-track
```

时间线配置：

| 配置 | 源码值 |
|---|---|
| 主轨模式 | `enable-main-track-mode=true` |
| 跨轨拖拽 | `enable-cross-track-drag=true` |
| 吸附 | `enable-snap=true` |
| 默认缩放 | `default-scale=1` |
| scale 按钮 | `["snap"]` |

轨道数量限制：

| 类型 | 最大数量 |
|---|---:|
| video | 5 |
| image | 3 |
| audio | 3 |
| subtitle | 2 |
| text | 2 |
| sticker | 2 |
| filter | 1 |
| effect | 2 |

工具栏按钮：

| 按钮 | 源码行为 | 失败或禁用 |
|---|---|---|
| reset | `videoTrackRef.reset()` | 无 |
| undo | `historyStore.undo()` | `!historyStore.canUndo` 禁用 |
| redo | `historyStore.redo()` | `!historyStore.canRedo` 禁用 |
| split | 按当前播放时间切割选中 clip，并写历史快照 | 没有选中 clip 禁用 |
| delete | 删除选中 clips，并写历史快照 | 没有选中 clip 时直接返回 |
| import | operationButtons 里配置了 import，但模板被注释 | 源码没有实际入口 |
| exportVideo | 调 `videoPreviewRef.exportVideo()` | 导出中禁用重复触发 |

拖拽素材到轨道：

```txt
1. mediaLibrary 把素材写入 dataTransfer：application/json 和 text/plain。
2. editVideo 接收 drop-media。
3. 如果素材是 transition，走转场逻辑。
4. 其他素材先按类型和 duration 找轨道。
5. 优先使用传入 trackId；轨道类型不一致或没空间则找同类型非主轨。
6. 找不到可用轨道就创建新轨道。
7. 创建 clip 后写 historyStore 快照。
8. 视频缺缩略图时加载缩略图。
9. 音频缺波形时加载波形。
```

默认时长：

| 素材类型 | 默认秒数 |
|---|---:|
| video | 5 |
| image | 5 |
| audio | 30 |
| subtitle | 3 |
| text | 3 |
| sticker | 3 |
| filter | 3 |
| effect | 3 |
| transition | 3 |

clip 主要字段：

| 类型 | 主要字段 |
|---|---|
| video | `sourceUrl`、`originalDuration`、`trimStart`、`trimEnd`、`playbackRate`、`thumbnails` |
| image | `sourceUrl`、`originalDuration`、`trimStart`、`trimEnd`、`playbackRate`、`thumbnails` |
| audio | `sourceUrl`、`originalDuration`、`trimStart`、`trimEnd`、`playbackRate`、`volume`、`waveformData` |
| subtitle | `text` |
| text | `text` |
| sticker | `sourceUrl` |
| filter | `filterType`、`filterValue` |
| effect | `effectType`、`effectDuration` |
| transition | `transitionType`、`transitionDuration` |

转场规则：

```txt
转场必须插在同一轨道相邻两个非 transition clip 之间。
如果当前位置找不到相邻 clip，提示“转场需要放在两个片段之间”。
同一位置已有转场时提示已存在，不重复添加。
点击添加转场默认 fade。
拖拽转场使用素材 subType。
默认转场时长 1 秒，起止时间跨越两个 clip 的连接点。
```

转场列表：

```txt
素材库展示：fade、slide、wipe、dissolve、zoom、rotate
渲染器支持：fade、dissolve、slide、slide-left、slide-right、slide-up、slide-down、wipe、wipe-left、wipe-right、wipe-up、wipe-down、zoom、zoom-in、zoom-out、rotate、circle、diamond、clock、blur
```

源码问题：

```txt
素材库展示的转场类型少于渲染器支持的类型。
operationButtons 里有 import，但模板注释掉了导入按钮。
时间线数据在参考项目里主要是前端状态，没有找到保存到后端的闭环。
```

## 9. WebAV mp4 导出

触发：

```txt
点击视频编辑器底部导出视频按钮。
editVideo/index.vue -> handleExport -> videoPreviewRef.exportVideo()
```

导出前检查：

```txt
avCanvas 不存在：抛 avCanvasNotInit
clipSpriteMap 为空：抛 noExportContent
正在播放：先暂停播放，并同步 playbackStore.pause()
```

导出流程：

```txt
1. avCanvas.createCombinator()
2. combinator.output().getReader()
3. 循环读取 Uint8Array chunks
4. new Blob(chunks, { type: "video/mp4" })
5. URL.createObjectURL(blob)
6. a.download = WebAV-export-${Date.now()}.mp4
7. a.click()
8. URL.revokeObjectURL(url)
```

WebAV 预览和导出支持：

```txt
AVCanvas
MP4Clip
AudioClip
ImgClip
VisibleSprite
renderTxt2ImgBitmap
transitionRenderers
filterEffect
```

会同步到 sprite 的属性：

```txt
rect x/y/w/h/angle
opacity
visible
flip
zIndex
```

时间单位风险：

```txt
vue-clip-track 使用秒。
WebAV 内部大量使用微秒。
VT Studio 实现导出时必须明确 timebase，不能在不同模块里混用秒和微秒。
```

源码问题：

```txt
WebAV 导出是浏览器端直接下载，不写数据库。
WebAV 导出没有进入任务中心。
WebAV 导出没有保存导出历史。
WebAV 导出不是剪映草稿导出。
```

## 10. 剪映草稿导出设计要求

这是 VT Studio 新增能力。参考项目没有对应实现，因此只能参考它的业务数据来源，不能照搬接口。

输入来源建议：

```txt
优先来源：M-008 视频工作台里已选中的视频轨道和候选视频。
可选来源：内置视频编辑器 vue-clip-track 的轨道和 clip。
第一版建议只选一个来源，避免两个时间线同时存在造成语义冲突。
```

建议第一版采用：

```txt
从 M-008 视频工作台构建剪映草稿时间线。
原因：这些数据来自数据库，和分镜、资产、视频候选关系更稳定。
内置视频编辑器可后续作为高级编辑增强，不作为第一版剪映导出的唯一数据源。
```

导出调用链：

```txt
renderer -> window.vtStudio.export.createJianyingDraft -> main/ipc -> main/services/export
```

导出步骤：

```txt
1. 读取 projectId、scriptId、导出配置。
2. 获取生产工作台视频轨道。
3. 构建统一 timeline。
4. 校验每个 clip 的素材路径。
5. 生成剪映草稿目录。
6. 写入草稿 JSON 和素材引用。
7. 记录导出任务或导出历史。
8. 返回草稿路径和结果摘要。
```

草稿目录规则：

```txt
不能写到 D:\project\vt-studio 源码目录。
默认可写到用户数据目录下的 exports。
也可以让用户选择导出目录。
导出目录必须记录在配置里，失败时能展示完整路径。
```

素材校验必须列出：

| 项 | 说明 |
|---|---|
| 轨道 id | 哪条视频轨道出问题 |
| clip id | 哪个时间线片段出问题 |
| 素材类型 | image/video/audio/subtitle/text |
| 来源类型 | storyboard/assets/video/audio/manual |
| 来源 id | 分镜 id、资产 id、视频 id 等 |
| 原始路径 | 数据库或 timeline 里的 filePath/sourceUrl |
| 失败原因 | 文件不存在、路径为空、权限不足、格式不支持 |

建议状态：

| 内部状态 | 中文展示 | 说明 |
|---|---|---|
| `pending` | 待导出 | 创建任务但未开始 |
| `validating` | 校验中 | 正在校验素材 |
| `running` | 导出中 | 正在写草稿 |
| `succeeded` | 导出成功 | 返回草稿目录 |
| `failed` | 导出失败 | 记录失败原因 |
| `cancelled` | 已取消 | 用户取消 |

## 12. 调用链路

| 用户动作 | Toonflow 前端调用 | Toonflow 后端文件 | 读写对象 | 成功反馈 | 失败反馈 |
|---|---|---|---|---|---|

## 14. 不能漏的逻辑

1. 剪映草稿导出不是 Toonflow 已有功能，必须作为 VT Studio 增强能力记录。
2. 导出必须最后做，前面的视频轨道、素材路径、分镜、资产关系先稳定。
3. 不能让 renderer 直接写剪映草稿文件，必须走 main 服务。
4. 导出前必须做素材校验，不能边写草稿边失败。
5. 缺失素材要能定位到轨道、clip、来源对象和路径。
6. 秒和微秒必须统一转换，不能把 WebAV 时间单位直接套到剪映草稿。
7. 参考项目的 WebAV 时间线没有后端保存闭环，VT Studio 不能依赖临时 Vue 状态做正式导出。
8. 如果剪映导出读取 M-008 视频工作台，必须以已选择候选视频为准，而不是随机取第一个候选。
9. 导出目录和 runtime 目录不能放项目源码目录。
10. 剪映导出偏差必须写入 `docs/04-对齐验收与偏差记录.md`。

## 17. 缺少的逻辑和实现细节

### F-010-001 快速预览分镜图片导出

参考项目快速预览导出的是分镜图片 zip，不是视频，也不是剪映草稿。

必须明确：

```txt
来源：当前剧本 storyboard.filePath
操作：勾选分镜、全选、拖拽排序、恢复排序、导出 zip
默认时长：预览播放空 duration 按 3 秒
zip 内容：选中分镜图片
失败处理：必须列出哪些图片拉取失败
保存位置：参考项目浏览器下载；VT Studio 可让用户选目录或写 downloads
```

### F-010-002 内置视频编辑器时间线构建

参考项目内置编辑器有前端时间线，但没有后端保存闭环。

必须明确：

```txt
轨道来源：vue-clip-track 前端状态
素材来源：mediaLibrary、分镜、资产、音频、视频
clip 字段：id、trackId、sourceUrl、mediaType、startTime、endTime、duration、offset、transition
时间单位：WebAV 用秒/毫秒混合风险，剪映导出前必须统一
正式导出：不能只依赖临时 Vue 状态
```

### F-010-003 WebAV mp4 导出

WebAV 是快速 mp4 导出，不等于剪映草稿。

必须明确：

```txt
执行位置：参考项目在 renderer 合成 Blob
输出：WebAV-export-时间戳.mp4
任务记录：源码未找到
历史记录：源码未找到
素材缺失：导出前必须校验，不能合成中途才失败
VT Studio 如果放 main：要重新验证 WebAV 环境支持
```

### F-010-004 / F-010-005 剪映导出前时间线构建和素材校验

剪映导出必须有确定时间线，不能临时猜。

必须明确：

```txt
第一版建议来源：M-008 视频工作台已选视频轨道
每条轨道：storyboardId、trackId、selectedVideoId、duration、prompt
每个 clip：素材类型、来源对象、文件路径、开始时间、时长
素材校验：路径为空、文件不存在、权限不足、格式不支持
错误展示：必须能定位到剧本、分镜、轨道、clip、文件路径
校验失败：阻止导出，不写半成品草稿
```

### F-010-006 Windows 剪映专业版草稿导出

参考项目没有实现剪映草稿导出，这是 VT Studio 增强能力。

必须明确：

```txt
执行位置：main/services/export，不允许 renderer 写文件
导出目录：用户选择目录或 userData/exports，不能放源码目录
草稿名称：项目名 + 剧本名 + 时间，必须清理非法字符
素材策略：第一版建议复制素材到草稿目录
输出内容：草稿目录、草稿 JSON、素材引用、导出摘要
成功反馈：显示草稿路径，提供打开目录按钮
失败反馈：结构化列出失败原因
任务中心：建议写导出任务或导出历史
```

### 不同导出的边界

```txt
剧本 zip：M-005，导出 txt 剧本
分镜图片 zip：M-010 快速预览，导出图片
```
