## 2. 功能边界

生产工作台是 Toonflow 的主生产链路，不是一个普通编辑页。它负责：

```txt
选择当前剧本
加载剧本对应 flowData
展示 VueFlow 流程图
编辑剧本、导演计划、分镜表
展示原始资产和衍生资产
新增、删除、生成衍生资产图片
管理分镜面板
生成分镜图片
编辑分镜图片工作流
打开视频工作台
生成视频提示词
生成视频候选
选择视频候选
管理视频轨道
接入 Production Agent 对话和工具调用
```

生产工作台不负责：

```txt
新建项目
配置模型供应商
配置项目默认模型
从小说生成剧本
剪映草稿最终导出
```

这些分别属于设置、项目管理、小说/剧本链路和 `M-010 导出`。

## 5. flowData 结构

前端 `FlowData` 来自 `flowBuilder.ts`：

| 字段 | 类型 | 说明 |
|---|---|---|
| `script` | string | 当前剧本正文 |
| `scriptPlan` | string | 导演计划 |
| `assets` | AssetItem[] | 原始资产和衍生资产 |
| `storyboardTable` | string | Markdown 分镜表 |
| `storyboard` | Storyboard[] | 分镜面板 |
| `workbench.videoList` | VideoList[] | 前端工作台字段，Agent schema 不包含 |

`AssetItem`：

| 字段 | 说明 |
|---|---|
| `id` | 父资产 id |
| `name` | 资产名 |
| `desc` | 资产描述，来自 `o_assets.describe` |
| `prompt` | 资产提示词 |
| `src` | 当前资产图片 URL |
| `state` | 图片状态 |
| `type` | `role/tool/scene/clip` |
| `flowId` | 图片工作流 id |
| `derive` | 衍生资产列表 |
| `errorReason` | 图片失败原因 |

`DeriveAsset`：

| 字段 | 说明 |
|---|---|
| `id` | 衍生资产 id |
| `assetsId` | 父资产 id |
| `name` | 衍生资产名 |
| `desc` | 衍生资产描述 |
| `prompt` | 衍生图片提示词 |
| `src` | 衍生图片 URL |
| `state` | `未生成/生成中/已完成/生成失败` |
| `type` | 跟随父资产类型 |
| `flowId` | 图片工作流 id |
| `errorReason` | 失败原因 |

`Storyboard`：

| 字段 | 说明 |
|---|---|
| `id` | 分镜 id |
| `index` | 分镜排序 |
| `duration` | 推荐视频时长 |
| `prompt` | 分镜图片提示词 |
| `videoDesc` | 分镜画面描述，视频提示词生成主要读取这个 |
| `associateAssetsIds` | 关联资产 id 列表 |
| `src` | 分镜图片 URL |
| `state` | `未生成/生成中/已完成/生成失败` |
| `reason` | 分镜图片失败原因 |
| `trackId` | 视频轨道 id |
| `flowId` | 图片工作流 id |
| `shouldGenerateImage` | 是否需要生成分镜图，0 否，1 是 |

重要差异：

```txt
Production Agent 的 flowDataSchema 只包含 script/scriptPlan/assets/storyboardTable/storyboard。
workbench 是前端工作台字段，不在 Agent 工具 schema 里。
```

## 6. 获取和保存流程数据

获取流程：

```txt
POST /production/getFlowData
参数：projectId, episodesId
```

后端逻辑：

```txt
1. 查 o_agentWorkData，条件 projectId + episodesId
2. 查当前 o_script，拿最新 content
3. 查 o_scriptAssets，得到当前剧本关联资产
4. 查父资产 o_assets + o_image
5. 查衍生子资产 o_assets.assetsId in 父资产 id
6. 没有 o_agentWorkData 时，返回默认 flowData
7. 有 o_agentWorkData 时，解析 data
8. 用数据库最新资产覆盖 flowData.assets
9. 用 o_storyboard 重建 flowData.storyboard
10. 用 o_assets2Storyboard 重建 associateAssetsIds
11. script 始终使用最新 o_script.content
```

默认 flowData：

```txt
script = scriptData.content
scriptPlan = ""
assets = 当前剧本资产 + derive
storyboardTable = ""
storyboard = []
workbench = { videoList: [] }
```

保存流程：

```txt
POST /production/saveFlowData
参数：projectId, episodesId, data
```

保存逻辑：

```txt
1. 如果 data.storyboard 全都有真实 id，按当前数组顺序更新 o_storyboard.index
2. 如果 o_agentWorkData 没有记录，插入 key=productionAgent
3. 如果已有记录，按 projectId + episodesId + key 更新 data
4. data 直接 JSON.stringify
```

VT Studio 落地要求：

```txt
flowData 只能存业务工作区快照。
真实资产、分镜、视频轨道必须以业务表为准。
保存 flowData 时不能覆盖数据库最新资产和分镜事实。
```

## 10. 衍生资产图片批量生成

触发来源：

```txt
Production Agent 工具 generate_deriveAsset
前端 store 监听 socket generateDeriveAsset
```

调用：

```txt
POST /production/assets/batchGenerateAssetsImage
参数：assetIds, projectId, scriptId, concurrentCount
```

后端逻辑：

```txt
1. 查项目 imageModel/imageQuality/artStyle
2. 查衍生资产 o_assets
3. 查父资产图片和描述
4. 根据 role/tool/scene 读取衍生资产视觉手册：
   role -> art_character_derivative
   tool -> art_prop_derivative
   scene -> art_scene_derivative
5. 给每个衍生资产插入 o_image，state=生成中
6. 更新 o_assets.imageId
7. 用 universalAi 生成图片 prompt，并写回 o_assets.prompt
8. 用项目 imageModel 生成图片
9. 保存到 /{projectId}/assets/{scriptId}/{type}/{uuid}.jpg
10. 成功写 o_image.state=已完成、filePath
11. 失败写 o_image.state=生成失败、errorReason
```

前端轮询：

```txt
computed assetsNotStateImageIds 找出 derive.state == 生成中
每 5 秒 POST /production/assets/pollingImage
返回 state/src/errorReason/prompt 后更新 derive
```

轮询接口：

```txt
POST /production/assets/pollingImage
参数：ids
查询 o_assets left join o_image where o_assets.id in ids and o_image.state != 生成中
```

## 12. 分镜新增、编辑、删除

新增单条分镜：

```txt
POST /production/storyboard/addStoryboard
参数：prompt, duration, state, videoDesc, shouldGenerateImage, src, scriptId, projectId
```

后端逻辑：

```txt
1. 用 Date.now() 创建 trackId
2. 插入 o_videoTrack：id=trackId, scriptId, projectId
3. 插入 o_storyboard
4. filePath = u.replaceUrl(src)
5. shouldGenerateImage 实际写 src ? 1 : 0
6. 返回新 id
```

批量新增分镜：

```txt
POST /production/storyboard/batchAddStoryboardInfo
参数：data[], scriptId, projectId
```

`data[]` 字段：

```txt
prompt
duration
track
state
src
videoDesc
shouldGenerateImage
associateAssetsIds
```

后端逻辑：

```txt
1. 逐条插入 o_storyboard
2. 写 o_assets2Storyboard
3. 查询当前剧本全部分镜
4. 按 track 分组
5. 相同 track 复用已有 trackId，否则新建 o_videoTrack
6. 计算同 track 下 duration 总和，更新 o_videoTrack.duration
7. 返回完整 storyboardData
```

编辑分镜信息：

```txt
POST /production/storyboard/editStoryboardInfo
参数：id, prompt, videoDesc
更新 o_storyboard.prompt/videoDesc
```

更新分镜图片：

```txt
POST /production/storyboard/updateStoryboardUrl
参数：id, url, flowId
更新 o_storyboard.filePath/flowId/state=已完成/shouldGenerateImage
```

删除单条：

```txt
POST /production/storyboard/removeFrame
参数：id
```

后端逻辑：

```txt
1. 查 o_storyboard
2. 如果有 flowId，删除 o_imageFlow
3. 如果同 track 只剩一条分镜，删除 o_videoTrack
4. 删除 o_storyboard
5. 删除 o_assets2Storyboard
```

批量删除：

```txt
POST /production/storyboard/batchDelete
参数：ids, projectId
```

后端逻辑：

```txt
1. 查分镜列表
2. 删除相关 o_imageFlow
3. 删除 o_storyboard
4. 删除 o_assets2Storyboard
```

源码问题：

```txt
addStoryboard 接收 shouldGenerateImage，但写库时用 src ? 1 : 0，入参被忽略。
addStoryboard 前端插入时传 flowId，但后端 schema 没有 flowId，新增分镜图片工作流 id 不会写进库。
removeFrame 删除最后一个 track 分镜会删 o_videoTrack，但 batchDelete 没有同等轨道清理逻辑。
删除分镜没有删除分镜图片文件。
batchDelete 返回“视频删除成功”，语义错误。
```

## 13. 分镜图片生成和轮询

调用：

```txt
POST /production/storyboard/batchGenerateImage
参数：storyboardIds, projectId, scriptId, concurrentCount, compulsory
```

前端调用：

```txt
productionAgentStore().batchGenerateStoryboard(ids, compulsory)
```

后端逻辑：

```txt
1. 查 o_storyboard
2. compulsory=true：所有选中分镜 state=生成中，shouldGenerateImage=1
3. compulsory=false：shouldGenerateImage=0 的分镜 state=未生成；shouldGenerateImage=1 的分镜 state=生成中
4. 查项目 imageModel/imageQuality/artStyle/videoRatio
5. 查 o_assets2Storyboard，按 rowid 保留资产顺序
6. 查关联资产 imageId
7. 返回当前分镜状态给前端
8. 后台按 concurrentCount 分批生成
9. 每条分镜把关联资产图片转 base64 作为 referenceList
10. 调 u.Ai.Image(project.imageModel).run
11. 保存到 /{projectId}/assets/{scriptId}/{uuid}.jpg
12. 成功写 o_storyboard.filePath/state=已完成
13. 失败写 o_storyboard.filePath=""、reason、state=生成失败
```

轮询：

```txt
computed storyboardNotStateImageIds 找出 state=生成中 的分镜
每 5 秒 POST /production/storyboard/pollingImage
```

## 17. 获取视频生成数据


```txt
1. 查 project.videoModel 和 project.mode
2. 如果没有 videoModel，返回“项目未配置视频模型”
3. 判断 mode 是否为数组多参
4. 查当前剧本所有 o_storyboard，按 index 排序
```

## 19. 视频候选生成

单轨生成：

```txt
POST /production/workbench/generateVideo
参数：projectId, scriptId, uploadData[], prompt, model, mode, resolution, duration, audio, trackId
```

批量生成：

```txt
POST /production/workbench/batchGenerateVideo
参数：projectId, scriptId, trackData[], model, mode, resolution, audio
```

生成前端校验：

```txt
单轨：弹确认后发起。
批量：必须选中轨道；选中轨道必须都有 prompt，否则提示并终止。
text 模式 uploadData 为空。
首尾帧和单图模式会截取允许数量的参考素材。
```

后端逻辑：

```txt
1. 解析 mode，JSON 数组则转数组
2. 查项目 videoRatio
3. uploadData 中 storyboard 读取 o_storyboard.filePath
4. uploadData 中 assets 读取 o_assets.imageId -> o_image.filePath/type
5. 参考素材转 base64

7. 立即返回 videoId
8. 调 u.Ai.Video(model).run
9. 参数：prompt/referenceList/mode/duration/aspectRatio/resolution/audio
10. 保存视频到 videoPath
11. 成功更新 o_video.state=生成成功
12. 失败更新 o_video.state=生成失败、errorReason
```

视频轮询：

```txt
POST /production/workbench/checkVideoStateList
参数：projectId, scriptId, videoIds
查询 state in 生成成功/生成失败 的 o_video
返回 src = getFileUrl(filePath)
```

候选视频操作：

| 操作 | 调用 | 说明 |
|---|---|---|
| 选择候选视频 | `/production/workbench/selectVideo` | 写 `o_videoTrack.videoId=videoId` |
| 删除候选视频 | `/production/workbench/delVideo` | 删除 `o_video`，并清空引用它的轨道 videoId |
| 单个下载 | 前端 fetch video.src | 浏览器下载 mp4 |
| 批量下载 | 前端 JSZip | 选中轨道的已选视频打包 zip |
| 播放预览 | 前端 video 弹窗 | 不走后端 |

源码问题：

```txt
生成成功状态写的是“生成成功”，但前端 VideoItem 类型和 getGenerateData 映射使用“已完成”。
checkVideoStateList 返回“生成成功”，前端直接赋给 item.state，和 UI 类型不一致。
generateVideo 里如果 assets 查询结果为空，直接访问 filePath.type 有报错风险。
delVideo 不删除视频文件。
```

## 25. 源码冲突和风险

1. `poster` 节点存在注释代码，但当前生产页未启用；VT Studio 不应把它列入当前开发目标。
2. Agent schema 不包含 `workbench`，但前端 `FlowData` 包含 `workbench`；开发时要分清 Agent 工作区和前端工作台数据。
3. `deleteAssetsDireve` 拼写错误，VT Studio 正式命名必须改成 `deleteDerivedAsset` 类似语义。
4. `o_videoTrack` 同时有 `videoId` 和 `selectVideoId`，源码实际使用 `videoId`；VT Studio 只能保留一个正式字段。
5. 视频成功状态混用 `生成成功` 和 `已完成`，VT Studio 必须统一。
6. `addStoryboard` 接收 `shouldGenerateImage` 却用 `src ? 1 : 0` 写入，语义错误。
7. `addStoryboard` 前端插入新分镜时传 `flowId`，后端 schema 没收，导致新分镜工作流 id 丢失。
8. 分镜和衍生资产删除没有完整删除图片文件，VT Studio 删除必须事务化并清理文件。
9. `batchDelete` 没有处理视频轨道清理，和 `removeFrame` 行为不一致。
10. 工作流图片 `generateFlowImage` 保存路径少斜杠，路径规范不统一。
11. `getGenerateData` 没配视频模型时返回 200 success 文案，不是业务错误结构；VT Studio 应返回明确错误。
12. `generateVideo` 查询 assets 文件时如果查不到记录会访问 `filePath.type`，有空引用风险。
13. `delVideo` 只删数据库，不删视频文件。
14. `deleteTrack` 只删轨道并清空分镜 trackId，不删轨道下视频候选。
15. `getMaterialData` 会加入 Toonflow 固定片尾素材，这个属于 M-006 已记录，VT Studio 不能保留 Toonflow 品牌痕迹。
16. Agent 工具 `get_flowData` 输入有 key，但前端 socket 回调返回完整 flowData，不按 key 过滤。
17. `workbench/index.vue` 的导入到剪辑台函数是空实现，不能把它当已完成能力。

## 28. VT Studio 落地要求

1. 生产页只能通过 `window.vtStudio -> main/ipc -> main/services` 访问数据库、文件和模型。
2. flowData 只保存工作区快照，业务事实以资产、分镜、轨道、视频表为准。
3. 分镜、资产、视频生成全部进入统一任务中心，并记录失败原因。
4. 图片、视频、音频预览 URL 必须由主进程受控生成，不能暴露任意本地路径。
5. 衍生资产、分镜、视频轨道、视频候选的删除必须清理关系、工作流和文件。
6. 视频工作台要为 M-010 导出保留清晰轨道数据，但不要在本模块实现剪映导出。
7. Agent 工具调用不能直接写 Vue 组件状态，必须通过明确的服务和状态同步。
8. Toonflow 的错拼字段、中文状态、旧接口名只作为参考事实，VT Studio 正式命名要统一。
9. `workbench` 前端字段和 Production Agent schema 要分开建模，不能混在同一个服务 DTO。
10. 后续每个子功能开工前，必须重新读取对应源码并创建 `docs/tasks/xxx-功能名.md`。
