## 3. 功能拆分

| 功能 ID | 功能 | 要做到什么 |
|---|---|---|
| F-005-001 | 新增剧本 | 打开新增弹窗，输入名称，上传/粘贴内容，控制单集字数上限，选择关联资产，提交后写入剧本和资产关系 |
| F-005-002 | 批量新增剧本 | 打开批量导入弹窗，上传/粘贴多集剧本，支持自定义集正则，支持 AI 生成正则，解析预览，勾选集数，按字数上限保存 |
| F-005-003 | 编辑剧本 | 点击卡片打开编辑弹窗，修改名称和内容，维护关联资产，保存后刷新列表 |
| F-005-004 | 删除/批量删除剧本 | 单条和批量删除都二次确认；提取中的剧本不允许批量删除；删除剧本、关联资产、Agent 数据、分镜、视频并清缓存 |
| F-005-005 | 搜索剧本 | 按当前项目和名称模糊查询，返回关联资产、提取状态、失败原因 |
| F-005-006 | 导出剧本 zip | 选择剧本后导出 zip，每个剧本一个 txt 文件，浏览器下载 `script_YYYY-MM-DD.zip` |
| F-005-007 | 从剧本提取资产 | 选择剧本，调用异步提取接口，先置为等待提取，再分组调用 `universalAi`，生成或复用资产并写入剧本资产关系 |
| F-005-008 | 资产提取状态轮询 | 对 `extractState=0` 的剧本每 3 秒轮询，状态变化后刷新剧本列表，失败显示原因 |

批量解析规则：

```txt
默认正则：/第\s*([0-9０-９零一二三四五六七八九十百千万]+)\s*集\s*([^\n\r]*)/g
自定义正则优先
自定义正则支持 /表达式/flags 格式；如果没有 g，会自动补 g
没有匹配但文本非空时，把全文作为第 1 集
匹配到多集时，正文取当前匹配结束到下个匹配开始之间的内容
按 index 升序排序
中文数字支持零、一、二、三、四、五、六、七、八、九、十、百、千
```

AI 正则：

```txt
前端取 content 前 2000 字调用 /script/getAiRegex
后端用 universalAi
system 要求返回一个 JavaScript 正则字符串
正则必须有两个捕获组：集数/章节编号、标题/名称
没有明显分隔模式时返回空字符串
返回值直接写入 customRegStr
```

导出 zip：

```txt
前端请求 /script/exportScript，responseType=blob
后端用 compressing.zip.Stream
每个选中剧本写入一个 txt：`${script.name}.txt`
响应 Content-Type=application/zip
响应文件名 scripts.zip
前端实际下载文件名 script_YYYY-MM-DD.zip
```

资产提取：

```txt
点击提取资产调用 /script/extractAssets
参数 scriptIds=selectedIds
参数 projectId=当前项目 id
参数 groupSize=otherSetting.assetsBatchGenereateSize
接口立即返回“开始提取资产”
后端先把选中剧本 extractState=2 等待提取
异步把待处理剧本改为 extractState=0 正在提取
AI 成功后写入/复用资产，重建 o_scriptAssets 关系，把 extractState=1
AI 失败或无资产时，把 extractState=-1 并写 errorReason
```

轮询：

```txt
notCompletedData = scripts.filter(s => s.extractState == 0)
只轮询正在提取中的剧本，不轮询等待提取 2
每 3 秒调用 /script/pollScriptAssets
后端查询 whereIn ids 且 whereNot extractState "生成中"
由于 extractState 实际是数字，whereNot "生成中" 会返回数字状态；只要有返回，前端刷新列表
没有 extractState=0 时停止轮询
```

## 7. 数据和状态

数据库：

```txt

字段：id, name, content, projectId, extractState, createTime, errorReason

Toonflow 剧本资产关系表参考名：o_scriptAssets
字段：scriptId, assetId
```

```txt
源码风险：编辑时 assets 为空不会删除旧关系，VT Studio 必须修正为始终同步关系
```


## 8. 不能漏的逻辑

1. 剧本卡片点击是编辑，勾选框和删除图标都要 stop 事件。
2. 新增/编辑剧本可以手动关联已有角色、道具、场景资产。
3. 批量导入必须支持自定义正则和 AI 获取正则。
4. 批量导入 row-key 使用 `index`，集数重复会影响选择，VT Studio 必须用稳定临时 id。
5. 提取资产接口是异步任务，HTTP 成功不代表提取完成。
6. `extractState=2` 是等待提取，`extractState=0` 才是正在提取。
7. 资产提取成功后会先删除本批剧本的旧 `o_scriptAssets`，再写新关系。
8. `getScrptApi` 是 Toonflow 拼写错误，VT Studio 必须修正为 `getScriptList` 或 `script.list` 语义。
9. `assetsBatchGenereateSize` 是拼写错误，VT Studio 必须修正为清晰命名。
10. 导出 zip 文件名用剧本 name 直接作为 txt 文件名，VT Studio 必须处理非法文件名和重名。
11. 删除剧本会删除分镜、视频和 Agent 工作数据，必须二次确认。
12. 提取中的剧本不允许批量删除；单条删除源码没有做这个保护，VT Studio 必须补齐。
13. 资产提取不写任务中心，如果 VT Studio 要统一任务记录，要记录偏差。

## 11. 缺少的逻辑和实现细节

### F-005-001 新增剧本

新增剧本不是只存标题和正文，还可以手动关联已有角色、场景、道具资产。

必须明确：

```txt
必填：剧本名称、剧本内容
关联资产：只选当前项目 role/scene/tool
新增后 extractState：默认未提取或待提取状态必须固定
资产关系：assets 为空也要明确写成无关系
成功后：关闭弹窗、刷新卡片、清空表单
```

### F-005-002 批量新增剧本

参考项目批量新增依赖用户正则和 AI 生成正则，保存前有预览和勾选。

必须明确：

```txt
支持文件：txt/docx
正则来源：用户输入或 universalAi 生成
解析结果：必须预览集数、名称、内容长度
勾选保存：不能解析完直接入库
长度限制：受设置 scriptEpisodeLength 影响
row-key：必须用稳定临时 id，不能用 index
```

### F-005-003 编辑剧本

参考源码编辑时 assets 为空不会删除旧关系，这是风险，VT Studio 必须修正。

必须明确：

```txt
可编辑：名称、内容、关联资产
保存关系：每次保存都以当前 assets 为准，先删旧关系再写新关系
内容变更后：已生成的分镜/视频是否失效要提示，不静默保留
提取中：禁止编辑或提示会影响提取结果
```

### F-005-004 删除/批量删除剧本

删除剧本会影响生产链路，不是普通删除。

必须明确：

```txt
删除前列出影响：Agent 工作区、分镜、视频、剧本资产关系
批量删除：提取中剧本不能删
单条删除：也要补提取中保护
文件清理：分镜图片、视频文件是否删除要跟项目删除规则一致
事务：数据库关系删除必须一体化
缓存：删除后清理 imageListCache 和当前生产上下文
```

### F-005-005 搜索剧本

参考项目没有分页源码事实，搜索返回当前项目匹配剧本。

必须明确：

```txt
搜索范围：当前项目
搜索字段：剧本名称
返回数据：剧本、关联资产、extractState、errorReason
空搜索：返回当前项目全部剧本
```

### F-005-006 导出剧本 zip

这是剧本文本导出，不是剪映导出。

必须明确：

```txt
导出内容：每个剧本一个 txt
zip 名称：script_YYYY-MM-DD.zip 或 VT Studio 统一命名
txt 文件名：必须处理非法字符、重名、超长文件名
导出位置：参考项目浏览器下载；VT Studio 可保存到用户选择目录或 downloads
失败列表：哪些剧本写入失败要显示
```

### F-005-007 / F-005-008 从剧本提取资产和轮询

资产提取是异步逻辑，HTTP 成功只代表进入队列。

必须明确：

```txt
开始：先把 extractState 写为等待/进行中
模型：universalAi，提示词类型 scriptAssetExtraction
输出：resultTool 结构化 newAssets/existingAssetRefs
已有资产匹配：参考项目按 name 精确匹配，VT Studio 要处理重名资产
成功：写 assets，写 scriptAssets 关系，extractState=succeeded
失败：extractState=failed，写 errorReason
异常退出：running 要修复为 failed；waiting 是否修复也要定规则
取消：参考源码未找到，VT Studio 如做取消必须写偏差
```
