## 2. 菜单入口和页面结构

入口：项目内顶部菜单「小说/原文」，仅 `projectType=novel` 的项目显示。  
页面作用：导入原文、管理章节、按选中章节触发事件分析、查看章节级事件摘要和失败原因。

主页面区域：

```txt
顶部操作栏：导入原文、批量删除、事件分析
搜索区：章节名搜索输入框、搜索按钮
原文章节表格
正文/事件预览弹窗
导入原文弹窗
编辑章节弹窗
```

主表格列：

```txt
多选
index：章节序号
reel：卷
chapter：章节名
chapterData：章节正文预览
event：事件摘要/生成中/失败原因
operation：编辑、删除
```

```txt

有卷结构时，按卷拆分，再按章节拆分，每卷内章节按 index 排序
中文数字支持零、一、二、三、四、五、六、七、八、九、十、百、千
```

## 5. 表单字段和校验

原文章节字段：

| 业务含义 | Toonflow 源码名 | 控件 | 必填 | 字段来源 | 说明 |
|---|---|---|---|---|---|
| 项目 ID | `projectId` | 隐式 | 是 | 当前项目 | 所有原文按项目隔离 |
| 章节序号 | `index` / `chapterIndex` | 解析/隐藏 | 是 | 解析结果或后端顺序生成 | 导入时后端忽略解析 index，按项目已有最大 chapterIndex 继续递增 |
| 卷 | `reel` | 解析/隐藏 | 是 | 解析结果 | 无卷时为 `正文卷` |
| 章节名 | `chapter` | 解析/输入框 | 否 | 解析结果或编辑 | 搜索只查这个字段 |
| 章节正文 | `chapterData` | 文本域 | 是 | 上传/粘贴/编辑 | 事件分析输入 |
| 事件摘要 | `event` | 文本域/展示 | 否 | 模型生成或编辑 | 存在 `o_novel.event` |
| 事件状态 | `eventState` | 状态展示 | 是 | 后端写入 | `0` 生成中，`1` 完成，`-1` 失败 |
| 失败原因 | `errorReason` | 预览弹窗 | 否 | 模型异常 | 失败时必须可见 |

导入弹窗字段：

| 业务含义 | Toonflow 源码名 | 控件 | 必填 | 字段来源 | 说明 |
|---|---|---|---|---|---|
| 上传文件 | `fileList` | `t-upload` | 否 | 本地文件 | 只允许 1 个文件，不自动上传到后端 |
| 粘贴内容 | `content` | textarea | 否 | 用户粘贴/文件读取 | content 和解析结果同时存在才能下一步 |
| 选中章节 | `selectedRowKeys` | 表格多选 | 是 | 用户选择 | 保存前必须至少选一章 |

其他设置字段：

| 业务含义 | Toonflow 源码名 | 控件 | 默认值 | 说明 |
|---|---|---|---|---|
| 章节拆分正则 | `otherSetting.chapterReg` | 设置页 textarea | `/第\s*([0-9０-９零一二三四五六七八九十百千万]+)\s*[章回节]\s*([^\n\r]*)/g` | 导入解析优先使用 |

## 7. 数据和状态

数据库：

```txt
Toonflow 原文章节表参考名：o_novel
字段：id, chapterIndex, reel, chapter, chapterData, projectId, eventState, event, errorReason, createTime

Toonflow 事件表参考名：o_event
字段：id, name, detail, createTime

Toonflow 事件章节关系表参考名：o_eventChapter
字段：id, eventId, novelId

VT Studio 不能直接照搬 o_ 表名。
```

章节导入写入：

```txt
后端先查当前 projectId 下最大 chapterIndex
每保存一章，chapterIndex = lastChapterIndex + 1
保存 reel、chapter、chapterData、projectId、createTime
新增章节 eventState 初始为 0
新增后立即启动 cleanNovel 生成章节事件摘要
addNovel 不等待 cleanNovel 全部完成就返回成功
```

事件分析写入：

```txt
generateEvents 先把选中章节 eventState=0、event=null
cleanNovel 逐章调用 universalAi
成功：更新 o_novel.event 为模型输出，eventState=1
失败：更新 o_novel.event=null，eventState=-1，errorReason=错误信息
fixDB 会在应用启动时把残留 eventState=0 改为 -1，并写 errorReason=软件退出导致失败
```

模型调用：

```txt
模型 key：universalAi
调用方式：u.Ai.Text("universalAi").invoke
提示词来源优先级：
  1. o_prompt where type=eventExtraction 的 useData
  2. o_prompt where type=eventExtraction 的 data
  3. utils/getPrompts.ts 中 getPrompts("event") 的默认提示词
模型输出会经过 stripThink
cleanNovel 不创建 o_tasks 任务记录
```

事件列表链路：

```txt
主页面事件摘要：写在 o_novel.event
event.vue 事件列表：读取 o_event/o_eventChapter
当前源码没有找到从 o_novel.event 写入 o_event/o_eventChapter 的 route
所以“章节事件摘要”和“事件列表管理”不是同一条完整闭环
```

本地文件：

```txt
导入原文源码只读取 txt/docx 内容，不保存原始文件副本
VT Studio 如果要保存原文件副本，属于增强能力，需要写偏差记录
```

任务记录：源码没有为原文导入或事件分析写 `o_tasks`。  
状态流转：`0 -> 1` 成功；`0 -> -1` 失败；启动修复把残留 `0 -> -1`。  
轮询/Socket：主页面用 3 秒轮询，无 Socket。  

## 8. 不能漏的逻辑

1. 导入不是直接全量保存，用户要在解析结果里勾选章节。
2. 导入后会自动开始事件摘要生成，新增章节初始 `eventState=0`。
3. 事件分析按钮只分析选中的章节，不是全项目一键分析。
4. 事件生成不写 `o_tasks`，如果 VT Studio 要纳入任务中心，必须作为偏差记录。
5. `editNodel` 是旧拼写，VT Studio 必须改成 `editNovel`。
6. `assetsBatchGenereateSize` 拼写错误但被复用为事件分析并发数，VT Studio 要改成清晰命名。
7. 主页面事件摘要存在 `o_novel.event`，`event.vue` 事件列表读取 `o_event/o_eventChapter`，两者当前源码没有打通。
8. `eventAnalysis.vue` 调用不存在的后端 route，不能作为实现依据。
9. 单条/批量删除章节源码用 `o_eventChapter` 的 `id` 去删 `o_event.id`，不是 `eventId`，这是源码风险，VT Studio 必须修正。
10. 导入预览表格 row-key 使用 `index`，不同卷出现相同章节序号时可能影响选择，VT Studio 必须用稳定临时 id。
11. 主页面单条删除成功提示源码会触发两次，VT Studio 必须修正为一次。
12. 事件分析接口返回后前端没有失败 catch，VT Studio 必须补失败提示。

## 11. 缺少的逻辑和实现细节

### F-004-001 小说/原文导入

参考项目导入是“解析 -> 勾选 -> 保存章节 -> 自动事件摘要”，不是上传后直接全量入库。

必须明确：

```txt
支持文件：txt/docx
拆章规则：来自设置里的 chapterReg
非法正则：保存设置时要拦截；导入时仍要兜底报错
预览列表：必须有稳定临时 id，不能用 chapterIndex 当 row-key
用户动作：勾选章节后保存
保存后：按当前项目最大 chapterIndex 继续累加
原文件副本：Toonflow 未保存；VT Studio 若保存，路径放项目业务目录并写偏差
```

### F-004-002 原文列表、搜索、分页

参考项目只确认按章节名搜索。正文、卷名、事件内容搜索没有源码事实。

必须明确：

```txt
列表范围：只查当前项目
搜索字段：先按 chapter 对齐参考项目
分页：后端分页，切搜索重置第 1 页
eventState 显示：未生成、生成中、成功、失败
失败原因：必须能看到 errorReason
```

### F-004-003 编辑原文章节

编辑章节会影响后续事件摘要和剧本 Agent 上下文。

必须明确：

```txt
可编辑字段：卷、章节名、正文
正文变更后：必须把事件摘要标记为需要重新分析，不能继续使用旧 event
保存后：刷新列表
如果该章节正在事件分析：禁止编辑或提示会取消/重置分析
```

### F-004-004 删除/批量删除原文

参考源码删除事件关系有明显风险，VT Studio 必须修正。

必须明确：

```txt
删除章节：同步删除章节事件关系
删除事件：必须按 eventId 删除，不按关系表 id 误删事件
批量删除：事务化处理，任一失败要回滚
正在事件分析的章节：禁止删除或先取消任务
删除后：刷新分页，当前页为空时回退上一页
```

### F-004-005 / F-004-006 事件分析生成和轮询

参考项目事件分析不写任务中心，靠章节字段和 3 秒轮询。

必须明确：

```txt
触发范围：只分析用户选中的章节
开始状态：eventState=0，event=null
成功状态：eventState=1，写 event
失败状态：eventState=-1，写 errorReason
异常退出修复：启动时把残留 eventState=0 改失败
VT Studio 如果进入任务中心：任务状态和章节 eventState 要同步
```

### F-004-007 事件管理

参考项目 `event.vue` 和主列表的 `o_novel.event` 没有找到闭环。

必须明确：
