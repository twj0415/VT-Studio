# UI 与参考项目

## UI 总原则

1. 工作台是生产工具，不做营销页。
2. 页面优先清晰、密集、可扫描。
3. 主流程不要隐藏关键状态。
4. 表格适合批量编辑，Inspector 适合辅助信息。
5. 不用说明文字替代真实控件。

## 工作台布局

```text
Header
StepBar
主工作区
辅助 Inspector 或抽屉
底部状态区
```

工作台 Header 必须能快速回到：

```text
我的作品
任务中心
AI 工具
创作资源
模型 / 工作流
系统设置
当前作品概览
```

不能让用户为了切换配置从生图页退三层。

## 分镜页

采用表格主编辑。

Inspector 可以放：

```text
校验问题
AI 助手
候选预览
高级字段
下游影响提示
```

Inspector 不能重复编辑：

```text
sourceText
visualDescription
imagePrompt
videoPrompt
```

## 进度显示

显示：

```text
5/8
当前 #3
已用时 2分15秒
成功 4
失败 1
```

不显示预计剩余时间。

## 危险操作

必须二次确认：

```text
删除作品
删除素材
清理未选候选
覆盖已选图或已选视频
删除 Provider
删除密钥
导出到外部目录
真实 real_generate
```

## waoowaoo 吸收点

1. 内容导入成熟：长文本检测、智能切分建议、手动创建兜底，落到内容创作页切分预览。
2. 流程推进感强：参考 `config → script → assets → storyboard → videos → voice`，本项目落为 `内容创作 → 分镜 → 生图 → 视频 → 合成 → 导出`。
3. 分镜不是空壳：字段包含 description、characters、location、shot_type、camera_move、source_text、video_prompt、photographyRules、actingNotes、candidateImages、imageUrl。
4. 提示词嵌在分镜和 AI 修改里：学 `ai-modify-shot-prompt`、`ai-modify-location`、AI data modal；本项目 prompt 跟 StoryboardItem 走。
5. 候选图机制完整：生成、确认、取消、撤回、预览、批量下载；本项目落为 ImageCandidate[]、selectedImageId、revision、清理未选。
6. 任务态专业：流式控制台、运行中 badge、进度提示；本项目落到任务中心、TaskStep、ProgressEvent、已用时、失败恢复。
7. 资产贯穿流程：角色、场景、配音服务创作流程；本项目落为 Asset / AssetReference / Bible 引用。

## Toonflow-web 吸收点

1. 项目和工作台结构：`project / novel / script / production / assets / settings` 分层清楚。
2. 生产画布关系：`script → scriptPlan → storyboardTable → storyboard → workbench`，本项目吸收“总纲 + 镜头”思想，但 MVP 用一个表格承接主编辑。
3. 分镜两层：内容创作管文本确认，分镜页管镜头生产字段。
4. 分镜卡片交互：插入镜头、批量删除、全选、预览、按比例缩放、批量生成图，本项目落为表格行操作、批量操作、候选预览。
5. 模型 / 工作流 / 设置分得细：Provider、模型映射、Prompt、Skill、测试、数据库设置，本项目落为 Provider、ProviderModel、WorkflowPreset、CreativeRule、测试记录。
6. 视频和资产管理：参考图、模型选择、生成历史、结果保存、资产中心、数据库导出思路，本项目落为素材库、候选历史、ExportRecord、诊断包。

## 不吸收

1. 不复制参考项目代码、UI、品牌。
2. 不照搬后端架构。
3. 不把小说、数字人、素材成片塞进当前必经主线。
4. 不新增第二套模板概念替代视频包。
