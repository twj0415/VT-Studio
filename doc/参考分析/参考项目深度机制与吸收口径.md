# 参考项目深度机制与吸收口径

> 本文用于解决一个问题：参考项目不能只写“有分镜、有视频包、有设置页”，必须写清它们“怎么做、为什么有效、我们吸收什么、不吸收什么、落到本项目哪里”。
>
> 新窗口接手时，如果要实现内容导入、分镜、生图、任务反馈、资产、视频包、模型 / 工作流、导出备份，必须先读本文，再看 `功能借鉴矩阵.md` 和对应功能模块文档。

---

## 一、筛选原则

不是参考项目有的都搬进来。筛选规则如下：

```text
1. 能强化当前 image_to_video 主线的，进入当前执行口径。
2. 能约束后续 TODO 数据结构和页面布局的，写进文档但不抢当前主线。
3. 只是参考项目自己的技术栈、品牌、页面外壳、接口命名，不吸收。
4. 有安全风险、任务丢失风险、明文密钥风险的，只记录避坑，不照搬。
5. 已经在本项目文档里有的，不重复造新概念；只补“怎么分层、怎么实现、怎么验收”。
```

写入文档的细度要求：

```text
1. 必须写“怎么实现”，不能只写“有这个功能”。
2. 必须至少落到：页面区域、字段、按钮、状态、数据写回、任务/日志、验收、不吸收项。
3. 如果是页面能力，必须能对应 `页面交互账本.md` 的主按钮、批量按钮、行内按钮、弹窗/抽屉和状态。
4. 如果是数据能力，必须能对应 `底层设计/数据结构.md` 或对应底层规范；没有字段定义时先补字段定义。
5. 如果是参考项目源码证据，必须写文件 / 组件 / route / store 名称；但不得复制代码、样式、Prompt 或接口命名。
6. 如果本项目当前代码还没实现，必须标注“目标结构 / TODO 落点 / 当前实现差异”，不能写成已完成事实。
```

本文不是替代总设计，而是给总设计补“机制细节”。冲突时按 `plan/阶段路线图.md` 的优先级判断；涉及 UI 执行口径时以 `doc/UI设计/全页面布局设计.md` 为准；涉及数据结构、任务、安全时以底层设计为准。

## 二、当前文档体检结论

现有文档方向基本正确，但有三个不足：

```text
1. 部分文档写了“学 waoowaoo 流程、学 Toonflow 分层”，但没有写清具体字段、布局和操作。
2. 视频包、创作规则、模型 / 工作流已经有分层结论，但新窗口还可能不知道“为什么这样分、各自管什么、不该管什么”。
3. 参考项目优点分散在多个文档里，后续执行容易只读 TODO，漏掉内容导入、候选、任务态、资产贯穿这些细节。
4. UI 执行文档已新增 AI 工具一级入口，旧文档若仍只列四个一级菜单，必须按五个入口修正。
```

本次补充后的阅读关系：

```text
参考项目深度机制与吸收口径.md   说明参考项目怎么做、我们怎么筛选吸收
功能借鉴矩阵.md                 把吸收点映射到模块和 TODO
页面交互账本.md                 把吸收点映射到页面按钮、状态、弹窗
UI设计/全页面布局设计.md         把吸收点转成前端布局和交互执行口径
底层设计/数据结构.md             定义字段，不在参考分析里重复定义唯一字段
```

当前统一后的一级入口：

```text
我的作品
AI 工具
创作资源
模型 / 工作流
系统设置
```

AI 工具只放独立小工具，不提供创建作品入口。主流程创建作品仍只能从“我的作品 / 开始创作 / 内容创作”进入。

## 三、waoowaoo-main 的强项和吸收口径

### 3.1 内容导入：长文本检测 + 智能切分建议 + 手动兜底

waoowaoo 不是直接把用户输入扔进分镜。它把输入内容当成前置处理阶段，先判断文本是否过长，再引导用户选择适合的处理方式，并保留手动创建兜底。

它的实现机制：

```text
StoryInputComposer / LongTextDetectionPrompt 负责输入体验。
长文本会触发检测和提示，不让用户直接进入不可控生成。
短文本、长文本、手动创建不是同一种交互。
进入后续阶段前，文本已经被整理成可处理的脚本 / 段落。
```

本项目吸收：

```text
内容创作页必须有输入模式：topic / paste / article。
paste 模式必须机械切分，不允许 AI 改写原文。
长文本要提示切分风险，并给出按段落 / 按行 / 按句的切分预览。
切分预览必须允许编辑、合并、拆分、删除。
AI 整理失败或用户不想用 AI 时，必须能手动确认段落进入分镜。
```

本项目不吸收：

```text
不把长内容小说 workflow 强行塞进当前 MVP。
不新增独立 script-editor 必经页，让用户重复编辑同一批旁白。
不复制 waoowaoo 的组件结构、文案和接口命名。
```

落地位置：

```text
页面：开始创作 / 内容创作
数据：Project.inputType、Project.sourceText、StoryboardItem.sourceText、StoryboardItem.narrationText
TODO：TODO-03、TODO-09 9.11、TODO-12 长内容增强
```

### 3.2 流程推进：config → script → assets → storyboard → videos → voice

waoowaoo 的强项是流程推进感。用户知道当前处于配置、脚本、资产、分镜、视频还是配音阶段。

它的实现机制：

```text
StageNavigation 显示当前阶段和可进入阶段。
ScriptStage 不是终点，而是进入 storyboard 的前置确认。
StoryboardStage / VideosStage 继续承接上一步产物。
运行中状态用 badge、进度区域、阶段反馈呈现，不只是按钮 loading。
```

本项目吸收：

```text
工作台必须固定主线：内容创作 → 分镜 → 生图 → 视频 → 合成。
StepBar 当前只放分镜 / 生图 / 视频 / 合成；内容创作在开始创作和工作台概览承接。
每一步必须展示：已完成数、当前项、失败项、阻断原因。
生成类任务要有阶段反馈，可参考流式控制台，但必须接本项目 Task / ProgressEvent。
```

本项目不吸收：

```text
不把 voice / TTS 放进当前必经 StepBar。
不做跨步骤自动推进到生图或视频完成。
不只靠前端状态推进，真实任务状态必须以后端持久化为准。
```

落地位置：

```text
页面：作品工作台、分镜、生图、视频、合成
数据：Task、TaskStep、ProgressEvent、Artifact
TODO：TODO-05、TODO-07、TODO-09、TODO-10
```

### 3.3 分镜不是空壳：镜头字段 + 生产字段 + 结果字段

waoowaoo 的分镜卡不是只有一张图，而是一个镜头生产单元。它包含文本、镜头描述、人物、地点、摄影、表演、视频提示词和候选图。

它的分镜字段强项：

```text
description
characters
location
shot_type
camera_move
srt_range
duration
source_text
video_prompt
photographyRules
actingNotes
candidateImages
imageUrl
```

本项目吸收，但按自己的字段体系落地：

```text
source_text              → sourceText
narration text / SRT     → narrationText / subtitleChunks 后续增强
description              → visualDescription
characters               → charactersJson / characterIdsJson
location                 → sceneDescription / locationId
shot_type                → shotSize（高级可选字段，对应 SQLite shot_size）
camera_move              → cameraMotion（高级可选字段，对应 SQLite camera_motion）
duration                 → durationSeconds
video_prompt             → videoPrompt
photographyRules         → 后续 advancedDirectingJson.photographyRules 或 TaskStep 输出摘要
actingNotes              → 后续 advancedDirectingJson.actingNotes 或 TaskStep 输出摘要
candidateImages          → ImageCandidate[]
imageUrl                 → selectedImageId 对应候选图
```

本项目执行要求：

```text
StoryboardItem 是当前主线最小工作单元。
分镜页不是只编辑画面描述，还要能看到文本、角色、场景、提示词、时长和状态。
高级字段可以先藏在 Inspector 或展开列里，但数据结构和 UI 入口不能把它们排除掉。
```

本项目不吸收：

```text
不直接照搬字段名。
不把全部高级字段强塞进首版表格导致密度失控。
不让 Inspector 和表格重复编辑同一字段。
```

### 3.4 提示词跟着分镜走，不做孤立大页面

waoowaoo 的提示词能力不是独立大页，而是嵌在镜头生产里。它有按镜头修改提示词、修改场景、查看 AI 数据的能力。

它的实现机制：

```text
ai-modify-shot-prompt：围绕当前镜头优化出图 / 镜头提示词。
ai-modify-location：围绕当前镜头或地点优化场景。
AI data modal：让用户查看 AI 生成 / 修改相关数据。
提示词修改发生在 storyboard / image 生产上下文里，不脱离镜头。
```

本项目吸收：

```text
提示词必须绑定 StoryboardItem，不做孤立“提示词大页面”。
分镜页可提供 AI 助手：优化画面描述、重写提示词、补角色 / 场景。
生图页必须提供“预览最终提示词”：展示角色、场景、创作规则、模型参数组装后的实际 prompt。
每次生成任务记录 rule_key / rule_id、模型、参数和输入快照，方便解释结果。
```

本项目不吸收：

```text
不把 prompt 正文硬编码进 Vue 页面。
不复制参考项目 Prompt / Skill 正文。
不允许 AI 修改覆盖用户锁定字段。
```

落地位置：

```text
页面：分镜 Inspector、生图 prompt 列、最终 prompt 弹窗
数据：StoryboardItem.imagePrompt、negativePrompt、videoPrompt、lockFlagsJson、TaskStep.input_json
规则：PromptSkill / 创作规则
TODO：TODO-06、TODO-08、TODO-09 9.11
```

### 3.5 候选图机制：生成、确认、取消、撤回、预览、批量下载

waoowaoo 对候选结果处理比较完整，不是生成一次就覆盖旧图。

它的实现机制：

```text
每个镜头保留 candidateImages。
用户可以预览候选图，再确认 imageUrl / final image。
生成、取消、撤回、批量下载围绕候选集合操作。
重生成不等于删除历史候选。
```

本项目吸收：

```text
ImageCandidate[] 是必须能力，不允许只存一个 imagePath。
selectedImageId 是用户确认结果，不能由重生成自动覆盖。
候选图支持预览、大图查看、选择、替换、清理历史。
批量生成期间逐行更新候选和状态。
后续批量下载 / 导出候选属于增强，但数据结构不能阻断。
```

本项目不吸收：

```text
不把远程 URL 当正式资产路径。
不把候选图自动写入素材库，只有用户“保存为角色 / 场景 / 风格参考”才进入素材库。
不让删除 / 合并分镜静默丢候选，必须提示影响。
```

### 3.6 资产贯穿流程：角色、场景、配音不是孤立模块

waoowaoo 的角色、场景、配音服务于创作流程，不只是放在资源页展示。

本项目吸收：

```text
角色、场景、风格资产要能被分镜和生图读取。
创作资源不是仓库陈列，而是生产上下文。
作品产物默认留在作品目录；可复用资产经用户确认才进入素材库。
TTS / 配音后续增强，但要能从 narrationText / subtitleChunks 接入。
```

本项目不吸收：

```text
不把 TTS 放进当前 image_to_video 必经主线。
不为了资产一致性默认生成所有角色三视图、表情图、姿态图。
不让素材库和任务产物混成一个概念。
```

## 四、Toonflow-web 的强项和吸收口径

### 4.1 项目和工作台分层清楚

Toonflow-web 的信息结构清楚，`project / novel / script / production / assets / settings` 分层让用户知道每类能力属于哪里。

本项目吸收为：

```text
我的作品：只管作品列表、继续制作、状态和输出记录。
开始创作 / 内容创作：只管输入、整理、切分、基础参数。
作品工作台：只管当前作品的分镜、生图、视频、合成、任务和导出。
创作资源：只管可复用的视频包、创作规则、素材库。
模型 / 工作流：只管 Provider、模型能力、workflow preset、测试记录。
系统设置：只管应用偏好、路径、sidecar、密钥安全、日志诊断。
```

禁止混淆：

```text
视频包不是模型配置。
创作规则不是视频包正文。
素材库不是作品目录。
模型 / 工作流不是创作资源。
系统设置不直接承载作品生产字段。
```

### 4.2 生产画布：script → scriptPlan → storyboardTable → storyboard → workbench

Toonflow-web 的生产画布强在产物关系直观。它不是单页把所有东西摊平，而是把脚本、规划、分镜总表、镜头卡片和工作台结果分层。

它的实现机制：

```text
script：原始脚本 / 文本产物。
scriptPlan：对脚本做结构规划。
storyboardTable：总纲式分镜表，适合看整体段落和结构。
storyboard：逐镜头卡片，适合编辑画面、插入镜头、生成图。
workbench：承接生成结果和后续处理。
```

本项目吸收：

```text
内容创作页承担 script / 文本确认能力。
分镜页承担 storyboardTable + storyboard 的合并版 MVP：左侧或表格看全局，行内编辑镜头字段。
当分镜数量变多时，要有行号跳转、搜索、筛选、批量操作，补足 storyboardTable 的总览价值。
后续长内容 workflow 可以拆出更完整的 scriptPlan / storyboardTable 层，但当前短视频主线先不新增必经页。
```

本项目不吸收：

```text
不把生产画布直接照搬成节点编辑器。
不让用户在导入前先做完整分镜。
不把 script-editor 作为当前必经步骤。
```

### 4.3 分镜两层结构：总纲 + 镜头

Toonflow-web 的 `storyboardTable` 管整体文本和结构，`storyboard` 管逐镜头卡片。这个结构解决了一个问题：长分镜既需要全局扫描，也需要单镜头精修。

本项目吸收：

```text
分镜表格必须保留全局扫描能力：行号、原文、角色、场景、画面描述、提示词、状态。
Inspector 只做辅助：校验问题、AI 助手、候选预览、高级字段，不重复编辑主字段。
长表格必须有行号快速跳转，后续可加搜索和状态筛选。
```

本项目不吸收：

```text
不同时做两套主编辑界面。
不让表格和右侧面板编辑同一个字段造成冲突。
```

### 4.4 分镜卡片交互：插入、批量删除、全选、预览、缩放、批量生成图

Toonflow-web 的逐镜头卡片交互比普通表格强，适合大量镜头的批量生产。

本项目吸收：

```text
表格行要支持新增、复制、删除、拆分、合并、上移、下移。
批量操作必须尊重锁定字段和已确认产物。
生成前可以全选 / 筛选未生成 / 筛选失败项。
预览要支持缩略图和大图弹窗。
缩放视图属于后续增强，但表格密度和响应式必须预留空间。
```

本项目不吸收：

```text
不把卡片式画布作为当前唯一分镜形态。
不牺牲表格密集编辑效率。
```

### 4.5 模型 / 工作流 / 设置集中管理

Toonflow-web 把 Provider、模型映射、Prompt、Skill、测试、数据库设置集中到设置中心。这一点值得学，但要按本项目安全规则重做。

本项目吸收为三层：

```text
Provider：连接配置、base_url、vendor、key_alias、连通性测试；不保存具体业务模型能力。
API 模型能力：model_name、capability、输入输出约束、价格 / 限流 / 最大尺寸等；不保存真实密钥。
Workflow Preset：ComfyUI / RunningHub / 本地 workflow 的 param_schema、node_map、output_map、能力声明和测试记录。
```

和创作资源的边界：

```text
创作规则：写“如何生成”，包括 prompt 模板、schema、校验规则。
视频包：写“默认用哪套策略”，引用创作规则、推荐模型能力、默认风格和参数。
模型 / 工作流：写“系统能调用什么”，不关心某个内容类型的创作策略。
```

禁止混淆：

```text
不要把 API key 放进视频包。
不要让视频包复制创作规则正文。
不要把模型能力写死在 Vue 页面。
不要执行用户上传 TS / JS Provider。
```

### 4.6 视频和资产管理：参考图、生成历史、资产中心、数据库导出

Toonflow-web 的资产和结果管理比较完整，能回看生成历史、保存结果、管理参考图和导出数据库。

本项目吸收：

```text
任务历史看 Artifact，素材库看 Asset。
ImageCandidate / VideoSegment 记录生成上下文快照。
用户确认复用后，候选图才能变成角色 / 场景 / 风格参考资产。
项目包导出必须收集受控资产、项目快照、导出记录和必要模板，但不能包含密钥和绝对路径。
```

本项目不吸收：

```text
不把所有生成结果自动塞进素材库。
不把数据库导出做成无脱敏裸导出。
不保存未授权外部文件引用。
```

## 五、提示词的最终口径

提示词不是没有，也不是单独做一个大页面。真正要学的是：

```text
提示词跟着分镜数据走。
提示词由创作规则、角色、场景、风格、模型能力和当前镜头共同组装。
用户编辑的是镜头级字段和可见 prompt，系统发送的是最终组装 prompt。
最终 prompt 必须可预览、可追踪、可诊断，但不能散落在 Vue 页面里。
```

页面分布：

```text
内容创作页：不编辑最终生图 prompt，只确认文本段落。
分镜页：编辑镜头画面、角色、场景、基础 imagePrompt / videoPrompt。
生图页：预览最终 prompt，生成候选图，选择 selectedImageId。
视频页：编辑 videoPrompt，生成候选视频，选择 selectedVideoSegmentId。
创作规则页：管理 prompt 模板、结构化输出 schema 和校验规则。
模型 / 工作流页：管理模型能力和 workflow 输入输出约束。
```

## 六、必须进入后续执行口径的参考点

这些点不能丢：

```text
1. 内容导入智能切分：长文本检测、切分建议、手动兜底。
2. 流程推进感：用户知道当前步骤、下一步、阻断原因和任务状态。
3. 分镜结构化：文本、画面、角色、场景、提示词、摄影 / 运镜高级字段、候选结果都挂在镜头上。
4. 提示词绑定镜头：AI 修改和最终 prompt 预览围绕 StoryboardItem。
5. 候选结果保留：生成、预览、确认、撤回 / 清理、历史解释。
6. 任务反馈专业化：不是普通 loading，而是 TaskStep、ProgressEvent、运行中状态、失败可恢复。
7. 资产贯穿流程：角色、场景、风格、参考图服务分镜和生成，不是孤立资源库。
8. 模型 / 工作流 / 设置分层：Provider、API 模型能力、Workflow Preset、测试记录分开。
9. 生产结果管理：预览、批量操作、历史记录、导出和项目包安全扫描。
```

这些点不能盲目搬：

```text
1. 不搬参考项目 UI 外壳。
2. 不搬 Prompt / Skill 正文。
3. 不搬明文密钥和用户代码热执行。
4. 不搬纯内存任务和 fire-and-forget。
5. 不把当前短视频主线膨胀成长内容 / 小说 / TTS / 数字人全链路。
```

## 七、参考机制落地规格

这一节是给后续写代码用的。不是再总结“学什么”，而是把参考点落到本项目页面、按钮、状态、字段和后端边界。

### 7.1 内容导入落地规格

| 项 | 执行口径 |
|---|---|
| 页面区域 | 输入来源 Tabs、主输入区、切分预览区、右侧作品设置、底部确认条 |
| 参考机制 | waoowaoo 的 `LongTextDetectionPrompt` 先提示长文本风险；`SmartImportWizard` 拆来源、解析、映射、确认；Pixelle 的 generate/fixed 双模式 |
| 必有按钮 | AI 生成 / 整理、切分预览、编辑段落、合并、拆分、删除、恢复原文、保存草稿、创建并进入分镜 |
| 必有状态 | empty_input、too_long、needs_split_preview、splitting、ai_generating、ai_failed、ready_to_confirm、confirmed、draft_saved |
| 数据写回 | Project.inputType、inputProcessMode、sourceText/sourceTextPath、targetSceneCount；StoryboardItem.sourceText/narrationText |
| 吸收 | 长文本检测、切分建议、手动确认兜底、paste 不改写 |
| 不吸收 | 不新增 script-editor 必经页，不把小说长内容塞进当前 MVP，不复制参考项目导入 UI |
| 验收 | paste 模式一字不改；长文本有风险提示；切分后用户可逐条编辑并确认；确认后分镜页读取的是已确认段落 |

### 7.2 分镜落地规格

| 项 | 执行口径 |
|---|---|
| 页面区域 | 顶部工具栏、表格主编辑区、Inspector 辅助区、底部保存 / 错误状态 |
| 参考机制 | waoowaoo 的镜头字段：description、characters、location、shot_type、camera_move、source_text、video_prompt、photographyRules、actingNotes、candidateImages、imageUrl；Toonflow-web 的 storyboardTable + storyboard 两层 |
| 必有按钮 | 生成分镜、重新生成未锁定、确认分镜、进入生图、新增、复制、删除、上移、下移、拆分、合并、锁定、单条重生成、跳转到指定行 |
| 主编辑字段 | sourceText、narrationText、visualDescription、characters/characterIds、sceneDescription/locationId、imagePrompt、negativePrompt、videoPrompt、durationSeconds |
| Inspector 只放 | 校验问题、AI 助手、候选图/视频预览、高级字段：shotSize/cameraMotion/composition/pace/transitionType；photographyRules/actingNotes 未落库前只作为建议数据 |
| 禁止 | 表格和 Inspector 同时编辑同一个主字段；删除/合并/拆分静默丢候选；AI 覆盖 lockFlagsJson 锁定字段 |
| 验收 | 未确认或必填缺失不能进入生图；锁定字段不被批量重生成覆盖；长分镜可跳转指定行 |

### 7.3 提示词落地规格

| 项 | 执行口径 |
|---|---|
| 页面分布 | 分镜页编辑镜头级 prompt；生图页预览最终 prompt；创作规则页管理模板和 schema |
| 参考机制 | waoowaoo 的 ai-modify-shot-prompt、ai-modify-location、AIDataModal；Toonflow 的 Prompt / Skill 管理 |
| 用户可编辑 | imagePrompt、negativePrompt、videoPrompt、画面描述、角色 / 场景引用 |
| 系统组装 | 创作规则 + Style Bible + Character Bible + Environment Bible + StoryboardItem + 模型 / workflow 限制 |
| 必有按钮 | 编辑提示词、单条重生成、锁定、复制、预览最终 prompt、查看当前创作规则 |
| 数据写回 | StoryboardItem.imagePrompt/negativePrompt/videoPrompt、lockFlagsJson、TaskStep.input_json 里的 rule_key/rule_id 和模型快照 |
| 禁止 | 不做孤立提示词大页面；不把 prompt 正文硬编码进 Vue；不伪造未接入后端的最终发送 prompt |

### 7.4 候选结果落地规格

| 项 | 执行口径 |
|---|---|
| 参考机制 | waoowaoo 的 candidateImages、select-candidate、cleanup-unselected-images、download-images、undo-regenerate |
| 图片产物 | 每个 StoryboardItem 有 ImageCandidate[]，用户选择后写 selectedImageId |
| 视频产物 | 每个 StoryboardItem 有 VideoSegment[]，用户选择后写 selectedVideoSegmentId |
| 必有按钮 | 生成、重新生成、预览、选择、确认、替换本地、清理未选、保存为素材；批量下载作为增强 |
| 状态 | not_generated、input_missing、running、failed、waiting_select、selected、confirmed |
| 存储 | 远程 URL / base64 / 临时文件必须转存到 StorageService 受控目录，DB 只存相对路径 |
| 禁止 | 不用单字段覆盖旧结果；不让重生成自动覆盖已选结果；不把所有候选自动塞进素材库 |

### 7.5 任务反馈落地规格

| 项 | 执行口径 |
|---|---|
| 参考机制 | waoowaoo 的运行中 badge、流式进度、阶段反馈；Pixelle 的 ProgressEvent；参考项目任务可靠性不足作为避坑 |
| 页面落点 | 生图 / 视频底部 TaskProgressPanel，工作台 Header 任务摘要，作品卡片失败恢复入口 |
| 必显示 | 总数、成功数、失败数、当前行、当前步骤、已用时、取消 / 重试入口 |
| 不显示 | 不可靠 ETA / 预计剩余时间 |
| 数据来源 | SQLite Task / TaskStep 为准，Tauri ProgressEvent 只做实时展示 |
| 禁止 | 不做 fire-and-forget；不靠前端 loading 冒充任务状态；取消后不删除已完成产物 |

资源消耗统计也归到任务反馈体系，不单独做一套账：

```text
工作台 Header / 作品卡片可以显示：已用 图片 N 次 / 视频 N 次 / LLM N 次。
数据来源优先从 TaskStep、Artifact、Provider 调用记录或 generation_context_snapshot 聚合。
如果当前阶段没有真实统计数据，页面只能显示“未接入统计”或隐藏该项，不能写假次数。
统计只展示次数和脱敏模型来源，不展示 API Key、完整请求头、完整 prompt 原文或本地绝对路径。
```

### 7.6 资产贯穿落地规格

| 项 | 执行口径 |
|---|---|
| 参考机制 | waoowaoo / Toonflow 的角色、场景、参考图贯穿分镜和生成 |
| 页面落点 | 分镜行角色 / 场景字段，生图参考图条，素材库详情和引用关系 |
| 数据边界 | 作品产物默认留在作品目录；用户显式保存后才登记 Asset / AssetReference |
| 注入方式 | 生成最终 prompt 或 ImageInputPlan 时读取 Style / Character / Environment Bible |
| 禁止 | 不默认生成所有三视图 / 表情图 / 姿态图；不把素材库和任务产物混成一个概念 |

### 7.7 视频包 / 创作规则 / 模型工作流落地规格

| 层 | 管什么 | 不管什么 | 页面 |
|---|---|---|---|
| 视频包 | 默认策略组合：默认规则引用、推荐模型能力、默认画幅、时长、分镜数、语气、风格 | 不保存规则正文、不保存密钥、不保存 Provider 连接、不作为模板创建入口 | 创作资源 > 视频包 |
| 创作规则 | prompt 模板、结构化输出 schema、参数模板、校验规则、测试样例 | 不保存密钥、不执行用户代码、不直接写业务表、不保存素材文件 | 创作资源 > 创作规则 |
| Provider 连接 | providerKind、vendor、baseUrl、authType、keyAlias、enabled | 不保存 modelName、不保存 workflow 参数、不保存真实 key | 模型 / 工作流 > Provider |
| API 模型能力 | modelName、abilityTypes、input/output modalities、limits、featureFlags | 不登记 ComfyUI/RunningHub workflow、不保存密钥 | 模型 / 工作流 > API 模型能力 |
| Workflow Preset | workflowKey/id/version、paramSchema、nodeMap、outputMap、limits | 不混进 provider_models、不保存 key、不执行未注册 workflow | 模型 / 工作流 > Workflow Preset |

后续实现必须按下面的引用链施工，不能自由发挥：

```text
视频包
  只保存 ruleRefs、recommendedExecutableRefs、assetRefs 和默认参数
  ruleRefs 指向创作规则的 rule_key / rule_id
  recommendedExecutableRefs 指向 providerModelId 或 workflowPresetId
  assetRefs 指向素材库 Asset / AssetReference

创作规则
  保存 prompt_body、output_schema、params_schema、校验规则和测试样例
  不直接写 StoryboardItem，不直接调用 Provider
  由内容创作、分镜、提示词、生图、视频等任务按 rule_key / rule_id 引用

模型 / 工作流
  Provider 连接只解决连接与 key_alias
  API 模型能力只落 provider_models
  Workflow Preset 只落 workflow_presets
  业务页面只读 list_executable_media_options，不直接拼 provider_models + workflow_presets
```

页面施工细节：

```text
视频包页：
  左侧包列表；中间基本信息；右侧绑定规则、推荐模型 / workflow、素材引用、引用关系。
  必有按钮：复制为用户包、绑定规则、绑定推荐模型能力、查看引用、启用 / 禁用、删除用户包。
  保存校验：不能保存真实密钥；不能内联复制 prompt 正文；不能引用 disabled 模型 / workflow；删除前检查作品引用。

创作规则页：
  左侧规则分类；中间规则列表；右侧规则正文、schema、参数、测试输出。
  必有按钮：复制为用户规则、保存、启用 / 禁用、测试输出、查看引用、恢复默认、删除用户规则。
  保存校验：builtin 不可直接编辑；output_schema 必须可解析；规则正文不能执行 JS / TS / Python；删除前检查视频包 / 作品 / 任务引用。

模型 / 工作流页：
  Provider、API 模型能力、Workflow Preset、测试记录四个 Tab。
  Provider 测连接不生成媒体；API 模型 dry_run 默认可用；real_generate 必须二次确认；Workflow Preset 保存只登记 schema / nodeMap / outputMap，不执行未注册 workflow。
  测试记录只保存脱敏请求摘要、脱敏响应摘要、结果、耗时、traceId 和错误码。
```

任务记录要求：

```text
每次内容生成 / 分镜 / 生图 / 视频任务都要在 TaskStep.input_json 或 generation_context_snapshot 里记录：
  使用的视频包 packId（如果有）
  rule_key / rule_id / source_type
  providerModelId 或 workflowPresetId
  模型 / workflow 约束摘要
  用户覆盖过的规则或参数

不得记录：
  API Key
  Provider 完整请求头
  本地绝对路径
  大段未脱敏 Provider 原始响应
```
