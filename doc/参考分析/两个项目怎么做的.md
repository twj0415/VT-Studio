# 参考项目源码证据索引

> 这篇只记录参考项目里“证明某种能力存在”的源码证据。它不是产品蓝图，也不是实现指令。
> 开发时必须先读 `doc/00-文档入口.md`、`开始之前/最终产品形态.md`、`开始之前/菜单与页面详细说明.md`、`参考分析/参考项目取舍总则.md`、`参考分析/参考方法与防漏机制.md`，再把这里当作证据补充。

## 一、使用边界

1. 不能复制参考项目代码、UI、品牌、素材、Prompt / Skill 正文和接口命名。
2. 不能因为某个参考项目用了某种技术栈，就改变本项目 `Vue3 + Tauri 2 + Rust + SQLite` 的架构。
3. 不能把参考项目的页面结构直接搬成本项目菜单。
4. 不能把参考项目的风险实现带进来，比如明文密钥、弱任务状态、静默吞错、用户代码热执行。
5. 这里的“源码位置”只用于确认能力、交互和工程链路，不等于可以复用实现。

Toonflow 明确不能带入本项目的问题：

1. `vm2` 执行用户 TS Provider 有沙箱逃逸和 RCE 风险。
2. 部分 Provider / TTS 实现不完整或静默吞错。
3. API Key / vendorConfig 明文存储不适合本项目。
4. 后台任务存在 fire-and-forget 风险，进程重启后恢复能力不足。
5. Toonflow-web 后端不可二开，不能假设完整服务端能力可用。

## 三、Pixelle 能力证据

| 能力 | 参考源码位置 | 可学习点 | 本项目落地规则 |
|---|---|---|---|
| generate / fixed 输入模式 | `web/components/content_input.py` | 主题生成和固定文案切分分开 | 本项目按入口自动决定模式；paste 不允许 AI 改写 |
| 主题生成段落 | `pixelle_video/utils/content_generators.py::generate_narrations_from_topic` | 主题 + 目标段数生成结构化段落 | 本项目用 PromptSkill + schema 校验，不复制提示词正文 |
| 文章浓缩段落 | `generate_narrations_from_content` | 文章浓缩和主题发散要分开 | `inputType=article` 使用浓缩策略 |
| 文案切分 | `split_narration_script` | 段落 / 行 / 句三种切分方式 | 本项目自己实现并加可编辑预览、合并、拆分、删除 |
| 结构化输出 | Pydantic 模型和校验逻辑 | LLM 输出必须可校验 | 本项目后端 schema 校验，不信任前端 |
| 视频模型能力矩阵 | `pixelle_video/services/api_media.py::VIDEO_MODEL_CAPABILITIES` | 不同视频模型约束差异很大 | 落到 `provider_models` 和 `workflow_presets`，不硬编码在页面 |
| ComfyUI / RunningHub | `comfy_base_service`、workflow 扫描逻辑 | API 模型和 workflow 双轨 | 本项目统一注册 workflow preset，校验 param_schema / node_map / output_map |
| TTS | `pixelle_video/services/tts_service.py` | 音频生成和时长回填 | 后续增强；不作为当前 image_to_video 主线前置 |
| HTML 模板渲染 | templates + Playwright | 字幕、封面、图文动效可模板化 | 落到模板安全规范和 PathGuard，不让模板访问任意路径 |
| FFmpeg 合成 | VideoService / concat 流程 | 真实视频片段需要合成、转码、探测 | 本项目用 FFmpeg sidecar、ffprobe、自检、脱敏日志 |
| 历史记录 | `history_manager.py`、`persistence.py` | 产物需要回看 | 本项目以 SQLite Task / Artifact / ExportRecord 为准 |

Pixelle 明确不能带入本项目的问题：

1. Streamlit / Python 应用架构不适合作为本项目主架构。
2. API Key 明文配置不符合本项目安全边界。
3. 能力矩阵硬编码在代码里，后期扩展成本高。
4. 任务和历史偏文件 / 内存状态，可靠性不足。
5. 部分错误直接上抛，缺少用户可恢复路径。

waoowaoo 的主要价值不是某个页面长相，而是短视频生产节奏、输入前置处理、分镜生产字段、候选图确认和任务反馈。

| 能力 | 参考源码位置 | 可学习点 | 本项目落地规则 |
|---|---|---|---|
| 首页输入入口 | `src/app/[locale]/home/page.tsx` 使用 `StoryInputComposer` | 用户先输入故事 / 文案 / 配置，再进入工作台 | 本项目“开始创作 / 内容创作”承担输入、视频包、画幅、时长、分镜数 |
| 长文本风险提示 | `src/components/story-input/LongTextDetectionPrompt.tsx` | 长文本先提示风险，不直接进入不可控生成 | 本项目 paste / article 必须有长文本提示和切分预览 |
| 智能导入向导 | `src/app/[locale]/workspace/[projectId]/page.tsx` 使用 `SmartImportWizard`，组件在 `modes/novel-promotion/components/SmartImportWizard.tsx` | 导入拆成来源、解析、映射、确认 | 本项目吸收“确认前置”，但不新增 script-editor 必经页 |
| 阶段导航 | `src/app/[locale]/workspace/[projectId]/modes/novel-promotion/StageNavigation.tsx` | config / script / assets / storyboard / videos / voice 的阶段感 | 本项目 StepBar 固定内容创作 → 分镜 → 生图 → 视频 → 合成，TTS 不进当前必经主线 |
| 脚本阶段 | `modes/novel-promotion/components/ScriptStage.tsx` | 文本确认在分镜前发生 | 本项目把逐条旁白 / 文案确认合并到内容创作切分预览 |

| 场景修改 | `src/app/api/novel-promotion/[projectId]/ai-modify-location/route.ts` | 针对地点 / 场景做 AI 修改 | 本项目场景字段和 Environment Bible 可由 AI 辅助，但不能覆盖锁定字段 |

| 清理未选候选 | `src/app/api/novel-promotion/[projectId]/cleanup-unselected-images/route.ts` | 候选集合可清理，但要围绕已选图 | 本项目清理未选候选是危险操作，二次确认，保留已引用资产 |
| 候选解析 | `components/storyboard/hooks/panel-candidate-runtime.ts` | candidateImages 是可解析集合，不是单路径 | 本项目必须用 ImageCandidate[]，不允许单 imagePath 覆盖历史 |

waoowaoo 不能带入本项目的问题：

1. 不复制页面代码、组件结构、样式、接口命名和 Prompt 正文。
2. 不因为它有 script 阶段就保留独立 script-editor 必经路由；本项目避免重复编辑旁白。
3. 不把远程 URL 当本项目正式资产路径；本项目必须转存到 StorageService 受控目录。
4. 不因为它流程清楚就删掉桌面端需要的 Task、Artifact、PathGuard、导出和诊断。
5. 不把 voice / TTS 放入当前 `image_to_video` 必经 StepBar。

## 五、huobao-drama 能力证据

huobao-drama 只作为短剧业务形态补充参考。

可学习：

1. 短剧项目 / 剧集 / 内容管理的业务菜单感。
2. 后续如果做长内容或短剧管理，可参考其“作品、剧集、素材、发布”的信息分层。

不能学习：

1. 不把当前工具扩成复杂 SaaS。
2. 不让短剧管理压过当前 image_to_video 主线。
3. 不把它作为生成链路、模型接入或桌面架构来源。

## 六、最终取舍

| 领域 | 主要证据来源 | 本项目采用方式 |
|---|---|---|
| 前端流程 | waoowaoo | 线性工作台，但保留桌面端任务、历史、候选和导出 |
| 创作资源 | Toonflow-app / Toonflow-web | 创作规则、视频包、Style / Character / Environment Bible |
| 长内容 | Toonflow | 作为后续 workflow，不阻塞当前短视频主线 |
| 真实生成 | Pixelle | ProviderManager + provider_models + workflow_presets |
| ComfyUI / RunningHub | Pixelle | workflow preset 注册和 schema 校验 |
| TTS / 字幕 / 封面 / BGM | Pixelle | 后续增强，不作为当前主线前置 |
| 视频合成 | Pixelle | FFmpeg sidecar + PathGuard + ffprobe + 脱敏日志 |
| 可靠性 | 参考项目都不足 | 本项目用持久化 Task / TaskStep / Artifact / ExportRecord 补强 |
| 安全 | 参考项目都不足 | keyring、SecretGuard、PathGuard、导出扫描补强 |

一句话：Toonflow 证明“内容组织和一致性资产”值得做，Pixelle 证明“真实生成和合成链路”必须扎实，waoowaoo 证明“前端流程要清楚”。最终实现只能落到本项目自己的架构、菜单、数据结构和 TODO。
