# 功能借鉴矩阵

> 这是防止产品做成“四不像”的核心账本。开发任何功能前，先看本表：该功能学谁、学什么、保留什么、按钮和状态不能少什么、对应哪个 TODO。
>
> 规则：参考项目只提供优点和避坑，本项目最终实现以 `doc/功能模块`、`doc/底层设计` 和 `plan/TODO` 为准。
> 机制细节先看 `参考项目深度机制与吸收口径.md`；本表只做模块映射，不能用一句“学某项目”替代字段、布局、状态和数据流设计。

## 一、总控矩阵

| 本项目模块 | MVP | 主要参考 | 必须吸收的优点 | 本项目必须保留 / 强化 | 禁止照搬 / 风险 | 对应 TODO |
|---|---|---|---|---|---|---|
| 我的作品 / 开始创作 | 是 | waoowaoo 流程、Toonflow 项目概念 | 入口简单、继续未完成、作品状态清晰 | 用户侧叫作品；点击开始创作直接进入工作台；Project 只做技术实体 | 不做传统空项目表单；不复制参考 UI | TODO-01、TODO-03 |
| 内容创作 / 内容导入 | 是 | waoowaoo 长文本检测与手动兜底、Pixelle generate/fixed、Toonflow 小说分章 | topic / paste / article 分流；长文本提示；切分预览；paste 不改写；手动确认段落 | `inputType` 和 `workflowType` 分离；内容创作产物只到 `sourceText / narrationText` 段落；自动保存草稿 | 不让脚本页成为当前主线必经；不把小说混入 MVP | TODO-01、TODO-03、TODO-09、TODO-12 |
| 分镜 | 是 | waoowaoo 镜头生产字段、Toonflow 总纲 + 镜头两层、Pixelle frame | 每行一个镜头；文本、画面、角色、场景、提示词、时长、状态、候选结果挂在镜头上；可锁定 / 重生成 | StoryboardItem 是主线最小工作单元；高级字段进入 Inspector 或展开列；表格负责主编辑 | 不做专业影视复杂分镜前置；不覆盖用户锁定内容；不做表格和 Inspector 双编辑 | TODO-02、TODO-03、TODO-05、TODO-09 |
| 生图提示词 | 是 | waoowaoo AI 修改提示词 / 场景、Pixelle 批量提示词、Toonflow 资产注入 | 提示词跟着分镜走；最终 prompt 可预览；批量生成、数量校验、重试、风格/角色/场景注入 | imagePrompt 可编辑、可锁定、可行内重生成；实际发送 prompt 由规则 + 镜头 + 资产 + 模型能力组装 | 不把大段 prompt 写进页面；不做孤立提示词大页；不让提示词和分镜错位 | TODO-03、TODO-06、TODO-09 |
| AI 生图 | 是 | Pixelle API/ComfyUI 双轨、Toonflow 多参考图 | 多候选、批量生成、失败重试、参考图一致性 | ImageCandidate[] + selectedImageId；远程结果转存本地 | 不只存单 imagePath；不按 modelName 写分支 | TODO-02、TODO-06、TODO-07 |
| AI 视频 | 是 | Pixelle i2v 能力矩阵、Toonflow VideoMode 语义 | 读取已选图、按能力校验、生成片段、多候选选择 | VideoSegment[] + selectedVideoSegmentId；VideoInputPlan | 不绕过 selectedImageId；text_to_video 不进 MVP 主线 | TODO-02、TODO-06、TODO-07 |
| 视频合成 | 是 | Pixelle FFmpeg VideoService | concat、转码兜底、ffprobe 检查、进度反馈 | Tauri/Rust sidecar；PathGuard；输出 final.mp4 | 不要求 TTS/字幕/BGM 才能合成 | TODO-07、TODO-09 |
| 导出 / 项目备份 | 是 | Toonflow 导入导出思路、自研安全 | 打开输出目录、项目包、历史可回看 | 不泄露密钥、绝对路径、未授权外部文件 | 不导出 keyring 内容；不把诊断包做成泄密入口 | TODO-09 |
| 设置 / 模型配置 | 是 | Toonflow-web 设置结构、Pixelle ConfigManager | Provider、API 模型能力、Workflow Preset、测试记录分层；测试入口 | keyring；provider_models；workflow_presets；dry_run/real_generate；模型能力不写在页面 | 不明文 key；不执行用户 TS Provider；不把模型能力塞进视频包 | TODO-04、TODO-06 |
| 创作规则 / Skill | 是 | Toonflow-app/web | builtin/user、规则集中管理、可编辑、绑定视频包 | MVP 只做可编辑和 schema；版本快照后置 | 不复制 Toonflow Skill 正文；不做可执行插件 | TODO-06、TODO-08、TODO-12 |
| 视频包 | 是 | Toonflow 内容策略包 | 包绑定规则、推荐模型能力、默认风格和基础参数 | 视频包只引用规则和默认配置，不复制规则正文；同一作品只能激活一套默认包，可在作品内覆盖 | 不把视频包做成模型配置、素材库或模板创建入口 | TODO-08 |
| 画风 / 角色 / 场景 | 增强 | Toonflow 资产一致性 | 画风库、角色/场景资产、多参考图、衍生图 | Bible + AssetReference + lockFlags | 不默认生成所有三视图；按 InputPlan 生成 | TODO-08 |
| 任务与历史 | 是 | Pixelle ProgressEvent、Toonflow 状态思路 | 进度、失败、取消、重试、历史 | 自研持久化队列、Step/Attempt/Artifact | 不用 fire-and-forget；不做纯内存任务 | TODO-05、TODO-09 |
| TTS | 后续 | Pixelle Edge TTS | 本地 Edge TTS 兜底、真实音频时长 | audioDurationSeconds 回填；可选启用 | 不照搬 Toonflow TTS 空壳；不阻塞 MVP 主线 | TODO-10 |
| 字幕 | 后续 | Pixelle HTML 渲染 | 模板渲染、分辨率目录、字幕帧 | 自研逐字/安全区/描边/换行；可选烧录 | 不启用字幕也能合成 final.mp4 | TODO-10 |
| 封面 | 后续 | Pixelle title + HTML 模板、Toonflow poster 概念 | 标题生成、模板封面、从已选图/首帧取图 | 多模板、智能选图、导出 cover.png | 不阻塞主视频导出 | TODO-10 |
| BGM / 模板动效 | 后续 | Pixelle FFmpeg + templates | BGM loop/volume、HTML 参数模板 | 淡入淡出、模板安全、PathGuard | 不变成当前主线前置 | TODO-10、TODO-12 |
| 素材成片 | 后续独立 | Pixelle asset_based | VLM 素材分析、素材派镜 | 独立 workflow material_edit；资产引用入库 | 不混入 image_to_video MVP | TODO-12 |
| 数字人口播 | 后续独立 | Pixelle digital_human | 人物图 + 声音 + 口播视频 | 独立 workflow digital_human | 不污染当前图生视频主线 | TODO-12 |
| 画布精修 | 后续增强 | Toonflow 图像流程 | 局部修图、流程节点、回写结果 | 精修产物作为新 ImageCandidate | 不覆盖原候选图 | TODO-12 |

## 二、页面按钮和操作账本

### 1. 我的作品

必须有：

```text
开始创作
打开作品
继续生成
搜索作品
按状态筛选
按更新时间排序
打开输出目录
删除 / 归档
```

必须显示：

```text
作品标题
封面或占位图
workflowType
当前阶段
分镜数量
生图进度
视频进度
合成状态
最后更新时间
失败原因摘要
```

参考：

```text
waoowaoo：线性短视频制作入口
Toonflow：项目概念和统计
本项目：作品卡片 + 工作台入口
```

验收：

```text
用户不需要先理解技术 Project。
开始创作后直接进入内容创作，不停在空表单。
失败作品能从失败步骤继续。
```


## 三、功能不能少的逻辑清单

| 逻辑 | 必须出现在哪里 | 参考来源 | 验收标准 |
|---|---|---|---|
| generate/fixed 双模式 | 内容创作 | Pixelle | paste 不改写；topic/article 调不同提示词 |
| 切分预览 | paste 文案 | Pixelle 基础上增强 | 用户能合并、拆分、删除段落 |
| 分镜锁定 | 分镜、提示词 | Toonflow 思路 + 自研 | 批量重生成跳过锁定字段 |
| 多候选保留 | 生图、视频 | Pixelle/Toonflow 均有候选思想 | 重生成不删除旧结果 |
| 选择最终产物 | 生图、视频 | waoowaoo 流程确认 | selectedImageId / selectedVideoSegmentId 必须写回 |
| 输入规划 | 生图、视频 | Pixelle 能力矩阵 + 自研 | required/optional/unused 明确，不需要的不生成 |
| 能力矩阵 | 模型/工作流 | Pixelle | 后端校验，不只前端禁用 |
| 参考图一致性 | 生图、角色/场景 | Toonflow | 按 Character/Environment/Style Bible 注入 |
| 远程转存 | 生图、视频、素材 | 自研安全 | URL 不作为最终主数据入库 |
| FFmpeg 兜底 | 合成 | Pixelle | 格式不一致能转码后合成 |
| 进度事件 | 生成、合成 | Pixelle ProgressEvent | 页面能看到当前步骤和失败原因 |
| 持久化任务 | 所有生成 | 自研 | 不能 fire-and-forget，不能纯内存丢任务 |
| keyring | 设置/Provider | 自研 | SQLite 只存 key_alias |
| 诊断脱敏 | 日志/导出 | 自研 | 不泄露 key、绝对路径、请求头 |

## 四、后续功能不能丢

这些不进 MVP 主线，但必须保留文档、入口规划和 TODO：

```text
TTS
字幕
封面
BGM
模板动效
数字人口播
素材成片
小说长内容
画布精修
首尾帧视频
尾帧续接
动作迁移
素材 VLM 分析
```

规则：

```text
1. 当前页面可以显示为后续能力或可选增强。
2. 不要在 MVP 删除字段和文档入口。
3. 不要让后续能力阻塞当前 image_to_video。
4. 后续能力接入时，仍要遵守候选、确认、任务、路径、安全规则。
```

## 五、开发前检查表

开发任何功能前先回答：

```text
1. 这个功能属于 MVP、增强，还是独立 workflow？
2. 主要参考哪个项目？
3. 参考项目里这个能力具体怎么实现？字段、页面区域、按钮、状态和数据流是什么？
4. 本项目筛选吸收什么，不吸收什么？
5. 参考项目里学的是流程、交互、工程链路，还是数据组织？
6. 哪些参考实现明确不能照搬？
7. 本项目已有的好设计必须保留什么？
8. 页面主按钮有哪些？
9. 行内按钮有哪些？
10. 状态有哪些？
11. 失败怎么恢复？
12. 旧结果是否保留？
13. 用户最终确认写回哪个字段？
14. 对应哪个 TODO？
```

如果这些问题答不出来，不能开始写代码。
