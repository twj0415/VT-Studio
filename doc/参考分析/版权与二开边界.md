# 版权与二开边界

> 本项目可以学习参考项目的产品思路、工程思想和功能取舍，但不能复制代码、UI、品牌、素材和专有内容。本文是后续二开和实现时的边界。

---

## 一、允许借鉴

允许借鉴的是抽象层面的设计思想：

```text
1. 生产流程：输入 → 分镜 → 生图 → 视频 → 合成。
2. 信息架构：作品、创作资源、模型/工作流、系统设置。
3. 通用交互：表格工作台、多候选选择、失败重试、缺项提示。
4. 工程模式：ProviderManager、能力矩阵、工作流预设、任务进度。
5. 功能思想：Skill 可编辑、角色一致性、素材分析、FFmpeg 合成。
6. 安全经验：不明文存密钥、路径受控、日志脱敏。
```

这些属于通用产品和工程思路，可以用本项目自己的代码、命名、视觉和数据结构重新实现。

---

## 二、禁止照搬

禁止：

```text
1. 复制参考项目源码文件、函数实现、组件实现。
2. 复制参考项目 UI 视觉、布局细节、图标组合、品牌色和 Logo。
3. 复制参考项目品牌名、菜单名、宣传文案、示例素材。
4. 复制 Toonflow 的 Skill 内容、内置画风包、故事包正文。
5. 复制 waoowaoo 的页面代码、组件结构和样式。
6. 复制 Pixelle 的 Python 实现到本项目作为业务代码。
7. 复制闭源或未开放后端的接口协议，假装能二开。
8. 把参考项目许可证不明确的资源打进本项目。
```

---

## 三、实现方式要求

本项目必须用自己的实现：

```text
前端：Vue3 + TypeScript + Pinia + Naive UI + Tailwind
桌面：Tauri 2
后端：Rust
数据：SQLite
媒体：FFmpeg sidecar
模板：受控 Chromium / HTML 模板
模型：ProviderManager + provider_models + workflow_presets
安全：keyring + PathGuard + SecretGuard
```

允许看参考项目源码理解流程，但落地时必须转成：

```text
本项目自己的 DTO
本项目自己的数据库字段
本项目自己的组件
本项目自己的样式系统
本项目自己的错误码
本项目自己的任务状态
```

## 四、Toonflow 相关边界

Toonflow 可学习：

```text
1. 创作规则 / Skill 管理思路。
2. 画风、角色、场景、道具资产组织。
3. 小说分章、事件提取、长内容 Agent 思路。
4. 桌面端创作资源入口。
```

Toonflow 不可照搬：

```text
1. 品牌名 Toonflow。
2. Electron + Express 代码。
3. 内置 skills 具体内容。
4. vm2 执行用户 Provider 代码。
5. 明文密钥方案。
6. UI 视觉和菜单细节。
```

## 七、二开验收边界

判断是否有版权风险，看这几个问题：

```text
1. 是否复制了参考项目源码？
2. 是否复制了参考项目 UI 视觉？
3. 是否复制了参考项目品牌或文案？
4. 是否复制了参考项目内置素材或 Skill 正文？
5. 是否保留了参考项目接口和数据结构命名？
6. 是否让用户误以为这是参考项目的改版？
```

如果任一项是“是”，需要重做。

安全做法：

```text
1. 功能可以同类，但实现必须自写。
2. 流程可以借鉴，但页面视觉必须自有。
3. 数据结构可以表达同一业务，但字段命名和边界以本项目文档为准。
4. 参考项目只出现在文档“参考来源”里，不出现在应用品牌和 UI 里。
```
