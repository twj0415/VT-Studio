# MVP 简化口径

> 本文用于防止第一版被后期架构拖慢。简化不是删功能，而是明确哪些是当前必须落地，哪些是保留设计、后续按 TODO 接入。

---

## 一、当前 MVP 目标

当前 MVP 只验收这条主线：

```text
输入文字
→ 分镜
→ 生图提示词
→ AI 生图
→ 选择最终图
→ 图生视频
→ 选择最终视频片段
→ 合成 final.mp4
→ 导出 / 打开输出目录
```

必须满足：

```text
1. 用户能编辑分镜。
2. 用户能编辑每行 imagePrompt / videoPrompt。
3. 每行能生成多张候选图。
4. 每行必须选择一张最终图才能进入视频阶段。
5. 每行能生成一个或多个视频片段。
6. 每行必须选择一个最终视频片段才能合成。
7. 合成必须生成 final.mp4。
8. 失败能定位到具体行和具体步骤。
9. Mock 和真实生成必须明确区分。
```

## 二、MVP 必做和暂缓

| 模块 | MVP 必做 | MVP 暂缓但保留 |
|---|---|---|
| 内容导入 | topic / paste / article；paste 固定文案不改写；创建作品草稿 | 小说整本分章、素材成片入口 |
| 分镜 | StoryboardItem 可编辑、排序、锁定、重生成、校验 | 专业导演分镜、多 Agent 长剧规划 |
| 生图提示词 | 批量生成、行内编辑、锁定、数量校验 | 高级提示词版本系统、复杂 Prompt 评估 |
| AI 生图 | 多候选图、已选图、重生成保留旧图、失败重试 | 角色三视图全自动批量生成、复杂控制图体系 |
| AI 视频 | image_to_video、读取 selectedImageId、生成 VideoSegment、选择最终片段 | text_to_video、首尾帧、动作迁移、尾帧续接 |
| 合成 | 拼接 selectedVideoSegmentId 输出 final.mp4 | 字幕烧录、BGM 混音、封面生成、模板包装 |
| 创作规则 | builtin/user、内置复制为用户规则、当前启用规则、输出 schema 校验 | SkillVersion、content_hash、schema_hash、PackSnapshot、SkillSnapshot 完整复现 |
| 密钥 | keyring 底座、真实 Key 不进 SQLite/日志/导出 | 复杂密钥轮换、团队密钥管理 |
| 任务 | 当前任务状态、失败记录、可重试 | 完整断点续跑、死信队列、任务租约、多进程恢复 |
| 模型配置 | Provider、provider_models、workflow_presets、连通性测试 | 模板市场式 workflow 分发 |

## 三、创作规则 MVP 口径

用户侧统一叫：

```text
创作规则
```

技术侧可以叫：

```text
PromptSkill
SkillDefinition
CreativeRule
```

MVP 只做：

```text
1. 内置规则 builtin：应用内置，不直接编辑。
2. 用户规则 user：用户复制内置规则后编辑。
3. 当前启用规则：视频包或项目可以引用当前启用规则。
4. 输出 schema：核心 LLM 输出必须 JSON parse + schema validate。
5. 规则集中存放：不能散落在 Vue 页面或 Rust service 字符串里。
```

MVP 不强制：

```text
1. 每次编辑都生成 SkillVersion。
2. 每次任务都生成完整 SkillSnapshot。
3. 计算 content_hash / schema_hash。
4. 历史任务完全可复现。
5. 视频包版本历史。
```

保留后续升级入口：

```text
规则表可以预留 version 字段。
任务表可以预留 snapshot_json 字段。
文档可以保留 SkillSnapshot 设计，但标记为后续增强。
```

## 四、密钥 MVP 口径

密钥安全不是本阶段功能亮点，但它是安全底线。

MVP 必做：

```text
1. 真实 API Key 只进入系统 keyring。
2. SQLite 只保存 key_alias。
3. 导出、备份、诊断包不包含真实 Key。
4. 日志不打印 API Key、Bearer Token、完整请求头。
5. Provider 配置保存前拒绝 secret-like 字段进入 config_json。
```

MVP 不做：

```text
1. 自己实现加密算法。
2. 密钥分享。
3. 密钥轮换策略。
4. 团队权限。
```

## 五、TTS / 字幕 / 封面 / BGM 口径

这些功能一个也不删，但不阻塞第一条 image_to_video 主线。

| 功能 | 当前处理 | 后续接入 |
|---|---|---|
| TTS | 不作为生图/视频前置 | TODO-10 接入 Edge TTS / API TTS，回填 audioDurationSeconds |
| 字幕 | 不阻塞合成 | TODO-10 接入字幕断句、样式、安全区、可选烧录 |
| 封面 | 不阻塞 final.mp4 | TODO-10 接入封面模板、标题生成、从已选图/视频首帧生成 |
| BGM | 不阻塞主线合成 | TODO-10 接入 BGM 选择、音量、loop、淡入淡出 |
| 模板动效 | 不作为当前必经步骤 | TODO-10/12 接入 image_slideshow 或包装模板 |

## 六、页面和按钮 MVP 口径

每个主线页面必须有这些操作能力：

| 页面 | 必须有的主按钮 | 必须有的辅助按钮 |
|---|---|---|
| 我的作品 | 开始创作、打开作品、继续生成 | 搜索、筛选、删除/归档、打开输出目录 |
| 内容创作 | 创建并进入分镜、保存草稿 | 选择视频包、设置画幅/时长/分镜数、切换输入来源 |
| 分镜 | 生成分镜、重新生成未锁定、确认分镜、进入生图 | 新增、删除、复制、上移/下移、拆分、合并、锁定、单条重生成 |
| 生图 | 批量生成全部、只生成未生成、只生成失败项、进入视频 | 编辑提示词、单行生成、重新生成、选择候选图、确认图片、查看大图 |
| 视频 | 批量生成全部、只生成未生成、只生成失败项、进入合成 | 编辑视频提示词、单行生成、重新生成、选择片段、确认片段、预览视频 |
| 合成 | 开始合成、重新合成、打开输出目录 | 检查片段、查看错误、预览 final.mp4 |
| 设置/模型 | 保存配置、测试连接、测试模型、扫描工作流 | 新增 Provider、新增模型、新增 workflow preset、启用/禁用、删除 |

按钮可以根据状态禁用，但必须显示缺项原因，不能静默不可点。

## 七、验收方式

开发时不能只问“页面有没有”。必须问：

```text
1. 入口有没有？
2. 主按钮有没有？
3. 辅助按钮有没有？
4. 状态有没有？
5. 缺项提示有没有？
6. 失败重试有没有？
7. 旧结果是否保留？
8. 用户选择是否写回？
9. 下一步是否校验前置条件？
10. 是否对应当前 TODO？
```
