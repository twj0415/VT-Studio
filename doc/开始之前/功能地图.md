## 一、当前主线地图

当前优先跑通的主 workflow：

```text
workflowType = image_to_video
```

当前产品入口：

```text
我的作品：管理作品、开始创作、打开作品工作台
AI 工具：独立小工具，只生成可复制 / 可保存的辅助结果，不创建作品
创作资源：管理视频包、创作规则、素材库
模型 / 工作流：管理 Provider、API 模型能力、ComfyUI / RunningHub 工作流
系统设置：主题、路径、密钥安全、默认值、日志诊断
```

当前作品生产主线：

```text
我的作品 / 开始创作
→ 作品工作台：内容创作
→ 分镜
→ 生图
→ 视频
→ 合成
→ 导出 / 历史
```

更细的完整能力链路：

```text
内容创作中的 AI 生成 / 整理
→ 开始创作
→ 导入 / 确认剧本
→ 提取角色 / 场景 / 风格 / 道具
→ 分镜
→ 判断生图需要什么资源
→ 按需生图
→ 判断生成视频需要什么资源
→ 生成视频
→ 字幕 / 独白 / 配音
→ 合成
→ 导出 / 历史回看
```

这条完整能力链路不是当前必经 StepBar。它的作用是说明数据和资源的先后关系，避免后续增强时漏掉角色、场景、TTS、字幕等能力：

```text
1. 这里的 AI 生成 / 整理属于内容创作能力，不是一级菜单“AI 工具”；它负责生成可用剧本，不直接生成视频。
2. 开始创作负责创建作品草稿、导入 / 确认剧本和设置基础信息。
3. 设定集负责提取角色、场景、风格、道具等可复用信息。
4. 分镜只引用角色 ID、场景 ID、道具 ID，不复制整份角色或场景设定。
5. 生图前先看当前选择的图片模型 / ComfyUI workflow 需要什么资源。
6. 生成视频前也先看当前选择的视频模型 / workflow 需要什么资源。
7. 模型需要什么就生成什么；模型不需要的资源不生成、不入库、不污染项目。
8. 独白 / 旁白优先落在 StoryboardItem.narrationText，不是单独再发明一套正文。
9. 字幕优先从 narrationText 或 sourceText 断句生成；启用 TTS 后，可用真实音频时长校准字幕时间轴。
10. TTS 只是把独白 / 旁白转成音频，未启用 TTS 时，当前 image_to_video 主线仍能生图、生成视频、合成。
11. 启用 TTS 且选择“音频驱动时长”时，必须用真实 audioDurationSeconds 反向校准分镜 / 视频片段时长。
12. 导出负责把 final.mp4、可选字幕、可选封面、作品包和历史记录交给用户，不负责重新生成内容。
```

边界：

```text
用户侧叫“作品”，技术侧可以继续叫 `Project`。
点击“开始创作”会自动创建作品草稿，并进入作品工作台。
内容创作是作品工作台第一步，不是独立一级菜单。
任务系统贯穿作品内部，不做全局任务中心入口。
素材库只放可复用资产，不默认收纳作品产物。
角色、场景、风格等文字设定由“创作规则 / Skill”输出并做 schema 校验；MVP 先做 builtin/user 可编辑和 schema 校验，完整版本化后续增强。
角色图、场景图、道具图、表情图、动作/姿态参考图等图片资源按模型 / workflow 的输入规划按需生成，不默认全量生成。
角色动作、表情、心情、服装变化、正面 / 侧面 / 背面方位图都属于角色资源包；它们不是单独的主流程步骤。
数据模板可以统一保存，但生成行为必须由 ImageInputPlan / VideoInputPlan 裁剪：required 才必须补，optional 由用户决定，unused 不生成。
```

## 五、参考分析怎么读

| 文档 | 用途 |
|---|---|
| `参考分析/参考项目取舍总则.md` | 规定参考项目只作为能力来源，不作为完整蓝图 |
| `参考分析/参考方法与防漏机制.md` | 规定如何拆参考项目，防止少按钮、少状态、少异常恢复、少数据写回 |
| `参考分析/功能借鉴矩阵.md` | 开发前必读的功能账本，防止少功能、少按钮、少状态 |
| `参考分析/页面交互账本.md` | 页面级交互账本，规定每个页面区域、按钮、状态、抽屉和参考来源 |
| `参考分析/MVP简化口径.md` | 明确第一版必须做什么，哪些后续增强不能删但不阻塞主线 |
| `参考分析/版权与二开边界.md` | 规定二开边界，不复制代码、UI、品牌、素材和 Skill 正文 |
| `参考分析/功能对比总表.md` | 历史功能对照稿，只作为证据补充 |
| `参考分析/两个项目怎么做的.md` | 源码证据索引，写具体功能时按需查文件/函数 |

写每篇功能文档或开始开发前，必须做五步：

```text
1. 先看参考项目取舍总则，确认该参考项目能学什么、不能学什么。
2. 再看参考方法与防漏机制，按七层拆解检查用户目标、页面、按钮、状态、数据、异常和本项目增强。
3. 再看功能借鉴矩阵，确认功能、按钮、状态、逻辑和 TODO 映射。
4. 涉及页面交互时，必须看页面交互账本。
5. 如需确认参考项目能力，再看功能对比总表和两个项目怎么做的。
6. 最后按需打开参考项目源码确认文件/函数还存在；不能把源码结构当本项目实现结构。
```

不能只凭记忆写。

## 六、模块之间的数据流

当前 `image_to_video` 主线数据流：

```text
开始创作 自动创建作品草稿（技术 Project / ProjectBible）
内容创作 输出 ScriptDraft / CleanNarrationList，并写入当前作品 StoryboardItem.sourceText / narrationText
设定集抽取 输出 StyleBible / CharacterBible / EnvironmentBible / PropBible 草稿，用户确认后入库
分镜 输出 StoryboardItem[]
生图提示词 写入 StoryboardItem.imagePrompt / negativePrompt
生图输入规划 根据 image_kind、模型能力、workflow preset、Bible 和分镜，返回 required / optional / unused
AI 生图 按需输出 ImageCandidate[]、角色参考图、场景参考图、风格参考图、道具图、尾帧图、控制图或封面图
分镜图选择 写入 StoryboardItem.selectedImageId
视频输入规划 根据 selectedImageId、VideoInputPlan、模型能力和 workflow preset，返回 required / optional / unused
AI 视频读取 selectedImageId 和 VideoInputPlan，输出 VideoSegment[]，并写入 StoryboardItem.selectedVideoSegmentId
视频合成读取 selectedVideoSegmentId，输出 CompositionTask / exports/final.mp4
任务与历史 写入 Task / TaskStep / History，显示在作品卡片和作品工作台内部
模型 / 工作流 提供 Provider、模型能力、workflow preset
系统设置 提供路径、默认值、密钥安全、诊断配置
创作资源 / 素材库 维护可复用 Asset / AssetReference
导出与项目备份 输出 final.mp4 / 可选 cover.png / 可选 subtitles / project_package.zip
```

后续增强数据流：

```text
脚本生成 可输出 script / narrations，作为分镜输入之一
TTS 可把 narrationText 转为 audioPath / audioDurationSeconds；启用音频驱动时可更新 durationSeconds
字幕 可从 narrationText / sourceText 生成 subtitleChunks / subtitles.json；有 TTS 时使用 audioDurationSeconds 做时间轴
封面 可写入 coverPath
模板与动效 可写入 renderedFramePath 或 slideshow segment
```

## 八、不要混在一起的边界

1. 用户侧叫“作品”，技术侧可以继续叫 `Project`。
2. 不做独立“新建项目”表单；点击“开始创作”直接进入作品工作台。
3. 内容创作是作品工作台第一步，负责文案生成、清洗、提炼、改编，并写入当前作品。
4. 角色、场景、风格、道具的文字设定由创作规则 / Skill 生成，并由 output_schema 约束；字段要求不是写死在页面里。
5. 用户可以编辑创作规则；MVP 阶段编辑内置规则时复制成用户规则，新规则只影响新任务。完整 SkillSnapshot / hash / 历史复现属于后续增强，不作为第一版强制门槛。
6. 分镜负责把输入文字拆成可生图、可图生视频的工作单元。
7. 分镜只引用角色、场景、道具 ID，不复制整份设定。
8. 生图提示词可以作为独立能力，也可以并入生图表格行内编辑。
9. AI 生图不只生成分镜图，还可生成角色参考图、场景参考图、风格参考图、道具图、尾帧图、控制图、封面图。
10. 是否生成某类图片由 ImageInputPlan 决定；required 必须补，optional 由用户选择，unused 不生成。
11. 角色正面图、侧面图、背面图、面部特写、表情表、服装细节、动作/姿态参考都属于角色资源包，不是独立主流程。
12. AI 视频中的 `image_to_video` 是当前主线；文生视频、首尾帧视频是扩展，但输入差异必须由 VideoInputPlan 处理。
13. 不同 API 模型、ComfyUI workflow、RunningHub workflow 需要的输入不同，前端不能按固定表单写死。
14. TTS、字幕、封面、模板动效是 TODO 中的后续增强，不是取消。
15. 视频合成优先拼接已确认视频片段，BGM/字幕/封面可后续增强。
16. 任务系统贯穿作品内部，但不做全局任务中心入口。
17. AI 工具是一级菜单，但只放爆款开头、标题生成、角色设定、多语言改写、画面描述优化等独立小工具；不能提供“创建作品 / AI 内容创作 / 输入主题生成完整视频”入口。
18. 创作资源管理视频包、创作规则和素材库；作品产物默认放作品文件夹，不自动进入素材库。
19. 模型 / 工作流管理 Provider、API 模型能力、ComfyUI / RunningHub；系统设置只管理路径、主题、安全、默认值和诊断。

边界清楚，AI 开发才不容易写成一坨。
