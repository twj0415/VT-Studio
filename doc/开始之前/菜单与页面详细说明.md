# 菜单与页面详细说明

> 本文定义最终有哪些菜单、每个菜单下面有哪些页面、每个页面有什么功能、按钮、状态和核心逻辑。后续 UI 优化和 TODO 重排都要对齐本文。

---

## 一、一级菜单总表

| 一级菜单 | 用户理解 | 页面 | 核心参考 |
|---|---|---|---|
| 我的作品 | 我正在做哪些视频 | 作品列表、作品详情、最近任务、输出记录 | waoowaoo 流程、Toonflow 项目概念 |
| AI 工具 | 我临时需要一个小 AI 辅助 | 爆款开头、标题生成、角色设定、多语言改写、画面描述优化 | waoowaoo / Toonflow 的 AI 辅助入口思路 |
| 创作资源 | 我用什么规则、风格和素材创作 | 视频包、创作规则、素材库 | Toonflow-app / Toonflow-web |
| 模型 / 工作流 | 我接入哪些模型和 workflow | Provider、API 模型能力、Workflow Preset、测试记录 | Toonflow-web、Pixelle |
| 系统设置 | 软件怎么运行和保护数据 | 应用偏好、路径 Sidecar、密钥安全、日志诊断、配置导入导出 | 本项目自研 |

作品工作台从“我的作品”进入，不作为一级菜单。

AI 工具不能包含“创建作品 / AI 内容创作 / 输入主题生成完整视频”这类入口；这些能力属于 `我的作品 → 开始创作 → 内容创作`。AI 工具只提供复制结果、保存历史或保存为素材，不直接创建新作品。

### 本文施工门禁

后续写页面或接口时，不能只按本文件的页面名自由发挥。凡是命中下列能力，必须同时按 `参考分析/参考项目深度机制与吸收口径.md` 第十一节和 `参考分析/页面交互账本.md` 复核：

```text
内容导入：必须能落到长文本提示、切分预览、段落编辑、手动确认、写入 StoryboardItem。
分镜：必须能落到文本层、镜头层、生产层、高级层、结果层，以及表格主编辑 + Inspector 辅助。
候选图 / 视频：必须能落到候选缩略图、当前已选、确认、取消、撤回、清理未选、保存为素材、任务状态。
视频包 / 创作规则 / 模型工作流：必须能落到引用链，不允许把它们混成一个 JSON。
任务态：必须能落到 Task / TaskStep 权威记录、ProgressEvent 实时展示、已用时和失败恢复，不显示 ETA。
资产贯穿：必须区分作品产物、候选产物、可复用 Asset、AssetReference。
```

如果当前代码还没有完整实现，页面只能显示“未接入 / 占位 / 待配置”，不能伪装成真实可用。

## 二、我的作品

### 页面目标

让用户看到所有作品，能开始新作品、继续未完成作品、打开输出结果、查看失败原因。

### 页面区域

```text
顶部工具栏：开始创作、搜索、状态筛选、排序
主体：作品卡片 / 紧凑列表
右侧抽屉：作品详情、最近任务、输出记录、失败信息
底部状态区：工作区、任务运行摘要
```

### 必须显示

```text
作品标题
封面或占位图
workflowType
当前阶段
分镜数量
生图进度
视频进度
合成状态
最近更新时间
失败摘要
资源消耗摘要：图片次数 / 视频次数 / LLM 次数（有统计时显示）
```

### 操作按钮

```text
开始创作
打开作品
继续生成
打开输出目录
复制作品
归档
删除
```

### 状态

```text
draft
storyboard_ready
image_running
image_failed
video_running
video_failed
composition_succeeded
composition_failed
exported
```

### 核心逻辑

```text
1. 开始创作立即创建 Project draft，并进入内容创作页。
2. 继续生成必须定位到失败或未完成步骤。
3. 已合成作品显示打开输出目录。
4. 删除作品必须二次确认，并说明是否删除媒体文件。
5. 作品卡片不展示技术字段，技术侧 Project 用户侧叫作品。
6. 资源消耗摘要从 Task / Artifact / Provider 调用记录聚合；没有真实统计时显示“未接入统计”，不能写假次数。
```

### 参考与增强

```text
借鉴 waoowaoo：线性短视频制作入口。
借鉴 Toonflow：项目概念和统计。
本项目增强：失败作品能从失败步骤继续，作品卡片直接显示恢复动作。
```

## 三、AI 工具

### 页面目标

提供不依赖当前作品的小型 AI 能力，减少用户在主流程之外临时写标题、开头、角色设定或改写文案的成本。

### 页面区域

```text
顶部：工具分类、搜索、最近使用
主体：工具卡片网格
右侧 / 详情页：输入区、参数区、结果区、历史区
```

### 工具清单

```text
爆款开头生成：输入主题，生成多种开头，结果可复制。
标题生成器：输入内容摘要，生成多个标题，结果可复制。
角色设定生成：输入角色描述，生成完整人设，可保存为素材或复制。
多语言改写：粘贴文案，改写为目标语言。
画面描述优化：粘贴画面描述，优化为更适合分镜 / 生图的描述。
```

### 操作按钮

```text
开始使用
生成
重新生成
复制结果
保存到工具历史
保存为素材（仅角色设定 / 画面描述类结果）
清空输入
返回工具列表
```

### 状态

```text
available
coming_soon
running
failed
succeeded
empty_history
```

### 核心逻辑

```text
1. AI 工具调用仍走 ProviderManager，不在页面直连模型。
2. 工具结果默认不写入作品，不创建 Project，不创建 StoryboardItem。
3. 用户选择“保存为素材”时，结果进入素材库或创作资源，不反向插入当前作品。
4. 工具历史只保存输入摘要、结果、模型来源和时间，不保存真实密钥。
5. 高成本或真实生成类工具仍必须遵守 dry_run / real_generate 的确认规则。
```

### 禁止事项

```text
1. 不提供“创建为新作品”主按钮。
2. 不重复主流程的 topic → 文案 → 分镜能力。
3. 不把工具结果自动写入某个正在编辑的作品。
4. 不把 prompt 正文写死在页面里。
```

### 参考与增强

```text
借鉴参考项目：AI 辅助能力贴近上下文，结果可复制 / 修改。
本项目增强：把独立小工具和作品主流程彻底分开，避免两个入口做同一件事。
```

## 四、作品工作台：内容创作

### 页面目标

把用户输入变成可分镜的内容草稿，并设置当前作品的基础参数。

### 页面区域

```text
左侧：输入来源 Tabs
中间：输入编辑区 / AI 结果预览 / 切分预览
右侧：视频包、画幅、目标时长、分镜数、内容语言
底部固定操作条：保存草稿、创建并进入分镜
```

### 输入来源

```text

文章 / 笔记
小说 / 长内容（后续）
素材说明（后续）
```

### 操作按钮

```text
保存草稿
AI 生成 / 整理
重新生成
确认结果
切分预览
编辑段落
合并段落
拆分段落
删除段落
移动段落
恢复原文
导入文件
创建并进入分镜
```

### 表单控件

```text
视频包选择器
画幅选择器
目标时长
目标分镜数
内容语言
切分方式：按段落 / 按行 / 按句
长文本风险提示
段落计数 / 字数统计
当前创作规则入口
切分预览列表
段落锁定 / 解锁
```

### 核心逻辑

```text
1. topic 走 AI 发散生成。
2. paste 走 fixed 模式，只机械切分，不改写。
3. article 走 AI 浓缩提炼。
4. paste 切分后必须给用户预览，可合并、拆分、删除。
5. 确认后写入 StoryboardItem.sourceText / narrationText。
6. 小说和素材入口保留，但不阻塞当前 MVP。
7. 长文本先提示风险，再让用户选择按段落 / 按行 / 按句切分；AI 整理失败时仍能手动确认段落。
8. 内容创作页承担逐条旁白 / 文案段落确认；不再让 script-editor 成为主流程必经步骤。
9. 创建并进入分镜前必须有至少一条已确认段落。
```

### 切分预览施工细节

参考 waoowaoo 的长文本检测和 SmartImportWizard，当前页面不是只放一个文本框，而是必须把“输入内容变成可确认段落”做完整：

```text
长文本提示：
  触发条件：字数、段落数或估算分镜数超过阈值。
  显示内容：风险原因、建议切分方式、预计段落数、是否继续。
  操作：智能切分、按段落切分、按行切分、按句切分、继续手动编辑。

切分预览列表：
  每行显示序号、段落正文、字数、建议时长、锁定状态、校验问题。
  行内操作：编辑、合并上一段、合并下一段、拆分、删除、上移、下移、恢复原文、锁定。
  批量操作：全部确认、仅确认有效段落、重新切分、恢复全部原文。

确认写回：
  确认前只存在输入草稿和切分预览，不进入 StoryboardItem 主表。
  确认后按顺序写 StoryboardItem.sourceText / narrationText。
  topic/article 模式的 AI 结果也必须先进入切分预览，用户确认后才能进入分镜。
```

兜底规则：

```text
AI 整理失败：保留用户原文和机械切分结果，允许手动确认进入分镜。
切分过细：提示可能导致视频碎片化，但不强行阻止。
切分过粗：提示可能导致单镜头信息过多，建议拆分。
用户编辑段落后：再次 AI 整理不得覆盖锁定段落。
```

### 数据写回

```text
Project.workflowType
Project.inputType
Project.inputProcessMode
Project.sourceText / sourceTextPath
Project.targetSceneCount
Project.segmentDurationSeconds
StoryboardItem.sourceText
StoryboardItem.narrationText
```

### 必须处理的状态

```text
empty_input
too_long
needs_split_preview
splitting
ai_generating
ai_failed
ready_to_confirm
confirmed
draft_saved
```

### 参考与增强

```text
借鉴 Pixelle：generate / fixed 双模式。
借鉴 waoowaoo：LongTextDetectionPrompt 先提示长文本风险，SmartImportWizard 把来源、解析、映射、确认拆开。
借鉴 Toonflow-web：长内容和脚本入口有批量添加、编辑、解析工具，但不把长内容全流程塞进当前短视频主线。
本项目增强：切分预览和可编辑确认，避免 Pixelle 切完直接生成；筛选吸收长文本检测、切分建议和手动兜底，不吸收独立 script-editor 必经页。
```

### 弹窗 / 抽屉

```text

```

### 状态

```text
draft
invalid
needs_review
confirmed
locked
```

行级状态还必须能表达：

```text
image_not_generated
image_waiting_select
video_not_generated
video_waiting_select
downstream_dirty
```

这些不是分镜主状态，而是提示当前字段是否影响后续产物。

### 核心逻辑

```text
1. 每条分镜必须有 sourceText 或 narrationText。
2. 进入生图前必须有 visualDescription、imagePrompt、durationSeconds。
3. 角色和场景优先引用 ID，没有设定集时先用文本描述。
4. lockFlagsJson 是唯一锁定来源。
5. 批量重生成不得覆盖锁定字段。
6. 删除、合并、拆分必须提示对候选图和视频片段的影响。
7. 景别、运镜、转场是高级可选字段，不阻塞主线。
8. 长分镜必须支持行号快速跳转，后续可加搜索和状态筛选。
9. Inspector 不允许和表格同时编辑 sourceText、visualDescription、imagePrompt、videoPrompt 等主字段。
10. AI 助手只能改未锁定字段；锁定字段必须由用户显式解锁。
```

### 数据写回

```text
StoryboardItem.sourceText
StoryboardItem.narrationText
StoryboardItem.visualDescription
StoryboardItem.charactersJson / characterIdsJson
StoryboardItem.sceneDescription / locationId
StoryboardItem.imagePrompt
StoryboardItem.negativePrompt
StoryboardItem.videoPrompt
StoryboardItem.durationSeconds
StoryboardItem.lockFlagsJson
StoryboardItem.shotSize / cameraMotion / transitionType（高级可选）
StoryboardItem.selectedImageId / selectedVideoSegmentId（只读展示）
```

### 分镜字段分层施工细节

后续实现不能只写“分镜字段很多”。每个字段必须落在固定层级，UI 也按层级组织：

```text
文本层：
  sourceText、narrationText、subtitleChunks（后续）
  来源：内容创作确认后的段落。
  主编辑：表格。

镜头层：
  visualDescription、charactersJson / characterIdsJson、sceneDescription / locationId、propsJson
  来源：AI 分镜或用户行内编辑。
  主编辑：表格。

生产层：
  imagePrompt、negativePrompt、videoPrompt、durationSeconds
  来源：分镜生成、AI 助手、用户编辑、创作规则。
  主编辑：表格；最终 prompt 只在生图页预览。

高级层：
  shotSize、cameraMotion、composition、pace、transitionType
  photographyRules / actingNotes 只作为 Inspector 建议或后续 advancedDirectingJson 扩展，未落库前不写临时字段
  来源：参考 waoowaoo 的摄影规则 / 表演指导和 Toonflow 的属性面板。
  主编辑：Inspector 或展开列，不挤占首屏主表格。

结果层：
  selectedImageId、selectedVideoSegmentId、image_status、video_status、downstream_dirty
  来源：生图 / 视频 / 任务系统。
  分镜页只读展示，不在分镜页选择候选。
```

Inspector 固定职责：

```text
可以放：当前行校验问题、AI 助手建议、候选图 / 视频只读预览、高级字段、下游影响提示。
不能放：sourceText、visualDescription、imagePrompt、videoPrompt 等主字段的第二套编辑器。
AI 助手应用建议时，必须回写表格字段，并检查 lockFlagsJson。
```

影响提示：

```text
删除行：提示会影响该行 ImageCandidate[] / VideoSegment[] / selectedImageId / selectedVideoSegmentId。
拆分行：提示原候选和视频片段不会自动复制到新行。
合并行：提示需要重新确认 imagePrompt / videoPrompt，且下游产物可能变脏。
移动行：不删除候选，但合成顺序会按新 StoryboardItem.index 变化。
批量重生成：跳过锁定字段，只更新未锁定字段。
```

### 参考与增强

```text
借鉴 waoowaoo：分镜卡包含 description、characters、location、shot_type、camera_move、source_text、video_prompt、photographyRules、actingNotes、candidateImages、imageUrl。
借鉴 Toonflow：script → scriptPlan → storyboardTable → storyboard → workbench 的总纲 + 镜头两层思路。
借鉴 Pixelle：线性 StoryboardFrame 执行模型。
本项目增强：MVP 用一个表格承接总览和主编辑，用 Inspector 承接辅助能力；不做两套主编辑界面，不让表格和 Inspector 双编辑。
```

### 核心逻辑

```text
1. 选择模型后必须请求 ImageInputPlan。
2. required 缺失不能生成。
3. optional 由用户选择是否补充。
4. unused 不展示为缺项，不生成，不入库。
5. 每行默认支持多张候选图。
6. 重生成追加新候选，不删除旧候选。

未注册或 disabled preset 不能进入生图 / 视频选择器。
```

### 测试记录

页面区域：

```text
顶部：按类型、结果、时间筛选
表格：时间、测试对象、模式 dry_run / real_generate、结果、耗时、traceId、错误码
右侧：脱敏请求摘要、脱敏响应摘要、恢复建议
```

规则：

```text
保存配置不自动真实生成。
dry_run 默认可用。
real_generate 必须二次确认并提示可能产生费用。
测试记录不保存密钥、请求头和绝对路径。
```

核心逻辑：

```text
Provider 只保存连接信息。
API 模型能力进 provider_models。
ComfyUI / RunningHub 进 workflow_presets。
真实 Key 只进 keyring。
保存配置不自动真实生成。
高成本 real_generate 必须二次确认。
测试记录只保存脱敏摘要、耗时、结果、错误码和 traceId。
```

### 分层引用链

后续实现模型选择器、视频包、创作规则和任务执行时，必须按这条链路施工：

```text
Provider：
  保存 providerKind、vendor、baseUrl、authType、keyAlias、enabled。
  只解决“连到谁、怎么鉴权”。

ProviderModel：
  保存某个 API 模型的 modelName、能力、输入输出模态、限制和 featureFlags。
  只解决“这个 API 模型能做什么”。

WorkflowPreset：
  保存 ComfyUI / RunningHub / 本地 workflow 的 workflowKey、workflowId、paramSchema、nodeMap、outputMap、limits。
  只解决“这个 workflow 怎么执行”。

CreativeRule：
  保存 prompt_body、output_schema、params_schema 和校验规则。
  只解决“怎么生成文本 / 结构化输入”。

VideoPack：
  保存默认参数、CreativeRule 引用、ProviderModel / WorkflowPreset 推荐引用、AssetReference。
  只解决“默认用哪套策略开始作品”。

ProjectRuntimeConfig：
  保存当前作品从视频包复制来的引用和用户覆盖项。
  任务运行时再把实际选择写入 Task / TaskStep 快照。
```

业务页选择模型时不直接拼 provider_models 和 workflow_presets 表，而是读取后端聚合后的可执行选项，例如 `list_executable_media_options`。这样页面只知道“可用于生图 / 视频的选项”，不需要理解 Provider 连接细节。
