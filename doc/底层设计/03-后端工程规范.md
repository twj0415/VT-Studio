# 后端工程规范

> 这篇定义 Tauri Rust 后端怎么分层、怎么封装服务、怎么组织 Pipeline、Provider、Storage 和错误。功能模块不得绕过这些规则。

## 一、后端职责

Rust 后端负责：

```text
生产线调度
任务状态机
Provider 调用
文件读写
路径安全
密钥读取
FFmpeg/FFprobe 调用
Chromium/模板渲染
SQLite 持久化
事件推送
重试和限流
```

前端只做交互和展示，不承载生产线逻辑。

## 三、分层规则

| 层 | 职责 |
|---|---|
| `commands` | 只接参数、调用 service、返回 DTO |
| `domain service` | 业务流程、状态流转、校验、组合 repository/provider/storage |
| `repository` | 只做数据库 CRUD，不写业务判断 |
| `provider` | 只处理外部模型协议，不写 DB |
| `pipeline` | 编排任务步骤，不直接绕过 service |
| `storage` | 统一文件读写、路径解析 |
| `security` | PathGuard、密钥、文件策略 |
| `utils` | 无业务语义工具函数 |

禁止：

```text
Command 里直接 reqwest 调模型
Command 里直接拼 ComfyUI / RunningHub 请求
Command 里直接拼 FFmpeg
Command 里直接读写文件
Command 里直接操作多张表拼业务
Provider 里写数据库
Repository 调 Provider
Service 绕过 PathGuard
业务 Service 按具体 model_name 写分支
业务代码直接读取 API Key
日志打印 API Key
前端传任意路径给后端直接使用
```

```text
list_executable_media_options

```

规则：

```text

```

不要混用 `fetch / load / query / doXxx / handleXxx`。

## 五、AppState 服务注入

所有 command 从 `State<AppState>` 拿服务。

```rust
pub struct AppState {
    pub db: DbPool,
    pub config_service: Arc<ConfigService>,
    pub dictionary_service: Arc<DictionaryService>,
    pub project_service: Arc<ProjectService>,
    pub task_service: Arc<TaskService>,
    pub storyboard_service: Arc<StoryboardService>,
    pub pipeline_engine: Arc<PipelineEngine>,
    pub provider_manager: Arc<ProviderManager>,
    pub storage_service: Arc<StorageService>,
    pub keyring_service: Arc<KeyringService>,
    pub event_emitter: Arc<EventEmitter>,
}
```

不要在 command 里临时 new service。

## 六、Pipeline 规范

每个生产步骤独立实现 `PipelineStep`。

```rust
#[async_trait]
pub trait PipelineStep {
    fn kind(&self) -> TaskStepKind;
    fn input_dependencies(&self) -> Vec<TaskStepKind>;
    fn can_retry(&self) -> bool;
    fn is_idempotent(&self) -> bool;
    async fn execute(&self, ctx: &mut PipelineContext) -> AppResult<StepOutput>;
}
```

推荐目录：

```text
services/pipeline/steps/
  project_init.rs
  script_generation.rs
  script_review.rs
  storyboard_generation.rs
  storyboard_review.rs
  image_prompt_generation.rs
  video_prompt_generation.rs
  tts_generation.rs
  image_generation.rs
  image_review.rs
  video_generation.rs
  subtitle_generation.rs
  asset_analysis.rs
  cover_generation.rs
  cover_review.rs
  template_rendering.rs
  segment_composition.rs
  final_composition.rs
  export.rs
  canvas_editing.rs
  cleanup.rs
```

为什么必须这样做：

```text
1. 每步可单独重试。
2. 每步可单独记录输入输出。
3. 每步可单测。
4. 断点续跑清楚。
5. 用户确认节点清楚。
6. 后期加 AI 视频、数字人、批量任务不污染主流程。
```

## 七、Provider 规范

业务层不允许直接 new provider。

必须：

```text
业务 Service
→ ProviderManager
→ 具体 Provider
```

ProviderManager 负责：

```text
选择 provider
读取 key_alias 对应密钥
校验模型能力
通过 ModelRegistry 校验 provider_models
通过 WorkflowRegistry 校验 workflow_presets
限流
重试
错误分类
日志脱敏
统一返回结构
```

Provider trait 示例：

```rust
#[async_trait]
pub trait LlmProvider {
    async fn chat(&self, req: LlmChatRequest) -> AppResult<LlmChatResponse>;
}

#[async_trait]
pub trait TtsProvider {
    async fn synthesize(&self, req: TtsRequest) -> AppResult<TtsResponse>;
}

#[async_trait]
pub trait ImageProvider {
    async fn generate(&self, req: ImageRequest) -> AppResult<ImageResponse>;
}

#[async_trait]
pub trait VideoProvider {
    async fn generate(&self, req: VideoRequest) -> AppResult<VideoResponse>;
}
```

ComfyUI / RunningHub 作为 `workflow` provider vendor 接入，不作为独立 ProviderKind。

模型适配硬规则：

```text
1. API 模型能力只从 provider_models / ModelRegistry 读取。
2. ComfyUI / RunningHub 工作流只从 workflow_presets / WorkflowRegistry 读取。
3. ImageProvider / VideoProvider 可以由 provider_model_id 或 workflow_preset_id 驱动，但业务 Service 不判断具体模型名。
4. Command 和功能模块不得直接拼 ComfyUI workflow JSON、RunningHub 参数或某家 API 私有字段。
5. workflow 执行细节看 `23-模型适配与工作流注册规范.md`。
```

## 八、FFmpeg 封装

不要业务代码拼字符串命令。

提供：

```text
FfmpegService
FfmpegCommandBuilder
FfprobeService
```

业务方法：

```text
create_segment_from_image_audio
concat_segments
mix_bgm
burn_subtitles
probe_media
extract_frame
```

所有输入输出路径必须先过 StorageService + PathGuard。

## 九、Storage 规范

所有文件都必须归入 FileBucket：

```text
workspace / projects / assets / templates / outputs / cache / temp / sidecars / logs
```

禁止：

```rust
std::fs::read(user_path)
```

必须：

```rust
storage_service.read_file(bucket, relative_path)
```

入库只存相对路径，不存绝对路径。

## 十、错误结构

后端错误统一结构：

```rust
pub struct AppErrorDto {
    pub code: String,
    pub kind: ErrorKind,
    pub message: String,
    pub detail: Option<String>,
    pub is_retryable: bool,
}
```

错误码格式见 `01-枚举字典与配置规范.md`。

前端优先用 `code` 做 i18n 映射，找不到再显示 `message`。

---

## 十一、事件推送

任务进度通过 Tauri event 推送，后端统一由 `EventEmitter` 发送，不在各处散发事件。

ProgressEvent 至少包含：

```text
task_id
step_kind
step_status
progress
item_id / image_id / segment_id（可选）
message_key（可选）
created_at
```

用户可见 message 用 i18n key，不直接从后端返回中文作为长期协议。

---

## 十二、日志脱敏

禁止打印：

```text
API Key
Bearer Token
完整密钥
完整 Provider 请求头
用户敏感文件全文
超长原始小说文本
```

日志可以打印：

```text
key_alias
provider_id
provider_kind
vendor
model_name
错误码
任务 ID
步骤 ID
相对路径
```
