# 测试策略与验收用例

> 这篇定义从零开发时必须具备的测试层级。目标不是追求形式覆盖率，而是保证任务、路径、Provider、FFmpeg、模板这些高风险部分不会烂掉。

## 一、测试分层

```text
Rust 单元测试
Repository / Migration 测试
Service 集成测试
Provider mock 测试
ModelRegistry / WorkflowRegistry 测试
Workflow preset contract test
FFmpeg smoke test
Template render test
Tauri command contract test
前端组件测试
端到端生成 smoke test
打包后自检测试
```

## 二、必须测试的后端模块

| 模块 | 必测内容 |
|---|---|
| PathGuard | ../、symlink、junction、UNC、file:// |
| StorageService | 读写、复制导入、相对路径入库 |
| TaskService | 创建、取消、重试、恢复、review |
| PipelineEngine | step 顺序、幂等、失败恢复 |
| ProviderManager | 能力校验、密钥脱敏、错误映射 |
| ModelRegistry | provider_models schema、能力查询、limits 校验、禁用模型过滤 |
| WorkflowRegistry | workflow_presets schema、param_schema、node_map、output_map、未注册 workflow 拒绝 |
| FfmpegService | probe、合成、拼接、BGM |
| TemplateRenderer | 参数校验、截图、禁止联网 |
| ConfigService | 默认值、迁移、schema 校验 |

## 三、PathGuard 安全用例

必须覆盖：

```text
../ escape
..\ escape
符号链接跳出 workspace
Windows junction 跳出 workspace
UNC 路径
file:// URL
绝对路径伪装相对路径
模板资源引用外部路径
```

期望：全部拒绝并返回 `security.path_escape` 或 `storage.path_denied`。

## 五、Provider mock 用例

```text
1. 认证失败 → provider.auth_failed，不重试。
2. 限流 → provider.rate_limited，可重试。
3. 超时 → provider.timeout，可重试。
4. 返回非法 JSON → repair loop。
5. 返回远程 URL → 下载转存本地。
6. 不支持能力 → provider.capability_unsupported。
7. providerModelId 不存在 → provider.model_not_found。
8. providerModelId 已禁用 → provider.model_disabled。
9. providerModelId 和 workflowPresetId 同时为空 → provider.selection_required。
10. providerModelId 和 workflowPresetId 默认同时有值 → provider.selection_conflict。
11. 返回错误里包含疑似 API Key → 日志和 AppErrorDto 必须脱敏。
```

## 六、Workflow Preset 用例

必须覆盖：

```text
1. scan_builtin_workflow_presets 扫描内置 ComfyUI / RunningHub preset。
2. preset.json 缺少 workflow_preset_id → workflow_preset.schema_invalid。
3. param_schema_json 非法 → workflow_preset.schema_invalid。
4. node_map_json 缺少必填节点 → workflow_preset.node_map_invalid。
5. output_map_json 缺少输出路径规则 → workflow_preset.output_map_invalid。
6. 未注册 workflowPresetId → workflow_preset.not_found。
7. 已禁用 workflowPresetId → workflow_preset.disabled。
8. 未注册 workflow key 不能被 ProviderManager 执行。
9. dry_run 只校验连接、鉴权、workflow 存在和参数完整性，不生成媒体。
10. real_generate 必须使用 sampleParams，并把输出写入测试目录。
11. RunningHub workflow_id 缺失 → workflow_preset.remote_id_required。
12. ComfyUI selfhost workflow 文件不存在 → workflow_preset.workflow_missing。
```

验收红线：

```text
任何功能模块都不能绕过 WorkflowRegistry 直接执行 workflow key。
任何 workflow preset 都不能保存真实 API Key。
任何 workflow 执行前都必须校验 param_schema / node_map / output_map。
```

## 九、Tauri command contract test

每个 command 至少测：

```text
成功返回 DTO shape
参数非法返回 AppErrorDto
错误 code 稳定
不返回 secretValue
```

模型适配相关 command 必测：

```text
list_workflow_presets
save_workflow_preset
test_workflow_preset
scan_builtin_workflow_presets
list_executable_media_options
```

额外断言：

```text
1. list_executable_media_options 只返回启用且可执行的 provider_models / workflow_presets。
2. WorkflowPresetDto 不返回真实 API Key。
3. ExecutableModelOptionDto 必须区分 sourceType=provider_model / workflow_preset。
4. save_workflow_preset 保存前必须校验 param_schema / node_map / output_map。
5. test_workflow_preset 的 dry_run 不产生媒体文件。
```

## 十一、MVP 验收链路

第一版端到端必须跑通：

```text
主题 / 文案 / 文章输入
→ 生成 / 编辑分镜
→ 生图表格页生成候选图
→ 每个分镜选择最终图
→ 图生视频生成视频片段
→ 确认视频片段
→ FFmpeg 合成 final.mp4
→ 导出 / 打开输出目录
→ 历史记录可回看
```

验收标准：

```text
1. final.mp4 可播放。
2. 生图页每行有原文、角色、提示词、候选图、已选图、状态、操作。
3. 图生视频读取已选图并生成片段。
4. 任务详情能看到每一步状态。
5. 失败步骤可重试。
6. 日志不包含 API Key。
7. 生图 / AI 视频候选项来自 list_executable_media_options。
8. provider_models 和 workflow_presets 至少各有一个 dry_run 用例通过。
9. 未注册 workflow 执行必须失败。
10. workflow preset 的 param_schema / node_map / output_map 校验失败必须阻断任务。
```
