# UI 详细设计与页面布局

> 这篇定义产品的**视觉语言、全局布局系统、核心页面详细设计、组件视觉规格和交互态**。
> 它回答的是“页面长什么样、怎么排、各种状态怎么呈现”，是设计层。
> 工程层（代码怎么组织、字典/i18n/主题怎么用）看 `02-前端工程规范.md` 和 `04-UI主题与国际化规范.md`，本篇不重复。
> 冲突时：token 定义以 `04` 为准，本篇只引用 token 名，不重新定义色值。当前开发顺序以项目 `plan/阶段路线图.md` 为准。

## 〇、这篇怎么用

- 设计目标：在 `04` 已定的“专业创作工具 / 低干扰 / 高可读 / 信息密度适中”基础上，叠加**现代简约高级感**。
- 表达形式：ASCII 线框图给布局骨架，文字规格给尺寸、间距、状态、交互。线框只示意区域关系，不代表像素级比例。
- 落地边界：本篇所有颜色只用 `04` 的 semantic / component token 名，所有文案走 i18n，不在这里写死中文 UI 文案。

## 一、视觉语言（高级感从哪来）

高级感不是靠装饰堆出来的，是靠**克制 + 一致 + 呼吸感**。四条主原则：

```text
1. 留白比线条更重要——用间距分组，少用分割线和边框。
2. 主色克制——大面积中性灰白，主色只用在“当前/主操作/选中”。
3. 层次靠明度和间距，不靠重阴影和重描边。
4. 一切对齐到 4px 栅格，节奏统一。
```

### 1.1 间距节奏（spacing scale）

统一 4px 基准，只用这几档，禁止随手写 13px / 18px：

```text
space-1 = 4px
space-2 = 8px
space-3 = 12px
space-4 = 16px
space-5 = 24px
space-6 = 32px
space-8 = 48px
```

页面级规则：面板内边距用 `space-5`（24px），卡片内边距用 `space-4`（16px），区块之间用 `space-6`（32px）。宁可多留白，不要塞满。

### 1.2 圆角（radius）

```text
radius-sm = 6px
radius-md = 10px
radius-lg = 14px
radius-full = 999px
```

### 1.3 描边与阴影

```text
边框：默认只用 1px color-border，且优先用在输入类控件。
阴影：只给下拉、弹窗、悬浮卡、拖拽中元素。
分层：page < panel < card，靠 color-bg-* 的明度差区分。
```

### 1.4 字体与字阶

```text
字体族：系统 UI 字体栈优先，数字与代码用等宽。
text-display 28/36
text-title   20/28
text-section 16/24
text-body    14/22
text-caption 12/18
字重：常规 400，强调 500，标题 600。
```

### 1.5 主色使用纪律

```text
允许：当前导航、当前 Step、页面唯一主操作、进度完成段、选中态。
禁止：整块背景铺主色、多 Primary、用主色做装饰。
```

## 二、全局布局系统（App Shell）

整个应用是一个固定外壳 + 可变主区。外壳常驻，主区随路由切换。

```text
┌──────────────────────────────────────────────────────────────────────┐
│ TopBar  作品名 / 当前阶段 / 全局操作                                  │ 52px
├────────┬─────────────────────────────────────────────────┬───────────┤
│ Side   │                 MainArea                        │ Inspector │
│ Nav    │              主编辑区 / 表格 / 列表                │ 右侧面板   │
│ 64/224 │                  自适应填充                       │ 320px     │
├────────┴─────────────────────────────────────────────────┴───────────┤
│ StatusBar  ● 任务运行中 3/8 ···················· [展开任务面板]        │ 36px
└──────────────────────────────────────────────────────────────────────┘
```

### 2.1 区域职责与尺寸

```text
TopBar     高 52px。作品切换、当前状态、全局操作。
SideNav    宽两态：展开 224px / 收起 64px。只放全局入口。
MainArea   自适应填充。主内容撑满可用宽度，不做 1280px 居中限制。
Inspector  宽 320px，可折叠。放选中对象属性、prompt、详情、预览。
StatusBar  高 36px。显示任务摘要、进度、当前状态。
```

### 2.2 三栏 / 两栏 / 表格页取舍

```text
三栏：分镜编辑、字幕、封面、详情编辑类页面。
两栏：作品列表、创作资源、模型 / 工作流、系统设置。
表格/列表工作台：生图页、图生视频页、批量检查类页面。
单栏：内容创作、视频预览全屏。
```

生图页和图生视频页不以中间大预览为核心，而是以按分镜行排列的表格 / 列表为核心。大图和视频预览使用弹窗、抽屉或行内小预览。

### 2.3 窗口断点

```text
最小窗口：1024 × 720。
≥ 1440         Inspector 默认展开，SideNav 默认展开。
1200 ~ 1440    Inspector 默认展开，SideNav 可保持展开。
1024 ~ 1200    Inspector 默认收起为图标条，SideNav 自动收起为 64px。
```

### 2.4 导航结构与流转

```text
全局图标栏（Rail）只放全局级入口：
  我的作品
  AI 工具
  创作资源
  模型 / 工作流
  系统设置

作品工作台不是全局导航项：它是某个具体作品的上下文，只能从作品列表、作品卡片或开始创作进入。
内容创作不是全局导航项，它是作品工作台第一步。
任务中心不做全局入口。任务进度、失败、重试、历史回看放在作品卡片和作品工作台内部。
素材库不做一级菜单，放在“创作资源”下面。
AI 工具是全局入口，但只放独立小工具，不放创建作品入口。
```

当前 `image_to_video` 主流程：

```text
我的作品 / 开始创作
  → 作品工作台：内容创作 → 分镜 → 生图 → 视频 → 合成
  → 导出 / 打开输出目录
```

设定集和生图提示词不是删除，而是作为分镜 / 生图页里的抽屉、面板或行内能力逐步增强，不作为当前 StepBar 的必经步骤。

TTS、字幕、封面、BGM、模板动效不是取消，而是 TODO 里的后续增强；不作为当前 StepBar 的必经步骤。

### 2.5 工作台 StepBar

当前主线 StepBar：

```text
┌──────────────────────────────────────────────────────────────────────┐
│   ① 分镜 ──── ② 生图 ──── ③ 视频 ──── ④ 合成                         │
│   ✓done       ●now       ○locked     ○locked                         │
└──────────────────────────────────────────────────────────────────────┘
```

状态：

```text
已完成：实心勾 + color-success，可点击回看。
当前：实心圆点 + color-primary，加粗。
未解锁：空心圆 + color-text-secondary，禁用，hover 提示“先完成上一步”。
连接线：完成段用 color-primary，未完成段用 color-border。
```

## 三、核心页面布局

### 3.1 我的作品页

```text
作品列表 / 搜索 / 筛选 / 开始创作 / 打开作品工作台
```

作品卡片显示：

```text
标题
视频包
画幅 / 时长
最新任务状态
更新时间
```

### 3.2 开始创作

不做独立“新建项目”表单。用户点击“开始创作”后，系统自动创建作品草稿，并进入作品工作台的“内容创作”步骤。

```text
我的作品
  点击开始创作
  自动创建作品草稿（技术 Project）
  进入作品工作台 / 内容创作
```

作品标题可以先自动生成，后续用户可在作品工作台里修改。

### 3.3 内容创作页

内容创作是作品工作台第一步，不是独立全局页面。

```text
内容来源 Tabs：
  一句话视频想法
  已有文案
  文章 / 长文
  小说 / 剧情
  素材说明

主编辑区：
  输入 / 粘贴 / 上传
  AI 生成或整理
  版本对比
  原文锁定

右侧：
  视频包选择
  使用的创作规则
  画幅 / 目标时长 / 分镜数量
  作品标题
  进入分镜
```

内容创作写入当前作品：

```text
ScriptDraft
CleanNarrationList
StoryboardItem.sourceText
StoryboardItem.narrationText
```

### 3.4 分镜页

建议布局：

```text
┌──────────┬──────────────────────────────┬───────────┐
│ 分镜列表 │ 当前分镜编辑                  │ 属性/AI助手 │
│          │ 原文/画面/角色/提示词/时长     │ 参数/操作    │
└──────────┴──────────────────────────────┴───────────┘
```

分镜主字段：

```text
原文片段
旁白 / 文案
画面描述
角色 / 主体
场景
出图提示词
视频动作提示词
时长
状态
```

景别、运镜、转场、构图是高级可选字段，不是主界面重点。

### 3.5 生图表格页

生图页必须是表格 / 列表工作台：

```text
每一行 = 一个分镜
```

建议布局：

```text
顶部：批量生成全部 / 只生成未生成 / 进入视频阶段
主体：生图表格
```

表格列：

```text
序号
原文
角色
出图提示词
生图输入要求
生成图片
候选图
已选图
状态
操作
```

行内操作：

```text
编辑提示词
查看当前创作规则
查看当前模型需要的输入
生成图片
重新生成
选择候选图
确认图片
查看大图
```

规则：

```text
生图前先根据图片类型和生图工具返回“生图输入规划”。
不同图片类型不要共用同一套提示词规则。
所有分镜都选择最终图后，才能进入视频阶段。
```

### 3.6 图生视频页

图生视频页建议同样是表格 / 列表工作台：

```text
每一行 = 一个分镜的视频片段生成任务
```

表格列：

```text
序号
已选图片
视频动作提示词
视频输入要求
时长
生成视频
视频预览
状态
操作
```

规则：

```text
视频生成前先根据当前视频模型返回“视频输入规划”。
普通图生视频通常只要首帧图。
首尾帧模型需要首帧图 + 尾帧图。
参考图模型需要角色参考图 / 风格参考图。
ComfyUI / RunningHub 工作流按 workflow preset 的参数 schema 暴露输入。
所有分镜都确认视频片段后，才能进入合成阶段。
```

### 3.7 合成页

```text
检查清单
片段列表
输出设置
合成进度
最终预览
打开输出目录
```

当前主线只要求：

```text
按分镜顺序拼接已确认视频片段，输出 final.mp4。
```

字幕、封面、BGM、模板动效是后续独立增强流程，不放在当前合成页右侧控制区。合成页只负责片段检查、FFmpeg 合成、输出信息和导出；后续增强如果接入，应作为 StepBar 扩展或导出前后独立步骤，不阻塞当前 final.mp4 合成。

### 3.8 创作资源页

创作资源放作品之外可复用的创作素材和规则。

```text
创作资源
  视频包
  创作规则
  素材库
```

视频包：

```text
内容策略组合。决定语气、结构、默认创作规则、审核规则和推荐模型。
视频包只引用创作规则，不直接编辑规则内容。
```

创作规则：

```text
集中编辑文案规则、分镜规则、设定集规则、生图规则、视频规则、审核规则。
技术实现可叫 Skill，但用户界面统一叫“创作规则”。
作品步骤里只显示当前规则名称、来源 builtin/user 和跳转入口；完整版本、hash、snapshot 后续增强。
```

素材库：

```text
只放可复用资产：角色、场景、风格、道具 / 产品、音频、用户上传素材。
作品生成的候选图、视频片段、final.mp4 默认属于作品文件夹。
只有用户主动“保存为素材”，才进入素材库。
```

### 3.9 模型 / 工作流页

模型 / 工作流是生成能力配置中心，不放在系统设置里。

```text
Provider 连接
API 模型能力
ComfyUI / RunningHub 工作流
连接测试
能力测试
输入 Schema 预览
```

模型能力必须能说明：

```text
支持的输入：text / image / audio / video / mask / pose / depth
支持的输出：image / video / audio / text
限制：时长、fps、画幅、参考图数量、prompt 长度
工作流：paramSchema、nodeMap、outputMap、requiredAssets
```

### 3.10 系统设置页

```text
外观与主题
密钥管理
路径与 sidecar
生成默认参数
配置导入导出
日志与诊断
```

系统设置只放全局运行参数和安全诊断，不放具体模型能力。模型、Provider、ComfyUI、RunningHub 放在“模型 / 工作流”。
密钥和导出安全规则以底层设计为准。

## 四、验收重点

每个页面完成后必须检查：

```text
1. 是否遵守当前 image_to_video 主线。
2. StepBar 是否为 分镜 → 生图 → 视频 → 合成。
3. 生图页是否为表格/列表，而不是中间大预览。
4. 图生视频是否读取已选图片。
5. 合成是否读取已确认视频片段。
6. TTS/字幕/封面是否作为后续增强，而不是当前阻塞项。
7. 页面是否撑满可用宽度。
8. 视觉是否对照 `开始之前/菜单与页面详细说明.md`、`参考分析/页面交互账本.md` 和 `04-UI主题与国际化规范.md`。
9. 状态、字典、颜色、文案是否走统一规范。
10. 页面是否没有直接 invoke。
```
