# 错误码总表与恢复策略

> 这篇定义全项目错误码、用户提示、是否可重试、恢复策略。错误结构看 `09-错误日志与事件规范.md`，枚举分类看 `01-枚举字典与配置规范.md`。

## 一、核心原则

```text
1. 错误码是稳定协议，不随便改名。
2. 前端用 code 映射 i18n 文案，不长期依赖后端中文 message。
3. detail 只给详情面板和诊断包，不直接弹给普通用户。
4. 是否可重试由后端判断。
5. 任务失败必须写入 task_steps.error_json。
```

---

## 二、错误结构

```ts
interface AppErrorDto {
  code: string
  kind: ErrorKind
  message: string
  detail?: string
  isRetryable: boolean
  recoverAction?: RecoverAction
}
```

`RecoverAction`：

```text
retry
resume_task
update_secret
manual_check
change_provider_or_plan
edit_input
choose_controlled_path
start_composition
configure_sidecar
check_media_file
regenerate_or_replace_media
restart_app_or_check_database
remove_secret_and_retry
reject_package
grant_media_permission
```

## 三、配置错误

| code | 含义 | 可重试 | 恢复策略 |
|---|---|---|---|
| config.missing | 配置不存在 | 否 | 重新初始化默认配置 |
| config.invalid | 配置格式错误 | 否 | 打开设置页修复 |
| config.version_unsupported | 配置版本不兼容 | 否 | 执行配置迁移 |
| config.snapshot_missing | 任务快照缺失 | 否 | 任务不可恢复，提示重新创建 |

## 四、校验错误

| code | 含义 | 可重试 | 恢复策略 |
|---|---|---|---|
| validation.required | 必填字段为空 | 否 | 前端定位字段 |
| validation.invalid_format | 格式不合法 | 否 | 修改输入 |
| validation.enum_invalid | 枚举值非法 | 否 | 使用统一枚举 |
| validation.schema_invalid | JSON schema 校验失败 | 可有限重试 | LLM repair loop 或人工修正 |
| validation.dependency_missing | 上游依赖缺失 | 否 | 回到缺失步骤重跑 |

## 五、Provider 错误

| code | 含义 | 可重试 | 恢复策略 |
|---|---|---|---|
| provider.auth_failed | 密钥无效或无权限 | 否 | 打开 Provider 设置 |
| provider.rate_limited | 限流 | 是 | 按 retry_after 或退避重试 |
| provider.timeout | 请求超时 | 是 | 自动重试 |
| provider.network_failed | 网络失败 | 是 | 自动重试，失败后提示检查网络 |
| provider.content_policy | 内容审核拒绝 | 可中性化后重试 | 中性化 prompt 后最多重试一次 |
| provider.invalid_response | 返回结构不合法 | 是 | repair loop 或重试 |
| provider.capability_unsupported | 模型不支持能力 | 否 | 更换模型或参数 |
| provider.output_missing | 供应商未返回产物 | 是 | 自动重试 |

## 十二、任务恢复策略

```text
可重试错误：自动退避重试，超过 max_attempts 后停在 failed。
不可重试错误：立即 failed，展示用户处理建议。
用户可处理错误：展示 update_secret / edit_input / choose_controlled_path 等 recoverAction。
系统缺失错误：停止相关任务，不继续下游步骤。
```
