# 数据结构（字段单一来源）

> 这是整个项目所有业务数据字段的**唯一定义出处**。任何功能文档说“字段定义看这里”，都是指本文件。
> 枚举、字典、配置取值不在本文件维护，统一看 `底层设计/01-枚举字典与配置规范.md`。

## 一、命名约定

代码落地时必须统一命名：

```text
Rust / SQLite / JSON 存储层：snake_case
前端 TypeScript：camelCase
Tauri 边界：用 serde rename 显式映射
枚举 code：snake_case 稳定值
```

字段规则：

| 类型 | 命名规则 | 示例 |
|---|---|---|
| ID | `_id` | `project_id` / `item_id` / `image_id` / `segment_id` |
| 路径 | `_path` | `image_path` / `audio_path` |
| URL | `_url` | `source_url` |
| 时长 | `_duration_seconds`，单位秒 | `audio_duration_seconds` |
| 数量 | `_count` | `target_scene_count` |
| 状态 | `_status` | `task_status` / `image_status` |
| 布尔 | `is_ / has_ / should_ / can_ / requires_` | `requires_user_confirmation` |

不要使用 `data / event / dur / cnt / flag / info / type / status` 这类含糊字段名作为核心业务字段。

## 三、整体层级

数据从大到小是四层：

```text
Project（项目）                 一次创作 = 一个项目，记录输入、workflow、内容语言、全局设定
  └── Bibles（设定集）           项目级统一设定：画风、角色、场景、负向词
  └── Storyboard（分镜表）        一个项目一张主分镜表
        └── StoryboardItem[]     一个分镜行，是当前 image_to_video 主线最小工作单元
              ├── ImageCandidate[]  生图候选图
              └── VideoSegment[]    图生视频片段
  └── CompositionTask[]          合成任务，输出 final.mp4
  └── Task（任务）               一次“生成”动作 = 一个任务，挂在项目下
        └── TaskStep[]           任务里的每一步
  └── NovelChapter[]             仅小说导入时有，分章后的原文 + 提取的剧情事件
```

一句话：**Project 是容器，Bibles 是统一设定，StoryboardItem 是内容行，ImageCandidate/VideoSegment 是生成产物，Task/Step 是干活记录。**

Project 不承载生成流程状态；生成中、成功、失败等从最新 Task 读取。

## 四、Project（项目）

用户从「内容导入 / 新建项目」进来，就建一条 Project。

```json
{
  "project_id": "proj_xxx",
  "title": "为什么普通人要养成阅读习惯",
  "workflow_type": "image_to_video",
  "input_type": "topic",
  "input_process_mode": "generate",
  "input_options_json": {},
  "source_text": "短文本原文；长文本/小说为空或仅存摘要",
  "source_text_path": null,
  "topic": "为什么要早睡",
  "content_category": "knowledge",
  "content_language": "zh-CN",
  "tone": "温和、真诚、有启发",
  "aspect_ratio": "vertical_9_16",
  "target_duration_seconds": 60,
  "target_scene_count": 8,
  "segment_duration_seconds": 4,
  "style_prompt": "clean cinematic vertical short video",
  "project_lifecycle": "draft",
  "cover_path": null,
  "cover_title": null,
  "cover_template_id": null,
  "cover_source_item_id": null,
  "created_at": 0,
  "updated_at": 0,
  "deleted_at": null
}
```

| 字段 | 含义 | 类型/取值来源 |
|---|---|---|
| project_id | 项目 ID | 字符串 ID |
| title | 项目标题 | 字符串 |
| workflow_type | 制作流程，例如 image_to_video / digital_human | WorkflowType |
| input_type | 从哪个入口来 | InputType |
| input_process_mode | 内容处理模式；paste 固定文案不得改写 | InputProcessMode |
| input_options_json | 入口相关参数，例如 `split_mode` | JSON object |
| source_text | 20KB 以内短文本原文；长文本可存摘要或为空 | nullable string |
| source_text_path | 超过 20KB 的原文文件相对路径；小说原文必须走文件 | nullable path |
| topic | 主题输入内容 | nullable string |
| content_category | 内容类别，例如 knowledge/story/product，可选 | 字典 |
| content_language | 视频内容语言 | ContentLanguage |
| tone | 语气风格 | 字符串 |
| aspect_ratio | 画幅协议 code | AspectRatio |
| target_duration_seconds | 目标总时长，单位秒 | 正数 |
| target_scene_count | 目标分镜数 | 正整数 |
| segment_duration_seconds | 单段视频默认时长 | 正数 |
| style_prompt | 全局画风 / 风格提示词 | nullable string |
| project_lifecycle | 项目生命周期 | ProjectLifecycle |
| cover_path | 封面图相对路径，后续增强 | nullable |
| cover_title | 封面标题，后续增强 | nullable |
| cover_template_id | 封面模板 ID，后续增强 | nullable |
| cover_source_item_id | 封面来源分镜行 ID，后续增强 | nullable |

### 4.1 草稿和自动保存

项目一旦通过内容导入创建，必须立即落库为 `project_lifecycle=draft`。用户后续在分镜、提示词、设定集、候选图选择中的编辑都应保存到 SQLite，不能只存在前端内存。

规则：

```text
1. 新建项目成功后立即有 Project 记录和默认 Bible 记录。
2. 页面表单可以有本地 dirty 状态，但离开页面前必须自动保存或提示用户保存。
3. StoryboardItem、Character Bible、Environment Bible、ImageCandidate 选择关系都必须可从 SQLite 恢复。
4. 应用异常退出后，重新打开项目应恢复到最近一次已保存草稿。
5. 生成类产物保存为候选或版本，不因草稿保存覆盖旧产物。
6. 前端 Store 只是缓存，不是草稿权威源。
```

```json
{
  "project_id": "proj_xxx",
  "topic": "为什么普通人要养成阅读习惯",
  "content_category": "knowledge",
  "content_language": "zh-CN",
  "tone": "温和、真诚、有启发",
  "aspect_ratio": "vertical_9_16",
  "negative_prompt": "photorealistic, colorful, messy background, distorted face"
}
```

### 5.2 Style Bible（画风设定）

```json
{
  "style_id": "minimal_stick_figure",

  "color_palette": ["black", "white", "light gray"],

  "negative_prompt": "photorealistic, colorful, complex background, 3d render, realistic face",
  "reference_image_path": null
}
```

### 5.3 Character Bible（角色设定）

```json
{
  "characters": [
    {
      "character_id": "main_character",
      "name": "小林",
      "role": "ordinary young office worker",
      "age": "around 28",
      "gender": "male",

      "reference_images_json": [],
      "seed": null

  ]
}
```

### 5.4 Environment Bible（场景设定）

```json
{
  "locations": [
    {
      "location_id": "study_room",
      "name": "安静书房",
      "location_type": "indoor study room",

```

StoryboardItem 里的 `character_ids_json` 存 `character_id`，`location_id` 存 `location_id`，`props_json` 存道具名。生图时按 ID 取对应 `visual_prompt` 拼进提示词，保证全片一致。

## 七、StoryboardItem（分镜行，当前主线最小工作单元）

`StoryboardItem` 是当前 `image_to_video` 主线每一步都会读写的核心结构。

```json
{
  "item_id": "item_001",
  "project_id": "proj_xxx",
  "storyboard_id": "sb_xxx",
  "item_index": 1,
  "item_role": "hook",

  "source_text": "很多人以为早睡只是少玩一会手机。",
  "narration_text": "很多人以为早睡只是少玩一会手机。",
  "subtitle_chunks": [],

  "visual_description": "清晨卧室里，年轻上班族醒来，阳光照进房间，状态清醒。",
  "characters_json": ["年轻上班族"],
  "scene_description": "清晨卧室",
  "character_ids_json": [],
  "location_id": null,
  "props_json": [],

  "image_prompt": "young office worker waking up refreshed in a clean bedroom, morning sunlight, cinematic, vertical composition",
  "negative_prompt": "low quality, distorted face, extra fingers",
  "video_prompt": "soft morning light moves across the room, character slowly opens eyes and sits up",

  "duration_seconds": 4.0,
  "selected_image_id": null,
  "selected_video_segment_id": null,

  "shot_size": null,
  "camera_motion": null,
  "composition": null,
  "pace": null,
  "transition_type": null,

  "image_status": "pending",
  "video_status": "pending",
  "subtitle_status": "pending",
  "audio_status": "pending",

  "lock_flags_json": {},
  "retry_count": 0,
  "last_error_json": null
}
```

### 字段分组速查

| 组 | 字段 | 谁写入 |
|---|---|---|
| 标识 | item_id / item_index / item_role | 分镜生成（03） |
| 原文/文案 | source_text / narration_text / subtitle_chunks | 内容导入/分镜/脚本增强（01/03/02/10） |
| 画面创作 | visual_description / characters_json / scene_description / character_ids_json / location_id / props_json | 分镜生成（03） |
| 提示词 | image_prompt / negative_prompt / video_prompt | 生图提示词（07）/AI视频（14） |
| 时长 | duration_seconds | 分镜生成；TTS 或视频生成可后续回填 |
| 选择结果 | selected_image_id / selected_video_segment_id | AI生图（08）/AI视频（14） |
| 高级可选 | shot_size / camera_motion / composition / pace / transition_type | 图生视频或高级编辑，可选 |
| 状态 | image_status / video_status / subtitle_status / audio_status | 各对应步骤，取值使用 TaskStepStatus |
| 控制 | lock_flags_json / retry_count / last_error_json | 任务系统（17） |

### 校验规则（开发时必须实现）

进入生图前必须满足：

1. `source_text` 或 `narration_text` 至少一个非空。
2. `visual_description` 不能为空。
3. `image_prompt` 不能为空；为空时必须先执行反推提示词。
4. `duration_seconds` 必须 > 0。
5. `character_ids_json` 非空时，必须存在对应 Character Bible 条目。
6. `location_id` 非空时，必须存在对应 Environment Bible 条目。

进入图生视频前必须满足：

1. 每条 StoryboardItem 都有 `selected_image_id`。

进入合成前必须满足：

1. 每条 StoryboardItem 都有 `selected_video_segment_id`。

不阻塞当前主线的高级字段：

```text
shot_size / camera_motion / composition / pace / transition_type
```

### lock_flags_json 重生成保护

`lock_flags_json` 是唯一锁定来源，不再使用 `is_user_locked`。它用于细粒度保护用户编辑成果，例如：

```json
{
  "storyboard": false,
  "source_text": true,
  "visual_description": false,
  "image_prompt": true,
  "video_prompt": false,
  "image": false,
  "video": false,
  "audio": false,
  "subtitle": false
}
```

批量重生成必须跳过对应锁定项，只有用户显式解锁后才能覆盖。

## 八、ImageCandidate（生图候选图）

每个 StoryboardItem 可以有多张候选图，用户必须选择一张作为图生视频输入。

```json
{
  "image_id": "img_001",
  "item_id": "item_001",
  "project_id": "proj_xxx",
  "image_path": "images/item_001/img_001.png",
  "prompt": "young office worker waking up refreshed...",
  "negative_prompt": "low quality, distorted face",
  "provider_model_id": "model_xxx",
  "workflow_preset_id": null,
  "status": "succeeded",
  "selected": false,
  "retry_count": 0,
  "last_error_json": null,
  "created_at": 0
}
```

规则：

1. `image_path` 只存工作区相对路径。
2. 同一个 `item_id` 下只能有一个 `selected=true`。
3. 重生成不得删除旧候选图，除非用户显式清理。
4. 远程 URL 必须下载转存本地后再入库。

## 九、VideoSegment（图生视频片段）

每个 StoryboardItem 可以有一个或多个视频片段，用户确认其中一个进入合成。

```json
{
  "segment_id": "seg_001",
  "item_id": "item_001",
  "project_id": "proj_xxx",
  "input_image_id": "img_001",
  "video_path": "videos/item_001/seg_001.mp4",
  "video_prompt": "soft morning light moves across the room...",
  "duration_seconds": 4.0,
  "provider_model_id": "video_model_xxx",
  "workflow_preset_id": null,
  "status": "succeeded",
  "selected": true,
  "retry_count": 0,
  "last_error_json": null,
  "created_at": 0
}
```

规则：

1. `input_image_id` 必须指向已存在的 ImageCandidate。
2. `video_path` 只存工作区相对路径。
3. 同一个 `item_id` 下只能有一个 `selected=true`。
4. 失败可以重试，不得破坏已成功片段。

## 十、CompositionTask（合成任务）

合成任务把已确认的视频片段按分镜顺序拼接为最终视频。

```json
{
  "task_id": "task_xxx",
  "project_id": "proj_xxx",
  "segment_ids_json": ["seg_001", "seg_002"],
  "output_path": "exports/final.mp4",
  "status": "succeeded",
  "progress": 100,
  "created_at": 0,
  "updated_at": 0
}
```

规则：

1. 合成顺序按 StoryboardItem.item_index，不按文件名排序。
2. `output_path` 只存工作区相对路径。
3. 诊断包和导出包不得泄露本地绝对路径。

## 十三、文件存储布局

完整文件规范见 `底层设计/08-文件存储与路径规范.md`。

项目和任务产物推荐布局：

```text
workspace/projects/proj_xxx/
  project.json        # 仅导出/诊断快照，不是主数据源
  bibles/             # 可选快照目录，权威数据仍以 SQLite 为准
  tasks/task_xxx/
    input/
    script/
    storyboard/
    prompts/
    audio/
    images/
    videos/
    subtitles/
    rendered/
    segments/
    cover/
    exports/
    logs/
```

SQLite 是结构化数据唯一权威源；`project.json` 和 `bibles/*.json` 只作为导出、诊断或项目包快照，不允许成为第二套主数据源。

入库一律存**相对路径**，绝不存绝对路径；所有读写经 StorageService + PathGuard。

原文存储规则：`source_text` 只保存 20KB 以内短文本；超过 20KB 的文章/长文写入 `input/source.txt`，库里存 `source_text_path`；小说整本原文必须写文件，分章内容写入 `novel_chapters.chapter_content`。
