# 资产一致性与设定集规范

> 这篇定义 Style Bible、Character Bible、Environment Bible、Prop Bible、资产版本和生成快照。目标是解决 AI 视频里角色、画风、场景容易漂移的问题。

## 一、核心原则

```text
1. 全片一致性来自 Bible，不靠每个 StoryboardItem 自由发挥。
2. StoryboardItem 只引用设定 ID，不复制整份设定。
3. 生图 / 视频提示词从 Bible + StoryboardItem 拼装。
4. 每次生成记录 generation_context_snapshot。
5. 用户锁定的资产不得被批量重生成覆盖。
6. 重生成产生新候选或新版本，不直接覆盖用户已选产物。
```

---

## 二、设定集类型

```text
Project Bible       项目总设定
Style Bible         画风设定
Character Bible     角色设定
Environment Bible   场景/地点设定
Prop Bible          道具设定，第一版可作为 props_json，后续独立表
```

---

## 三、StoryboardItem 引用规则

StoryboardItem 使用：

```json
{
  "character_ids_json": ["main_character"],
  "location_id": "study_room",
  "props_json": ["book", "desk_lamp"],
  "visual_description": "主角坐在清晨卧室里，阳光从窗边照进来"
}
```

生成提示词时：

```text
Style Bible.style_prompt
+ Character Bible.visual_prompt
+ Environment Bible.visual_prompt
+ StoryboardItem.visual_description
+ StoryboardItem.characters_json / scene_description
+ 可选高级字段：shot_size / camera_motion / composition
+ Project negative_prompt
```

高级字段不是当前主线必填，不得阻塞生图。

## 四、资产版本

每个资产应记录：

```text
asset_id
asset_kind
source_asset_id
derived_from_asset_id
revision
relative_path
prompt
provider_id
model_name
seed
created_at
```

规则：

```text
1. 重生成产生新 revision，不直接覆盖旧资产。
2. ImageCandidate 指向具体候选图文件。
3. StoryboardItem.selected_image_id 指向当前选中的 ImageCandidate。
4. VideoSegment 指向具体视频片段文件。
5. StoryboardItem.selected_video_segment_id 指向当前选中的 VideoSegment。
6. 历史任务保留当时使用的 revision。
```

## 五、Reference Image 规则

参考图角色：

```text
character_front_view
character_side_view
character_back_view
character_full_body
character_face_closeup
character_expression_sheet
character_outfit
style
scene_wide_view
scene_layout_view
scene_detail_view
scene_day_variant
scene_night_variant
composition
first_frame
last_frame
other
```

角色不能只依赖单张参考图。Character Bible 第一版可以保留 `reference_image_path` 作为主参考图，但必须预留 `reference_images_json`，用于三视图、多表情和服装锁定。

```json
{
  "character_id": "main_character",
  "reference_images_json": [
    { "role": "character_front_view", "relative_path": "assets/characters/main/front.png" },
    { "role": "character_side_view", "relative_path": "assets/characters/main/side.png" },
    { "role": "character_back_view", "relative_path": "assets/characters/main/back.png" },
    { "role": "character_expression_sheet", "relative_path": "assets/characters/main/expressions.png" }
  ]
}
```

场景同理，Environment Bible 第一版可以保留 `reference_image_path`，但必须预留 `reference_images_json`，用于空间布局、全景、局部细节和昼夜变体。

规则：

```text
1. 所有参考图必须是本地相对路径。
2. 远程 URL 必须先转存。
3. Provider 能力矩阵限制 maxReferenceImages。
4. 超出模型能力时后端拒绝，不靠前端禁用。
5. 角色三视图优先级高于普通构图参考图。
6. 场景参考图不得混入角色图，避免把人物固化进背景。
7. 模型不支持多参考图时，后端按 front_view / full_body / scene_wide_view 优先级裁剪，不由前端静默丢弃。
```

## 六、生成快照

每次图片/视频生成记录：

```json
{
  "item_id": "item_001",
  "image_id": "img_001",
  "segment_id": null,
  "provider_id": "provider_xxx",
  "vendor": "dashscope",
  "model_name": "xxx",
  "prompt": "...",
  "negative_prompt": "...",
  "seed": 123,
  "reference_images": [],
  "style_bible_version": 1,
  "character_bible_versions": {},
  "environment_bible_version": 1,
  "created_at": 0
}
```

保存位置：

```text
task_steps.output_json
artifacts.metadata_json
ImageCandidate / VideoSegment metadata
```

---

## 七、用户锁定

锁定项进入 `StoryboardItem.lock_flags_json`：

```json
{
  "character": true,
  "location": false,
  "image_prompt": true,
  "video_prompt": false,
  "image": true,
  "video": false
}
```

规则：

```text
1. 批量重生成跳过锁定项。
2. 用户显式解锁后才能覆盖。
3. 锁定不影响最终合成读取当前已选产物。
```

---

## 八、VLM 反推设定

用户上传参考图时可用 VLM：

```text
参考图
→ VLM 分析
→ 生成 style/character/environment 草稿
→ 用户确认
→ 写入 Bible
```
