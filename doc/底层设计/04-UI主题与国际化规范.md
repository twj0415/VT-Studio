# UI 主题与国际化规范

> 这篇定义前端整体视觉风格、主题切换、多语言、状态色和用户可见文案规则。

## 一、产品风格定位

本产品是 AI 短视频生产工具，不是营销官网。

视觉定位：

```text
专业创作工具
清爽
低干扰
高可读
信息密度适中
状态清楚
```

不要做：

```text
满屏渐变
玻璃拟态
随机光斑
营销官网式大卡片
无意义动画
过度 AI 味装饰
```

---

## 二、主布局

推荐主框架：

```text
左侧：模块导航
顶部：当前项目 / 当前任务 / 全局操作
中间：主编辑区
右侧：属性面板 / 预览 / 日志
底部：任务进度 / 状态栏
```

典型页面：

### 分镜编辑页

```text
左：StoryboardItem 列表
中：当前分镜预览 / 图片 / 字幕
右：StoryboardItem 表单属性
底：任务步骤进度
```

### 生图表格页

当前 `image_to_video` 主线中，生图页不是普通图片网格，也不是中间大预览。

```text
上：批量生成全部 / 只生成未生成 / 进入视频阶段
中：按分镜排列的表格 / 列表工作台
行：序号 / 原文 / 角色 / 反推提示词 / 候选图 / 已选图 / 状态 / 操作
预览：大图用弹窗、抽屉或 hover 预览，不占据主布局中心
```

### 设置页

```text
左：设置分类
右：配置表单
底：检测结果 / 测试结果
```

## 三、主题枚举

> 本产品不把主题理解成“一个主色”。主题是完整视觉皮肤：背景、面板、卡片、边框、文字、阴影、辉光和强调色必须成套设计。

### ThemePreset（主题）

```text
graphite / aurora / ember / porcelain / sandstone
```

每套主题是**一整套完整配色**，可以是深色，也可以是浅色：

| code | 定位 |
|---|---|
| graphite | 石墨深蓝，冷静工程感，默认主题 |
| aurora | 极光墨青，更有生命力的深色创作工具 |
| ember | 琥珀暖夜，偏剪辑/创作工作室氛围 |
| porcelain | 瓷白晨雾，浅色、干净、阅读友好 |
| sandstone | 沙岩暖白，浅暖色，柔和但不发黄 |

切换主题时，必须同时切换：

```text
canvas / page / panel / card / card-hover
border / border-strong
text-primary / text-secondary / text-muted
accent / accent-soft / accent-line / accent-ink
shadow / glow / scrollbar / tooltip / blur surface
```

禁止只替换 `accent` 或只替换组件库 primary color。
状态色（pending/running/succeeded/failed 等）是功能语义色，**不随主题变**。

### LayoutDensity（密度）

```text
compact / comfortable
```

`ThemePreset` 和 `LayoutDensity` 是两个独立维度。示例：

```json
{
  "theme_preset": "graphite",
  "layout_density": "comfortable"
}
```

## 五、Token 分层

### Primitive tokens

基础色和基础尺寸：

```text
gray-50
gray-100
blue-500
red-500
spacing-4
radius-8
```

### Semantic tokens

语义变量：

```text
color-bg-page
color-bg-panel
color-text-primary
color-text-secondary
color-border
color-primary
color-danger
color-warning
color-success
```

### Component tokens

组件变量：

```text
button-primary-bg
card-bg
input-bg
sidebar-bg
timeline-step-active
```

组件样式只能用 token，不要到处写死色值。

## 六、状态色

统一状态色 token：

```text
status.pending
status.running
status.retrying
status.succeeded
status.failed
status.cancelled
status.skipped
status.waiting_user
```

建议语义：

| 状态 | 颜色语义 |
|---|---|
| pending | 灰色 |
| running | 蓝色 |
| retrying | 橙色 |
| succeeded | 绿色 |
| failed | 红色 |
| cancelled | 灰紫色 |
| skipped | 浅灰 |
| waiting_user | 紫色或靛蓝 |

页面不允许自己判断颜色，必须通过字典 `colorToken` 或主题 token。

## 七、多语言概念

必须区分两种语言：

```text
AppLocale       软件界面语言
ContentLanguage 视频内容语言
```

### AppLocale

```text
zh-CN / en-US
```

用于：

```text
菜单
按钮
提示
状态文案
设置页
表单 label
错误提示
```

### ContentLanguage

```text
zh-CN / en-US / ja-JP / ko-KR
```

第一版只实现 `zh-CN / en-US`。

用于：

```text
脚本生成语言
字幕语言
TTS voice 默认值
Prompt 输出语言
字幕断句规则
```

## 九、字典文案规则

字典项不直接存中文，只存：

```text
label_key
description_key
```

示例：

```json
{
  "code": "running",
  "label_key": "dict.taskStatus.running"
}
```

前端展示：

```ts
t(item.labelKey)
```

---

## 十、错误文案规则

后端返回：

```json
{
  "code": "provider.auth_failed",
  "message": "Provider authentication failed",
  "kind": "provider",
  "is_retryable": false
}
```

前端优先用 `code` 映射：

```text
errors.provider.auth_failed
```

找不到映射时，再显示后端 `message`。

---

## 十一、组件文案规则

禁止组件中直接写：

```text
生成中
失败
开启
关闭
确认
取消
```

必须：

```ts
t('common.confirm')
t('dict.taskStatus.running')
```

功能文档里的中文说明可以保留，但代码实现必须走 i18n。

---

## 十二、动效原则

只保留有意义动效：

```text
进度变化
状态切换
面板展开
图片生成完成提示
错误定位
```

不要做：

```text
漂浮装饰
随机光斑
无意义加载动画
影响性能的背景动画
```
