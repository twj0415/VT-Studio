# 日志追踪与诊断包规范

> 这篇定义 tracing 字段、日志等级、任务日志、FFmpeg/Provider 脱敏和诊断包导出。日志不能成为密钥泄露渠道。

## 一、核心原则

```text
1. 日志用于排错，不用于保存业务数据全文。
2. 所有日志必须脱敏。
3. 每条任务相关日志必须尽量带 task_id / step_id / item_id / image_id / segment_id。
4. Provider 请求只记录摘要，不记录完整请求头和密钥。
5. 诊断包默认不包含用户原始长文和真实密钥。
```

---

## 二、日志字段

推荐结构化字段：

```text
trace_id
project_id
task_id
task_step_id
step_kind
item_id
image_id
segment_id
provider_id
provider_kind
vendor
model_name
error_code
duration_ms
retry_count
relative_path
```

禁止字段：

```text
api_key
authorization
bearer_token
secret_value
absolute_user_path
full_novel_text
full_prompt_with_sensitive_data
```

---

## 三、日志等级

```text
trace   极细调试，默认关闭
debug   开发调试，正式版默认关闭
info    关键流程，如任务开始/结束
warn    可恢复异常，如重试、降级
error   不可自动恢复或步骤失败
```

规则：

```text
1. Provider 失败用 warn/error，成功不打印完整响应。
2. FFmpeg stderr 只在失败时截断保存。
3. 用户取消不是 error，使用 info。
```

---

## 四、日志文件布局

```text
workspace/logs/app.log
workspace/logs/error.log
workspace/projects/proj_xxx/tasks/task_xxx/logs/task.log
workspace/projects/proj_xxx/tasks/task_xxx/logs/ffmpeg.log
```

滚动策略：

```text
app.log 单文件 10MB，最多 7 个。
error.log 单文件 10MB，最多 7 个。
task.log 跟随任务保留。
```

---

## 五、Provider 日志

允许记录：

```json
{
  "provider_id": "provider_xxx",
  "vendor": "dashscope",
  "model_name": "xxx",
  "task_id": "task_xxx",
  "step_kind": "image_generation",
  "latency_ms": 1234,
  "error_code": "provider.timeout"
}
```

禁止记录：

```text
完整 headers
完整 api_key
完整 Authorization
供应商原始大响应
```

Prompt 日志：

```text
1. 默认不记录完整 prompt。
2. 需要诊断时只记录 prompt_hash、长度、模板版本。
3. 用户显式导出诊断包时，可选择包含 prompt 摘要，仍不得包含密钥。
```

---

## 六、FFmpeg 日志

规则：

```text
1. 保存 args 摘要，不保存绝对路径。
2. stderr 最多保留最后 200 行或 32KB。
3. 失败时写入 task_step.error_json 的 stderr_summary。
4. 诊断包可包含脱敏后的 ffmpeg.log。
```

---

## 七、诊断包

导出内容：

```text
app.log 脱敏版
error.log 脱敏版
任务 task.log
task_steps 摘要
错误码和错误详情
配置摘要
sidecar 版本
系统信息
```

默认不包含：

```text
真实 API Key
Authorization Header
keyring 内容
用户原始小说全文
用户导入的私密媒体文件
绝对路径
```

可选包含：

```text
项目 manifest
模板文件
失败产物截图
FFmpeg stderr 摘要
```

---

## 八、脱敏规则

必须替换：

```text
sk-xxxx → sk-***
Bearer xxxx → Bearer ***
C:\Users\Name\... → <user_path>/...
D:\private\... → <local_path>/...
```

密钥检测：

```text
1. 写日志前 SecretGuard 扫描。
2. 导出诊断包前二次扫描。
3. 命中疑似密钥时阻断导出并提示用户。
```
