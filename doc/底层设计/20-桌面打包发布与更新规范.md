# 桌面打包发布与更新规范

> 这篇定义 Tauri 桌面应用打包、sidecar、数据目录、首次启动、更新和备份恢复。桌面软件不能只考虑开发环境能跑。

## 一、支持范围

第一版优先：

```text
Windows 10/11 x64
```

后续再评估：

```text
macOS Apple Silicon / Intel
Linux x64
```

---

## 二、打包内容

必须包含：

```text
Tauri app
前端静态资源
Rust 后端
FFmpeg sidecar
FFprobe sidecar
内置模板
内置 Prompt/Skill
内置字体/图标资源
默认配置 schema
数据库 migrations
```

Chromium：

```text
使用 Playwright sidecar 控制随包 Chromium，必须明确 driver、浏览器可执行文件、授权和离线打包策略。
第一版建议固定随应用打包或首次启动受控下载，不允许运行时随意联网拉未知二进制。
```

---

## 三、应用数据目录

不要把用户数据写进安装目录。

推荐：

```text
AppData/Roaming/vt-ai-short-video-maker/
  workspace/
  config/
  logs/
  db/app.sqlite
```

工作区可由用户迁移，但迁移必须通过设置页受控操作。

---

## 四、首次启动

流程：

```text
1. 创建应用数据目录。
2. 初始化 SQLite。
3. 执行 migrations。
4. 初始化默认配置。
5. 初始化内置字典。
6. 扫描内置模板和 Skill。
7. 检查 sidecar。
8. 进入欢迎/设置引导。
```

---

## 五、sidecar 自检

启动自检：

```text
ffmpeg -version
ffprobe -version
Chromium 可启动
模板默认截图测试（可延迟执行）
```

失败：

```text
1. 不影响打开应用。
2. 设置页显示缺失项。
3. 相关任务禁止启动。
```

---

## 六、自动更新

第一版可以先不做自动更新，但必须保留设计：

```text
1. 更新前备份数据库。
2. 更新后执行 migration。
3. migration 失败时提示恢复。
4. sidecar 版本随应用版本管理。
```

---

## 七、备份恢复

备份内容：

```text
SQLite 数据库
workspace/projects
workspace/assets
用户模板
用户 Skill
配置文件
```

不备份：

```text
真实 API Key
keyring 内容
临时 cache
可重新生成的 temp
```

恢复后：

```text
1. 检查路径。
2. 重建 asset 引用。
3. 检查 Provider key_alias 是否可用。
4. 不可用则提示用户重新输入密钥。
```

---

## 八、代码签名和安全

发布版要求：

```text
1. 安装包签名。
2. sidecar 来源固定。
3. 不执行用户自定义二进制。
4. 更新源 HTTPS。
5. 不在安装目录写用户数据。
```

---

## 九、授权合规

必须确认：

```text
FFmpeg 分发授权
Playwright driver / Chromium 授权
字体授权
模板素材授权
参考项目代码许可证边界
```

文档参考可以借鉴思路，不能直接复制不兼容许可证代码。
