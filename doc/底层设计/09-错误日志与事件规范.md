# 错误日志与事件规范

> 这篇定义错误结构、错误码、日志脱敏和进度事件。错误枚举取值以 `01-枚举字典与配置规范.md` 为准。

## 一、错误结构

后端统一错误 DTO：

```rust
pub struct AppErrorDto {
    pub code: String,
    pub kind: ErrorKind,
    pub message: String,
    pub detail: Option<String>,
    pub is_retryable: bool,
}
```

前端统一错误结构：

```ts
export interface AppError {
  code: string
  kind: ErrorKind
  message: string
  detail?: string
  isRetryable: boolean
}
```

## 二、错误码规范

错误码格式：

```text
领域.具体错误
```

示例：

```text
config.missing
provider.auth_failed
provider.rate_limited
task.step_failed
storage.path_denied
ffmpeg.failed
template.render_failed
```

完整错误分类见 `01-枚举字典与配置规范.md`。

## 三、前端错误展示

```text
Toast：短错误，比如保存失败
Dialog：需要用户处理，比如 API Key 无效
LogPanel：完整错误和技术细节
TaskDetail：任务步骤错误、重试入口
```

前端展示优先级：

```text
1. 用 error.code 映射 i18n 文案。
2. 找不到 i18n 时显示 error.message。
3. detail 只在详情或日志面板展示。
```

## 四、日志脱敏

禁止打印：

```text
API Key
Bearer Token
完整密钥
完整 Provider 请求头
用户敏感文件全文
超长原始小说文本
```

允许打印：

```text
key_alias
provider_id
provider_kind
vendor
model_name
错误码
任务 ID
步骤 ID
相对路径
耗时
重试次数
```

## 五、ProgressEvent

任务进度事件通过 Tauri event 推送。

建议结构：

```json
{
  "task_id": "task_xxx",
  "step_kind": "image_generation",
  "step_status": "running",
  "progress": 0.45,
  "item_id": "item_003",
  "message_key": "progress.image.generating",
  "created_at": 1718000000000
}
```

规则：

```text
1. progress 范围 0~1。
2. step_kind / step_status 使用统一枚举。
3. 用户可见文案用 message_key，不长期依赖后端中文。
4. 事件只做通知，最终状态以数据库为准。
```

## 六、任务失败记录

TaskStep 失败时必须记录：

```text
step_kind
step_status=failed
error_json
retry_count
finished_at
```

可重试错误要保留：

```text
原始错误码
Provider 响应摘要
是否可重试
下一次重试建议
```

不可重试错误要清楚提示用户处理方式，如重新配置 Key、降低并发、修改 prompt。
