# 素材导入与素材成片

> 这是给 AI 开发用的功能说明书。读完这一篇，就能把“用户已有图片/视频素材 → 自动组织成短视频”这个入口从零写出来。
> 配套阅读：资产管理看 `20-资产库与媒体管理.md`；VLM/Provider 看 `底层设计/12-Provider接口契约与能力矩阵Schema.md`；任务执行看 `底层设计/11-PipelineEngine与持久化队列规范.md`。

## 一、这个功能是什么

素材导入与素材成片用于用户已经有图片、视频、产品图、人物图、素材包的场景。系统先分析素材，再把素材分配给分镜，最后生成口播、字幕、模板和成片。

典型场景：

```text
1. 用户上传 10 张产品图 → 自动生成带字幕的产品介绍短视频。
2. 用户上传一批旅行照片 → 自动生成旅行回忆短视频。
3. 用户上传几段视频素材 → 自动剪成口播/混剪视频。
```

1. 批量导入图片/视频

5. 素材绑定 StoryboardItem

## 子功能 1：批量导入图片/视频

### 界面

- 拖拽上传区域。
- 支持多选文件。
- 显示缩略图、文件名、类型、大小、时长。
- 支持删除、重命名、标记主素材。

### 系统逻辑

1. 用户选择文件。
2. 后端通过 PathGuard 校验原始路径。
3. 复制到 `workspace/assets` 或当前项目 `input/`。
4. 生成 Asset 记录。
5. 图片读取宽高，视频读取时长和首帧。

### 数据写入

```json
{
  "asset_id": "asset_xxx",
  "asset_kind": "user_media",
  "media_kind": "image",
  "relative_path": "assets/images/product_001.png",
  "preview_path": "assets/images/product_001_preview.png",
  "metadata_json": {}
}
```

---

## 子功能 2：素材 VLM 分析

### 系统逻辑

1. 对图片或视频首帧调用 VLM。
2. 输出结构化描述。
3. 写入 `assets.metadata_json.analysis`。
4. 用户可以编辑分析结果。

### 输出示例

```json
{
  "description": "一瓶白色包装的护肤精华，放在浅色桌面上",
  "tags": ["护肤品", "白色包装", "桌面", "产品图"],
  "dominant_colors": ["white", "beige"],
  "has_person": false,
  "usable_as": ["product", "cover", "scene_visual"]
}
```

### 坑提醒

- VLM 分析是建议，不是事实。重要字段需要用户确认。
- 不能把远程 URL 作为素材路径入库。

---

## 子功能 3：素材分组和筛选

### 界面

- 全部素材。
- 图片 / 视频 / 音频分类。
- 标签筛选。
- 可标记：主图、封面候选、产品图、人物图、背景图。

### 系统逻辑

1. 根据 VLM tags 生成初始分类。
2. 用户确认后写入 Asset metadata。
3. 后续脚本和分镜优先使用被标记素材。

---

## 子功能 4：基于素材生成脚本和分镜

### 系统逻辑

1. 读取用户补充说明、目标视频类型、目标时长。
2. 读取素材分析摘要。
3. 调 LLM 生成脚本和 StoryboardItem[]。
4. 每个 StoryboardItem 必须绑定至少一个素材或说明无需素材。
5. 进入 storyboard_review。

### 结构化输出

```json
{
  "title": "这款精华为什么适合通勤党",
  "storyboard_items": [
    {
      "item_index": 1,
      "narration": "很多人护肤最大的问题，不是产品不够多，而是早上真的没时间。",
      "asset_ids": ["asset_product_001"],
      "visual_goal": "展示产品外观和使用场景"
    }
  ]
}
```

## 子功能 5：素材绑定 StoryboardItem

StoryboardItem 扩展使用：

```json
{
  "item_id": "item_001",
  "source_asset_ids": ["asset_xxx"],
  "selected_image_id": "img_asset_001",
  "selected_video_segment_id": null
}
```

规则：

```text
1. 如果 StoryboardItem 使用已有图片，可以创建来源为素材的 ImageCandidate，并跳过 image_generation。
2. 如果 StoryboardItem 使用已有视频，可以创建来源为素材的 VideoSegment，并走音画对齐。
3. 素材绑定关系写 asset_references。
```

## 子功能 6：素材成片任务

TaskKind：

```text
asset_based_pipeline
```

推荐 Step：

```text
asset_analysis
script_generation
script_review
storyboard_generation
storyboard_review
tts_generation
subtitle_generation
template_rendering
segment_composition
final_composition
export
```

说明：

```text
素材成片默认不强制 AI 生图，因为素材本身就是画面来源。
如果用户要求补图，再进入 image_generation。
```
