# 脚本生成

> 这是给 AI 开发用的功能说明书。读完这一篇，就能把“脚本生成 / 文本预处理”模块从零写出来，不用再问。
> 配套阅读：字段定义看 `底层设计/数据结构.md`；模型调用看 `底层设计/Provider与安全.md`；内容来源看 `功能模块/01-内容导入.md`；当前开发顺序看项目 `plan/阶段路线图.md`。

## 一、这个功能是什么

脚本生成模块把内容导入模块输出的归一化输入，变成更适合分镜使用的「脚本 / 旁白段落 / 文本段」。

重要边界：

```text
当前 image_to_video 主线不强制经过独立脚本页。
开始创作后进入作品工作台，内容创作确认后可以直接进入分镜，由分镜模块基于输入文字生成 StoryboardItem[]。
脚本生成是文本预处理 / 高级编辑 / 后续增强能力。
```

它不是取消，而是优先级变化：

```text
当前主线：输入文字 → 分镜 → 生图 → 图生视频 → 合成
增强链路：输入文字 → 脚本生成/编辑 → 分镜 → 生图 → 图生视频/字幕/TTS/合成
```

---

## 二、输入类型策略

| 输入类型 | 处理方式 | 输出 | 当前主线关系 |
|---|---|---|---|
| 主题 topic | AI 发散生成 N 段短视频旁白 | narrations[] | 可作为分镜生成前的可选预处理 |
| 文章 article | AI 浓缩提炼成 N 段旁白 | narrations[] | 可作为分镜生成前的可选预处理 |
| 文案 paste | 不改写，只使用已切好的文本段 | narrations[] / textSegments[] | 当前主线可直接作为 StoryboardItem.sourceText |
| 小说 novel | 基于章节事件生成剧情脚本 | script / episodes / narrations[] | 后续长内容 workflow |

核心原则：

```text
短视频脚本走 Pixelle 的结构化 JSON 直出。
长内容/小说改编走 Toonflow 的分层 Agent 思路。
paste 固定文案模式绝对不能 AI 改写。
```

---

## 三、子功能清单

1. 主题生成短视频脚本
2. 文章 / 笔记浓缩脚本
3. 固定文案通过确认
4. 小说事件生成剧情脚本
5. 脚本编辑与确认
6. 标题生成

---

## 子功能 1：主题生成短视频脚本

### 界面

如果启用独立脚本页：

- 顶部显示来自内容导入的主题、内容类别、目标时长、目标分镜数。
- 中间是「生成中的脚本段落列表」。
- 每段显示：段号、旁白文本、预估字数、预估时长。
- 底部按钮：重新生成 / 确认脚本 / 进入分镜。

如果不启用独立脚本页：

- 分镜模块可内部调用本能力，直接生成 StoryboardItem[]。

### 用户操作

```text
可选脚本增强：
用户从主题输入进入 → 系统生成 N 段旁白 → 用户检查 → 确认 → 进入分镜

当前主线：
用户从主题输入进入 → 直接进入分镜 → 分镜模块生成分镜行
```

### 系统逻辑

1. 读取 `Project.input_type=topic`。
2. 取 `topic / content_category / target_duration_seconds / target_scene_count / content_language`。
3. 根据目标时长计算每段字数范围。
4. 调 LLM，要求返回 JSON：`{ "narrations": ["..."] }`。
5. 校验返回数量：
   - 多于 `target_scene_count`：截断。
   - 少于 `target_scene_count`：自动重试一次；还少则失败并提示用户调小段数。
6. 写入脚本草稿产物，例如 `script_generation.output_json.draft_narrations`。
7. 如果启用脚本确认，则用户确认后写入 `script_review.output_json.confirmed_narrations`。
8. 如果不启用脚本页，则分镜模块可直接读取草稿或生成结果，转成 StoryboardItem.sourceText / narrationText。

### 数据结构示意

```json
{
  "project_id": "proj_xxx",
  "input_type": "topic",
  "input_process_mode": "generate",
  "topic": "为什么要早睡",
  "target_scene_count": 8,
  "narrations": [
    "很多人以为早睡只是自律，其实它先改变的是你的清醒感",
    "当你不再熬夜，大脑会有更多时间修复白天的疲惫"
  ],
  "script_status": "succeeded"
}
```

### 参考来源

- Pixelle `pixelle_video/utils/content_generators.py` → `generate_narrations_from_topic(...)`。
- Pixelle `pixelle_video/prompts/topic_narration.py` → `build_topic_narration_prompt(...)`。
- Pixelle 校验逻辑：`narrations` 数量多则截断，少则抛错。

### 坑提醒

- 不要把主题当文章处理。主题是发散生成，文章是浓缩提炼。
- 不要让 AI 返回 Markdown 段落，必须 JSON，否则后续分镜解析不稳。
- 不要生成太长旁白；TTS 和字幕增强会受影响。

## 子功能 2：文章 / 笔记浓缩脚本

### 界面

- 左侧显示原文摘要和字数。
- 右侧显示 AI 生成的短视频旁白段落。
- 提供「保留原文重点」提示开关。
- 按钮：重新浓缩 / 确认脚本 / 进入分镜。

### 系统逻辑

1. 读取 `Project.input_type=article` 和 `source_text / source_text_path`。
2. 检查原文长度，超长则提示改用小说导入或分段处理。
3. 调 `generate_narrations_from_content` 同类逻辑，要求 AI 只基于原文提炼，不杜撰。
4. 校验 `narrations` 数量。
5. 写入 `script_generation.output_json.draft_narrations`。
6. 如果启用脚本确认，则确认后写入 `script_review.output_json.confirmed_narrations`。
7. 当前主线也允许分镜模块直接读取原文并生成 StoryboardItem，不强制独立脚本页。

### 参考来源

- Pixelle `pixelle_video/utils/content_generators.py` → `generate_narrations_from_content(...)`。
- Pixelle `pixelle_video/prompts/content_narration.py` → `build_content_narration_prompt(...)`。

### 坑提醒

- 文章脚本不能“自由发挥”，必须围绕原文。
- 原文过长时不要一次塞进模型上下文，后续需要分块摘要策略。

## 子功能 3：固定文案通过确认

### 界面

- 展示由内容导入切好的段落列表。
- 每段可编辑、合并、拆分、删除。
- 显示「固定文案模式：不会 AI 改写」。

### 系统逻辑

1. 读取 `input_process_mode=fixed`。
2. 不调用 LLM。
3. 切分后的文本可以直接作为 StoryboardItem.sourceText。
4. 如果启用脚本页，也可以创建脚本草稿，但内容必须原样保留。
5. 校验每段非空。

### 参考来源

- Pixelle `pixelle_video/utils/content_generators.py` → `split_narration_script(...)`。

### 坑提醒

- 绝不能“顺手优化文案”。固定文案模式的承诺就是一字不改。

## 子功能 4：小说事件生成剧情脚本

### 界面

- 左侧：章节 / 事件列表。
- 右侧：生成的剧情脚本或分集脚本。
- 支持选择章节范围。
- 按钮：生成本集脚本 / 重新生成 / 通过。

### 系统逻辑

1. 读取 NovelChapter[] 中 `event_status=succeeded` 的章节事件。
2. 按用户选择的章节范围拼接事件序列。
3. 先生成故事骨架，再生成改编策略，再生成具体脚本。
4. 每次最多处理有限章节，避免上下文过长。
5. 输出结构化脚本，后续再拆分镜。

### 参考来源

- Toonflow `src/agents/scriptAgent/index.ts`：ScriptAgent 总入口。
- Toonflow 旧拆解结论：故事骨架 → 改编策略 → 逐集剧本。
- Toonflow `src/agent/skillsTools.ts`：Skill 文件加载机制。

### 坑提醒

- 不要一次把整本小说塞给 LLM。
- 事件必须结构化，否则角色/地点/冲突无法稳定传给分镜。
- 长内容脚本失败要能从当前章节范围重试。

---

## 子功能 5：脚本编辑与确认

### 界面

- 段落卡片列表。
- 每段可编辑旁白。
- 支持重新生成单段。
- 支持锁定段落。
- 底部「确认脚本，进入分镜」。

### 系统逻辑

1. 用户编辑时只改脚本文本，不影响原始输入。
2. 用户锁定段落后，批量重生成不能覆盖。
3. 如果使用 `script_review`，点击确认后从 `waiting_user` 置为 `succeeded`。
4. 当前主线不强制 `script_review`，分镜可直接从 Project 输入生成。

### 坑提醒

- 不要把脚本确认写死成所有 workflow 的前置条件。
- 如果 workflow 启用了脚本确认，则确认前不能把脚本草稿当最终输入。

---

## 子功能 6：标题生成

### 系统逻辑

1. 如果用户填了标题，优先用用户标题。
2. 如果没有标题：
   - 主题很短：可直接截断作为标题。
   - 文章/长文：调 LLM 生成 15 字以内标题。
3. 标题用于历史列表、封面增强、导出元数据。

### 参考来源

- Pixelle `pixelle_video/utils/content_generators.py` → `generate_title(...)`。
- Pixelle `pixelle_video/prompts/title_generation.py`。

### 坑提醒

- 标题不要过长，封面排版会崩。
- 不要生成标题党到偏离原内容。

## 四、这个模块对外提供什么

可选输出给分镜模块：

```json
{
  "project_id": "proj_xxx",
  "title": "为什么普通人要早睡",
  "content_language": "zh-CN",
  "content_category": "knowledge",
  "narrations": ["第一段旁白", "第二段旁白"],
  "script_review_status": "succeeded"
}
```

当前主线说明：

```text
没有脚本输出时，03-分镜.md 直接读取 Project.sourceText / topic 生成 StoryboardItem[]。
有脚本输出时，03-分镜.md 可把 narrations[] 映射为 StoryboardItem.sourceText / narrationText。
```
