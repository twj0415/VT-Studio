# 视频合成

> 这是给 AI 开发用的功能说明书。读完这一篇，就能把“视频合成”模块从零写出来，不用再问。
> 配套阅读：技术栈看 `底层设计/技术栈与架构.md`；文件安全看 `底层设计/Provider与安全.md`；FFmpeg 细则看 `底层设计/13-FFmpegSidecar与媒体处理规范.md`。

## 一、这个功能是什么

视频合成模块负责把已确认的视频片段按分镜顺序合成为最终 MP4。

当前 `image_to_video` 主线中，合成的核心输入是：

```text
StoryboardItem.selectedVideoSegmentId
VideoSegment.videoPath
VideoSegment.durationSeconds
```

当前主线输出：

```text
CompositionTask.outputPath
exports/final.mp4
```

TTS、字幕、封面、BGM、模板动效不是不做，而是后续增强项；它们可以参与合成，但不能阻塞当前主线的“多个图生视频片段 → final.mp4”。

核心方案：FFmpeg / FFprobe sidecar。

---

## 二、子功能清单

1. 环境检测
2. 主线片段拼接
3. 片段格式检查与转码兜底
4. 合成任务进度
5. 输出最终 MP4
6. BGM / 字幕 / 封面增强（后续 TODO）

---

## 子功能 1：环境检测

### 系统逻辑

1. 启动时检查 `ffmpeg.exe / ffprobe.exe` 是否存在。
2. 检查是否可执行。
3. 读取版本。
4. 缺失则在设置页明确提示。
5. 合成前再次确认 sidecar 可用，避免最后一步才报模糊错误。

### 参考来源

- Pixelle `pixelle_video/services/video.py::check_ffmpeg`：检查系统 FFmpeg。

### 我们的改动

Pixelle 依赖系统安装 FFmpeg；我们桌面端要 sidecar 打包。`resources/bin` 是打包源目录，运行时由 StorageService 解析到 FileBucket `sidecars`：

```text
resources/bin/ffmpeg.exe
resources/bin/ffprobe.exe
→ sidecars/ffmpeg.exe
→ sidecars/ffprobe.exe
```

---

## 子功能 2：主线片段拼接

### 系统逻辑

1. 读取 `StoryboardItem[]`。
2. 校验每条 item 都有 `selectedVideoSegmentId`。
3. 根据 selectedVideoSegmentId 找到 `VideoSegment.videoPath`。
4. 按 StoryboardItem.index 排序。
5. 收集所有视频片段路径。
6. 优先用 concat demuxer 快速拼接。
7. 如果编码、分辨率、帧率不一致，降级转码或 concat filter。
8. 输出 `exports/final.mp4`。
9. 写入 `CompositionTask.outputPath`。

### 参考来源

- Pixelle `pixelle_video/services/video.py::concat_videos`。
- Pixelle `_concat_demuxer` / `_concat_filter`。

### 坑提醒

- 拼接顺序只能按 StoryboardItem.index，不能按文件名字符串排序。
- 所有路径必须过 StorageService / PathGuard。
- 入库只存工作区相对路径。

---

## 子功能 3：片段格式检查与转码兜底

### 系统逻辑

1. 用 ffprobe 读取每个 VideoSegment 的编码、分辨率、帧率、时长。
2. 如果全部一致，走快速 concat。
3. 如果不一致，进入转码兜底。
4. 转码输出临时规范片段。
5. 再拼接最终视频。

### 坑提醒

- 不同图生视频 Provider 输出格式可能不一致，不能假设都能直接 concat。
- 临时文件必须放在任务工作区，不能散落到用户目录。
- 失败时保留已完成片段，方便断点续跑。

---

## 子功能 4：合成任务进度

### 界面

合成页显示：

```text
片段检查清单
总片段数
已确认片段数
输出路径
合成进度
失败原因
重新合成
打开输出目录
```

### 系统逻辑

1. 创建 CompositionTask。
2. 状态从 pending → running → succeeded / failed。
3. 进度按检查、转码、拼接、写入结果分段上报。
4. 失败时记录错误码和可恢复建议。

### 坑提醒

- 不要只显示“合成失败”，必须说明是 sidecar 缺失、路径无效、片段损坏还是编码不一致。

---

## 子功能 5：输出最终 MP4

### 系统逻辑

1. 确认所有 StoryboardItem 都有 selectedVideoSegmentId。
2. 拼接视频片段。
3. 输出到 `exports/final.mp4`。
4. 写历史记录。
5. 前端显示预览和打开目录按钮。

### 数据结构

```json
{
  "taskId": "task_xxx",
  "projectId": "proj_xxx",
  "segmentIds": ["seg_001", "seg_002"],
  "outputPath": "exports/final.mp4",
  "status": "succeeded",
  "progress": 100
}
```

### 坑提醒

- 输出路径必须过 PathGuard。
- 合成失败要保留已完成片段，方便断点续跑。
- 不要把本地绝对路径写入导出包或诊断包。

## 子功能 6：BGM / 字幕 / 封面增强（后续 TODO）

这些不是不做，而是后续增强：

```text
09 配音 TTS
10 字幕
11 封面
12 模板与动效
BGM 混音
```

### BGM

- BGM 开关。
- BGM 选择。
- 音量滑块，默认低于人声。
- 循环模式：loop / once。
- 淡入淡出。

参考：

- Pixelle `pixelle_video/services/video.py::add_bgm`。
- Pixelle `pixelle_video/services/video.py::_add_bgm_to_video`。

### 字幕

- 可以基于分镜文案或 TTS 结果生成字幕。
- 可选择是否烧录进最终视频。
- 不应阻塞无字幕的主线合成。

### 封面

- 可以从已选图或视频首帧生成封面。
- 可在导出阶段关联 `cover.png`。
- 不应阻塞 final.mp4 输出。

---

## 三、这个模块对外提供什么

写回：

```text
CompositionTask.outputPath
TaskResult.finalVideoPath
历史记录
```

最终用户拿到：

```text
final.mp4
可选 cover.png
可选 subtitles
project metadata
```
