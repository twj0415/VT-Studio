# 导出与项目备份

> 这是给 AI 开发用的功能说明书。导出不仅是 final.mp4，还包括封面、字幕、项目包、历史任务和备份恢复。
> 配套阅读：文件规范看 `底层设计/08-文件存储与路径规范.md`；打包发布看 `底层设计/20-桌面打包发布与更新规范.md`。

## 一、这个功能是什么

导出与项目备份负责把项目结果安全地交给用户，同时保证项目可迁移、可恢复、可回看。

支持：

```text
导出 final.mp4
导出封面 cover.png
导出字幕 subtitles.json / srt
导出项目包
导入项目包
备份当前工作区
恢复备份
```

## 子功能 1：导出视频

### 界面

- 选择导出位置。
- 显示视频格式、分辨率、时长、大小。
- 打开目录按钮。

### 系统逻辑

1. 确认任务已成功。
2. 找到 `exports/final.mp4`。
3. PathGuard 校验目标路径。
4. 复制到用户选择位置。
5. 写历史记录。

---

## 子功能 2：导出字幕和封面

可选项：

```text
include_cover
include_subtitles_json
include_srt
```

输出：

```text
final.mp4
cover.png
subtitles.json
subtitles.srt
```

规则：

```text
1. 字幕 json 用项目内部格式。
2. srt 用通用格式。
3. 文件名默认跟项目标题，但要清理非法字符。
```

## 子功能 3：导出项目包

项目包结构：

```text
project_package.zip
  manifest.json
  db_export.json
  projects/proj_xxx/
    project.json        # 导出快照，不是本地运行时主数据源
  assets/
  templates/user/
```

本地运行时以 SQLite 为准；项目包用于迁移和恢复，导入时必须把 `db_export.json` 写回 SQLite。

manifest：

```json
{
  "app": "vt-ai-short-video-maker",
  "package_version": 1,
  "project_id": "proj_xxx",
  "title": "项目标题",
  "created_at": 0,
  "contains_secrets": false
}
```

禁止导出：

```text
真实 API Key
keyring 内容
Authorization header
临时 cache
系统绝对路径
```

## 子功能 4：导入项目包

流程：

```text
1. 选择 zip。
2. 校验 manifest。
3. 解压到 temp。
4. 防 Zip Slip 路径穿越。
5. 从 db_export.json 导入项目数据到 SQLite。
6. 重写 project_id 或处理冲突。
7. 复制 assets/templates。
8. 重新生成必要的 project.json 快照。
9. 提示用户重新配置 Provider 密钥。
```

规则：

```text
1. 导入包里的路径必须是相对路径。
2. 不信任包内任何脚本。
3. 不自动执行包内模板 JS。
```

## 子功能 5：工作区备份恢复

备份包含：

```text
SQLite 数据库
workspace/projects
workspace/assets
workspace/templates/user
workspace/prompts/user
```

不包含：

```text
真实密钥
cache
temp
logs 可选
```

恢复后检查：

```text
1. migration。
2. 文件引用是否存在。
3. Provider key_alias 是否可用。
4. sidecar 是否正常。
```

---

## 子功能 6：导出安全检查

导出前必须扫描：

```text
1. 是否包含疑似 API Key。
2. 是否包含绝对路径。
3. 是否包含工作区外文件引用。
4. zip 路径是否安全。
```

命中风险：

```text
阻止导出
提示具体风险
允许用户重新生成安全包
```
