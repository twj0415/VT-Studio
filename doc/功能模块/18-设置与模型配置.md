# 设置与模型配置

> 这是给 AI 开发用的功能说明书。读完这一篇，就能把“设置与模型配置”模块从零写出来。
> 配套阅读：Provider 安全看 `底层设计/Provider与安全.md`；模型适配和 workflow preset 看 `底层设计/23-模型适配与工作流注册规范.md`；配置分层和枚举看 `底层设计/01-枚举字典与配置规范.md`；主题和语言看 `底层设计/04-UI主题与国际化规范.md`。

## 一、这个功能是什么

设置与模型配置负责管理：

```text
AppConfig       软件偏好：界面语言、主题、密度
SystemConfig    工作区、输出目录、sidecar 路径
ProviderConfig  供应商连接配置、密钥别名、base_url
ProviderModel   API 模型能力矩阵
WorkflowPreset  ComfyUI / RunningHub 工作流预设
PipelineConfig  默认生成参数、并发数、重试次数
UiConfig        高级选项、布局偏好
ExportConfig    导出格式、码率、分辨率策略
```

同时负责：

```text
LLM / TTS / Image / Video / VLM / workflow Provider
ComfyUI / RunningHub workflow vendor
密钥管理
API 模型连通性测试
Workflow preset dry-run / real-generate 测试
路径和 sidecar 检测
配置导入导出
```

核心红线：**真实 API Key 不进 SQLite，不进 yaml/json，不进导出文件，只进系统钥匙串。**

模型配置分三块：

```text
Provider 连接配置       providers
API 模型能力配置        provider_models
Workflow Preset 配置    workflow_presets
```

这三块不能混在一个表单里。Provider 连接只解决“连到谁”，API 模型能力解决“这个 API 模型能做什么”，workflow preset 解决“这个 ComfyUI/RunningHub 工作流怎么执行”。

## 子功能 1：应用偏好设置

### 字段

```json
{
  "app_locale": "zh-CN",
  "theme_preset": "graphite",
  "layout_density": "comfortable"
}
```

### 系统逻辑

1. 用户修改界面语言、主题、密度。
2. 写入 AppConfig。
3. 前端即时切换 UI。
4. 不影响项目的 `content_language`。

### 坑提醒

- `app_locale` 是软件界面语言。
- `content_language` 是视频内容语言。
- 两者不能混用。

## 子功能 2：系统路径和 sidecar 检测

### 检测项

```text
workspace_dir
output_dir
template_dir
ffmpeg_path
ffprobe_path
chromium_path
```

### 系统逻辑

1. 启动时检测。
2. 设置页显示状态。
3. 缺失时给明确修复提示。
4. 路径写入 SystemConfig。
5. 所有路径最终必须通过 PathGuard。

### 坑提醒

- 不要等用户合成到最后一步才发现 FFmpeg 不存在。
- 不要让前端传任意路径直接给 FFmpeg 使用。

## 子功能 3：Provider 配置

统一 ProviderConfig：

```json
{
  "provider_id": "provider_xxx",
  "provider_kind": "llm",
  "vendor": "openai_compatible",
  "display_name": "DeepSeek",
  "base_url": "https://api.deepseek.com/v1",
  "key_alias": "deepseek_main",
  "is_enabled": true,
  "config": {
    "temperature": 0.7,
    "max_tokens": 4096
  }
}
```

ProviderKind：

```text
llm / tts / image / video / vlm / workflow
```

ComfyUI / RunningHub 是：

```text
provider_kind = workflow
vendor = comfyui / runninghub
```

Provider 配置只保存连接信息，不保存 API 模型能力，也不保存 workflow 参数。

```text
providers.config_json      供应商连接配置
provider_models            API 模型能力
workflow_presets           ComfyUI / RunningHub workflow preset
```

### 参考项目取舍

- 借鉴 Pixelle `pixelle_video/services/llm_service.py::LLMService` 的 OpenAI-compatible 接入。
- 借鉴 Pixelle `pixelle_video/config/manager.py::ConfigManager` 的配置更新思路。
- 借鉴 Toonflow `src/routes/setting/vendorConfig/*` 的供应商设置页结构。
- 不照搬 Pixelle / Toonflow 的明文密钥存储。
- 不照搬 Toonflow 的用户 TS Provider 热执行机制。

### 坑提醒

- Pixelle 的 config.yaml 明文存 api_key，不能照搬。
- Toonflow vendorConfig 明文存 key，也不能照搬。
- ProviderConfig 不允许保存 `model_name`，具体模型进入 `provider_models`。

## 子功能 4：模型能力矩阵

每个 API 模型必须声明能力，后端校验，不能只靠前端 UI 禁用。

示例：

```json
{
  "provider_model_id": "pm_xxx",
  "provider_id": "provider_xxx",
  "provider_kind": "video",
  "vendor": "dashscope",
  "model_name": "wanx-video",
  "ability_types": ["text_to_video", "image_to_video"],
  "input_modalities": ["text", "image"],
  "output_modalities": ["video"],
  "feature_flags": [
    "reference_image",
    "duration",
    "resolution",
    "aspect_ratio"
  ],
  "limits": {
    "supported_aspect_ratios": ["vertical_9_16", "horizontal_16_9", "square_1_1"],
    "duration_seconds": { "min": 2, "max": 15, "integer": true },
    "resolutions": ["720p", "1080p"]
  }
}
```

规则：

```text
1. provider_models 只登记 API 模型，不登记 ComfyUI / RunningHub workflow。
2. 每个 provider_model 保存 ability_types / input_modalities / output_modalities / feature_flags / limits。
3. 设置页保存前必须做 schema 校验。
4. 前端模型选择器不直接读 provider_models，而是读 list_executable_media_options。
```

这里的 `model_name` 是 `provider_models` 的供应商侧真实模型名，只能由 `ModelRegistry / ProviderAdapter` 使用；业务功能不得按它写分支。

### 参考项目取舍

- 借鉴 Pixelle `pixelle_video/services/api_media.py::APIProviderMediaService.VIDEO_MODEL_CAPABILITIES` 的能力声明。
- 不照搬硬编码矩阵，第一版落到 `provider_models` 并由后端校验。

## 子功能 5：Workflow Preset 配置

Workflow Preset 用来注册 ComfyUI / RunningHub 工作流。

示例：

```json
{
  "workflow_preset_id": "wf_img_kontext_v1",
  "provider_id": "provider_comfyui_local",
  "vendor": "comfyui",
  "workflow_key": "selfhost/image_kontext_v1",
  "display_name": "Kontext 生图",
  "workflow_version": "1.0.0",
  "ability_types": ["text_to_image", "image_to_image"],
  "input_modalities": ["text", "image"],
  "output_modalities": ["image"],
  "limits": {
    "supported_aspect_ratios": ["vertical_9_16", "horizontal_16_9", "square_1_1"]
  },
  "param_schema": {},
  "node_map": {},
  "output_map": {}
}
```

系统逻辑：

```text
1. 内置 preset 由 scan_builtin_workflow_presets 扫描 resources/workflows。
2. 用户新增 preset 时必须选择 provider_id 和 vendor。
3. 保存前校验 param_schema / node_map / output_map。
4. RunningHub preset 必须填写 workflow_id。
5. ComfyUI selfhost preset 必须能解析 workflow_key 对应文件。
6. 启用后的 preset 才能出现在 list_executable_media_options。
```

坑提醒：

- 不允许执行未注册 workflow。
- 不允许设置页上传 JS / TS / Python Provider 代码。
- Workflow Preset 不保存 API Key，只引用 provider_id 对应的 key_alias。

## 子功能 6：密钥管理

### 系统逻辑

1. 用户新增 Key。
2. 后端生成 `key_alias`。
3. 真实 Key 写系统钥匙串。
4. DB 只保存 `provider_id / provider_kind / vendor / base_url / key_alias`。
5. 日志只打印 `key_alias`，不打印真实 Key。

### 技术选择

- Rust `keyring` crate。
- Windows Credential Manager。
- macOS Keychain。
- Linux Secret Service。

### 坑提醒

- 不要把 Key 写进日志、SQLite、yaml、json、导出文件。

## 子功能 7：模型连通性测试

### 界面

- 每个 Provider、API 模型、Workflow Preset 旁边有「测试」按钮。
- 测试按钮提供 `dry_run` 和 `real_generate` 两种模式。
- 显示成功/失败、耗时、错误原因。

### 系统逻辑

1. Provider 连接测试：只检查 base_url、鉴权、服务可达。
2. API 模型 `dry_run`：校验 provider_model 配置、能力矩阵、鉴权，不生成高成本媒体。
3. LLM `real_generate`：发送短 prompt。
4. TTS `real_generate`：生成一句测试音频。
5. Image `real_generate`：生成低成本小图。
6. Video `real_generate`：必须用户明确点击，默认不自动触发。
7. Workflow preset `dry_run`：检查服务可达、workflow 是否存在、param_schema / node_map / output_map 是否完整。
8. Workflow preset `real_generate`：用 sampleParams 真实生成样例媒体，并写入测试输出目录。

规则：

```text
1. 保存配置时只做 schema 校验，不自动真实生成。
2. 高成本 real_generate 必须二次确认。
3. 测试失败返回脱敏错误，不返回真实 API Key。
4. Workflow preset 测试必须调用 test_workflow_preset。
```

### 参考项目取舍

- 借鉴 Toonflow `src/routes/setting/vendorConfig/modelTest.ts` 的模型测试入口。
- 借鉴 Toonflow `src/routes/setting/vendorConfig/modelTest/imageTest.ts` 和 `videoTest.ts` 的按类型测试思路。
- 不照搬 Toonflow 的明文 key 和静默吞错。

## 子功能 8：Pipeline 默认参数

PipelineConfig 示例：

```json
{
  "default_content_category": "knowledge",
  "default_aspect_ratio": "vertical_9_16",
  "default_content_language": "zh-CN",
  "default_target_duration_seconds": 60,
  "default_target_scene_count": 8,
  "max_concurrent_provider_calls": 2,
  "retry_max_attempts": 3,
  "retry_backoff_seconds": 2
}
```

### 系统逻辑

1. 创建项目时读取默认值。
2. 用户可在项目里覆盖。
3. 创建任务时把运行参数写入任务快照。
4. 任务运行和断点续跑使用快照，不受后续设置变化影响。

---

## 子功能 9：配置导入导出

### 系统逻辑

1. 导出配置时不包含真实 API Key。
2. 导出 providers、provider_models、workflow_presets 和 `key_alias` 占位。
3. 导入后提示用户重新录入密钥。
4. 导入前校验配置版本、枚举 code、能力矩阵 schema、workflow preset schema。
5. 导入 workflow preset 时不执行 workflow，只保存为禁用状态，用户确认后再启用。

### 参考项目取舍

- 借鉴 Toonflow `src/routes/setting/dbConfig/exportData.ts` / `importData.ts` 的导入导出思路。
- 不照搬明文密钥导出。

### 坑提醒

- 导出文件泄露 Key 是严重安全事故。
