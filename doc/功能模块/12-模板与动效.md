# 模板与动效

> 这是给 AI 开发用的功能说明书。读完这一篇，就能把“模板与动效”模块从零写出来。
> 配套阅读：模板类型和参数类型看 `底层设计/01-枚举字典与配置规范.md`；主题 token 看 `底层设计/04-UI主题与国际化规范.md`；文件路径看 `底层设计/08-文件存储与路径规范.md`。

## 一、这个功能是什么

模板与动效模块负责把图片、字幕、标题等内容渲染成画面帧或图文视频片段。

它要做，但不是当前 `image_to_video` 主线的前置条件。

当前主线优先走：

```text
已选图片 → 图生视频 → 合成 final.mp4
```

模板与动效主要服务：

```text
1. 字幕 / 封面渲染
2. 纯图文成片 image_slideshow workflow
3. 不使用图生视频 Provider 时的降级方案
4. 后续包装、标题、轻动效增强
```

核心方案：**HTML 模板 + Playwright sidecar 控制随包 Chromium 截图**。

## 子功能 1：模板目录和分辨率

### 目录约定

```text
templates/
  vertical_9_16/
    frame_default.html
    frame_modern.html
    cover_default.html
  horizontal_16_9/
    frame_full.html
  square_1_1/
    frame_minimal_framed.html
```

### 系统逻辑

1. 根据项目 `aspectRatio` 选择模板目录。
2. `aspectRatio` 使用协议 code：`vertical_9_16 / horizontal_16_9 / square_1_1`。
3. 模板文件名按 TemplateType 区分：`frame_`、`cover_`、`subtitle_`、`transition_`、`layout_`。
4. 解析模板尺寸。

### 参考来源

- Pixelle `templates/1080x1920/*.html`、`templates/1920x1080/*.html`、`templates/1080x1080/*.html`。
- Pixelle `pixelle_video/utils/template_util.py::parse_template_size`。
- Pixelle `pixelle_video/utils/template_util.py::get_template_type`。

## 子功能 2：模板参数 DSL

### 语法

```text
{{param}}
{{param=default}}
{{param:type}}
{{param:type=default}}
```

支持类型以 `TemplateParamType` 为准：

```text
text / number / color / bool / select / image / font / range / json
```

### 系统逻辑

1. 扫描 HTML 模板中的参数占位符。
2. 忽略内置参数：title / text / image / index。
3. 解析 TemplateParamType。
4. 生成可编辑表单。
5. 用户修改后保存到 `templateParams`。
6. `select` 类型必须引用统一字典，不允许模板自己写散乱选项。
7. 颜色、字体、间距优先使用主题 token。

### 参考来源

- Pixelle `pixelle_video/services/frame_html.py::HTMLFrameGenerator.parse_template_parameters`。

### 坑提醒

- 模板参数名要语义明确，如 `subtitle_color`，不要叫 `color1`。
- 不要把用户可见中文写死在模板参数里，展示文案走 i18n。

## 子功能 3：单帧渲染

### 系统逻辑

1. 读取 StoryboardItem 的已选图片，或读取 ImageCandidate.imagePath。
2. 读取 StoryboardItem.subtitleChunks。
3. 读取模板 HTML。
4. 注入变量：title、text、image、index、subtitle 样式参数。
5. 通过 Playwright sidecar 打开随包 Chromium 渲染 HTML。
6. 截图输出 `rendered/item_001_frame.png`。
7. 写入 renderedFramePath 或 Artifact。
8. `renderStatus=succeeded`。

### 参考来源

- Pixelle `pixelle_video/services/frame_html.py::HTMLFrameGenerator.generate_frame`。
- Pixelle `pixelle_video/services/frame_processor.py::_step_compose_frame`。

### 坑提醒

- 模板里引用的图片路径必须是 PathGuard 允许的本地路径。
- 渲染过程不能联网加载未知资源。
- 模板资源必须来自 `templates` 或项目允许目录。

## 子功能 4：基础动效参数

### 默认动效

先做稳定、简单的动效：

- 轻微推近 zoom in
- 轻微平移 pan
- 淡入淡出 fade
- 字幕逐字高亮
- 背景模糊 / 遮罩

### 系统逻辑

1. 动效参数写进模板参数或 StoryboardItem 的高级可选字段。
2. 单帧截图只产基础画面。
3. 图文视频片段可由 `13-视频合成.md` 用 FFmpeg 或 HTML 序列处理。
4. 动效类型使用统一 `TransitionType` 或模板参数字典。

### 坑提醒

- 不要为了“高级感”堆无意义动效。动效服务内容节奏。
- 动效时长不能写死，必须跟 StoryboardItem.durationSeconds 协调。
- 不要把模板渲染变成当前 image_to_video 主线的合成前置条件。

## 子功能 5：渲染失败重试

### 系统逻辑

1. Playwright sidecar / Chromium 截图失败时重试一次。
2. 如果浏览器断开，重启 browser 后再试。
3. 失败写 `renderStatus=failed` 和错误信息。
4. 错误结构遵守 `底层设计/09-错误日志与事件规范.md`。

### 参考来源

- Pixelle `pixelle_video/services/frame_html.py::_reset_browser`。
- Pixelle `pixelle_video/services/frame_html.py::generate_frame` 中截图失败重置浏览器的处理。

## 四、这个模块对外提供什么

写回：

```json
{
  "itemId": "item_001",
  "renderedFramePath": "rendered/item_001_frame.png",
  "renderStatus": "succeeded"
}
```

使用方：

- `10-字幕.md` 可用模板显示复杂字幕。
- `11-封面.md` 可用模板生成封面。
- `13-视频合成.md` 可在 image_slideshow 或降级方案中用 renderedFramePath + audioPath 生成视频片段。
- 当前 image_to_video 主线不依赖模板渲染即可合成。
