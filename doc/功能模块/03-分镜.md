# 分镜

> 这是给 AI 开发用的功能说明书。读完这一篇，就能把“分镜”模块从零写出来，不用再问。
> 配套阅读：字段定义看 `底层设计/数据结构.md`；当前开发顺序看项目 `plan/阶段路线图.md`。

## 一、这个功能是什么

分镜模块负责把用户输入的文字拆成一组可编辑的 `StoryboardItem[]`。

当前主线不是专业影视分镜，也不是必须先通过独立脚本页，而是服务这条链路：

```text
输入文字 → 分镜 → 生图 → 图生视频 → 合成
```

分镜要解决：

```text
这一段原文是什么
这一行要表达什么画面
出现谁
在哪里
画面描述是什么
出图提示词是什么
图生视频动作提示词是什么
持续多久
当前状态是什么
```

景别、运镜、转场、构图可以保留为高级可选字段，但不是当前主线必填项，也不能阻塞进入生图。

---

## 二、子功能清单

1. 根据输入文字生成分镜
2. 分镜编辑
3. 分镜排序 / 新增 / 删除 / 合并 / 拆分
4. 分镜确认与锁定
5. 单条分镜重生成
6. 分镜校验

## 子功能 1：根据输入文字生成分镜

### 界面

- 左侧：分镜列表或分镜表格。
- 中间：当前分镜编辑区。
- 右侧：AI 辅助 / 参数 / 操作。
- 顶部显示：项目名、workflowType、画幅、内容语言、目标分镜数。
- 按钮：生成分镜 / 重新生成未锁定 / 确认分镜 / 进入生图。

### 用户操作

```text
新建项目输入文字
→ 进入分镜页
→ 系统生成分镜草稿
→ 用户编辑 / 拆分 / 合并 / 排序
→ 确认分镜
→ 进入生图表格页
```

### 系统逻辑

1. 读取 Project 的 `sourceText / topic / inputType / workflowType`。
2. 如果 `inputType=paste`，优先按切分规则生成分镜原文段，不能改写原文。
3. 如果 `inputType=topic/article`，可以调用 LLM 生成分镜草稿。
4. 读取 Project Bible / Style Bible / Character Bible / Environment Bible。
5. 生成 `StoryboardItem[]`，数量尽量接近 `targetSceneCount`。
6. 每个 StoryboardItem 写入：
   - `itemId`
   - `projectId`
   - `index`
   - `sourceText`
   - `narrationText`
   - `visualDescription`
   - `characters`
   - `sceneDescription`
   - `imagePrompt`
   - `negativePrompt`
   - `videoPrompt`
   - `durationSeconds`
   - `selectedImageId=null`
   - `selectedVideoSegmentId=null`
   - `status=pending`
7. 初始时长使用估算值。TTS 是后续增强，不要求当前主线必须用真实音频时长回填。
8. 写入 Storyboard 草稿。

### 数据结构示意

以 `底层设计/数据结构.md` 为准，示意：

```json
{
  "itemId": "item_001",
  "projectId": "proj_xxx",
  "index": 1,
  "sourceText": "很多人以为早睡只是少玩一会手机。",
  "narrationText": "很多人以为早睡只是少玩一会手机。",
  "visualDescription": "清晨卧室里，年轻上班族醒来，阳光照进房间，状态清醒。",
  "characters": ["年轻上班族"],
  "sceneDescription": "清晨卧室",
  "imagePrompt": "young office worker waking up refreshed in a clean bedroom, morning sunlight, cinematic, vertical composition",
  "negativePrompt": "low quality, distorted face, extra fingers",
  "videoPrompt": "soft morning light moves across the room, character slowly opens eyes and sits up",
  "durationSeconds": 4,
  "selectedImageId": null,
  "selectedVideoSegmentId": null,
  "status": "pending",
  "lockFlagsJson": {}
}
```

### 参考来源

- Toonflow `src/agents/productionAgent/tools.ts`：分镜面板和内容组织思路。
- Pixelle `pixelle_video/models/storyboard.py`：线性 frame 执行模型。
- Pixelle `pixelle_video/pipelines/standard.py::initialize_storyboard`：按段落创建 frames。

### 坑提醒

- paste 模式不能改写用户原文。
- `index` 是展示顺序，不是主键。
- `itemId` 必须用 uuid / nanoid，不得用 `Date.now()`。
- 不要让 AI 随便生成无来源角色 ID、场景 ID；没有设定集时可以先用文本描述。

## 子功能 2：分镜编辑

### 界面

每个 StoryboardItem 至少支持编辑：

- 原文片段 `sourceText`
- 旁白 / 文案 `narrationText`
- 画面描述 `visualDescription`
- 角色 / 主体 `characters`
- 场景描述 `sceneDescription`
- 出图提示词 `imagePrompt`
- 负面提示词 `negativePrompt`
- 视频动作提示词 `videoPrompt`
- 单段时长 `durationSeconds`

高级可选字段：

- 景别
- 运镜
- 构图
- 节奏
- 转场

高级字段可以给图生视频提示词提供上下文，但不能作为主线必填。

### 用户操作

用户点开某条分镜 → 修改原文、画面、角色、提示词或时长 → 保存。

### 系统逻辑

1. 表单绑定当前 StoryboardItem。
2. 保存前校验必要字段。
3. 修改 `sourceText / narrationText` 后，后续字幕/TTS/视频提示词可标记需更新。
4. 修改 `visualDescription / characters / sceneDescription / imagePrompt` 后，已生成候选图如果存在，要提示是否保留或重生成。
5. 修改 `videoPrompt` 后，已生成视频片段如果存在，要提示是否保留或重生成。

### 坑提醒

- 不要用户改了提示词，但旧图还被当成完全匹配。
- 不要一改一个字段就把全项目状态全部重置。
- 状态重置要按影响范围做。

## 子功能 3：分镜排序 / 新增 / 删除 / 合并 / 拆分

### 界面

- 支持拖拽排序。
- 每行有复制、删除、上移、下移。
- 有「新增分镜」。
- 有「拆分分镜」。
- 有「合并相邻分镜」。

### 系统逻辑

1. 排序后重排 `index`。
2. 新增分镜时生成新的 `itemId`，状态初始化为 pending。
3. 删除分镜时如果已有候选图、视频片段，要提示会影响后续合成。
4. 合并分镜时合并 sourceText / narrationText，并清空或重算 imagePrompt / videoPrompt。
5. 拆分分镜时生成两个新的或更新当前 item，后续候选图/视频片段需要重新确认。
6. 保存后更新 `sceneCount / storyboardItemCount`。

### 坑提醒

- `index` 只表示顺序，不要拿它当数据库主键。
- 已选图和视频片段与分镜强绑定，删除/合并/拆分必须处理引用。

## 子功能 4：分镜确认与锁定

### 界面

- 每条分镜有锁定状态。
- 顶部有「确认全部分镜」。
- 未通过校验时必须指出第几条缺什么。

### 系统逻辑

1. 用户确认单条分镜后，可写入 `lockFlagsJson.storyboard=true`。
2. 批量重生成时跳过锁定项。
3. 用户点击「确认全部分镜」后，进入分镜确认状态。
4. 未确认分镜不能进入生图。

### 坑提醒

- `lockFlagsJson` 是保护用户编辑成果的唯一锁定来源。
- 不要批量重生成覆盖用户锁定内容。

---

## 子功能 5：单条分镜重生成

### 界面

- 每条分镜有「重生成画面描述」。
- 可选择只重生成：画面描述 / 出图提示词 / 视频动作提示词。

### 系统逻辑

1. 读取当前 item 的 sourceText 和上下文前后各一条 item。
2. 调 LLM 只返回当前需要重生成的字段。
3. 不覆盖用户锁定字段。
4. 如果重生成 imagePrompt，已有候选图状态应提示用户是否重生成。
5. 如果重生成 videoPrompt，已有视频片段状态应提示用户是否重生成。

### 坑提醒

- 单条重生成不能破坏整体角色/场景一致性。
- 不要覆盖 locked 字段。

---

## 子功能 6：分镜校验

### 校验规则

进入生图前必须检查：

1. StoryboardItem 数量 > 0。
2. 每条 `sourceText` 或 `narrationText` 至少一个非空。
3. 每条 `visualDescription` 非空。
4. 每条 `imagePrompt` 非空；如果为空，必须先反推/生成。
5. 每条 `durationSeconds > 0`。
6. 如果使用 `characterIds[]`，必须存在对应 Character Bible 条目。
7. 如果使用 `locationId`，必须存在对应 Environment Bible 条目。

不阻塞当前主线的字段：

```text
shotSize
cameraMotion
transitionType
composition
pace
```

### 失败提示

- 明确告诉用户哪一条分镜缺什么。
- 不要只弹“校验失败”。

## 四、这个模块对外提供什么

输出完整 Storyboard：

```json
{
  "storyboardId": "sb_xxx",
  "projectId": "proj_xxx",
  "title": "为什么要早睡",
  "itemCount": 8,
  "items": []
}
```

后续模块围绕 StoryboardItem 写字段：

- `07-生图提示词.md` 写 `imagePrompt / negativePrompt`
- `08-AI生图.md` 写 `ImageCandidate[] / selectedImageId`
- `14-AI视频.md` 读取 `selectedImageId`，写 `VideoSegment[] / selectedVideoSegmentId`
- `13-视频合成.md` 读取 `selectedVideoSegmentId`，写 `CompositionTask / final.mp4`
- `09-配音TTS.md`、`10-字幕.md`、`11-封面.md` 是后续增强能力，不是当前主线前置条件
