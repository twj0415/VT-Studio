# 分镜

> 这是给 AI 开发用的功能说明书。读完这一篇，就能把“分镜”模块从零写出来，不用再问。
> 配套阅读：字段定义看 `底层设计/数据结构.md`；当前开发顺序看项目 `plan/阶段路线图.md`。

## 一、这个功能是什么

分镜模块负责把用户输入的文字拆成一组可编辑的 `StoryboardItem[]`。

当前主线不是专业影视分镜，也不是必须先通过独立脚本页，而是服务这条链路：

```text
输入文字 → 分镜 → 生图 → 图生视频 → 合成
```

分镜要解决：

```text
这一段原文是什么
这一行要表达什么画面
出现谁
在哪里
画面描述是什么
出图提示词是什么
图生视频动作提示词是什么
持续多久
当前状态是什么
```

景别、运镜、转场、构图可以保留为高级可选字段，但不是当前主线必填项，也不能阻塞进入生图。

### 参考项目补充机制

waoowaoo 的分镜不是空壳。它的单个镜头既有内容字段，也有生产字段和结果字段：

```text
description
characters
location
shot_type
camera_move
source_text
video_prompt
photographyRules
actingNotes
candidateImages
imageUrl
```

Toonflow-web 的强项是把分镜分成两层：`storyboardTable` 管全局文本和结构，`storyboard` 管逐镜头卡片和生成操作。当前主线不拆两个必经页面，但要吸收这种“两层视角”：

```text
表格主编辑 = storyboardTable 的总览能力 + 当前主编辑入口
Inspector 辅助 = storyboard 的单镜头辅助能力，不重复编辑主字段
```

本项目落地规则：

```text
1. 表格内联编辑是主入口，负责 sourceText / narrationText / visualDescription / 角色 / 场景 / imagePrompt / videoPrompt / durationSeconds。
2. Inspector 只放校验问题、AI 助手、候选图 / 视频预览和高级字段。
3. 高级字段使用本项目规范名：`shotSize / cameraMotion / composition / pace / transitionType`；waoowaoo 的 `photographyRules / actingNotes` 先作为 `advancedDirectingJson` 或 Inspector 建议数据进入后续扩展，落库前不得临时发明 `cameraMove / shotType` 等平行字段。
4. 分镜表格必须有行号快速跳转；后续可加搜索和状态筛选。
5. 删除、合并、拆分必须提示对 ImageCandidate[]、VideoSegment[] 和合成状态的影响。
```

### 分镜字段分层

后续实现不要把分镜理解成“一个镜头图”。一个 StoryboardItem 要分成五层：

| 层 | 字段 | 页面位置 | 写入者 | 下游影响 |
|---|---|---|---|---|
| 文本层 | sourceText、narrationText、subtitleChunks | 表格主列 | 内容创作 / 分镜编辑 | TTS、字幕、视频节奏 |
| 镜头层 | visualDescription、charactersJson、characterIdsJson、sceneDescription、locationId、propsJson | 表格主列 | 分镜生成 / 用户编辑 | 生图 prompt、资产引用 |
| 生产层 | imagePrompt、negativePrompt、videoPrompt、durationSeconds | 表格主列或行内编辑 | 创作规则 / 用户编辑 | 生图、视频、合成时长 |
| 高级层 | shotSize、cameraMotion、composition、pace、transitionType；后续可扩展 advancedDirectingJson 承载 photographyRules / actingNotes | Inspector / 展开列 | AI 助手 / 高级编辑 | 提升镜头质量，不阻塞主线 |
| 结果层 | ImageCandidate[]、selectedImageId、VideoSegment[]、selectedVideoSegmentId | Inspector 只读预览 / 生图视频页 | 生图 / 视频确认 | 决定能否进入后续步骤 |

主编辑边界：

```text
1. 表格是 sourceText / visualDescription / imagePrompt / videoPrompt 等主字段的唯一主编辑入口。
2. Inspector 可以展示这些字段的摘要，但不能再提供同字段编辑器。
3. Inspector 的 AI 助手可以生成修改建议，应用时必须回写表格主字段，并遵守 lockFlagsJson。
4. 结果层在分镜页只读展示；选择候选图在生图页，选择视频片段在视频页。
5. 高级层字段可以先折叠，但字段名和影响范围必须固定，不允许实现时临时发明。
```

### AI 助手落地

AI 助手不是一个独立聊天页，只围绕当前行工作：

```text
可做：
  优化 visualDescription
  重写 imagePrompt / negativePrompt
  重写 videoPrompt
  补充角色 / 场景描述
  生成 shotSize / cameraMotion / composition / pace / transitionType 建议
  生成 photographyRules / actingNotes 建议，但在未落库前只能作为 Inspector 建议或 TaskStep 输出摘要
  解释当前行为什么不能进入生图

不可做：
  覆盖锁定字段
  自动删除候选图或视频片段
  绕过用户确认直接进入生图
  把 prompt 正文写死在 Vue 页面
```

### 参考字段到本项目字段映射

参考项目字段只能作为来源证据，代码和 DTO 必须使用本项目字段。后续实现遇到字段名不一致时，以 `底层设计/数据结构.md` 为准。

| 参考项目字段 | 本项目字段 / 落点 | 说明 |
|---|---|---|
| `description` | `visualDescription` / `visual_description` | 镜头画面描述 |
| `characters` | `charactersJson` 或 `characterIdsJson` | 有 Bible 时优先存角色 ID |
| `location` | `sceneDescription` 或 `locationId` | 有 Location Bible 时优先存 locationId |
| `shot_type` | `shotSize` / `shot_size` | 不使用 `shotType` 新字段 |
| `camera_move` | `cameraMotion` / `camera_motion` | 不使用 `cameraMove` 新字段 |
| `source_text` | `sourceText` / `source_text` | 来自内容创作确认段落 |
| `video_prompt` | `videoPrompt` / `video_prompt` | 视频动作提示词 |
| `photographyRules` | 后续 `advancedDirectingJson.photographyRules` 或 TaskStep 输出摘要 | TODO 未落库前不写入临时字段 |
| `actingNotes` | 后续 `advancedDirectingJson.actingNotes` 或 TaskStep 输出摘要 | TODO 未落库前不写入临时字段 |
| `candidateImages` | `ImageCandidate[]` | 选择动作在生图页 |
| `imageUrl` | `selectedImageId` 指向的候选图 | 不保存单独 imageUrl 主字段 |

## 子功能 1：根据输入文字生成分镜

### 界面

- 左侧：分镜列表或分镜表格。
- 中间：当前分镜编辑区。
- 右侧：AI 辅助 / 参数 / 操作。
- 顶部显示：项目名、workflowType、画幅、内容语言、目标分镜数。
- 按钮：生成分镜 / 重新生成未锁定 / 确认分镜 / 进入生图。

### 用户操作

```text
我的作品 / 开始创作
→ 内容创作确认输入文字
→ 进入分镜页
→ 系统生成分镜草稿
→ 用户编辑 / 拆分 / 合并 / 排序
→ 确认分镜
→ 进入生图表格页
```

### 系统逻辑

1. 读取 Project 的 `sourceText / topic / inputType / workflowType`。
2. 如果 `inputType=paste`，优先按切分规则生成分镜原文段，不能改写原文。
3. 如果 `inputType=topic/article`，可以调用 LLM 生成分镜草稿。
4. 读取 Project Bible / Style Bible / Character Bible / Environment Bible。
5. 生成 `StoryboardItem[]`，数量尽量接近 `targetSceneCount`。
6. 每个 StoryboardItem 写入：
   - `itemId`
   - `projectId`
   - `index`
   - `sourceText`
   - `narrationText`
   - `visualDescription`
   - `characters`
   - `sceneDescription`
   - `imagePrompt`
   - `negativePrompt`
   - `videoPrompt`
   - `durationSeconds`
   - `selectedImageId=null`
   - `selectedVideoSegmentId=null`
   - `status=pending`
7. 初始时长使用估算值。TTS 是后续增强，不要求当前主线必须用真实音频时长回填。
8. 写入 Storyboard 草稿。

### 数据结构示意

以 `底层设计/数据结构.md` 为准，示意：

```json
{
  "itemId": "item_001",
  "projectId": "proj_xxx",
  "index": 1,
  "sourceText": "很多人以为早睡只是少玩一会手机。",
  "narrationText": "很多人以为早睡只是少玩一会手机。",
  "visualDescription": "清晨卧室里，年轻上班族醒来，阳光照进房间，状态清醒。",
  "characters": ["年轻上班族"],
  "sceneDescription": "清晨卧室",
  "imagePrompt": "young office worker waking up refreshed in a clean bedroom, morning sunlight, cinematic, vertical composition",
  "negativePrompt": "low quality, distorted face, extra fingers",
  "videoPrompt": "soft morning light moves across the room, character slowly opens eyes and sits up",
  "durationSeconds": 4,
  "selectedImageId": null,
  "selectedVideoSegmentId": null,
  "status": "pending",
  "lockFlagsJson": {}
}
```

### 参考来源

- Toonflow `src/agents/productionAgent/tools.ts`：分镜面板和内容组织思路。
- Pixelle `pixelle_video/models/storyboard.py`：线性 frame 执行模型。
- Pixelle `pixelle_video/pipelines/standard.py::initialize_storyboard`：按段落创建 frames。

### 坑提醒

- paste 模式不能改写用户原文。
- `index` 是展示顺序，不是主键。
- `itemId` 必须用 uuid / nanoid，不得用 `Date.now()`。
- 不要让 AI 随便生成无来源角色 ID、场景 ID；没有设定集时可以先用文本描述。

## 子功能 2：分镜编辑

### 界面

每个 StoryboardItem 至少支持编辑：

- 原文片段 `sourceText`
- 旁白 / 文案 `narrationText`
- 画面描述 `visualDescription`
- 角色 / 主体 `characters`
- 场景描述 `sceneDescription`
- 出图提示词 `imagePrompt`
- 负面提示词 `negativePrompt`
- 视频动作提示词 `videoPrompt`
- 单段时长 `durationSeconds`

高级可选字段：

- 景别
- 运镜
- 构图
- 节奏
- 转场

高级字段可以给图生视频提示词提供上下文，但不能作为主线必填。

### 用户操作

用户点开某条分镜 → 修改原文、画面、角色、提示词或时长 → 保存。

### 系统逻辑

1. 表单绑定当前 StoryboardItem。
2. 保存前校验必要字段。
3. 修改 `sourceText / narrationText` 后，后续字幕/TTS/视频提示词可标记需更新。
4. 修改 `visualDescription / characters / sceneDescription / imagePrompt` 后，已生成候选图如果存在，要提示是否保留或重生成。
5. 修改 `videoPrompt` 后，已生成视频片段如果存在，要提示是否保留或重生成。

### 坑提醒

- 不要用户改了提示词，但旧图还被当成完全匹配。
- 不要一改一个字段就把全项目状态全部重置。
- 状态重置要按影响范围做。

## 子功能 3：分镜排序 / 新增 / 删除 / 合并 / 拆分

### 界面

- 支持拖拽排序。
- 每行有复制、删除、上移、下移。
- 有「新增分镜」。
- 有「拆分分镜」。
- 有「合并相邻分镜」。

### 系统逻辑

1. 排序后重排 `index`。
2. 新增分镜时生成新的 `itemId`，状态初始化为 pending。
3. 删除分镜时如果已有候选图、视频片段，要提示会影响后续合成。
4. 合并分镜时合并 sourceText / narrationText，并清空或重算 imagePrompt / videoPrompt。
5. 拆分分镜时生成两个新的或更新当前 item，后续候选图/视频片段需要重新确认。
6. 保存后更新 `sceneCount / storyboardItemCount`。

### 坑提醒

- `index` 只表示顺序，不要拿它当数据库主键。
- 已选图和视频片段与分镜强绑定，删除/合并/拆分必须处理引用。

## 子功能 4：分镜确认与锁定

### 界面

- 每条分镜有锁定状态。
- 顶部有「确认全部分镜」。
- 未通过校验时必须指出第几条缺什么。

### 系统逻辑

1. 用户确认单条分镜后，可写入 `lockFlagsJson.storyboard=true`。
2. 批量重生成时跳过锁定项。
3. 用户点击「确认全部分镜」后，进入分镜确认状态。
4. 未确认分镜不能进入生图。

### 坑提醒

- `lockFlagsJson` 是保护用户编辑成果的唯一锁定来源。
- 不要批量重生成覆盖用户锁定内容。

---

## 子功能 5：单条分镜重生成

### 界面

- 每条分镜有「重生成画面描述」。
- 可选择只重生成：画面描述 / 出图提示词 / 视频动作提示词。

### 系统逻辑

1. 读取当前 item 的 sourceText 和上下文前后各一条 item。
2. 调 LLM 只返回当前需要重生成的字段。
3. 不覆盖用户锁定字段。
4. 如果重生成 imagePrompt，已有候选图状态应提示用户是否重生成。
5. 如果重生成 videoPrompt，已有视频片段状态应提示用户是否重生成。

### 坑提醒

- 单条重生成不能破坏整体角色/场景一致性。
- 不要覆盖 locked 字段。

---

## 子功能 6：分镜校验

### 校验规则

进入生图前必须检查：

1. StoryboardItem 数量 > 0。
2. 每条 `sourceText` 或 `narrationText` 至少一个非空。
3. 每条 `visualDescription` 非空。
4. 每条 `imagePrompt` 非空；如果为空，必须先反推/生成。
5. 每条 `durationSeconds > 0`。
6. 如果使用 `characterIds[]`，必须存在对应 Character Bible 条目。
7. 如果使用 `locationId`，必须存在对应 Environment Bible 条目。

不阻塞当前主线的字段：

```text
shotSize
cameraMotion
transitionType
composition
pace
```

### 失败提示

- 明确告诉用户哪一条分镜缺什么。
- 不要只弹“校验失败”。

## 四、这个模块对外提供什么

输出完整 Storyboard：

```json
{
  "storyboardId": "sb_xxx",
  "projectId": "proj_xxx",
  "title": "为什么要早睡",
  "itemCount": 8,
  "items": []
}
```

后续模块围绕 StoryboardItem 写字段：

- `07-生图提示词.md` 写 `imagePrompt / negativePrompt`
- `08-AI生图.md` 写 `ImageCandidate[] / selectedImageId`
- `14-AI视频.md` 读取 `selectedImageId`，写 `VideoSegment[] / selectedVideoSegmentId`
- `13-视频合成.md` 读取 `selectedVideoSegmentId`，写 `CompositionTask / final.mp4`
- `09-配音TTS.md`、`10-字幕.md`、`11-封面.md` 是后续增强能力，不是当前主线前置条件
