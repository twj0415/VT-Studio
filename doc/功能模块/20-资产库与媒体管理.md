# 资产库与媒体管理

> 这是给 AI 开发用的功能说明书。资产库统一管理图片、视频、音频、BGM、字体、参考图、模板资源和任务产物引用，避免文件散落。
> 配套阅读：文件规范看 `底层设计/08-文件存储与路径规范.md`；数据库表看 `底层设计/05-数据库Schema与迁移规范.md`。

## 一、这个功能是什么

资产库负责管理所有可复用媒体资源：

```text
图片
视频
音频
BGM
字体
角色参考图
场景参考图
风格参考图
用户上传素材
模板资源
```

资产库不替代任务产物目录。任务产物仍在 `tasks/task_xxx/`，需要复用时再登记为 Asset。

## 子功能 1：资产导入

### 界面

- 上传按钮。
- 拖拽导入。
- 分类选择：图片 / 视频 / 音频 / BGM / 字体 / 参考图。
- 导入后显示预览。

### 系统逻辑

1. 用户选择文件。
2. PathGuard 校验读取权限。
3. 复制到 `workspace/assets/{kind}/`。
4. 读取元数据。
5. 生成 assets 表记录。

### 数据结构

```json
{
  "asset_id": "asset_xxx",
  "asset_kind": "bgm",
  "display_name": "轻快背景音乐",
  "relative_path": "assets/bgm/light.mp3",
  "media_kind": "audio",
  "metadata_json": {
    "duration_seconds": 62.5
  }
}
```

## 子功能 3：资产预览

预览规则：

```text
image：显示缩略图
video：提取首帧 + 时长
audio/bgm：波形或播放按钮
font：显示字体样张
json/html：显示文件信息，不直接执行
```

所有预览都必须走 StorageService，不直接拼文件路径。

## 子功能 4：资产引用关系

引用表：

```json
{
  "reference_id": "ref_xxx",
  "asset_id": "asset_xxx",
  "owner_kind": "storyboard_item",
  "owner_id": "item_001",
  "usage_kind": "source_material"
}
```

用途：

```text
1. 防止删除正在使用的资产。
2. 追踪某张图被哪些 StoryboardItem / ImageCandidate / VideoSegment 使用。
3. 项目导出时知道要带哪些素材。
```

---

## 子功能 5：资产删除和清理

删除规则：

```text
1. 默认软删除。
2. 被引用资产不能物理删除。
3. 内置资产不能删除，只能隐藏。
4. 用户原始文件不删除，只删除工作区副本。
5. 清理任务临时产物不影响历史记录需要的最终产物。
```

---

## 子功能 6：内置资产和用户资产

内置资产：

```text
内置 BGM
内置字体
内置模板资源
内置示例图
```

规则：

```text
1. is_builtin=true。
2. 应用升级可更新内置资产。
3. 用户不能覆盖内置资产，只能复制后自定义。
```
