# 资产库与媒体管理

> 这是给 AI 开发用的功能说明书。资产库统一管理图片、视频、音频、BGM、字体、参考图、模板资源和任务产物引用，避免文件散落。
> 配套阅读：文件规范看 `底层设计/08-文件存储与路径规范.md`；数据库表看 `底层设计/05-数据库Schema与迁移规范.md`。

## 一、这个功能是什么

资产库负责管理所有可复用媒体资源：

```text
图片
视频
音频
BGM
字体
角色参考图
场景参考图
风格参考图
用户上传素材
模板资源
```

资产库不替代任务产物目录。任务产物仍在 `tasks/task_xxx/`，需要复用时再登记为 Asset。

### 参考项目补充机制

waoowaoo 和 Toonflow 的共同优点是资产贯穿生产流程，而不是孤立陈列：

```text
waoowaoo：角色、地点、分镜图、候选图、确认图服务 storyboard / videos 阶段。
Toonflow-web：assets、deriveAsset、referenceList、batchGenerateImage 让角色 / 场景 / 参考图参与分镜图生成。
```

本项目吸收：

```text
1. 资产库只放可复用资产；作品生成产物默认留在项目目录。
2. 用户点击“保存为角色参考 / 场景参考 / 风格参考”后，候选图才登记为 Asset。
3. 分镜和生图通过 AssetReference 读取角色、场景、风格参考图。
4. ImageInputPlan / VideoInputPlan 决定哪些资产 required、optional、unused。
5. 项目导出只收集被引用且受控的资产，不导出未授权外部文件。
```

本项目不吸收：

```text
1. 不把所有生成图自动塞进素材库。
2. 不把素材库当任务产物目录。
3. 不把远程 URL 当正式资产路径。
4. 不裸导出数据库和本地绝对路径。
```

### 四类对象边界

后续实现素材库时，不能把任务产物、候选产物和可复用素材混成一个表面概念：

| 对象 | 负责什么 | 是否出现在素材库 | 典型来源 |
|---|---|---|---|
| Artifact | 某次任务生成 / 转码 / 导出的产物索引，服务历史和诊断 | 默认不作为素材卡片展示 | Provider、FFmpeg、导出 |
| ImageCandidate / VideoSegment | 当前作品分镜下的候选图 / 候选视频 | 不自动进入素材库 | 生图页、视频页 |
| Asset | 用户可复用、可管理的素材 | 是 | 用户导入、用户显式保存候选为参考 |
| AssetReference | 素材被谁使用和用途 | 在详情面板显示 | 角色、场景、风格、视频包、分镜、项目 |

登记规则：

```text
生成候选图 / 视频片段：
  写 ImageCandidate / VideoSegment，并写 Artifact。
  不自动写 Asset。

用户点击“保存为角色参考 / 场景参考 / 风格参考”：
  保留原 ImageCandidate。
  新增或复用 Asset。
  新增 AssetReference，owner 指向对应 Bible 或 StoryboardItem，usage_kind 表示 character_reference / location_reference / style_reference。

用户导入素材：
  PathGuard 校验。
  复制到 workspace/assets/{kind}/。
  新增 Asset。
  如绑定到项目 / 角色 / 场景 / 视频包，再新增 AssetReference。

导出项目包：
  只按 Project、StoryboardItem、Bible、VideoPack 相关 AssetReference 收集素材。
  不扫描整个 workspace。
```

按钮到数据写回的对应关系：

| 用户动作 | 写入对象 | 必须校验 | 不允许 |
|---|---|---|---|
| 导入素材 | Asset，可选 AssetReference | PathGuard、mime、大小、相对路径 | 不长期引用用户原始绝对路径 |
| 候选保存为角色参考 | Asset + AssetReference(owner=character_bible 或 storyboard_item, usage=character_reference) | 候选文件存在且在受控目录 | 不移动或删除原 ImageCandidate |
| 候选保存为场景参考 | Asset + AssetReference(owner=location_bible 或 storyboard_item, usage=location_reference) | 同上 | 不把场景图误写成角色图 |
| 候选保存为风格参考 | Asset + AssetReference(owner=style_bible, usage=style_reference) | 同上 | 不自动应用到所有分镜 |
| 视频包绑定默认素材 | AssetReference(owner=video_pack, usage=对应用途) | video_pack 存在，asset 未删除 | 不复制素材文件内容进视频包 |
| 解除引用 | 删除 AssetReference | 不能破坏当前已确认产物；必要时提示下游影响 | 不删除 Asset 文件和原候选 |
| 删除素材 | Asset soft delete；无引用时可物理删除工作区副本 | 先查 AssetReference；内置资源禁止删除 | 不删除用户原始文件；不删除被引用文件 |

删除保护细节：

```text
1. delete_asset 返回被阻断时，错误必须携带可展示引用摘要：owner_kind、owner_id、usage_kind、project_id。
2. UI 不能只弹“删除失败”，必须列出被哪些作品 / 分镜 / 视频包 / 任务使用。
3. 解除引用后再删，必须重新查询引用，不能靠前端旧状态判断。
4. 清理未选候选不走 delete_asset；它属于作品候选清理，只能清未选、未引用、可再生产物。
5. 项目包导出必须调用按引用收集路径的后端服务，不允许前端扫描目录。
```

参考项目到本项目的筛选：

```text
吸收：
  waoowaoo 的资产服务生产流程，不只是资源陈列。
  Toonflow-web 的参考图、资产派生、批量生成和引用关系思路。
  资产能服务分镜、生图、视频和视频包。

不吸收：
  不把所有生成结果自动塞进素材库。
  不用远程 URL 当正式资产路径。
  不做裸数据库导出。
  不把资产库、作品目录、任务历史混成一个概念。
```

删除规则：

```text
1. 删除 Asset 前必须列出引用它的 owner_kind / owner_id / usage_kind。
2. 被引用素材默认只能隐藏或解除引用后删除。
3. 解除引用不删除原候选图 / 原任务 Artifact。
4. 清理未选候选只能处理未选、未引用、可再生的作品候选产物。
```

## 子功能 1：资产导入

### 界面

- 上传按钮。
- 拖拽导入。
- 分类选择：图片 / 视频 / 音频 / BGM / 字体 / 参考图。
- 导入后显示预览。

### 系统逻辑

1. 用户选择文件。
2. PathGuard 校验读取权限。
3. 复制到 `workspace/assets/{kind}/`。
4. 读取元数据。
5. 生成 assets 表记录。

### 数据结构

```json
{
  "asset_id": "asset_xxx",
  "asset_kind": "bgm",
  "display_name": "轻快背景音乐",
  "relative_path": "assets/bgm/light.mp3",
  "media_kind": "audio",
  "metadata_json": {
    "duration_seconds": 62.5
  }
}
```

## 子功能 4：资产引用关系

引用表：

```json
{
  "reference_id": "ref_xxx",
  "asset_id": "asset_xxx",
  "owner_kind": "storyboard_item",
  "owner_id": "item_001",
  "usage_kind": "source_material"
}
```

用途：

```text
1. 防止删除正在使用的资产。
2. 追踪某张图被哪些 StoryboardItem / ImageCandidate / VideoSegment 使用。
3. 项目导出时知道要带哪些素材。
4. 生图 / 视频输入规划时判断参考图是否可用。
```

引用关系规则：

```text
1. owner_kind 可以是 project、storyboard_item、image_candidate、video_segment、character_bible、style_bible、location_bible、video_pack、task、composition_task、export。
2. usage_kind 必须表达用途，例如 character_reference、style_reference、location_reference、pose_reference、depth_reference、mask_reference、source_material、selected_image、selected_video、bgm、font、template_resource、generated_image、generated_video、final_export、task_artifact。
3. 删除资产前必须查询引用关系；被引用资产只能隐藏或提示先解除引用。
4. 候选图保存为资产时，要同时保留原 ImageCandidate 和新 Asset 引用，不能把原候选图移动走。
```

---

## 子功能 5：资产删除和清理

删除规则：

```text
1. 默认软删除。
2. 被引用资产不能物理删除。
3. 内置资产不能删除，只能隐藏。
4. 用户原始文件不删除，只删除工作区副本。
5. 清理任务临时产物不影响历史记录需要的最终产物。
```
