# 画风设定

> 这是给 AI 开发用的功能说明书。读完这一篇，就能把“画风设定”模块从零写出来，不用再问。
> 配套阅读：Style Bible 字段看 `底层设计/数据结构.md`。

## 一、这个功能是什么

画风设定负责定义全片统一的视觉美学。它不直接生成图片，而是给后续 `07-生图提示词.md` 和 `08-AI生图.md` 提供统一约束。

核心产物是 Style Bible：

```text
style_id
style_prompt
color_palette
lighting
composition
negative_prompt
reference_image_path
```

没有画风设定，AI 生图会一张像插画、一张像照片、一张像 3D，整片不统一。

---

## 二、子功能清单

1. 选择内置画风
2. 自定义画风
3. 从参考图反推画风
4. 画风负面词
5. 保存为画风预设

---

## 子功能 1：选择内置画风

### 界面

- 画风卡片网格。
- 每张卡片包含：名称、预览图、适合场景、提示词摘要。
- 常见内置：极简线稿、扁平插画、水彩、赛博霓虹、治愈手绘、胶片写实、黑白漫画。

### 用户操作

用户选择「极简线稿」→ 系统把对应字段写入 Style Bible。

### 系统逻辑（一步步）

1. 读取内置画风列表。
2. 用户选择某个 `style_id`。
3. 写入项目的 Style Bible。
4. 后续生成 image_prompt 时自动注入 `style_prompt / color_palette / lighting / composition / negative_prompt`。

### 数据结构

```json
{
  "style_id": "minimal_line_art",
  "style_prompt": "minimal black and white line art, clean hand-drawn strokes, simple shapes, white background",
  "color_palette": ["black", "white", "light gray"],
  "lighting": "flat clean lighting",
  "composition": "centered composition, clear empty space",
  "negative_prompt": "photorealistic, colorful, 3d render, complex background"
}
```

### 参考证据 / 本项目落地

- Toonflow `src/routes/artStyle/getArtStyle.ts`：画风列表。
- Toonflow `skills/art_skills/*`：画风技能文件。
- Pixelle `pixelle_video/config/schema.py` 的 `ImageSubConfig.prompt_prefix`：提供了全局风格前缀思路，但太弱，只能作为最低兜底。

### 坑提醒

- 不要只做一个 `prompt_prefix` 字符串。画风要拆成风格、色彩、光线、构图、负面词，后续才可控。

---

## 子功能 2：自定义画风

### 界面

表单字段：

- 画风名称
- 风格描述
- 色彩倾向
- 光线
- 构图倾向
- 负面词
- 预览图（可选）

### 系统逻辑

1. 校验名称非空。
2. 把用户输入转成 Style Bible。
3. 保存到项目；可选保存到全局预设。
4. 提供「测试生成一张预览图」按钮。

### 坑提醒

- 不要让用户填一坨中文描述就直接拼进英文生图 prompt。后续 `07-生图提示词.md` 要统一转成模型适合的英文提示词。

---

## 子功能 3：从参考图反推画风

### 界面

- 上传参考图。
- 显示图片预览。
- 按钮：「分析画风」。
- 输出：风格描述、色彩、光线、构图、负面词建议。

### 系统逻辑（一步步）

1. 用户上传参考图。
2. 文件经 PathGuard 存入项目 assets 目录。
3. 调 VLM 分析图片风格。
4. 要求 VLM 返回结构化字段：`style_prompt / color_palette / lighting / composition / negative_prompt`。
5. 用户确认后写入 Style Bible。

### 参考证据 / 本项目落地

- Toonflow `src/routes/artStyle/extractStylePrompt.ts`：用图像模型从参考图反推画风提示词。
- Pixelle `pixelle_video/services/image_analysis.py` 和 `pixelle_video/services/api_services/vlm_client.py`：VLM 分析服务思路。

### 坑提醒

- 参考图只能作为风格参考，不要默认把图中具体人物也带入，人物应由 Character Bible 管。
- 远程 URL 或上传图都要转存本地相对路径。

---

## 子功能 4：画风负面词

### 界面

- 默认负面词区域。
- 用户可追加。
- 提供常用勾选项：真实照片、3D、复杂背景、畸形手、文字水印、低清、脸崩。

### 系统逻辑

1. 内置负面词 + 风格负面词 + 用户负面词合并。
2. 去重。
3. 写入 `negative_prompt`。
4. 生图提示词阶段注入。

### 坑提醒

- 负面词不能越堆越长，太长会稀释重点。要有长度上限。

---

## 子功能 5：保存为画风预设

### 界面

- 「保存为预设」按钮。
- 输入预设名。
- 可选：是否设为默认。

### 系统逻辑

1. 从当前 Style Bible 复制字段。
2. 保存到全局画风预设表。
3. 下次创建项目可直接选择。

### 参考证据 / 本项目落地

- Toonflow `src/routes/artStyle/addArtStyle.ts`。
- Toonflow `src/routes/artStyle/editArtStyle.ts`。

---

## 四、这个模块对外提供什么

输出 Style Bible：

```json
{
  "style_id": "minimal_line_art",
  "style_prompt": "minimal black and white line art...",
  "color_palette": ["black", "white"],
  "lighting": "flat clean lighting",
  "composition": "centered composition",
  "negative_prompt": "photorealistic, 3d render"
}
```

被这些模块使用：

- `07-生图提示词.md`：拼接进 image_prompt。
- `08-AI生图.md`：作为模型调用参数来源。
- `11-封面.md`：封面图也要跟随同一画风。
