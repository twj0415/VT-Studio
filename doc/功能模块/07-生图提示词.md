# 生图提示词

> 这是给 AI 开发用的功能说明书。读完这一篇，就能把“生图提示词 / 反推提示词”模块从零写出来，不用再问。
> 配套阅读：分镜字段看 `底层设计/数据结构.md` 和 `功能模块/03-分镜.md`；画风/角色/场景看 04~06；当前开发顺序看 `plan/阶段路线图.md`。

## 一、这个功能是什么

把每条分镜 `StoryboardItem` 变成图片模型可用的：

```text
imagePrompt
negativePrompt
```

在页面上，这个能力可以独立执行，也可以直接并入“生图表格页”的行内反推区域。

当前主线中的位置：

```text
输入文字 → 分镜 → 生图提示词/反推 → AI 生图 → 图生视频 → 合成
```

它不负责真正调用图片模型，只负责生成稳定、可控、数量正确、方便用户编辑的出图提示词。

### 参考项目补充机制

waoowaoo 的提示词能力不是孤立页面，而是嵌在镜头生产上下文里：

```text
ai-modify-shot-prompt：围绕当前镜头优化出图 / 镜头提示词。
ai-modify-location：围绕当前镜头或地点优化场景。
AIDataModal：查看 / 调整 AI 生成相关数据，例如摄影规则和演技指导。
```

Toonflow-web 的 Prompt / Skill 管理证明：规则需要集中管理和可编辑，但不代表用户要在一个孤立大页面里编辑每条分镜 prompt。

本项目落地规则：

```text
1. 分镜页编辑镜头级 imagePrompt / videoPrompt。
2. 生图页提供“预览最终 prompt”按钮。
3. 创作规则页管理 prompt 模板、结构化输出 schema 和测试样例。
4. 最终发送给模型的 prompt 由后端按创作规则、角色、场景、画风、模型能力和 StoryboardItem 组装。
5. 如果最终 prompt 组装还没接入，页面必须显示“未接入最终组装”，不能假装展示真实发送内容。
6. AI 修改不得覆盖 lockFlagsJson 锁定字段。
```

## 二、输入和输出

输入：

```text
StoryboardItem.sourceText
StoryboardItem.narrationText
StoryboardItem.visualDescription
StoryboardItem.characters
StoryboardItem.sceneDescription
Style Bible
Character Bible
Environment Bible
可选高级字段：shotSize / cameraMotion / composition / emotion
```

输出：

```text
StoryboardItem.imagePrompt
StoryboardItem.negativePrompt
```

注意：

```text
shotSize / cameraMotion / transitionType 不是当前主线必填项。
它们可以作为增强上下文，但不能阻塞提示词生成。
```

5. 审核失败后的中性化改写
6. 提示词锁定

## 子功能 1：批量反推 imagePrompt

### 界面

当前推荐在生图表格页中展示：

```text
每一行 = 一个分镜
列：原文 / 角色 / 反推提示词 / 候选图 / 已选图 / 状态 / 操作
```

提示词相关操作：

```text
反推全部提示词
只反推空提示词
重推当前行
编辑当前行提示词
锁定当前行提示词
```

### 系统逻辑

1. 读取已确认的 `StoryboardItem[]`。
2. 跳过 `lockFlagsJson.imagePrompt=true` 且已有 imagePrompt 的 item。
3. 按批次处理，默认每批 10 条。
4. 调 LLM，要求返回 JSON：`imagePrompts[]`，数量必须等于本批 item 数。
5. 校验数量：不一致则重试，最多 3 次。
6. 写入每条 item 的 `imagePrompt`。
7. 合并负面词，写入 `negativePrompt`。
8. 已生成候选图的 item，如果提示词变化，标记需要用户确认是否重新生图。

### 参考来源

- Pixelle `pixelle_video/utils/content_generators.py::generate_image_prompts`：`batch_size=10`、`max_retries=3`、返回数量必须匹配。
- Pixelle `pixelle_video/prompts/image_generation.py::build_image_prompt_prompt`。

### 坑提醒

- 不要一条一条调 LLM，慢且贵。
- 不要允许返回数量不一致继续往下跑，否则提示词会错位。
- 不要“少了就补空字符串”，那会导致后续生成垃圾图。

## 子功能 2：行内编辑 / 单条重生成

### 界面

每行显示：

```text
出图提示词 imagePrompt
负面提示词 negativePrompt
编辑
重推
锁定
复制
```

### 系统逻辑

1. 用户手动编辑后，设置 `lockFlagsJson.imagePrompt=true`。
2. 单条重生成只改当前 item。
3. 如果当前 item 已生成候选图，重生成 prompt 后不能直接覆盖候选图，要提示“提示词已变化，是否重新生成图片”。
4. 用户锁定后，批量反推不得覆盖。

### 坑提醒

- 不要批量覆盖用户手动改过的提示词。
- 不要因为提示词变化就静默删除旧候选图。

---

## 子功能 3：注入画风 / 角色 / 场景

### 系统逻辑

每条 item 的最终提示词由这些部分组成：

```text
画风 Style Bible
+ 角色 Character Bible 或 item.characters
+ 场景 Environment Bible 或 item.sceneDescription
+ 分镜画面描述 item.visualDescription
+ 可选：构图 / 情绪 / 运动意图
+ 道具 / 关键元素
```

拼接顺序建议：

```text
[style]
[main subject / characters]
[location / environment]
[action / visual description]
[composition / lighting / color palette]
[details]
```

生图页最终 prompt 弹窗必须展示：

```text
基础 imagePrompt
negativePrompt
注入的角色设定
注入的场景设定
注入的画风规则
当前创作规则 key / rule_id
模型或 workflow 约束摘要
最终将提交给 ProviderManager 的 prompt 预览
```

### 参考来源

- Pixelle `pixelle_video/utils/prompt_helper.py::build_image_prompt`：基础 prompt + prefix 的组合。
- Toonflow `src/routes/production/storyboard/batchGenerateImage.ts`：按分镜关联资产收集参考图。

### 坑提醒

- 角色描述从 Character Bible 取，不要让 LLM 每次重新发明。
- 场景描述从 Environment Bible 取，不要同一个地点每条都变。
- 负面词要合并去重，不能无限堆。

## 子功能 4：数量校验和重试

### 系统逻辑

1. 每批输入 N 条 StoryboardItem。
2. LLM 必须返回 N 条 prompt。
3. 如果 JSON 解析失败：重试。
4. 如果数量不一致：重试。
5. 3 次失败后，该批标记失败，写 TaskStep 错误。

### 参考来源

- Pixelle `generate_image_prompts` 的 JSON parse + count mismatch retry。

### 坑提醒

- 不能把提示词错位写入别的分镜。

---

## 子功能 5：审核失败后的中性化改写

### 系统逻辑

1. AI 生图或图生视频返回审核失败。
2. 读取失败 prompt。
3. 调 LLM 做中性化改写，保留画面意图，去掉敏感表达。
4. 用改写后的 prompt 重试。
5. 记录原始 prompt、改写 prompt 和失败原因。

### 参考来源

- Pixelle `pixelle_video/services/api_media.py::_neutralize_video_prompt`。

### 坑提醒

- 中性化只处理审核风险，不要改掉核心剧情。
- 原 prompt 要保留，方便排查。

---

## 子功能 6：提示词锁定

### 系统逻辑

1. 用户编辑或锁定提示词后，写入 `lockFlagsJson.imagePrompt=true`。
2. 批量反推跳过锁定项。
3. 用户显式解锁后才允许覆盖。

## 四、这个模块对外提供什么

写回 StoryboardItem：

```json
{
  "itemId": "item_001",
  "imagePrompt": "young office worker waking up refreshed in a clean bedroom, morning sunlight, cinematic, vertical composition",
  "negativePrompt": "low quality, distorted face, extra fingers",
  "status": "image_prompt_ready"
}
```
