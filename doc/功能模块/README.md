# 功能模块总索引

> 本文件把 `doc/功能模块` 下所有功能模块按当前开发优先级重新整理。
> 这里是“功能模块索引 + 开发阅读入口”，不是替代单个模块文档。
> 真正开发顺序仍以项目 `plan/阶段路线图.md` 的 TODO 为准。
> 所有模块都必须服务本项目最终产品，不得按参考项目做页面拼贴。

---

## 0. 当前产品主线

当前优先跑通的 workflow：

```text
workflowType = image_to_video
```

主流程：

```text
输入文字
→ 分镜
→ 生图
→ 图生视频
→ 合成
```

页面主线：

```text
我的作品 / 开始创作
→ 作品工作台 / 内容创作
→ 分镜工作台
→ 生图表格页
→ 视频表格页
→ 合成与导出
```

关键原则：

```text
1. doc/功能模块 是完整需求库，不等于当前开发顺序。
2. TTS、字幕、封面、BGM 不是不做，而是后续增强 TODO。
3. 数字人口播、素材成片、纯图文成片是独立 workflow，不混进 image_to_video 主线。
4. 生图页必须是表格 / 列表工作台：每行一个分镜。
5. 图生视频读取生图阶段已选图片，不能绕过图片选择。
6. 参考项目只提供能力证据，模块文档必须写成本项目落地规则。
7. AI 工具是独立小工具集合，不能提供创建作品入口，不能和内容创作的主题模式重复。
```

## 0.1 每个模块统一施工模板

后续修改或开发任何模块，单篇功能文档至少要写清这些内容：

| 必写项 | 要回答的问题 | 防漏目的 |
|---|---|---|
| 目标 | 用户进来要完成什么 | 防止做成工具堆 |
| 页面位置 | 属于哪个一级菜单、哪个工作台页 | 防止入口散乱 |
| 页面区域 | 顶栏、左栏、表格、预览、抽屉、弹窗分别有什么 | 防止 UI 缺块 |
| 操作按钮 | 主按钮、批量按钮、行内按钮、恢复按钮、危险按钮、辅助按钮 | 防止只做主按钮 |
| 状态 | 未开始、缺输入、运行中、失败、待选择、已选择、已确认、已跳过 | 防止页面无恢复路径 |
| 数据读写 | 读哪些 DTO / 表 / 文件，写哪些字段 | 防止前端本地状态替代真实状态 |
| 后端服务 | Tauri command、Rust service、Provider、Pipeline、Storage 如何协作 | 防止页面直连模型或文件 |
| 参考证据 | 参考项目哪个能力值得学 | 防止无来源乱设计 |
| 本项目增强 | 比参考项目补强什么 | 防止只复刻缺点 |
| 禁止事项 | 不能照搬什么、不能删什么 | 防止版权和架构风险 |
| 验收标准 | 页面、数据、异常、测试如何算通过 | 防止写完不能验 |
| 对应 TODO | 落在哪个 TODO 阶段 | 防止临时加戏 |

如果单篇模块还没有按这个模板写完整，开发前必须先补齐关键项，再开始写代码。

执行细度补充：

```text
1. 参考项目优点不能只写一句话，必须拆到字段、页面区域、按钮、状态、数据流和验收。
2. 已有概念不能重复造名词，只补它到底怎么分层、怎么保存、怎么显示、怎么校验。
3. 如果文档写了“视频包 / 创作规则 / 模型工作流分层”，必须说明：视频包保存哪些引用，创作规则保存哪些正文和 schema，Provider 保存什么，provider_models 保存什么，workflow_presets 保存什么。
4. 如果文档写了“分镜参考 waoowaoo / Toonflow”，必须说明：分镜行有哪些字段，主编辑在表格还是 Inspector，候选图 / 视频怎么保留，AI 助手改什么字段，锁定如何生效。
5. 如果文档写了“任务态参考 waoowaoo / Pixelle”，必须说明：页面显示什么、DB 记录什么、ProgressEvent 只做什么、不显示什么。
6. 单篇模块如果不能回答这些问题，开发者必须先补文档和 TODO，不能直接实现。
```

深度施工口径：

```text
1. 内容导入必须按“输入风险提示 → 切分 / AI 整理 → 段落映射编辑 → 用户确认 → 写入 StoryboardItem”描述清楚。
2. 分镜必须按五层字段写：文本层、镜头层、生产层、高级层、结果层；不能只写“有分镜卡”。
3. 候选图 / 候选视频必须写清：生成追加候选、预览、选择、确认、取消、撤回、清理未选、保存为素材；不能只写“支持多候选”。
4. 任务态必须写清：Task / TaskStep 是权威源，ProgressEvent 是实时展示，页面显示总数 / 成功 / 失败 / 当前行 / 已用时，不显示 ETA。
5. 视频包必须按引用链施工：VideoPack → CreativeRule / ProviderModel / WorkflowPreset / AssetReference → ProjectRuntimeConfig → TaskStep 快照。
6. 模型 / 工作流必须按六层拆开：Provider 连接、API 模型能力、Workflow Preset、测试记录、创作规则、视频包。
7. 资产必须按 Artifact / Candidate / Asset / AssetReference 区分；作品产物默认不等于可复用素材。
8. 如果当前代码没落地对应表或字段，模块文档必须标注“目标结构，TODO 落点，当前差异”，不能写成已完成。
```

参考强项落地门禁：

```text
1. 开发任何模块前，先到 `参考分析/参考项目深度机制与吸收口径.md` 的“参考强项写入文档的审计表”查该模块是否命中参考强项。
2. 命中后必须继续读表里列出的“必读落点”，再读当前模块文档；只读本索引的一句话不够。
3. 命中内容导入、流程推进、分镜、候选、生产画布、模型 / 工作流、视频包或资产贯穿时，还必须读 `参考项目深度机制与吸收口径.md` 第十一节“深度施工证据索引与吸收细则”。
4. 命中第十二节施工卡时按这个索引读：内容导入 12.1；流程推进 / 任务反馈 12.2；分镜 12.3；提示词 12.4；候选图 / 候选视频 12.5；生产画布 / workbench 12.6；模型 / 工作流 12.7；资产贯穿 12.8；视频包 / 创作规则 / 模型工作流 12.9；导出备份 12.10。
5. 如果当前模块文档没有写清参考项目怎么实现、我们怎么筛选、字段怎么落库、页面怎么布局、按钮和状态有哪些，先补模块文档和当前 TODO。
6. 如果当前代码尚未实现目标字段或表，模块文档必须写“当前实现差异 / TODO 落点”，不能把目标结构写成已完成事实。
```

## 0.2 参考项目到本项目的映射规则

| 参考项目 | 只学习这些 | 本项目落地方式 | 明确不做 |
|---|---|---|---|
| waoowaoo-main | 前端线性流程、短视频制作节奏、步骤推进感 | 保留 Vue3/Tauri 工作台，用“我的作品 → 内容创作 → 分镜 → 生图 → 视频 → 合成”承载 | 不复制 UI、代码结构、接口命名 |
| Toonflow-app | 创作资源、Skill、画风/角色/场景/道具、一致性资产 | 落到“创作资源 / 创作规则 / 视频包 / 素材库”和 Bible 数据结构 | 不搬 Electron/Express/vm2，不复制品牌和 Skill 正文 |
| Toonflow-web | Prompt/Skill 管理页、模型配置页、管理端信息结构 | 落到“模型 / 工作流”和“创作规则”页面 | 不假设后端可二开，不复刻服务端闭源能力 |
| Pixelle-Video | 真实生成链路、ComfyUI/RunningHub/API 双轨、TTS、字幕、FFmpeg | 落到 ProviderManager、workflow_presets、FFmpeg sidecar、任务队列 | 不搬 Streamlit/Python 架构，不明文存 key |
| huobao-drama | 短剧业务菜单、内容管理形态 | 仅作为后续短剧内容管理参考 | 不扩成复杂 SaaS，不作为核心架构来源 |

## 0.3 不能变成“四不像”的硬标准

1. 一级菜单只能围绕本项目：`我的作品 / AI 工具 / 创作资源 / 模型 / 工作流 / 系统设置`。这里的“模型 / 工作流”是一个一级菜单名，不是两个菜单。
2. 工作台主线只能围绕当前作品：`内容创作 / 分镜 / 生图 / 视频 / 合成 / 导出历史`。
3. 视觉、按钮、路由文案要服务本项目，不直接沿用参考项目文案。
4. 生成类页面必须支持批量、单行、重试、取消或跳过、查看日志、清理历史产物。
5. 候选图和视频片段必须可回看、可选择、可保留历史，不允许静默覆盖。
6. 模型能力来自后端能力矩阵和输入规划，不允许页面按模型名硬编码。
7. 后续能力可以后移，但不能从文档和 TODO 消失。
8. 页面开发完成后必须对照 `参考分析/页面交互账本.md` 和本索引复核。

## 7. 模块边界红线

1. 内容导入不负责最终生成，只负责创建项目和归一化输入。
2. 脚本生成不是当前主线必经页面。
3. 分镜阶段不强制景别、运镜、转场。
4. 生图提示词可以并入生图表格页行内。
5. AI 生图必须支持候选图和已选图，不再是单 `imagePath`。
6. 图生视频必须读取已选图。
7. 合成必须优先支持视频片段直接拼接。
8. TTS、字幕、封面、BGM 是 TODO 增强项，不是删除项。
9. 数字人口播、素材成片、纯图文成片是独立 workflow。
10. 所有真实密钥只进系统钥匙串。
