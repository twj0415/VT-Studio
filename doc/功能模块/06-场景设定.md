# 场景设定

> 这是给 AI 开发用的功能说明书。读完这一篇，就能把“场景设定”模块从零写出来，不用再问。
> 配套阅读：Environment Bible 字段看 `底层设计/数据结构.md`。

## 一、这个功能是什么

场景设定负责定义视频里反复出现的地点和环境，让不同镜头里的空间保持一致。

核心产物是 Environment Bible：

```text
location_id
name
location_type
layout
lighting
color_palette
props
visual_prompt
reference_image_path
reference_images_json
```

StoryboardItem 里只存 `location_id` 的 ID，或在未建立场景库前使用 `scene_description` 文本；不要在每条分镜里重复写完整场景设定。

---

## 二、子功能清单

1. 新建场景
2. 从脚本/分镜自动抽取场景
3. 上传场景参考图
4. 生成场景设定图
5. 道具列表管理
6. 场景一致性注入到生图

---

## 子功能 1：新建场景

### 界面

表单字段：

- 场景名称
- 场景类型：室内/室外/城市/自然/办公室/家/商店等
- 空间布局
- 光线
- 色彩
- 常见道具
- 场景视觉描述
- 参考图（可选）

### 用户操作

用户创建「安静书房」→ 填布局和光线 → 保存。

### 系统逻辑

1. 校验场景名称非空。
2. 生成 `location_id`。
3. 把字段组合成 `visual_prompt`。
4. 写入 Environment Bible。
5. 分镜 StoryboardItem 的 `location_id` 只引用 `location_id`。

### 数据结构

```json
{
  "location_id": "study_room",
  "name": "安静书房",
  "location_type": "indoor study room",
  "layout": "wooden desk near the window, bookshelf on the left wall, warm desk lamp",
  "lighting": "warm soft evening light",
  "color_palette": "brown, cream, dark green",
  "props": ["bookshelf", "wooden desk", "desk lamp", "notebook"],
  "visual_prompt": "a cozy indoor study room, wooden desk near the window, bookshelf on the left wall, warm desk lamp",
  "reference_image_path": null,
  "reference_images_json": []
}
```

### 参考证据 / 本项目落地

- Toonflow 的统一资产表思路：角色/场景/道具都作为 assets 管。
- Toonflow `src/agents/productionAgent/tools.ts`：资产写入工具。
- Toonflow `src/routes/production/assets/batchGenerateAssetsImage.ts`：批量生成资产图。

### 坑提醒

- 不要只在每条分镜里自由写场景。场景要复用，否则同一个“书房”每张图都不一样。

---

## 子功能 2：从脚本/分镜自动抽取场景

### 界面

- 按钮：「识别场景」。
- 候选场景列表。
- 支持合并相似场景。

### 系统逻辑

1. 读取脚本或分镜画面描述。
2. 调 LLM 抽取地点和常见道具。
3. 返回结构化场景数组。
4. 用户确认后写入 Environment Bible。
5. 自动把 StoryboardItem.location_id 映射到对应 `location_id`。

### 参考证据 / 本项目落地

- Toonflow `src/routes/script/extractAssets.ts`：从剧本抽角色/场景/道具。
- Pixelle `pixelle_video/pipelines/asset_based.py::generate_content`：LLM 结构化输出并做路径/引用校验的思路。

### 坑提醒

- “办公室”“公司”“工位”可能是同一场景，要让用户合并。
- 不要把一次性背景都变成全局场景；只把会复用或对一致性重要的地点入库。

---

## 子功能 3：上传场景参考图

### 界面

- 场景详情页支持上传参考图。
- 可设置为主参考图。

### 系统逻辑

1. 上传文件经 PathGuard 存储。
2. 写 `reference_image_path` 相对路径。
3. 生图时作为参考图传给模型。

### 参考证据 / 本项目落地

- Toonflow `src/utils/ai.ts`：`referenceList`。
- Toonflow `src/routes/production/storyboard/batchGenerateImage.ts::getAssetsImageBase64`。

### 坑提醒

- 场景参考图不要混入角色参考图。角色和场景要分开传，提示词里也要明确。

---

## 子功能 4：生成场景设定图

### 界面

- 按钮：「生成场景图」。
- 支持重生成。
- 支持设为参考图。

### 多视图要求

场景一致性不能只靠单张背景图。至少预留以下参考图角色：

```text
scene_wide_view
scene_layout_view
scene_detail_view
scene_day_variant
scene_night_variant
```

第一版可以先只生成 `scene_wide_view`，但数据结构必须支持多视图和昼夜变体。

### 系统逻辑

1. 用 Environment Bible 的 `visual_prompt` + Style Bible 生成场景图提示词。
2. 调 ImageProvider。
3. 转存结果。
4. 用户确认后写入 `reference_image_path`。

### 坑提醒

- 场景设定图最好不要出现主要角色，否则后续用作场景参考时会把角色带进去。

---

## 子功能 5：道具列表管理

### 界面

- 场景内有道具列表。
- 可新增/删除/编辑道具。
- 常用道具可被 StoryboardItem.props_json 引用。

### 系统逻辑

1. 道具先作为字符串列表管理，可升级为 Prop Bible。
2. StoryboardItem.props_json 只能引用当前项目已定义道具或临时道具。
3. 生图提示词阶段把道具和场景一起拼入。

### 坑提醒

- 道具不要塞进 `location_id` 字段；语义要分开。

---

## 子功能 6：场景一致性注入到生图

### 系统逻辑

1. `07-生图提示词.md` 读取 StoryboardItem.location_id 或 scene_description。
2. 根据 `location_id` 找 Environment Bible。
3. 拼入 `visual_prompt / lighting / color_palette / props`。
4. 如果有场景参考图，`08-AI生图.md` 加入参考图列表。
5. 同时处理角色参考图和场景参考图，避免覆盖。

### 参考证据 / 本项目落地

- Toonflow 分镜生成图时收集关联资产参考图的思路：`src/routes/production/storyboard/batchGenerateImage.ts`。

### 坑提醒

- 同一个场景在早晨/夜晚可能光线不同。默认可用 `lighting` 覆盖，可做 scene variant。

---

## 四、这个模块对外提供什么

输出 Environment Bible：

```json
{
  "locations": [
    {
      "location_id": "study_room",
      "name": "安静书房",
      "visual_prompt": "...",
      "reference_image_path": "assets/locations/study_room.png"
    }
  ]
}
```

被这些模块使用：

- `03-分镜.md`：StoryboardItem.location_id 引用场景 ID。
- `07-生图提示词.md`：拼场景描述。
- `08-AI生图.md`：传场景参考图。
