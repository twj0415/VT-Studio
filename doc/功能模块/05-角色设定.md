# 角色设定

> 这是给 AI 开发用的功能说明书。读完这一篇，就能把“角色设定”模块从零写出来，不用再问。
> 配套阅读：Character Bible 字段看 `底层设计/数据结构.md`。

## 一、这个功能是什么

角色设定负责定义片中固定人物，让多张 AI 图片里的角色尽量长得一致、穿着一致、身份一致。

核心产物是 Character Bible：

```text
character_id
name
role
age
gender
face
body
clothing
signature_items
visual_prompt
negative_prompt
reference_image_path
reference_images_json
seed
```

角色设定不是每个分镜行重新描述一个人，而是先定义角色，再让 StoryboardItem 用 `character_ids_json` 引用角色 ID。

---

## 二、子功能清单

1. 新建角色
2. 从脚本自动抽取角色
3. 上传角色参考图
4. 生成角色设定图
5. 生成三视图 / 表情表
6. 衍生资产：表情/姿态/服装变化
7. 角色一致性注入到生图

---

## 子功能 1：新建角色

### 界面

表单字段：

- 角色名称
- 角色身份
- 年龄感
- 性别
- 脸部特征
- 体型
- 服装
- 标志物
- 性格备注
- 负面词

### 用户操作

用户填写“小林，28岁，上班族，圆框眼镜，白衬衫深蓝开衫” → 保存。

### 系统逻辑（一步步）

1. 校验角色名称非空。
2. 生成 `character_id`，使用语义化 slug 或 uuid，不能用显示名当主键。
3. 把表单字段组合成 `visual_prompt`。
4. 写入 Character Bible。
5. 后续 StoryboardItem 的 `character_ids_json` 只存 `character_id`。

### 数据结构

```json
{
  "character_id": "main_character",
  "name": "小林",
  "role": "ordinary young office worker",
  "age": "around 28",
  "gender": "male",
  "face": "oval face, calm expression, short black hair",
  "body": "slim build, average height",
  "clothing": "white shirt, dark blue cardigan, black trousers",
  "signature_items": ["round glasses", "brown notebook"],
  "visual_prompt": "a slim young Asian man around 28, short black hair, round glasses, white shirt, dark blue cardigan, black trousers",
  "negative_prompt": "different face, different age, different hairstyle, different clothing",
  "reference_image_path": null,
  "reference_images_json": [],
  "seed": null
}
```

### 抄谁

- Toonflow `src/agents/productionAgent/tools.ts`：资产和衍生资产写入工具。
- Toonflow `src/routes/assetsGenerate/generateAssets.ts`：单个资产生成。
- Toonflow `src/routes/assetsGenerate/batchGenerateImageAssets.ts`：批量资产图生成。

### 坑提醒

- 不要把 `name` 当唯一 ID。用户会改名，也可能重名。
- 字段要拆细，不要只存一条 `description`，否则后续不好局部修改。

## 子功能 2：从脚本自动抽取角色

### 界面

- 按钮：「从脚本识别角色」。
- 弹出候选角色列表。
- 每个候选可编辑、合并、删除。

### 系统逻辑

1. 读取已确认脚本或 Storyboard。
2. 调 LLM 抽取人物：姓名/身份/外貌/服装/标志物。
3. 返回结构化数组。
4. 用户确认后写入 Character Bible。
5. 更新 StoryboardItem.character_ids_json 引用。

### 抄谁

- Toonflow `src/routes/script/extractAssets.ts`：从剧本抽角色/场景/道具的思路。
- Pixelle `pixelle_video/pipelines/asset_based.py::VideoScript`：Pydantic 结构化输出思路。

### 坑提醒

- 自动抽取结果必须让用户确认，不能直接覆盖已有角色。
- 同一个角色不同叫法要提示合并。

---

## 子功能 3：上传角色参考图

### 界面

- 每个角色卡片支持上传参考图。
- 显示参考图缩略图。
- 可删除/替换。

### 系统逻辑

1. 上传文件经 PathGuard 存到项目 assets 目录。
2. 写入 `reference_image_path`，只存相对路径。
3. 生图时把参考图转为模型需要的格式。
4. 如果模型不支持参考图，则降级为只用 `visual_prompt`。

### 抄谁

- Toonflow `src/utils/ai.ts`：`referenceList` 传多参考图。
- Toonflow `src/routes/production/storyboard/batchGenerateImage.ts::getAssetsImageBase64`：把资产图片转 base64 给生图。

### 坑提醒

- 不要把用户本地绝对路径直接入库。
- 不要让前端直接传任意系统路径给后端生成器。

---

## 子功能 4：生成角色设定图

### 界面

- 按钮：「生成角色图」。
- 可选：正面半身、全身、表情图。
- 显示生成结果，支持重生成/设为参考图。

### 三视图要求

角色一致性不能只靠一张图。至少预留以下参考图角色：

```text
character_front_view
character_side_view
character_back_view
character_full_body
character_face_closeup
character_expression_sheet
character_outfit
```

第一版可以先生成 `front_view / full_body`，但数据结构必须支持三视图和表情表。

### 系统逻辑

1. 用 Character Bible 的 `visual_prompt` + Style Bible 的 `style_prompt` 生成图片提示词。
2. 调 ImageProvider。
3. 下载/转存结果到本地。
4. 用户确认后写入 `reference_image_path`。

### 抄谁

- Toonflow `src/routes/assetsGenerate/generateAssets.ts`。
- Toonflow `src/routes/assetsGenerate/batchGenerateImageAssets.ts`。

### 坑提醒

- 角色设定图应该先于批量分镜图生成，否则后面没稳定参考。

---

## 子功能 5：生成三视图 / 表情表

### 系统逻辑

1. 使用已确认的主参考图、Character Bible 和 Style Bible。
2. 生成正面、侧面、背面、全身、面部特写和表情表。
3. 每张图都作为 Asset 入库，并写入 `reference_images_json`。
4. 用户可选择其中一张作为 `reference_image_path` 主参考图。
5. 后续分镜生图优先读取 `reference_images_json`，模型不支持多参考图时再降级为主参考图。

### 坑提醒

- 三视图必须锁定同一角色身份、年龄、发型、服装和标志物。
- 表情表不能生成成另一个人。
- 生成失败不能覆盖旧参考图。

---

## 子功能 6：衍生资产

### 界面

- 在角色详情里新增「衍生姿态/表情」。
- 字段：衍生名称、描述、参考图。
- 例如：开心、惊讶、背影、坐姿、奔跑。

### 系统逻辑

1. 衍生资产挂到父角色 `character_id`。
2. 生成时用父角色参考图 + 衍生描述。
3. 分镜可选择具体衍生资产。

### 抄谁

- Toonflow `src/agents/productionAgent/tools.ts::add_deriveAsset`。
- Toonflow `src/agents/productionAgent/tools.ts::generate_deriveAsset`。

### 坑提醒

- 衍生资产不能变成新角色。它必须继承父角色的核心外貌和服装。

---

## 子功能 7：角色一致性注入到生图

### 系统逻辑

1. `07-生图提示词.md` 读取 StoryboardItem.character_ids_json 或 characters_json。
2. 按 ID 找 Character Bible。
3. 把 `visual_prompt` 拼入分镜 image_prompt。
4. 如果有 `reference_images_json`，`08-AI生图.md` 按参考图角色传给支持多参考图的模型。
5. 如果模型不支持多参考图，则降级使用 `reference_image_path` 主参考图。
6. 如果多个角色同镜，明确区分位置和身份，避免串脸。

### 抄谁

- Toonflow `src/routes/production/storyboard/batchGenerateImage.ts`：按分镜关联资产收集参考图。
- Toonflow `src/utils/ai.ts`：`referenceList`。

### 坑提醒

- 角色 prompt 不要每个镜头重新自由生成，要从 Character Bible 取。
- 多角色同框要写清“左边/右边/前景/背景”，否则模型容易混。

## 四、这个模块对外提供什么

输出 Character Bible：

```json
{
  "characters": [
    {
      "character_id": "main_character",
      "name": "小林",
      "visual_prompt": "...",
      "reference_image_path": "assets/characters/main_character.png",
      "reference_images_json": []
    }
  ]
}
```

被这些模块使用：

- `03-分镜.md`：StoryboardItem.character_ids_json 引用角色 ID。
- `07-生图提示词.md`：拼角色视觉描述。
- `08-AI生图.md`：传角色参考图。
