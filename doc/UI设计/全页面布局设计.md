## 执行口径

本文是后续前端页面布局、导航、交互和实现优先级的执行依据。若本文与较早的 UI 草案、参考分析或页面说明冲突，以本文为准；若涉及数据库 Schema、后端接口、安全、日志、Provider 能力或文件存储规则，以对应底层设计和 TODO 文档为准。

### 参考项目校准

本口径已按 `D:\project\短视频\waoowaoo-main` 和 `D:\project\短视频\Toonflow-web-master` 重新校准：

- `waoowaoo-main` 的工作台不是“导入前先完成完整分镜”，而是先做文本 / 配置 / 资产准备，再执行 `script-to-storyboard`，之后进入 storyboard、videos、voice 等阶段。
- `Toonflow-web-master` 的生产画布主链是 `script → scriptPlan → storyboardTable → storyboard → workbench`，分镜同样是脚本之后的生产节点，不是文案导入前置物。
- 两个参考项目都证明：文本到分镜之间需要脚本 / 旁白确认能力，但不证明本项目必须保留一个独立 `script-editor` 必经路由。
- 参考项目的具体机制以 `doc/参考分析/参考项目深度机制与吸收口径.md` 为准；本文件只把其中会影响页面布局和交互的部分落成 UI 执行口径。

本项目落地取舍：

- 内容创作页负责输入、AI 整理、切分预览、逐条旁白 / 文案段落确认，产物是 `sourceText / narrationText` 段落。
- 分镜页负责把已确认段落变成镜头级 `StoryboardItem`，继续编辑画面描述、角色 / 场景、出图提示词、视频提示词和分镜状态。
- `script-editor` 不作为主流程必经独立页；如后续保留脚本高级编辑，只能作为可选增强，不能让用户在内容创作和脚本页重复编辑同一批旁白。
- 提示词跟着分镜数据走：分镜页编辑镜头级字段，生图页预览最终 prompt，创作规则页管理 prompt 模板和 schema，不新建孤立的“提示词大页面”。
- 候选结果是生产链路核心：生图和视频页必须保留候选、选择、预览、清理历史、失败重试等操作，不允许只显示单个结果路径。
- 资产贯穿流程：角色、场景、风格、参考图服务分镜和生图，不是孤立素材陈列；作品产物默认留在作品目录，用户确认复用后才进入素材库。

冲突处理规则：

- 不新增与主流程重复的创建入口；创建作品统一从“开始创作 / 内容创作”进入。
- 不保留两套编辑同一字段的页面或面板；主编辑入口唯一，辅助面板只做校验、预览和 AI 辅助。
- 不显示不可靠 ETA；批量任务只显示完成数、当前项和已用时。
- 合成页只负责片段检查、FFmpeg 合成、输出信息和导出；字幕、封面、BGM、模板动效作为后续独立步骤处理。
- 视频包是创作配置模板的唯一产品概念；不再另设“模板创建”入口。
- 步骤之间保留人工确认，不做“自动推进到生图/视频完成”的跨步骤跳过。

### 页面施工防自由发挥清单

后续写 UI 时，只要命中下面页面，必须在 TODO 或实现说明里回答“字段从哪里来、写回哪里、当前缺口是什么”。不能只按视觉草图补一个空壳。

```text
内容创作：
  必须有长文本提示、切分预览、段落行内编辑、确认写回；未确认段落不能进入 StoryboardItem。

分镜：
  表格是主编辑；Inspector 只做校验、AI 助手、候选预览和高级字段；字段按文本层 / 镜头层 / 生产层 / 高级层 / 结果层拆。

生图：
  候选图必须是 ImageCandidate[] 缩略图集合；重生成追加 revision；确认后写 selectedImageId；最终 prompt 预览要标注来源。

视频：
  候选视频必须是 VideoSegment[]；生成必须读取 selectedImageId；确认后写 selectedVideoSegmentId；清理未选不得删已确认片段。

视频包：
  页面展示默认参数、规则引用、推荐模型 / workflow、素材引用；只保存引用，不保存 prompt 正文、密钥、Provider 连接或任务产物。

创作规则：
  页面管理 prompt_body / output_schema / params_schema 和测试样例；不直接调用 Provider，不直接写 StoryboardItem。

模型 / 工作流：
  Provider、ProviderModel、WorkflowPreset、测试记录是四个 Tab；业务页只读后端聚合的可执行选项。
```

### 2.5 规则

```text
1. AI 工具是独立能力，不承担"创建作品"入口
2. 工具结果只支持复制、保存到历史或保存为素材，不直接创建新作品
3. 工具结果保存在本地，有历史记录
4. 后续新增工具只需注册到工具列表，不改 Shell 结构
5. 工具调用走已有的 ProviderManager，不另建调用链路
```

### 2.6 MVP

爆款开头生成先做，其他工具标"即将推出"。主题生成完整作品只保留在"开始创作 / 内容创作"主流程里，避免两个入口做同一件事。

### 4.2 三种输入模式

| 模式 | 主编辑区行为 | AI 按钮 |
|---|---|---|
| 主题 | 输入一句话，AI 发散生成完整文案 | "AI 生成" → 调 LLM |
| 已有文案 | 直接粘贴，机械切分预览 | "切分预览" → 不调 LLM |
| 文章 | 粘贴长文，AI 浓缩提炼 | "AI 整理" → 调 LLM |

### 4.2.1 切分预览施工细节

内容创作页吸收 waoowaoo 的长文本检测和 SmartImportWizard 的“来源 → 解析 → 映射 → 确认”机制，但不新增独立脚本编辑页。

```text
长文本提示区：
  位置：输入框上方或输入框内顶部。
  内容：风险原因、建议切分方式、预计段落数。
  操作：[智能切分] [按段落] [按行] [按句] [继续手动编辑]。

切分预览区：
  位置：主编辑区下半部分或独立 Tab。
  行内容：序号、段落正文、字数、建议时长、锁定状态、校验问题。
  行内按钮：编辑、拆分、合并上一段、合并下一段、删除、上移、下移、恢复原文、锁定。
  批量按钮：重新切分、全部确认、仅确认有效段落。

写回：
  确认前只保存为当前内容草稿和切分预览。
  确认后按顺序写 StoryboardItem.sourceText / narrationText。
  AI 整理失败时仍允许用机械切分结果手动确认进入分镜。
```

不能做：


### 6.1.1 Inspector 展开规则


### 6.1.2 分镜字段和布局分层

这里吸收 waoowaoo 的镜头生产字段和 Toonflow-web 的 storyboardTable / storyboard 两层思路，但本项目只保留一个表格主编辑界面。

| 层级 | 字段 | UI 位置 | 规则 |
|---|---|---|---|
| 文本层 | `sourceText`、`narrationText` | 表格主列 | 来自内容创作确认段落，可行内编辑 |
| 镜头层 | `visualDescription`、角色、场景、道具 | 表格主列 | 负责“画什么”，可由 AI 助手建议 |
| 生产层 | `imagePrompt`、`negativePrompt`、`videoPrompt`、`durationSeconds` | 表格主列 | 负责生图 / 视频输入，锁定后批量重生成不得覆盖 |
| 高级层 | `shotSize`、`cameraMotion`、`composition`、`pace`、`transitionType`；`photographyRules / actingNotes` 后续进入 `advancedDirectingJson` | Inspector 或展开列 | 可选增强，不阻塞 MVP 进入生图；未落库前不临时造字段 |
| 结果层 | `selectedImageId`、`selectedVideoSegmentId`、下游状态 | 表格只读标签 + Inspector 预览 | 选择动作在生图 / 视频页完成 |

Inspector 内容固定分 Tab：

```text
校验：当前行缺失字段、锁定冲突、下游变脏提示。
AI 助手：优化画面描述、重写 imagePrompt、补充场景、改写 videoPrompt；应用时回写表格字段。
预览：只读展示候选图、已选图、视频候选、已选视频。
高级：景别、运镜、转场、摄影规则、表演指导。
```

删除 / 合并 / 拆分 / 插入时的弹窗必须说明影响：

```text
ImageCandidate[] 是否保留、迁移或变成下游变脏。
VideoSegment[] 是否保留、迁移或变成下游变脏。
selectedImageId / selectedVideoSegmentId 是否仍有效。
合成顺序是否因为 index 变化而改变。
```
