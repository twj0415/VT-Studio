# 测试体系

> 目标：把默认安全验证、可选真实 smoke、发布前验收分开，避免把 mock 结果写成真实能力通过。

## 默认验证

默认验证不触发外部 Provider、不产生费用、不访问网络、不要求真实 FFmpeg sidecar。

```powershell
.\scripts\verify-mvp.ps1
```

覆盖：

- Rust format：`cargo fmt --check`
- Rust 导出 / 备份 / 诊断：`cargo test export`
- Rust 错误码策略：`cargo test task_error_registry_covers_todo_09_error_codes`
- Rust 日志脱敏与轮转：`cargo test log_service`
- Rust 全量单测：`cargo test`
- 前端类型检查：`pnpm --dir src typecheck`
- 前端构建：`pnpm --dir src build`

快速模式：

```powershell
.\scripts\verify-mvp.ps1 -SkipFullCargoTest -SkipFrontendBuild
```

## Rust 测试层

- Repository / Migration：数据库迁移、Repository CRUD、任务状态机、资产引用。
- Service 集成：ProviderManager、SceneService、FFmpegService、ExportService、LogService。
- 安全：PathGuard、SecretGuard、错误 DTO 脱敏、导出 / 导入 / 诊断 Zip Slip。
- 导出恢复：final.mp4 导出、项目包导出 / 导入、工作区备份 / 恢复、诊断包。

## 前端测试层

当前默认自动化为：

- `vue-tsc --noEmit`
- `vite build`

后续如果引入组件测试，只能覆盖 UI 状态和交互，不替代 Tauri command / SQLite / 文件系统测试。

## Mock 与真实能力边界

- Mock E2E 只能证明页面流程和本地状态流，不证明真实 Provider 成功。
- 真实 Provider smoke 必须用户显式提供密钥和费用确认。
- 真实 FFmpeg smoke 必须工作区存在 `sidecars/ffmpeg.exe` 与 `sidecars/ffprobe.exe`。
- 真实导出外部目录、真实媒体附件诊断包、真实删除文件都不能默认跑。

## 发布前人工 Smoke

发布前至少跑：

```text
打开桌面应用
创建作品
输入内容
确认分镜
生成/选择图片候选
生成/确认视频片段
合成 final.mp4
导出 final.mp4
导出项目包
导入项目包
备份工作区
导出诊断包
打开输出目录
```

如果缺少真实 sidecar 或真实 Provider，必须在记录中写明“未执行真实 smoke”，不能写成通过。

详细步骤、失败恢复项和记录模板见 `plan/MVP-smoke-test.md`。该清单是 TODO-09 的 MVP smoke 固定口径；后续 UI 收口和发布前验收必须按这个文件记录。

发布版安装包 smoke 见 `plan/release-smoke-test.md`，它必须先通过 `pnpm run verify:release-resources` 和 `pnpm run tauri:build:windows`，不能用浏览器 Mock 或 Tauri dev 结果替代。
