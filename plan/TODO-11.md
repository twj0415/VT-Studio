### 【】11.1 明确第一版发布平台

**问题：**  
多平台同时处理会拖慢 MVP，Windows 路径和 sidecar 问题已经足够复杂。

**改法：**

第一版优先：

```text
Windows 10/11 x64
```

后续再扩展：

```text
macOS
Linux
```

**验证：**

- Windows 10/11 x64 打包可安装、可启动。
- 工作区默认路径合理。
- 中文路径、空格路径可用。

**风险：**  
Windows junction、UNC、权限和杀软拦截要重点测试。

---

### 【】11.2 整理随包资源清单

**问题：**  
开发环境文件齐全不代表安装包包含运行所需资源。

**改法：**

安装包必须包含：

```text
前端静态资源
Rust 后端
FFmpeg
FFprobe
内置模板
内置 PromptSkill
内置字体/图标资源
默认配置 schema
SQLite migrations
Playwright driver / Chromium 策略
默认字典
```

**验证：**

- 全新机器安装后无需源码目录即可启动。
- 缺少非核心 sidecar 时给明确提示。
- migrations 可执行。

**风险：**  
不要依赖开发机上的全局 ffmpeg、node、pnpm、chrome。

---

### 【】11.3 实现首次启动初始化

**问题：**  
发布版首次启动要自动准备数据目录、数据库和内置资源。

**改法：**

首次启动流程：

```text
创建应用数据目录
初始化 SQLite
执行 migrations
初始化默认配置
初始化内置字典
扫描内置模板
扫描 PromptSkill
检查 sidecar
进入欢迎/设置引导
```

**验证：**

- 删除数据目录后重新启动可完整初始化。
- 初始化失败有可恢复错误。
- 不会覆盖用户已有配置。

**风险：**  
初始化过程不能写入源码目录或安装目录。

---

### 【】11.4 实现 sidecar 自检

**问题：**  
FFmpeg、FFprobe、Chromium 是打包后最容易失效的部分。

**改法：**

自检：

```text
ffmpeg -version
ffprobe -version
Chromium 可启动
默认模板可截图
工作区可读写
SQLite 可打开
```

**验证：**

- 设置页或诊断页显示自检状态。
- sidecar 缺失不阻止打开应用。
- 相关任务启动前必须检查 sidecar。

**风险：**  
sidecar 路径必须受控，不能让用户输入任意可执行路径直接执行。

---

### 【】11.5 建立发布版 smoke test

**问题：**  
源码环境测试通过不代表安装包可用。

**改法：**

发布版 smoke test：

```text
安装应用
首次启动
初始化数据库
打开系统设置
检查 sidecar
从“我的作品”点击“开始创作”
Mock 主线跑通
真实 FFmpeg 小样例合成
导出 final.mp4
重启恢复项目
导出诊断包
卸载/保留数据策略检查
```

**验证：**

- 全新 Windows 环境通过。
- 中文路径和空格路径通过。
- 离线环境可打开应用。

**风险：**  
真实 Provider smoke test 需要显式密钥和用户确认，不能默认跑。

---

### 【】11.6 建立安装包签名与更新预留

**问题：**  
未签名安装包容易被拦截；更新机制后续要预留安全边界。

**改法：**

第一版记录：

```text
代码签名策略
安装包生成流程
版本号规则
更新源 HTTPS 要求
更新包签名校验预留
回滚策略预留
```

**验证：**

- 版本号能在应用和诊断包中显示。
- 更新相关配置不含密钥。

**风险：**  
不要从非 HTTPS 或未签名来源自动更新。

---

### 【】11.7 建立授权合规清单

**问题：**  
FFmpeg、Chromium、Playwright、字体、模板素材都有授权风险。

**改法：**

检查并记录：

```text
FFmpeg license
FFprobe license
Chromium license
Playwright license
Naive UI / Vue / 依赖 license
字体授权
图标授权
模板素材授权
参考项目许可证边界
```

**验证：**

- 安装包包含 license 说明。
- 不使用来源不明字体/模板/素材。
- 参考项目代码不直接复制污染许可证。

**风险：**  
特别注意 FFmpeg 编码器组合的 LGPL/GPL 边界。

---

### 【】11.8 建立发布前检查清单

**问题：**  
发布动作不可逆，必须有固定 checklist。

**改法：**

发布前检查：

```text
前端 typecheck
前端 build
cargo test
Tauri build
发布版 smoke test
sidecar 自检
诊断包脱敏检查
项目包导出检查
许可证检查
版本号检查
安装卸载检查
```

**验证：**

- 每次发布前按 checklist 执行。
- 失败项不得跳过发布。

**风险：**  
不要只以开发模式 `tauri:dev` 能跑作为发布依据。
